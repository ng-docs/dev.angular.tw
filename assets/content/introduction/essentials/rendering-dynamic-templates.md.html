渲染動態範本

edit

使用 Angular 的範本語法建立動態 HTML。

到目前為止，你學到的內容使你可以將應用拆分為 HTML 元件，但這限制了你只能使用靜態範本（即內容不變）。下一步是學習如何使用 Angular 的範本語法來建立動態 HTML。

渲染動態資料

當你需要在範本中顯示動態內容時，Angular 使用雙花括號語法來區分靜態和動態內容。

這是一個 TodoListItem 元件的簡化範例。

      
@Component({
  selector: 'todo-list-item',
  template: `
    <p>Title: {{ taskTitle }}</p>
  `,
})
export class TodoListItem {
  taskTitle = 'Read cup of coffee';
}

當 Angular 渲染該組件時，你會看到輸出：

      
<p>Title: Read cup of coffee</p>

這種語法宣告了 HTML 內部動態資料屬性之間的 內嵌。因此，每當資料變化時，Angular 會自動更新 DOM，以反映該屬性的新值。

動態屬性

當你需要在 HTML 元素上動態設定標準 DOM 屬性的值時，該屬性會用方括號包裹，以通知 Angular 宣告的值應被解釋為類似 JavaScript 的語句（ 帶有一些 Angular 增強功能），而不是純字串。

例如，在你的 HTML 中動態更新屬性的一個常見例子是，根據表單是否有效來確定是否應停用表單送出按鈕。

將所需的屬性用方括號括起來，告訴 Angular 分配的值是動態的（即不是靜態字串）。

      
@Component({
  selector: 'sign-up-form',
  template: `
    <button type="submit" [disabled]="formIsInvalid">Submit</button>
  `,
})
export class SignUpForm {
  formIsInvalid = true;
}

在這個例子中，因為 formIsInvalid 是 true，渲染的 HTML 將會是：

      
<button type="submit" disabled>Submit</button>

動態屬性

如果你想動態繫結自定義的 HTML 屬性（例如， aria-， data- 等），你可能會傾向於用相同的方括號包裹自定義屬性。

      
@Component({
  standalone: true,
  template: `
    <button [data-test-id]="testId">Primary CTA</button>
  `,
})
export class AppBanner {
  testId = 'main-cta';
}

不幸的是，這樣做是無效的，因為自定義的 HTML 屬性不是標準的 DOM 屬性。為使其按預期工作，我們需要用 attr. 字首來修飾自定義的 HTML 屬性。

      
@Component({
  standalone: true,
  template: `
    <button [attr.data-test-id]="testId">Primary CTA</button>
  `,
})
export class AppBanner {
  testId = 'main-cta';
}
