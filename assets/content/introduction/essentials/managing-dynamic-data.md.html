管理動態資料

edit

定義元件的狀態和行為來管理動態資料。

現在我們已經學習了元件的基本結構，讓我們學習如何定義元件的資料（即狀態）和行為。

什麼是狀態？

元件可以讓你整潔地封裝應用某些離散部分的職責。例如，一個 SignUpForm 元件可能需要在允許使用者執行特定動作之前跟蹤表單是否有效。因此，元件需要跟蹤的各種屬性通常被稱為「狀態」。

定義狀態

要定義狀態，你可以在元件內使用 類別欄位語法。

例如，使用 TodoListItem 元件，建立兩個你想要跟蹤的屬性：

  1. taskTitle — 任務的標題
  2. isComplete — 任務是否完成
      
// todo-list-item.component.ts
@Component({ ... })
export class TodoListItem {
  taskTitle = '';
  isComplete = false;
}

更新狀態

當你想更新狀態時，通常透過在元件類別中定義方法來實現，這些方法可以使用 this 關鍵字訪問各種類別中的屬性。

      
// todo-list-item.component.ts
@Component({ ... })
export class TodoListItem {
  taskTitle = '';
  isComplete = false;
  completeTask() {
    this.isComplete = true;
  }
  updateTitle(newTitle: string) {
    this.taskTitle = newTitle;
  }
}
