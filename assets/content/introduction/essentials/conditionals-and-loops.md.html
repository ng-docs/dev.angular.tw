條件和迴圈

edit

基於動態資料有條件地顯示和/或重複內容。

使用像 Angular 這樣的框架的優勢之一是它為開發者常遇到的常見問題提供了內建解決方案。這些問題的例子包括：基於某個條件顯示內容，基於應用資料渲染一系列條目等。

為了解決這個問題，Angular 使用內建的控制流區塊，這些控制流區塊告訴框架何時以及如何渲染你的範本。

條件渲染

開發者遇到的最常見場景之一是希望根據條件在範本中顯示或隱藏內容。

一個常見的例子是是否根據使用者的許可權級別在螢幕上顯示某些控制元件。

@if 控制區塊

類似於 JavaScript 的 if 語句，Angular 使用 @if 控制區塊有條件地隱藏和顯示範本及其內容的部分。

      
// user-controls.component.ts
@Component({
  standalone: true,
  selector: 'user-controls',
  template: `
    @if (isAdmin) {
      <button>Erase database</button>
    }
  `,
})
export class UserControls {
  isAdmin = true;
}

在此範例中，只有當 isAdmin 屬性為 true 時，Angular 才會渲染 <button> 元素。否則，它不會出現在頁面上。

@else 控制區塊

雖然 @if 控制區塊在許多情況下都很有用，但在條件不滿足時顯示備用 UI 也是很常見的。

例如，在 UserControls 元件中，與其顯示空白螢幕，不如讓使用者知道他們看不到任何內容是因為他們沒有經過身份驗證。

當你需要備用時，類似於 JavaScript 的 else 子句，可以新增 @else 控制區塊來實現相同的效果。

      
// user-controls.component.ts
@Component({
  standalone: true,
  selector: 'user-controls',
  template: `
    @if (isAdmin) {
      <button>Erase database</button>
    } @else {
      <p>You are not authorized.</p>
    }
  `,
})
export class UserControls {
  isAdmin = true;
}

渲染清單

開發者遇到的另一個常見場景是需要渲染一些清單條目。

@for 控制區塊

類似於 JavaScript 的 for...of 迴圈，Angular 提供了 @for 控制區塊來渲染重複的元素。

      
<!-- ingredient-list.component.html -->
<ul>
  @for (ingredient of ingredientList; track ingredient.name) {
    <li>{{ ingredient.quantity }} - {{ ingredient.name }}</li>
  }
</ul>

    
      
// ingredient-list.component.ts
@Component({
  standalone: true,
  selector: 'ingredient-list',
  templateUrl: './ingredient-list.component.html',
})
export class IngredientList {
  ingredientList = [
    {name: 'noodles', quantity: 1},
    {name: 'miso broth', quantity: 1},
    {name: 'egg', quantity: 2},
  ];
}

然而，不同於標準的 for...of 迴圈，你可能注意到還有一個額外的 track 關鍵字。

track 屬性

當 Angular 使用 @for 渲染元素清單時，這些條目可能會在後續更改或移動。Angular 需要透過任何重新排序跟蹤每個元素，通常是將條目的某個屬性作為唯一識別符號或鍵。

這確保了對清單的任何更新都能在 UI 中正確反映，並且在 Angular 內部得到正確的跟蹤，特別是在有狀態元素或動畫的情況下。

為此，我們可以使用 track 關鍵字為 Angular 提供唯一的鍵。
