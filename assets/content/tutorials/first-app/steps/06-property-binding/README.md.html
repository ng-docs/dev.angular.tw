向元件的樣板新增屬性繫結

edit

本教程課程示範了如何向樣板新增屬性繫結，並使用它將動態資料傳遞給元件。

你將學到什麼

  • 你的應用在 HomeComponent 樣板中具有資料繫結。
  • 你的應用將資料從 HomeComponent 傳送到 HousingLocationComponent。

對輸入（Input）的概念性預覽

在本課程中，你將繼續執行從父元件向子元件共享資料的過程，方法是使用屬性繫結將資料繫結到樣板中的那些屬性（Property）。

屬性繫結使你能夠將變數連線到 Angular 樣板中的 Input。然後，資料會動態繫結到 Input。

要獲得更深入的解釋，請參閱屬性繫結指南。

  1. 更新 HomeComponent 樣板

    此步驟將屬性繫結新增到 <app-housing-location> 標籤。

    在程式碼編輯器中：

    1. 導向到 src/app/home/home.component.ts

    2. 在 @Component 裝飾器的 template 屬性中，更新程式碼以對應以下程式碼：

      新增 housingLocation 屬性繫結

            
      import {Component} from '@angular/core';import {CommonModule} from '@angular/common';import {HousingLocationComponent} from '../housing-location/housing-location.component';import {HousingLocation} from '../housinglocation';@Component({  selector: 'app-home',  imports: [CommonModule, HousingLocationComponent],  template: `    <section>      <form>        <input type="text" placeholder="Filter by city" />        <button class="primary" type="button">Search</button>      </form>    </section>    <section class="results">      <app-housing-location [housingLocation]="housingLocation"></app-housing-location>    </section>  `,  styleUrls: ['./home.component.css'],})export class HomeComponent {  readonly baseUrl = 'https://angular.dev/assets/images/tutorials/common';  housingLocation: HousingLocation = {    id: 9999,    name: 'Test Home',    city: 'Test city',    state: 'ST',    photo: `${this.baseUrl}/example-house.jpg`,    availableUnits: 99,    wifi: true,    laundry: false,  };}

      當向元件標籤新增屬性繫結時，我們使用 [attribute] = "value" 語法來通知 Angular，分配的值應被視為來自元件類別的屬性（Property），而不是字串值。

      右側的值是來自 HomeComponent 的屬性（Property）的名稱。

  2. 確認程式碼仍然有效

    1. 儲存你的更改並確認應用沒有任何錯誤。
    2. 在繼續下一步之前，請更正任何錯誤。

摘要：在本課程中，你新增了一個新的屬性繫結，並傳入了對類別屬性（Property）的引用。現在，HousingLocationComponent 可以訪問可用於自定義元件顯示的資料。
