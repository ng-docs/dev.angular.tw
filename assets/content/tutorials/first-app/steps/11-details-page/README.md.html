将详情页面集成到应用中

edit

本课程示范了如何将详情页面连接到应用程序中。

重要： 我们建议使用本地环境学习路由。

你将学到什么

当这节课结束时，应用程序将支持路由到详情页面。

带参数路由的概念性预览

每个住房位置都有特定的详情，当用户导航至该条目的详情页面时应显示这些详情。要实现此目标，你需要使用路由参数。

路由参数使你能够将动态信息作为路由 URL 的一部分。要确定用户点击了哪个房屋位置，你要使用 HousingLocation 类型的 id 属性。

  1. 为你的应用创建一个新的服务

    在第 10 课中，你在 src/app/routes.ts 中添加了第二个路由，其中包括一个识别路由参数 id 的特殊段：

          
    'details/:id'

    在这种情况下，:id 是动态的，并且会根据代码请求路由的方式而变化。

    1. 在 src/app/housing-location/housing-location.component.ts 中，向 section 元素添加一个锚标签，并包含 routerLink 指令：

      在 housing-location.component.ts 中添加带有 routerLink 指令的锚标签

            
      import {Component, Input} from '@angular/core';
      import {CommonModule} from '@angular/common';
      import {HousingLocation} from '../housinglocation';
      import {RouterModule} from '@angular/router';
      @Component({
        selector: 'app-housing-location',
        standalone: true,
        imports: [CommonModule, RouterModule],
        template: `
          <section class="listing">
            <img
              class="listing-photo"
              [src]="housingLocation.photo"
              alt="Exterior photo of {{ housingLocation.name }}"
              crossorigin
            />
            <h2 class="listing-heading">{{ housingLocation.name }}</h2>
            <p class="listing-location">{{ housingLocation.city }}, {{ housingLocation.state }}</p>
            <a [routerLink]="['/details', housingLocation.id]">Learn More</a>
          </section>
        `,
        styleUrls: ['./housing-location.component.css'],
      })
      export class HousingLocationComponent {
        @Input() housingLocation!: HousingLocation;
      }

      routerLink 指令使 Angular 的路由器能够在应用中创建动态链接。赋予 routerLink 的值是一个包含两部分的数组：路径的静态部分和动态数据。

      为了在模板中使用 routerLink，从 '@angular/router' 中添加 RouterLink 和 RouterOutlet 的文件级别导入，然后更新组件的 imports 数组，以包含 RouterLink 和 RouterOutlet。

    2. 此时，你可以确认路由在应用中正常工作。在浏览器中，刷新主页并点击某个住房位置的 "Learn More" 按钮。

  2. 获取路由参数

    在此步骤中，你将在 DetailsComponent 中获取路由参数。目前，应用显示 details works!。接下来，你将更新代码以显示使用路由参数传递的 id 值。

    1. 在 src/app/details/details.component.ts 中更新模板以导入你需要在 DetailsComponent 中使用的函数、类和服务：

      更新文件级别导入

            
      import {Component, inject} from '@angular/core';
      import {CommonModule} from '@angular/common';
      import {ActivatedRoute} from '@angular/router';
      import {HousingService} from '../housing.service';
      import {HousingLocation} from '../housinglocation';
      @Component({
        selector: 'app-details',
        standalone: true,
        imports: [CommonModule],
        template: `
          <article>
            <img
              class="listing-photo"
              [src]="housingLocation?.photo"
              alt="Exterior photo of {{ housingLocation?.name }}"
              crossorigin
            />
            <section class="listing-description">
              <h2 class="listing-heading">{{ housingLocation?.name }}</h2>
              <p class="listing-location">{{ housingLocation?.city }}, {{ housingLocation?.state }}</p>
            </section>
            <section class="listing-features">
              <h2 class="section-heading">About this housing location</h2>
              <ul>
                <li>Units available: {{ housingLocation?.availableUnits }}</li>
                <li>Does this location have wifi: {{ housingLocation?.wifi }}</li>
                <li>Does this location have laundry: {{ housingLocation?.laundry }}</li>
              </ul>
            </section>
          </article>
        `,
        styleUrls: ['./details.component.css'],
      })
      export class DetailsComponent {
        route: ActivatedRoute = inject(ActivatedRoute);
        housingService = inject(HousingService);
        housingLocation: HousingLocation | undefined;
        constructor() {
          const housingLocationId = Number(this.route.snapshot.params['id']);
          this.housingLocation = this.housingService.getHousingLocationById(housingLocationId);
        }
      }

    2. 更新 @Component 装饰器的 template 属性以显示值 housingLocationId：

            
      template: `<p>details works! {{ housingLocationId }}</p>`,

    3. 使用以下代码更新 DetailsComponent 类的主体：

            
      export class DetailsComponent {
               route: ActivatedRoute = inject(ActivatedRoute);
               housingLocationId = -1;
               constructor() {
                   this.housingLocationId = Number(this.route.snapshot.params['id']);
               }
           }

      此代码使 DetailsComponent 能够访问 ActivatedRoute 路由特性，从而使你能够访问有关当前路由的数据。在 constructor 中，代码将从路由获取的 id 参数从字符串转换为数字。

    4. 保存所有更改。

    5. 在浏览器中，点击某个住房位置的 "Learn More" 链接，并确认页面上显示的数值与数据中该位置的 id 属性匹配。

  3. 自定义 DetailComponent

    路由已能在应用程序中正常工作，接下来要更新 DetailsComponent 的模板，以显示由路由参数的房屋位置表示的特定数据。

  4. 为 HomeComponent 添加导航

    在前一课中，你更新了 AppComponent 的模板，加入了 routerLink。添加该代码使你的应用能够在点击标志时导航回 HomeComponent。

    1. 确认你的代码与以下内容一致：

      向 AppComponent 添加 routerLink

            
      import {Component} from '@angular/core';
      import {HomeComponent} from './home/home.component';
      import {RouterLink, RouterOutlet} from '@angular/router';
      @Component({
        selector: 'app-root',
        standalone: true,
        imports: [HomeComponent, RouterLink, RouterOutlet],
        template: `
          <main>
            <a [routerLink]="['/']">
              <header class="brand-name">
                <img class="brand-logo" src="/assets/logo.svg" alt="logo" aria-hidden="true" />
              </header>
            </a>
            <section class="content">
              <router-outlet></router-outlet>
            </section>
          </main>
        `,
        styleUrls: ['./app.component.css'],
      })
      export class AppComponent {
        title = 'homes';
      }

      你的代码可能已经是最新的，但还是请再次确认。

摘要：在本课中，你添加了路由以显示详情页面。

你现在知道如何：

  • 使用路由参数将数据传给路由
  • 使用 routerLink 指令，通过动态数据创建路由
  • 使用路由参数，从 HousingService 检索数据以显示特定的住房位置信息。
