为组件添加一个输入参数

edit

本教程演示了如何创建一个组件 @Input() 并将其用于向组件传递数据以进行自定义。

你将学到什么

你的应用程序的 HousingLocationComponent 模板有一个 HousingLocation 属性来接收输入。

输入属性的概念性预览

输入属性允许组件共享数据。数据共享的方向是从父组件到子组件。

在本课中，你将在 HousingLocationComponent 组件中定义 @Input() 属性，这将使你能够自定义组件中显示的数据。

在 使用输入属性接受数据 和 使用输出进行自定义事件 指南中了解更多。

  1. 导入 Input 装饰器

    这一步将 Input 装饰器导入到类中。

    在代码编辑器中：

    1. 导航到 src/app/housing-location/housing-location.component.ts
    2. Update the file imports to include Input and HousingLocation:

      在 src/app/housing-location/housing-location.component.ts 中导入 HousingLocationComponent 和 Input。

            
      import {Component, Input} from '@angular/core';
      import {CommonModule} from '@angular/common';
      import {HousingLocation} from '../housinglocation';
      @Component({
        selector: 'app-housing-location',
        standalone: true,
        imports: [CommonModule],
        template: `
          <p>housing-location works!</p>
        `,
        styleUrls: ['./housing-location.component.css'],
      })
      export class HousingLocationComponent {
        @Input() housingLocation!: HousingLocation;
      }

  2. 添加 Input 属性

    1. 在同一文件中，向 HousingLocationComponent 类添加一个名为 housingLocation 的类型为 HousingLocation 的属性。在属性名称后面添加一个 !，并在前面加上 @Input() 装饰器：

      在 src/app/housing-location/housing-location.component.ts 中导入 HousingLocationComponent 和 Input。

            
      import {Component, Input} from '@angular/core';
      import {CommonModule} from '@angular/common';
      import {HousingLocation} from '../housinglocation';
      @Component({
        selector: 'app-housing-location',
        standalone: true,
        imports: [CommonModule],
        template: `
          <p>housing-location works!</p>
        `,
        styleUrls: ['./housing-location.component.css'],
      })
      export class HousingLocationComponent {
        @Input() housingLocation!: HousingLocation;
      }

      你必须添加 !，因为输入框期望传递一个值。在这种情况下，没有默认值。在我们的示例应用中，我们知道这个值会被传递进来 —— 这是刻意设计的。这个感叹号被称为非空断言操作符，它告诉 TypeScript 编译器该属性的值不会是 null 或 undefined。

    2. 保存你的更改并确认应用程序没有任何错误。

    3. 在继续下一步之前更正所有错误。

总结：在这节课中，你创建了一个用 @Input() 装饰的新属性。你还使用了非空断言运算符，通知编译器新属性的值不会是 null 或 undefined。
