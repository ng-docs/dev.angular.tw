Angular 服務

edit

本教學課程課程示範了如何建立一個 Angular 服務，並使用依賴注入將其包含在你的應用中。

你將學到什麼

你的應用有一個服務來為你的應用提供資料。 在本課程結束時，該服務將從本地靜態資料中讀取資料。 在後麵的課程中，你將更新該服務以從 Web 服務獲取資料。

服務的概念預覽

本教學課程介紹了 Angular 服務和依賴注入。

Angular 服務

Angular 服務 為你提供了一種方法來分離 Angular 應用資料和函式，這些資料和函式可以被你的應用中的多個元件使用。 為了被多個元件使用，服務必須是可注入的。 可注入的且被元件使用的服務將成為該組件的依賴項。 元件依賴於這些服務，沒有它們就無法執行。

依賴注入

依賴注入 是一種機制，用於管理應用的元件的依賴項以及其他元件可以使用的服務。

  1. 為你的應用建立一個新服務

    此步驟為你的應用建立一個可注入的服務。

    在 IDE 的 終端 窗格中：

    1. 在你的專案目錄中，導向到 first-app 目錄。

    2. 在 first-app 目錄中，執行此指令以建立新服務。

            
      ng generate service housing --skip-tests

    3. 執行 ng serve 以建立應用並將其服務於 http://localhost:4200。

    4. 確認應用建立時沒有錯誤。 在繼續下一步之前，更正任何錯誤。

  2. 將靜態資料新增到新服務

  3. 將新服務注入到 HomeComponent 中

    此步驟將新服務注入到你的應用的 HomeComponent 中，以便它可以從服務中讀取應用的資料。 在後麵的課程中，你將把靜態資料替換為實時資料來源，以便像在真實應用中一樣獲取資料。

    在 IDE 的 編輯 窗格中，在 src/app/home/home.component.ts 中：

    1. 在 src/app/home/home.component.ts 的頂部，將 inject 新增到從 @angular/core 匯入的條目中。 這會將 inject 函式匯入到 HomeComponent 類別中。

      更新到 src/app/home/home.component.ts

摘要： 在本課程中，你向你的應用新增了一個 Angular 服務，並將其注入到 HomeComponent 類別中。 這劃分了你的應用獲取資料的方式。 目前，新服務從靜態資料陣列中獲取其資料。 在後麵的課程中，你將重構該服務以從 API 端點獲取其資料。
