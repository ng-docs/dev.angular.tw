建立介面

edit

本教程課程示範了如何建立一個介面並將其包含在你的應用元件中。

你將學到什麼

  • 你的應用有了一個新的介面，可以將其用作資料型別。
  • 你的應用擁有一個新介面的實例，其中包含範例資料。

介面的概念預覽

介面是你的應用的自定義資料型別。

Angular 使用 TypeScript 以充分利用在強型別程式設計環境中工作。 強型別檢查降低了你的應用中的一個元素向另一個元素傳送格式不正確的資料的可能性。 這種型別不對應錯誤會被 TypeScript 編譯器捕獲，並且許多此類錯誤也可以在你的 IDE 中捕獲。

在本課程中，你將建立一個介面來定義表示單個房屋位置資料的屬性（Property）。

  1. 建立一個新的 Angular 介面

    此步驟在你的應用中建立一個新介面。

    在你的 IDE 的 終端 窗格中：

    1. 在你的專案目錄中，導向到 first-app 目錄。

    2. 在 first-app 目錄中，執行此指令以建立新介面。

            
      ng generate interface housinglocation

    3. 執行 ng serve 以建立應用併為 http://localhost:4200 啟動伺服器。

    4. 在瀏覽器中，開啟 http://localhost:4200 以檢視你的應用。

    5. 確認應用建立時沒有錯誤。 在繼續下一步之前，更正所有錯誤。

  2. 向新介面新增屬性（Property）

    此步驟將屬性（Property）新增到你的應用所需的介面，以表示房屋位置。

    1. 在你的 IDE 的 終端 窗格中，啟動 ng serve 指令（如果尚未執行），以建立應用併為 http://localhost:4200 啟動伺服器。

    2. 在你的 IDE 的 編輯 窗格中，開啟 src/app/housinglocation.ts 檔案。

    3. 在 housinglocation.ts 中，將預設內容替換為以下程式碼，以使你的新介面與此範例對應。

      更新 src/app/housinglocation.ts 以對應此程式碼

            
      export interface HousingLocation {  id: number;  name: string;  city: string;  state: string;  photo: string;  availableUnits: number;  wifi: boolean;  laundry: boolean;}

    4. 儲存你的更改並確認應用未顯示任何錯誤。 在繼續下一步之前，更正所有錯誤。

    至此，你已經定義了一個介面，該介面表示有關房屋位置的資料，包括 id、name 和位置訊息。

  3. 為你的應用建立測試環境

    你有一個介面，但你尚未使用它。

    在此步驟中，你將建立一個介面的實例，併為其賦予一些範例資料。 你還不會在你的應用中看到此範例資料。 在該範例資料出現之前，還需要完成一些課程。

    1. 在你的 IDE 的 終端 窗格中，執行 ng serve 指令（如果尚未執行），以建立應用併為你的應用啟動伺服器，地址為 http://localhost:4200。

    2. 在你的 IDE 的 編輯 窗格中，開啟 src/app/home/home.component.ts。

    3. 在 src/app/home/home.component.ts 中，在現有的 import 語句之後新增此匯入語句，以便 HomeComponent 可以使用新的介面。

      在 src/app/home/home.component.ts 中匯入 HomeComponent

            
      import {Component} from '@angular/core';import {CommonModule} from '@angular/common';import {HousingLocationComponent} from '../housing-location/housing-location.component';import {HousingLocation} from '../housinglocation';@Component({  selector: 'app-home',  imports: [CommonModule, HousingLocationComponent],  template: `    <section>      <form>        <input type="text" placeholder="Filter by city" />        <button class="primary" type="button">Search</button>      </form>    </section>    <section class="results">      <app-housing-location></app-housing-location>    </section>  `,  styleUrls: ['./home.component.css'],})export class HomeComponent {  readonly baseUrl = 'https://angular.dev/assets/images/tutorials/common';  housingLocation: HousingLocation = {    id: 9999,    name: 'Test Home',    city: 'Test city',    state: 'ST',    photo: `${this.baseUrl}/example-house.jpg`,    availableUnits: 99,    wifi: true,    laundry: false,  };}

    4. 在 src/app/home/home.component.ts 中，將空的 export class HomeComponent {} 定義替換為以下程式碼，以在該組件中建立新介面的單個實例。

      將範例資料新增到 src/app/home/home.component.ts

            
      import {Component} from '@angular/core';import {CommonModule} from '@angular/common';import {HousingLocationComponent} from '../housing-location/housing-location.component';import {HousingLocation} from '../housinglocation';@Component({  selector: 'app-home',  imports: [CommonModule, HousingLocationComponent],  template: `    <section>      <form>        <input type="text" placeholder="Filter by city" />        <button class="primary" type="button">Search</button>      </form>    </section>    <section class="results">      <app-housing-location></app-housing-location>    </section>  `,  styleUrls: ['./home.component.css'],})export class HomeComponent {  readonly baseUrl = 'https://angular.dev/assets/images/tutorials/common';  housingLocation: HousingLocation = {    id: 9999,    name: 'Test Home',    city: 'Test city',    state: 'ST',    photo: `${this.baseUrl}/example-house.jpg`,    availableUnits: 99,    wifi: true,    laundry: false,  };}

    5. 確認你的 home.component.ts 檔案與此範例類似。

      src/app/home/home.component.ts

            
      import {Component} from '@angular/core';import {CommonModule} from '@angular/common';import {HousingLocationComponent} from '../housing-location/housing-location.component';import {HousingLocation} from '../housinglocation';@Component({  selector: 'app-home',  imports: [CommonModule, HousingLocationComponent],  template: `    <section>      <form>        <input type="text" placeholder="Filter by city" />        <button class="primary" type="button">Search</button>      </form>    </section>    <section class="results">      <app-housing-location></app-housing-location>    </section>  `,  styleUrls: ['./home.component.css'],})export class HomeComponent {  readonly baseUrl = 'https://angular.dev/assets/images/tutorials/common';  housingLocation: HousingLocation = {    id: 9999,    name: 'Test Home',    city: 'Test city',    state: 'ST',    photo: `${this.baseUrl}/example-house.jpg`,    availableUnits: 99,    wifi: true,    laundry: false,  };}

      透過將 HousingLocation 型別的 housingLocation 屬性（Property）新增到 HomeComponent 類，我們能夠確認資料與介面的描述對應。 如果資料不滿足介面的描述，則 IDE 有足夠的訊息為我們提供有用的錯誤。

    6. 儲存你的更改並確認應用沒有任何錯誤。 開啟瀏覽器並確認你的應用仍然顯示訊息『housing-location works!』。

    7. 在繼續下一步之前，請更正所有錯誤。

總結： 在本課程中，你建立了一個介面，為你的應用建立了一個新的資料型別。 這種新的資料型別使你可以指定 HousingLocation 資料在何處是必需的。 這種新的資料型別還使你的 IDE 和 TypeScript 編譯器可以確保在需要 HousingLocation 資料的地方使用它。
