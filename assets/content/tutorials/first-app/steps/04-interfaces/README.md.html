创建一个接口

edit

本教程的课程演示了如何创建接口并将其包含在应用程序的组件中。

你将学到什么

  • 应用程序有了一个可以用作数据类型的新接口。
  • 应用程序有了一个带有示例数据的新接口实例。

接口的概念性预览

接口 是应用程序的自定义数据类型。

Angular 使用 TypeScript 来充分发挥在强类型编程环境中工作的优势。强类型检查降低了应用程序中的一个元素向另一个元素发送错误格式数据的可能性。这种类型不匹配错误会被 TypeScript 编译器捕获，许多此类错误也可以在你的 IDE 中捕获。

在这一课中，你将创建一个接口来定义表示关于单个房屋位置的数据的属性。

  1. 创建一个新的 Angular 接口

    此步骤会在你的应用中创建一个新接口。

    在你的 IDE 的 终端 窗格中：

    1. 在你的项目目录中，导航到 first-app 目录。

    2. 在 first-app 目录中，运行此命令以创建新接口。

            
      ng generate interface housinglocation

    3. 运行 ng serve 来构建应用程序并将其提供给 http://localhost:4200。

    4. 在浏览器中，打开 http://localhost:4200 以查看应用程序。

    5. 确认此应用构建没有错误。在继续下一步之前，修正任何错误。

  2. 向新接口添加属性

    此步骤会将属性添加到应用程序用以代表房屋位置的接口中。

    1. 在 IDE 的终端窗格中，启动 ng serve 命令（如果它尚未运行）以构建应用程序并用其在 http://localhost:4200 上启动开发服务器。

    2. 在 IDE 的编辑窗格中，打开 src/app/housinglocation.ts 文件。

    3. 在 housinglocation.ts 中，将默认内容替换为以下代码，来让你的新接口与此示例一致。

      更新 src/app/housinglocation.ts 以匹配此代码

            
      export interface HousingLocation {
        id: number;
        name: string;
        city: string;
        state: string;
        photo: string;
        availableUnits: number;
        wifi: boolean;
        laundry: boolean;
      }

    4. 保存你的更改并确认应用程序未显示任何错误。在继续下一步之前更正所有错误。

    到目前为止，你已定义了一个表示有关住房位置的数据的接口，包括 id、 name 和位置信息。

  3. 为你的应用程序创建测试房屋

    你有了一个接口，但还没有使用它。

    在此步骤中，你将创建接口实例并为其赋值一些示例数据。但你仍然无法在应用程序中看到此示例数据。在那之前，还有另一些课程需要完成。

    1. 在 IDE 的终端窗格中，运行 ng serve 命令（如果它没在运行）以构建应用程序并在 http://localhost:4200 上用它提供服务。

    2. 在 IDE 的编辑窗格中，打开 src/app/home/home.component.ts 文件。

    3. 在 src/app/home/home.component.ts 中，在现有 import 语句之后添加此导入语句，以便 HomeComponent 可以使用新接口。

      在 src/app/home/home.component.ts 中导入 HomeComponent。

            
      import {Component} from '@angular/core';
      import {CommonModule} from '@angular/common';
      import {HousingLocationComponent} from '../housing-location/housing-location.component';
      import {HousingLocation} from '../housinglocation';
      @Component({
        selector: 'app-home',
        standalone: true,
        imports: [CommonModule, HousingLocationComponent],
        template: `
          <section>
            <form>
              <input type="text" placeholder="Filter by city" />
              <button class="primary" type="button">Search</button>
            </form>
          </section>
          <section class="results">
            <app-housing-location></app-housing-location>
          </section>
        `,
        styleUrls: ['./home.component.css'],
      })
      export class HomeComponent {
        readonly baseUrl = 'https://angular.dev/assets/tutorials/common';
        housingLocation: HousingLocation = {
          id: 9999,
          name: 'Test Home',
          city: 'Test city',
          state: 'ST',
          photo: `${this.baseUrl}/example-house.jpg`,
          availableUnits: 99,
          wifi: true,
          laundry: false,
        };
      }

    4. 在 src/app/home/home.component.ts 中，用如下代码替换空定义 export class HomeComponent {}，以在组件中创建新接口的单个实例。

      在 src/app/home/home.component.ts 中添加示例数据。

            
      import {Component} from '@angular/core';
      import {CommonModule} from '@angular/common';
      import {HousingLocationComponent} from '../housing-location/housing-location.component';
      import {HousingLocation} from '../housinglocation';
      @Component({
        selector: 'app-home',
        standalone: true,
        imports: [CommonModule, HousingLocationComponent],
        template: `
          <section>
            <form>
              <input type="text" placeholder="Filter by city" />
              <button class="primary" type="button">Search</button>
            </form>
          </section>
          <section class="results">
            <app-housing-location></app-housing-location>
          </section>
        `,
        styleUrls: ['./home.component.css'],
      })
      export class HomeComponent {
        readonly baseUrl = 'https://angular.dev/assets/tutorials/common';

          id: 9999,
          name: 'Test Home',
          city: 'Test city',

        };
      }

    5. 确认你的 home.component.ts 文件与此示例匹配。

      src/app/home/home.component.ts

            
      import {Component} from '@angular/core';
      import {CommonModule} from '@angular/common';
      import {HousingLocationComponent} from '../housing-location/housing-location.component';
      import {HousingLocation} from '../housinglocation';
      @Component({
        selector: 'app-home',
        standalone: true,
        imports: [CommonModule, HousingLocationComponent],
        template: `
          <section>
            <form>
              <input type="text" placeholder="Filter by city" />
              <button class="primary" type="button">Search</button>
            </form>
          </section>
          <section class="results">
            <app-housing-location></app-housing-location>
          </section>
        `,
        styleUrls: ['./home.component.css'],
      })
      export class HomeComponent {
        readonly baseUrl = 'https://angular.dev/assets/tutorials/common';
        housingLocation: HousingLocation = {
          id: 9999,
          name: 'Test Home',
          city: 'Test city',
          state: 'ST',
          photo: `${this.baseUrl}/example-house.jpg`,
          availableUnits: 99,
          wifi: true,
          laundry: false,
        };
      }

      通过将类型为 HousingLocation 的 housingLocation 属性添加到 HomeComponent 类中，我们可以确认数据是否与接口的描述匹配。如果数据不符合接口的描述，IDE 将有足够的信息为我们提供有用的错误提示。

    6. 保存你的更改并确认应用程序没有任何错误。打开浏览器并确认应用程序仍然能显示消息 “housing-location works!”

    7. 在继续下一步之前更正所有错误。

总结：在这节课中，你创建了一个接口，为你的应用程序创建了一个新的数据类型。 这种新的数据类型使你能够指定需要 HousingLocation 数据的位置。 这种新的数据类型还使你的 IDE 和 TypeScript 编译器能够确保 HousingLocation 数据在需要的地方被使用。
