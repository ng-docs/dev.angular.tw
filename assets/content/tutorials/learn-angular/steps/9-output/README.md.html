使用 @Output 进行组件通信

edit

在使用组件时，可能需要通知其他组件某些事情已经发生。也许是一个按钮被点击了，一个条目被添加/移除出列表，或发生了一些其他重要更新。在这种情况下，组件需要与父组件通信。

Angular 使用 @Output 装饰器来启用这种行为。

在此活动中，你将学习如何使用 @Output 装饰器和 EventEmitter 与组件进行通信。

要创建从子组件到父组件的通信路径，请在类属性上使用 @Output 装饰器，并将其赋值为 EventEmitter 类型：

child.component.ts

      
@Component({...})
class ChildComponent {
    @Output() incrementCountEvent = new EventEmitter<number>();
}

现在，该组件可以生成父组件可以监听的事件。通过调用 emit 方法来触发事件：

child.component.ts

      
class ChildComponent {
    ...
    onClick() {
        this.count++;
        this.incrementCountEvent.emit(this.count);
    }
}

emit 函数将生成一个与 EventEmitter 实例类型相同的事件。

好了，现在轮到你试试了。通过完成这些任务来完善代码：

  1. 添加一个 @Output 属性

    通过添加一个名为 addItemEvent 的输出属性来更新 child.component.ts，确保将 EventEmitter 类型设置为 string。

  2. 完成 addItem 方法

    在 child.component.ts 中更新 addItem 方法；使用以下代码作为逻辑：

    child.component.ts

          
    addItem() {
      this.addItemEvent.emit('🐢');
    }

  3. 更新 AppComponent 模板

    在 app.component.ts 中更新模板，通过添加以下代码来监听发出的事件：

          
    <app-child (addItemEvent)="addItem($event)" />

    现在，每次点击“Add Item”按钮时，该按钮都会向列表中添加一个新条目。
