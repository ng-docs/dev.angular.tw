建立可注入的服務

edit

Angular 中的依賴注入 (DI) 是該框架最強大的屬性之一。可以將依賴注入視為 Angular 在執行時提供你的應用所需的資源的能力。依賴項可以是服務或其他一些資源。

你可以從 Angular 文件的依賴注入 瞭解更多訊息。現在，你將練習建立 injectable 資源。

在本活動中，你將學習如何建立一個可注入的服務。

使用服務的一種方式是充當與資料和 API 互動的方式。為了使服務可複用，你應該將邏輯保留在服務中，並在需要時在整個應用中共享它。

要使服務符合被 DI 體系注入的條件，請使用 @Injectable 裝飾器。例如：

      
@Injectable({    providedIn: 'root'})class UserService {    // methods to retrieve and return data}

@Injectable 裝飾器通知 DI 體系，UserService 可在類別中被請求。providedIn 設定此資源可用的作用域。目前，理解 providedIn: 'root' 意味著 UserService 對整個應用可用就足夠了。
