创建可注入服务

edit

Angular 中的依赖注入（DI）是框架最强大的特性之一。可以将依赖注入视为 Angular 在运行时为你的应用 提供所需资源的能力。依赖项可以是服务或其他资源。

你可以在 Angular 文档中了解更多关于依赖注入的信息。现在，你将练习创建 injectable 资源。

在这个活动中，你将学习如何创建一个可注入的服务。

使用服务的一种方式是作为与数据和 API 交互的方式。为了使服务可重用，你应该将逻辑保留在服务中，并在需要时在整个应用程序中共享它。

要使服务符合依赖注入系统的注入资格，请使用 @Injectable装饰器。例如：

      
@Injectable({
    providedIn: 'root'
})
class UserService {
    // methods to retrieve and return data
}

@Injectable装饰器会通知依赖注入系统， UserService可以在类中请求。 providedIn设置了该资源可用的范围。现在，理解 providedIn: 'root'意味着 UserService对整个应用程序可用已经足够了。
