最佳化影象

edit

影象是許多應用的重要組成部分，並且可能是導致應用效能問題（包括 Core Web Vitals 分數偏低）的主要原因。

影象最佳化可能是一個複雜的主題，但 Angular 使用 NgOptimizedImage 指令為你處理了大部分工作。在本活動中，你將學習如何使用 NgOptimizedImage 來確保你的影象被高效載入。

  1. 匯入 NgOptimizedImage 指令

    為了利用 NgOptimizedImage 指令，首先從 @angular/common 函式庫中匯入它，並將其新增到元件的 imports 陣列中。

          
    import { NgOptimizedImage } from '@angular/common';@Component({  imports: [NgOptimizedImage],  ...})

  2. 將 src 屬性更新為 ngSrc

    要啟用 NgOptimizedImage 指令，請將 src 屬性替換為 ngSrc。這適用於靜態影象源（即 src）和動態影象源（即 [src]）。

          
    import { NgOptimizedImage } from '@angular/common';@Component({  template: `    ...    <li>      Static Image:      <img ngSrc="/assets/logo.svg" alt="Angular logo" width="32" height="32" />    </li>    <li>      Dynamic Image:      <img [ngSrc]="logoUrl" [alt]="logoAlt" width="32" height="32" />    </li>    ...  `,  imports: [NgOptimizedImage],})

  3. 新增 width 和 height 屬性

    請注意，在上面的程式碼範例中，每個影象都具有 width 和 height 屬性。為了防止佈局偏移，NgOptimizedImage 指令要求每個影象都具有這兩個尺寸屬性。

    在無法或不想為影象指定靜態 height 和 width 的情況下，你可以使用 fill 屬性來告訴影象充當『背景影象』，填充其包含元素：

          
    <div class="image-container"> //Container div has 'position: "relative"'  <img ngSrc="www.example.com/image.png" fill /></div>

    注意：為了使 fill 影象正確渲染，其父元素必須使用 position: "relative"、position: "fixed" 或 position: "absolute" 進行樣式設定。

  4. 優先處理重要影象

    對於載入效能而言，最重要的最佳化之一是優先處理任何可能是 『LCP 元素』 的影象，LCP 元素是頁面載入時螢幕上最大的圖形元素。為了最佳化載入時間，請務必將 priority 屬性新增到你的『英雄影象』或你認為可能是 LCP 元素的任何其他影象。

          
    <img ngSrc="www.example.com/image.png" height="600" width="800" priority />

  5. 可選：使用影象載入器

    NgOptimizedImage 允許你指定一個 圖片載入器，它會告知該指令如何格式化你的圖片 URL。使用載入器允許你用簡短的相對 URL 定義你的圖片：

          
    providers: [  provideImgixLoader('https://my.base.url/'),]

    最終 URL 將是『https://my.base.url/image.png』

          
    <img ngSrc="image.png" height="600" width="800" />

    影象載入器不僅僅是為了方便——它們還允許你使用 NgOptimizedImage 的全部功能。要了解有關這些最佳化以及流行的 CDN 的內建載入器的更多訊息，請點選此處。

透過將此指令新增到你的工作流程中，藉助 Angular 🎉，你的影象現在可以使用最佳實踐進行載入

如果你想了解更多訊息，請檢視 NgOptimizedImage 的文件。繼續保持出色的工作，接下來讓我們學習路由。
