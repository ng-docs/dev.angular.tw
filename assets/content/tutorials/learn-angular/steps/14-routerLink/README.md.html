使用 RouterLink 進行導向

edit

在此應用的當前狀態下，當我們點選應用內的內部連結時，整個頁面都會重新整理。雖然這在小型應用中可能並不明顯，但對於內容較多的大型頁面，這可能會對效能產生影響，因為使用者必須重新下載資產並再次執行計算。

在本次實踐中，你將學習如何利用 RouterLink 指令來充分利用 Angular Router。

  1. 匯入 RouterLink 指令

    在 app.component.ts 中，將 RouterLink 指令匯入新增到 @angular/router 中已有的匯入語句中，並將其新增到元件裝飾器的 imports 陣列中。

          
    ...import { RouterLink, RouterOutlet } from '@angular/router';@Component({  imports: [RouterLink, RouterOutlet],  ...})

  2. 向樣板新增 routerLink

    要使用 RouterLink 指令，請將 href 屬性替換為 routerLink。使用此更改更新樣板。

          
    import { RouterLink, RouterOutlet } from '@angular/router';@Component({  ...  template: `    ...    <a routerLink="/">Home</a>    <a routerLink="/user">User</a>    ...  `,  imports: [RouterLink, RouterOutlet],})

現在，當你點選導向欄中的連結時，你應該看不到任何閃爍，並且只有頁面本身的內容（即 router-outlet）發生了變化 🎉

很好，路由相關的學習很棒。這只是 Router API 的冰山一角，要了解更多訊息，請檢視 Angular Router 文件。
