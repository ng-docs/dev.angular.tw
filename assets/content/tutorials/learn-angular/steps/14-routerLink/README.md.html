使用 RouterLink 进行导航

edit

在应用的当前状态下，当我们点击应用内存在的内部链接时，整个页面会刷新。虽然这在小型应用中看起来无关紧要，但对于拥有更多内容的大型页面，这可能会对性能产生影响，因为用户必须重新下载资产并重新运行计算。

在此活动中，你将学习如何利用 RouterLink 指令充分使用 Angular 路由。

  1. 导入 RouterLink 指令

    在 app.component.ts 中，将 RouterLink 指令导入现有的 @angular/router 导入语句，并将其添加到你的组件装饰器的 imports 数组中。

          
    ...
    import { RouterLink, RouterOutlet } from '@angular/router';
    @Component({
      standalone: true,
      imports: [RouterLink, RouterOutlet],
      ...
    })

  2. 向模板添加 routerLink

    要使用 RouterLink 指令，将 href 属性替换为 routerLink。用此更改更新模板。

          
    import { RouterLink, RouterOutlet } from '@angular/router';
    @Component({
      ...
      standalone: true,
      template: `
        ...
        <a routerLink="/">Home</a>
        <a routerLink="/user">User</a>
        ...
      `,
      imports: [RouterLink, RouterOutlet],
    })

现在点击导航中的链接时，你应该不会看到任何闪烁，只看到页面内容本身（即 router-outlet）发生了变化 🎉

很好，你已经学习了关于 Angular 路由的一些知识。这只是 Router API 的表面，想了解更多请查看 Angular 路由文档。
