基於注入的依賴注入

edit

建立一個可注入的服務是 Angular 中依賴注入 (DI) 體系的第一步。如何將服務注入到元件中？Angular 有一個名為 inject() 的便捷函式，可以在適當的上下文中使用。

注意： 注入上下文超出了本教學課程的範圍，但如果您想了解更多訊息，可以在 Angular 文件 中找到更多訊息。

在本活動中，您將學習如何注入服務並在元件中使用它。

通常，使用 DI 體系提供的值初始化類別屬性會很有幫助。這是一個範例：

      
@Component({...})class PetCareDashboardComponent {    petRosterService = inject(PetRosterService);}

  1. 注入 CarService

    在 app.component.ts 中，使用 inject() 函式注入 CarService 並將其賦值給名為 carService 的屬性

    注意：請注意屬性（Property） carService 和類別 CarService 之間的區別。

  2. 使用 carService 實例

    呼叫 inject(CarService) 會為您提供 CarService 的一個實例，您可以在應用中使用它，並存儲在 carService 屬性中。

    在 AppComponent 的 constructor 函式中，新增以下實現：

          
    constructor() {    this.display = this.carService.getCars().join(' ⭐️ ');}

  3. 更新 AppComponent 範本

    使用以下程式碼更新 app.component.ts 中的元件範本：

          
    template: `<p>Car Listing: {{ display }}</p>`,

您剛剛將您的第一個服務注入到了一個元件中 - 非常棒！在您完成關於 DI 的這一節之前，您將學習一種替代語法，用於將資源注入到您的元件中。
