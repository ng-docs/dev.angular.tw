可延遲檢視

edit

有時在應用開發中，你最終會得到許多需要在你的應用中引用的元件，但由於各種原因，其中一些元件不需要立即載入。

也許它們位於可見區域下方，或者是直到稍後才互動的重型元件。 在這種情況下，我們可以使用可延遲檢視稍後載入其中一些資源。

在此活動中，你將學習如何使用可延遲檢視來延遲載入元件樣板的某個部分。

  1. 在 comments 元件周圍新增一個 @defer 控制區塊

    在你的應用中，部落格文章頁面在文章詳情之後有一個評論元件。

    用 @defer 控制區塊包裹評論元件以延遲載入它。

          
    @defer {  <comments />}

    上面的程式碼是如何使用基本的 @defer 控制區塊的範例。 預設情況下，當瀏覽器空閒時，@defer 將載入 comments 元件。

  2. 新增一個佔位符

    向 @defer 控制區塊新增一個 @placeholder 控制區塊。 @placeholder 控制區塊是你放置 html 的位置，它將在延遲載入開始之前顯示。 @placeholder 控制塊中的內容是急性載入的。

          
    @defer {  <comments />} @placeholder {  <p>Future comments</p>}

  3. 新增一個載入區塊

    向 @defer 控制區塊新增一個 @loading 控制區塊。 @loading 控制區塊是你放置 html 的位置，它將在當正在主動獲取延遲內容但尚未完成時顯示。 @loading 控制塊中的內容是急性載入的。

          
    @defer {  <comments />} @placeholder {  <p>Future comments</p>} @loading {  <p>Loading comments...</p>}

  4. 新增最短持續時間

    @placeholder 和 @loading 部分都有可選引數，以防止在快速載入時發生閃爍。 @placeholder 具有 minimum，而 @loading 具有 minimum 和 after。 向 @loading 控制區塊新增 minimum 持續時間，以便它至少渲染 2 秒。

          
    @defer {  <comments />} @placeholder {  <p>Future comments</p>} @loading (minimum 2s) {  <p>Loading comments...</p>}

  5. 新增一個視口觸發器

    可延遲檢視有許多觸發器選項。 新增一個視口觸發器，以便內容在進入視口後延遲載入。

          
    @defer (on viewport) {  <comments />}

  6. 新增內容
