什麼是可延遲檢視？

edit

一個完全渲染的 Angular 頁面可能包含許多不同的元件、指令和通道。雖然頁面的某些部分應該立即顯示給使用者，但可能有些部分可以等待稍後顯示。 Angular 的 可延遲檢視 ，使用 @defer 語法，可以幫助你加速你的應用，它會告訴 Angular 延遲下載頁面中不需要立即顯示的部分的 JavaScript。

在本次實踐中，你將學習如何使用可延遲檢視來延遲載入元件樣板的某個部分。

  1. 向樣板的某個部分新增一個 @defer 控制區塊

    在你的 app.component.ts 中，用 @defer 控制區塊包裹 article-comments 元件以延遲載入它。

          
    @defer {  <article-comments />}

    預設情況下，當瀏覽器空閒時，@defer 會載入 article-comments 元件。

    在瀏覽器的開發者控制台中，你可以看到 article-comments-component 延遲載入塊檔案是單獨載入的（具體的檔案名可能因執行而異）：

          
    Initial chunk files | Names                      |  Raw sizechunk-NNSQHFIE.js   | -                          | 769.00 kB | main.js             | main                       | 229.25 kB | Lazy chunk files    | Names                      |  Raw sizechunk-T5UYXUSI.js   | article-comments-component |   1.84 kB |

做得好！你已經學習了可延遲檢視的基礎知識。
