@angular/core

ChangeDetectorRef

類
程式碼

提供變更檢測功能的基礎類。 變更檢測樹會收集所有要檢查變更的檢視。 使用這些方法可以從樹中新增和移除檢視、啟動變更檢測，並顯式地將檢視標記為 dirty，這意味著它們已更改，需要重新渲染。

使用變更檢測鉤子定義自定義變更檢測

API

  
    abstract class ChangeDetectorRef {}

markForCheck

void

當檢視使用 ChangeDetectionStrategy#OnPush (checkOnce) 變更檢測策略時，會顯式地將該檢視標記為已更改，以便可以再次檢查它。

當輸入屬性已更改或檢視中已觸發事件時，元件通常會被標記為 dirty（需要重新渲染）。呼叫此方法以確保即使未發生這些觸發器，也檢查元件。

@returnsvoid

detach

void

將此檢視從變更檢測樹中分離。 分離的檢視在重新附著之前不會被檢查。 與 detectChanges() 結合使用以實現本地變更檢測檢查。

分離的檢視在變更檢測執行期間不會被檢查，直到它們被重新附著，即使它們被標記為 dirty 也是如此。

@returnsvoid

detectChanges

void

檢查此檢視及其子檢視。與 ChangeDetectorRef#detach 結合使用以實現本地變更檢測檢查。

@returnsvoid

checkNoChanges

void

檢查變更檢測器及其子檢測器，如果檢測到任何更改，則丟擲錯誤。

在開發模式下使用，以驗證執行變更檢測是否不會引入其他更改。在生產模式下調用它不起作用。

@deprecated

這是一個僅用於測試的 API，在生產介面中沒有位置。當應用在開發模式下執行時，checkNoChanges 已經是 ApplicationRef tick 的一部分。對於更精細的 checkNoChanges 驗證，請使用 ComponentFixture。

@returnsvoid

reattach

void

將先前分離的檢視重新附著到變更檢測樹。 預設情況下，檢視會附著到樹。

@returnsvoid

使用須知

以下範例示範了如何修改預設的變更檢測行為，以便在需要時執行顯式檢測。

將 markForCheck() 與 CheckOnce 策略結合使用

以下範例為一個元件設定了 OnPush 變更檢測策略（CheckOnce，而不是預設的 CheckAlways），然後在一段時間後強制執行第二次檢查。

          
@Component({  selector: 'app-root',  template: `Number of ticks: {{ numberOfTicks }}`,  changeDetection: ChangeDetectionStrategy.OnPush,  standalone: false,})class AppComponent {  numberOfTicks = 0;  constructor(private ref: ChangeDetectorRef) {    setInterval(() => {      this.numberOfTicks++;      // require view to be updated      this.ref.markForCheck();    }, 1000);  }}

分離變更檢測器以限制檢查發生的頻率

以下範例定義了一個元件，該組件具有一個大型的只讀資料清單，預計該清單會不斷變化，每秒多次。 為了提高效能，我們希望檢查和更新清單的頻率低於實際發生更改的頻率。為此，我們分離元件的變更檢測器，並每五秒執行一次顯式本地檢查。

          
class DataListProvider {  // in a real application the returned data will be different every time  get data() {    return [1, 2, 3, 4, 5];  }}@Component({  selector: 'giant-list',  template: ` <li *ngFor="let d of dataProvider.data">Data {{ d }}</li>`,  standalone: false,})class GiantList {  constructor(    private ref: ChangeDetectorRef,    public dataProvider: DataListProvider,  ) {    ref.detach();    setInterval(() => {      this.ref.detectChanges();    }, 5000);  }}@Component({  selector: 'app',  providers: [DataListProvider],  template: ` <giant-list></giant-list>`,  standalone: false,})class App {}

重新附著已分離的元件

以下範例建立了一個顯示實時資料的元件。 當 live 屬性設定為 false 時，該組件會將其變更檢測器從主變更檢測器樹中分離，並在該屬性變為 true 時重新附著它。

          
class DataProvider {  data = 1;  constructor() {    setInterval(() => {      this.data = 2;    }, 500);  }}@Component({  selector: 'live-data',  inputs: ['live'],  template: 'Data: {{dataProvider.data}}',  standalone: false,})class LiveData {  constructor(    private ref: ChangeDetectorRef,    public dataProvider: DataProvider,  ) {}  @Input()  set live(value: boolean) {    if (value) {      this.ref.reattach();    } else {      this.ref.detach();    }  }}@Component({  selector: 'app',  providers: [DataProvider],  template: `    Live Update: <input type="checkbox" [(ngModel)]="live" />    <live-data [live]="live"></live-data>  `,  standalone: false,})class App1 {  live = true;}
