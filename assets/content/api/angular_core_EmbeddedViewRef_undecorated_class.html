@angular/core

EmbeddedViewRef

類
程式碼

表示檢視容器中的 Angular 檢視。 嵌入式檢視可以從宿主元件之外的其他元件引用，宿主元件的樣板定義了它，或者它可以由 TemplateRef 獨立定義。

ViewContainerRef

API

  
    abstract class EmbeddedViewRef<C> extends ViewRef {}

context

C

此檢視的上下文，繼承自錨點元素。

rootNodes

any[]

此嵌入式檢視的根節點。

destroy

void

銷燬此檢視以及與之關聯的所有資料結構。

@returnsvoid

destroyed

boolean

報告此檢視是否已被銷燬。

onDestroy

void

一個生命週期鉤子，為檢視提供額外的開發者定義清理功能。

@paramcallbackFunction

一個處理函式，用於清理與檢視關聯的開發者定義資料。當呼叫 destroy() 方法時調用。

@returnsvoid

markForCheck

void

當檢視使用 ChangeDetectionStrategy#OnPush (checkOnce) 變更檢測策略時，顯式地將該檢視標記為已更改，以便可以再次檢查它。

當輸入屬性（Inputs）發生更改或檢視中觸發事件時，元件通常被標記為髒（需要重新渲染）。呼叫此方法以確保即使這些觸發器未發生，也檢查元件。

@returnsvoid

detach

void

將此檢視從變更檢測樹中分離。 分離的檢視在重新附著之前不會被檢查。 與 detectChanges() 結合使用以實現本地變更檢測檢查。

分離的檢視在變更檢測執行期間不會被檢查，直到它們被重新附著，即使它們被標記為髒。

@returnsvoid

detectChanges

void

檢查此檢視及其子檢視。與 ChangeDetectorRef#detach 結合使用以實現本地變更檢測檢查。

@returnsvoid

checkNoChanges

void

檢查變更檢測器及其子檢測器，如果檢測到任何更改，則丟擲錯誤。

在開發模式下使用，以驗證執行變更檢測是否不會引入其他更改。在生產模式下調用它是一個空操作。

@deprecated

這是一個僅用於測試的 API，不應出現在生產介面中。 當應用在開發模式下執行時，checkNoChanges 已經是 ApplicationRef 變更檢測週期的一部分。要進行更細粒度的 checkNoChanges 驗證，請使用 ComponentFixture。

@returnsvoid

reattach

void

將先前分離的檢視重新附著到變更檢測樹。 預設情況下，檢視會附著到樹上。

@returnsvoid

說明

表示檢視容器中的 Angular 檢視。 嵌入式檢視可以從宿主元件之外的其他元件引用，宿主元件的樣板定義了它，或者它可以由 TemplateRef 獨立定義。

檢視中元素的 Property 可以更改，但檢視中元素的結構（數量和順序）不能更改。透過在檢視容器中插入、移動或刪除巢狀檢視來更改元素的結構。

使用須知

以下樣板分解為兩個獨立的 TemplateRef 實例， 一個外部的和內部的。

          
Count: {{items.length}}<ul>  <li *ngFor="let  item of items">{{item}}</li></ul>

這是外部的 TemplateRef：

          
Count: {{items.length}}<ul>  <ng-template ngFor let-item [ngForOf]="items"></ng-template></ul>

這是內部的 TemplateRef：

          
  <li>{{item}}</li>

外部和內部的 TemplateRef 實例按如下方式組裝成檢視：

          
<!-- ViewRef: outer-0 -->Count: 2<ul>  <ng-template view-container-ref></ng-template>  <!-- ViewRef: inner-1 --><li>first</li><!-- /ViewRef: inner-1 -->  <!-- ViewRef: inner-2 --><li>second</li><!-- /ViewRef: inner-2 --></ul><!-- /ViewRef: outer-0 -->
