ServerRoutePrerenderWithParams

interface
開發者預覽版

使用靜態站點生成 (SSG) 模式的伺服器路由設定，包括對帶引數路由的支援。

RenderModeServerRoutePrerenderPrerenderFallback

API

  
    interface ServerRoutePrerenderWithParams extends Omit<ServerRoutePrerender, 'fallback'> {}

fallback

PrerenderFallback | undefined

如果 SSG 路徑未預渲染，則使用的可選策略。這與帶引數的 URL 路由尤其相關，在這些路由中，某些路徑可能不會在建立時預渲染。

此屬性決定瞭如何處理對未預渲染路徑的請求：

  • PrerenderFallback.Server：使用伺服器端渲染 (SSR) 在請求時動態生成頁面。
  • PrerenderFallback.Client：使用用戶端渲染 (CSR) 在用戶端獲取和渲染頁面。
  • PrerenderFallback.None：沒有備用方案；如果路徑未預渲染，伺服器將不處理該請求。

getPrerenderParams

() => Promise<Record<string, string>[]>

一個回傳 Promise 的函式，該 Promise 求解為一個物件陣列，每個物件都表示一個帶有 URL 引數的路由路徑。此函式在注入器上下文中執行，允許訪問 Angular 服務和依賴項。

path

string

與此路由關聯的路徑。

headers

Record<string, string> | undefined

要包含在此路由回應中的可選附加標頭。

renderMode

RenderMode.Prerender

指定路由使用靜態站點生成 (SSG) 模式。
