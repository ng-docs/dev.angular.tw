@angular/forms

NgControl

類別
程式碼

所有基於 FormControl 的指令所擴充套件的基底類別。它將 FormControl 物件繫結到一個 DOM 元素。

API

控制元件

AbstractControl<any, any> | null

對底層控制元件的引用。

value

any

報告控制元件的值（如果存在），否則為 null。

valid

boolean | null

報告控制元件是否有效。如果當前值不存在驗證錯誤，則認為控制元件有效。如果控制元件不存在，則回傳 null。

invalid

boolean | null

報告控制元件是否無效，這意味著輸入值中存在錯誤。如果控制元件不存在，則回傳 null。

pending

boolean | null

報告控制元件是否處於待定狀態，這意味著正在進行非同步驗證，並且輸入值的錯誤尚不可用。如果控制元件不存在，則回傳 null。

disabled

boolean | null

報告控制元件是否已停用，這意味著該控制元件在 UI 中被停用，並且免於驗證檢查，並從祖先控制元件的聚合值中排除。如果控制元件不存在，則回傳 null。

enabled

boolean | null

報告控制元件是否已啟用，這意味著該控制元件包含在祖先控制元件的有效性或值的計算中。如果控制元件不存在，則回傳 null。

errors

ValidationErrors | null

報告控制元件的驗證錯誤。如果控制元件不存在，則回傳 null。

pristine

boolean | null

報告控制元件是否為 pristine，這意味著使用者尚未更改 UI 中的值。如果控制元件不存在，則回傳 null。

dirty

boolean | null

報告控制元件是否為 dirty，這意味著使用者已更改 UI 中的值。如果控制元件不存在，則回傳 null。

touched

boolean | null

報告控制元件是否為 touched，這意味著使用者已在其上觸發了 blur 事件。如果控制元件不存在，則回傳 null。

status

string | null

報告控制元件的驗證狀態。可能的值包括：『VALID』、『INVALID』、『DISABLED』和『PENDING』。如果控制元件不存在，則回傳 null。

untouched

boolean | null

報告控制元件是否為已觸碰狀態，表示使用者尚未在其上觸發 blur 事件。如果控制元件不存在，則回傳 null。

statusChanges

Observable<any> | null

回傳一個多播的可觀察者，每當為控制元件計算驗證狀態時，它都會發出該狀態。如果控制元件不存在，則回傳 null。

valueChanges

Observable<any> | null

回傳一個多播的控制元件值變更可觀察者，每當控制元件的值在 UI 中或以程式設計方式更改時，它都會發出新的值。 如果控制元件不存在，則回傳 null。

path

string[] | null

回傳一個陣列，該陣列表示從頂層表單到此控制元件的路徑。 每個索引都是該層級控制元件的字串名稱。

validator

ValidatorFn | null

同步驗證器函式，由所有向此指令註冊的同步驗證器組成。

asyncValidator

AsyncValidatorFn | null

非同步驗證器函式，由所有向此指令註冊的非同步驗證器組成。

hasError

boolean

報告具有給定路徑的控制元件是否具有指定的錯誤。

@paramerrorCodestring

要檢查的錯誤程式碼

@parampathstring | (string | number)[] | undefined

控制元件名稱的清單，用於指定如何從當前控制元件移動到應查詢錯誤的控制元件。

@returnsboolean
用法說明

例如，對於以下 FormGroup：

          
form = new FormGroup({  address: new FormGroup({ street: new FormControl() })});

從根表單到 'street' 控制元件的路徑將是 'address' -> 'street'。

可以透過以下兩種格式之一提供給此方法：

  1. 字串控制元件名稱的陣列，例如 ['address', 'street']
  2. 一個字串中以句點分隔的控制元件名稱清單，例如 'address.street'

如果未提供路徑，則此方法檢查當前控制元件上的錯誤。
