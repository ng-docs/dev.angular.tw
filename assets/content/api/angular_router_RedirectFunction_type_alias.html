用于处理路径匹配 Route 配置时的重定向的函数类型。

RedirectFunction 确实可以访问完整的 ActivatedRouteSnapshot 接口。但在路由匹配阶段，某些数据无法准确知晓。例如，求解器直到稍后才运行，因此任何已求解的标题都不会填充。对于延迟加载的组件也是如此。这对于所有快照（直到根节点）也适用，因此包含父级（根、父、pathFromRoot）的属性也被排除。而且，由于完整的路由匹配尚未发生，因此 firstChild 和 children 也不可用。
