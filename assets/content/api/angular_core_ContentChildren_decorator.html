說明

用於設定內容查詢的 Property 裝飾器。

用於從內容 DOM 獲取元素或指令的 QueryList。 每當新增、刪除或移動子元素時，查詢清單都會更新，並且查詢清單的 changes 可觀察者將發出新值。

內容查詢在 ngAfterContentInit 回呼被呼叫之前設定。

不會檢索其他元件樣板中的元素或指令，因為元件的樣板對於其祖先元件來說始終是一個黑盒子。

元資料 Property：

  • selector - 用於查詢的指令類型或名稱。
  • descendants - 如果為 true，則包括元素的所有後代。如果為 false，則僅查詢元素的直接子元素。
  • emitDistinctChangesOnly - 僅當 QueryList 結果已更改時， QueryList#changes 可觀察者才會發出新值。當 false 時，即使 QueryList 沒有更改，changes 可觀察者也可能會發出。 ** 注意：*** 此設定選項已棄用，它將被永久設定為 true 並在未來的 Angular 版本中刪除。
  • read - 用於從查詢到的元素中讀取不同的令牌。

支援以下選擇器。

  • 任何帶有 @Component 或 @Directive 裝飾器的類
  • 作為字串的樣板引用變數（例如，使用 @ContentChildren('cmp') 查詢 <my-component #cmp></my-component>）
  • 當前元件的子元件樹中定義的任何提供者（例如，@ContentChildren(SomeService) someService: SomeService）
  • 透過字串令牌定義的任何提供者（例如，@ContentChildren('someToken') someTokenVal: any）
  • 一個 TemplateRef（例如，使用 @ContentChildren(TemplateRef) template; 查詢 <ng-template></ng-template>）

此外，多個字串選擇器可以用逗號分隔（例如，@ContentChildren('cmp1,cmp2')）

read 支援以下值：

  • 任何帶有 @Component 或 @Directive 裝飾器的類
  • 此查詢的 selector 所對應的元件的注入器上定義的任何提供者
  • 透過字串令牌定義的任何提供者（例如，{provide: 'token', useValue: 'val'}）
  • TemplateRef、ElementRef 和 ViewContainerRef

使用須知

這是一個關於如何使用 ContentChildren 裝飾器的簡單示範。

          
import {AfterContentInit, ContentChildren, Directive, QueryList} from '@angular/core';@Directive({  selector: 'child-directive',  standalone: false,})class ChildDirective {}@Directive({  selector: 'someDir',  standalone: false,})class SomeDir implements AfterContentInit {  @ContentChildren(ChildDirective) contentChildren!: QueryList<ChildDirective>;  ngAfterContentInit() {    // contentChildren is set  }}

Tab-pane 範例

這是一個稍微更真實的範例，展示了如何使用 ContentChildren 裝飾器來實現一個 tab pane 元件。

          
import {Component, ContentChildren, Directive, Input, QueryList} from '@angular/core';@Directive({  selector: 'pane',  standalone: false,})export class Pane {  @Input() id!: string;}@Component({  selector: 'tab',  template: `    <div class="top-level">Top level panes: {{ serializedPanes }}</div>    <div class="nested">Arbitrary nested panes: {{ serializedNestedPanes }}</div>  `,  standalone: false,})export class Tab {  @ContentChildren(Pane) topLevelPanes!: QueryList<Pane>;  @ContentChildren(Pane, {descendants: true}) arbitraryNestedPanes!: QueryList<Pane>;  get serializedPanes(): string {    return this.topLevelPanes ? this.topLevelPanes.map((p) => p.id).join(', ') : '';  }  get serializedNestedPanes(): string {    return this.arbitraryNestedPanes ? this.arbitraryNestedPanes.map((p) => p.id).join(', ') : '';  }}@Component({  selector: 'example-app',  template: `    <tab>      <pane id="1"></pane>      <pane id="2"></pane>      <pane id="3" *ngIf="shouldShow">        <tab>          <pane id="3_1"></pane>          <pane id="3_2"></pane>        </tab>      </pane>    </tab>    <button (click)="show()">Show 3</button>  `,  standalone: false,})export class ContentChildrenComp {  shouldShow = false;  show() {    this.shouldShow = true;  }}
