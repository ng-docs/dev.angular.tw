用于配置内容查询的参数装饰器。

用于从内容 DOM 中获取元素或指令的 QueryList。任何时候添加、移除或移动子元素，查询列表将会更新，并且查询列表的更改可观察者将会发出新值。

在调用 ngAfterContentInit 之前设置的内容查询。

不检索其他组件模板中的元素或指令，因为组件模板对其祖先来说始终是黑匣子。

元数据属性：

  • selector - 要查询的指令类型或名称。
  • descendants - 如果为 true，则包括元素的所有后代。如果为 false，则仅查询元素的直接子元素。
  • emitDistinctChangesOnly - 此 QueryList#changes 可观察者只有在 QueryList 结果发生变化时才会发出新值。当 false 时，即使 QueryList 未发生变化， changes 可观察者也可能会发出新值。 ** 注意：*** 此配置选项已 弃用，在未来的 Angular 版本中将永久设置为 true 并移除。
  • read - 用于从查询到的元素中读取不同的令牌。

支持下列选择器：。

  • 任何带有 @Component 或 @Directive 装饰器的类
  • 一个字符串形式的模板引用变量（例如，查询 <my-component #cmp></my-component> 和 @ContentChildren('cmp')）
  • 在当前组件的子组件树中定义的任何提供者（例如 @ContentChildren(SomeService) someService: SomeService）
  • 通过字符串标记定义的任何提供者（例如 @ContentChildren('someToken') someTokenVal: any）
  • 一个 TemplateRef（例如，查询 <ng-template></ng-template> 和 @ContentChildren(TemplateRef) template;）

此外，多个字符串选择器可以用逗号分隔（例如 @ContentChildren('cmp1,cmp2')）

read 支持以下值：

  • 任何带有 @Component 或 @Directive 装饰器的类
  • 在此查询的 selector 匹配的组件注入器上定义的任何提供程序
  • 通过字符串标记定义的任何提供程序（例如 {provide: 'token', useValue: 'val'}）
  • TemplateRef、 ElementRef 和 ViewContainerRef
