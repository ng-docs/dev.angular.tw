interface ControlValueAccessor {
}

定义一个接口，该接口充当 Angular 表单 API 和 DOM 中的原生元素之间的桥梁。

实现此接口以创建与 Angular 表单集成的自定义表单控件指令。

writeValue

void

将新值写入元素。

当请求从模型到视图的编程更改时，表单 API 会调用此方法以写入视图。

@paramobjany

元素的新值

@returnsvoid
Usage notes

向元素写入值

以下示例将一个值写入原生 DOM 元素。

          
            writeValue(value: any): void {  this._renderer.setProperty(this._elementRef.nativeElement, 'value', value);}

registerOnChange

void

注册一个回调函数，该控件的值在 UI 中更改时将调用该回调函数。

当值从视图传播到模型时，表单 API 会在初始化时调用此方法以更新表单模型。

在你自己的值访问器中实现 registerOnChange 方法时，请保存给定的函数，以便你的类在适当的时机调用它。

@paramfnany

要注册的回调函数

@returnsvoid
Usage notes

存储变更函数

以下示例将所提供的函数存储为内部方法。

          
            registerOnChange(fn: (_: any) => void): void {  this._onChange = fn;}

当用户界面中的值更改时，请调用已注册的函数以允许表单 API 自行更新：

          
            host: {   '(change)': '_onChange($event.target.value)'}

registerOnTouched

void

注册一个在初始化时由表单 API 调用的回调函数，以在失焦时更新表单模型。

在你自己的值访问器中实现 registerOnTouched ，请保存给定函数，以便你的类在应将控件视为失焦或“已接触过”时调用它。

@paramfnany

要注册的回调函数

@returnsvoid
Usage notes

存储回调函数

以下示例将所提供的函数存储为内部方法。

          
            registerOnTouched(fn: any): void {  this._onTouched = fn;}

在 blur（或等效事件）时，你的类应调用已注册的函数以允许表单 API 自行更新：

          
            host: {   '(blur)': '_onTouched()'}

setDisabledState

void

当控件状态更改为 “DISABLED” 或从 “DISABLED” 更改时，表单 API 要调用的函数。根据其状态，它会启用或禁用适当的 DOM 元素。

@paramisDisabledboolean

要在元素上设置的禁用状态

@returnsvoid
Usage notes

以下是将 disabled 属性写入原生 DOM 元素的示例：

          
            setDisabledState(isDisabled: boolean): void {  this._renderer.setProperty(this._elementRef.nativeElement, 'disabled', isDisabled);}
