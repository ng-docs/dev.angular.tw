writeValue

void

向元素寫入新值。

當請求從模型到檢視的程式設計式更改時，表單 API 會呼叫此方法以寫入檢視。

@paramobjany

元素的新值

@returnsvoid
用法說明

向元素寫入一個值

以下範例向原生 DOM 元素寫入一個值。

          
writeValue(value: any): void {  this._renderer.setProperty(this._elementRef.nativeElement, 'value', value);}

registerOnChange

void

註冊一個回呼函式，當控制元件的值在 UI 中更改時，將呼叫該函式。

當值從檢視傳播到模型時，表單 API 會在初始化時調用此方法以更新表單模型。

在你自己的值訪問器中實現 registerOnChange 方法時，請儲存給定的函式，以便你的類在適當的時候呼叫它。

@paramfnany

要註冊的回呼函式

@returnsvoid
用法說明

儲存更改函式

以下範例將提供的函式儲存為內部方法。

          
registerOnChange(fn: (_: any) => void): void {  this._onChange = fn;}

當 UI 中的值更改時，呼叫已註冊的函式，以允許表單 API 自行更新：

          
host: {   '(change)': '_onChange($event.target.value)'}

registerOnTouched

void

註冊一個回呼函式，當在 blur 時更新表單模型時，表單 API 會在初始化時調用該函式。

在你自己的值訪問器中實現 registerOnTouched 時，請儲存給定的函式，以便在應將控制元件視為已失焦或『已觸碰』時，你的類可以呼叫它。

@paramfnany

要註冊的回呼函式

@returnsvoid
用法說明

儲存回呼函式

以下範例將提供的函式儲存為內部方法。

          
registerOnTouched(fn: any): void {  this._onTouched = fn;}

在 blur（或等效操作）時，你的類應呼叫已註冊的函式，以允許表單 API 自行更新：

          
host: {   '(blur)': '_onTouched()'}

setDisabledState

void

當控制元件狀態更改為『DISABLED』或從『DISABLED』更改時，表單 API 呼叫的函式。根據狀態，它會啟用或停用相應的 DOM 元素。

@paramisDisabledboolean

要在元素上設定的停用狀態

@returnsvoid
用法說明

以下是如何將 disabled 屬性寫入原生 DOM 元素的範例：

          
setDisabledState(isDisabled: boolean): void {  this._renderer.setProperty(this._elementRef.nativeElement, 'disabled', isDisabled);}

描述

定義了一個介面，它充當 Angular 表單 API 和 DOM 中的原生元素之間的橋樑。

實現此介面以建立一個與 Angular 表單整合的自定義表單控制元件指令。
