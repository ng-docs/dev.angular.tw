从 领域模型创建一个 FormControl实例，并将其绑定到表单控件元素。

FormControl实例跟踪控件的值、用户交互和验证状态，并保持视图与模型同步。如果在父表单中使用，此指令还会作为子控件向表单注册。

此指令可以单独使用，也可以作为较大表单的一部分。使用 ngModel选择器激活它。

它接受领域模型作为可选的 Input。如果你使用 []语法对 ngModel进行单向绑定，更改组件类中的领域模型的值会设置视图中的值。如果你使用 [()]语法（也称为“香蕉盒语法”）进行双向绑定，UI中的值始终会同步回类中的领域模型。

要检查关联的 FormControl（如验证状态）的属性，使用 ngModel作为键将指令导出到本地模板变量中（例如： #myVar="ngModel"）。然后可以使用指令的 control属性访问控件。然而，大多数常用的属性（如 valid和 dirty）也存在于控件上，可直接访问。查看 AbstractControlDirective中直接可用的属性的完整列表。

在 <form>标签内使用 ngModel时，你还需要提供 name属性，以便将控件注册到父表单中。

在父表单的上下文中，通常不需要包括单向或双向绑定，因为父表单会为你同步值。通过使用 ngForm将其导出到本地模板变量中（如： #f="ngForm"）访问其属性。在表单提交时需要使用该变量。

如果你确实需要将初始值填充到表单中，使用单向绑定 ngModel往往是足够的，只要你在提交时使用导出的表单值而不是领域模型的值。

通过选项设置 ngModel name 属性

以下示例显示了设置 name 属性的另一种方式。这里，在自定义表单控件组件中使用了名为 name 的属性。为了仍然能够指定 NgModel 的 name，必须使用 ngModelOptions 输入来指定它。

          
            <form>  <my-custom-form-control name="Nancy" ngModel [ngModelOptions]="{name: 'user'}">  </my-custom-form-control></form><!-- form value: {user: ''} -->

Usage notes

例如，对于以下 FormGroup：

          
            form = new FormGroup({  address: new FormGroup({ street: new FormControl() })});

从根表单到 'street' 控件的路径是 'address' -> 'street'。

调用此方法有两种形式：

  1. 控件名称的字符串数组，如 ['address', 'street']
  2. 以一个字符串表示的句号分隔的控件名称列表，如 'address.street'

如果没有给出路径，则此方法检查当前控件上的错误。
