標識觸發導向的呼叫或事件。 imperative 觸發器是對 router.navigateByUrl() 或 router.navigate() 的呼叫。

restoredState

{ [k: string]: any; navigationId: number; } | null | undefined

先前提供給 pushState 呼叫的導向狀態， 當導向由 popstate 事件觸發時。否則為 null。

狀態物件由 NavigationExtras 定義，並且包含任何 開發者定義的狀態值，以及路由器為每次路由器轉場/導向分配的唯一 ID。

從路由器的角度來看，路由器永遠不會『後退』。 當用戶單擊瀏覽器中的後退按鈕時， 會建立一個新的導向 ID。

使用此先前狀態物件中的 ID 來區分新創建的狀態和由 popstate 事件回傳的狀態，以便您可以恢復某些 記住的狀態，例如捲動位置。
