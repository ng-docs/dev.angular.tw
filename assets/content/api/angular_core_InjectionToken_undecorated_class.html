描述

建立一個可在依賴注入提供者中使用的令牌。

每當你要注入的型別未被具體化（沒有執行時表示）時，例如當注入介面、可調用型別、陣列或引數化型別時，請使用 InjectionToken。

InjectionToken 以 T 為引數，T 是將由 Injector 回傳的物件的型別。這提供了額外的型別安全級別。

重要提示：請確保在提供者和注入呼叫中都使用 InjectionToken 的相同實例。在不同的地方建立 InjectionToken 的新實例，即使具有相同的描述，也會被 Angular 的依賴注入系統視為不同的令牌，從而導致 NullInjectorError。

          
const TOKEN = new InjectionToken<MyInterface>('SomeToken');// Setting up the provider using the same token instanceconst providers = [  {provide: TOKEN, useValue: {someProperty: 'exampleValue'}}, // Mock value for MyInterface];// Creating the injector with the providerconst injector = Injector.create({providers});// Retrieving the value using the same token instanceconst myInterface = injector.get(TOKEN);// myInterface is inferred to be MyInterface.

在建立 InjectionToken 時，你可以選擇性地指定一個工廠函式，該函式回傳（可能透過建立）引數化型別 T 的預設值。這將使用此工廠作為提供者來建立 InjectionToken，就像它在應用的根注入器中顯式定義一樣。如果接受零引數的工廠函式需要注入依賴項，則可以使用 inject 函式來完成。正如你在下面的搖樹最佳化 InjectionToken 範例中看到的那樣。

此外，如果指定了 factory，你還可以指定 providedIn 選項，該選項會覆蓋上述行為，並將令牌標記為屬於特定的 @NgModule（注意：此選項現已棄用）。如上所述，'root' 是 providedIn 的預設值。

providedIn: NgModule 和 providedIn: 'any' 選項已棄用。

使用須知

基本範例

普通 InjectionToken 

          
const BASE_URL = new InjectionToken<string>('BaseUrl');      const injector = Injector.create({        providers: [{provide: BASE_URL, useValue: 'http://localhost'}],      });      const url = injector.get(BASE_URL);      // Note: since `BASE_URL` is `InjectionToken<string>`      // `url` is correctly inferred to be `string`      expect(url).toBe('http://localhost');

可搖樹最佳化的 InjectionToken 

          
class MyService {        constructor(readonly myDep: MyDep) {}      }      const MY_SERVICE_TOKEN = new InjectionToken<MyService>('Manually constructed MyService', {        providedIn: 'root',        factory: () => new MyService(inject(MyDep)),      });      const instance = injector.get(MY_SERVICE_TOKEN);      expect(instance instanceof MyService).toBeTruthy();      expect(instance.myDep instanceof MyDep).toBeTruthy();
