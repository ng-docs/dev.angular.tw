控件指令的基类。

abstract class AbstractControlDirective {
}

控件指令的基类。

此类仅在 ReactiveFormsModule 和 FormsModule 内部使用。

control

AbstractControl<any, any>

对基础控件的引用。

value

any

报告控件的值（如果存在），否则为 null。

valid

boolean

报告控件是否有效。如果当前值不存在验证错误，则控件被视为有效。如果控件不存在，则返回 null。

invalid

boolean

报告控件是否无效，表示输入值中存在错误。如果控件不存在，则返回 null。

pending

boolean

报告控件是否处于挂起状态，这意味着正在进行异步验证，错误尚未对输入值可用。如果控件不存在，则返回 null。

disabled

boolean

报告该控件是否被禁用，这意味着该控件在 UI 中被禁用，并且免于进行验证检查，并被排除在祖先控件的聚合值之外。如果控件不存在，则返回 null。

enabled

boolean

报告控件是否被启用，这意味着控件已包含在祖先的有效性计算或值计算中。如果控件不存在，则返回 null。

errors

ValidationErrors

报告控件的验证错误。如果控件不存在，则返回 null。

pristine

boolean

报告控件是否为原始状态，原始的意思是用户尚未更改过 UI 中的值。如果控件不存在，则返回 null。

dirty

boolean

报告控件是否脏状态，脏的意思是用户已更改过 UI 中的值。如果控件不存在，则返回 null。

touched

boolean

报告控件是否被接触过，被接触过的意思是用户已在控件上触发过 blur 事件。如果控件不存在，则返回 null。

status

string

报告控件的验证状态。可能的值包括：“VALID”、“INVALID”、“DISABLED” 和 “PENDING”。如果控件不存在，则返回 null。

untouched

boolean

报告控件是否未被接触过，未被接触过的意思是用户尚未在其上触发过 blur 事件。如果控件不存在，则返回 null。

statusChanges

Observable<any>

返回一个多播的可观察者，它会发出为此控件计算过的验证状态。如果控件不存在，则返回 null。

valueChanges

Observable<any>

返回控件值变更的多播可观察者，它会在每次控件的值在 UI 中或以编程方式更改时触发。如果控件不存在，则返回 null。

path

string[]

返回一个数组，该数组表示从顶级表单到此控件的路径。每个索引是该级别上控件的字符串名称。

validator

ValidatorFn

同步验证器函数，由使用此指令注册的所有同步验证器组合而成。

asyncValidator

AsyncValidatorFn

异步验证器函数，由使用此指令注册的所有异步验证器组合而成。

reset

void

如果此控件存在，则使用所提供的值重置它。

hasError

boolean

报告给定路径下的控件是否具有指定的错误。

@paramerrorCodestring

要检查的错误代码

@parampathstring | (string | number)[]

控件名称列表，用于指定如何从当前控件移至要查询错误的控件。

@returnsboolean
Usage notes

例如，对于以下 FormGroup：

          
            form = new FormGroup({  address: new FormGroup({ street: new FormControl() })});

从根表单到 'street' 控件的路径是 'address' -> 'street'。

调用此方法有两种形式：

  1. 控件名称的字符串数组，如 ['address', 'street']
  2. 以一个字符串表示的句号分隔的控件名称列表，如 'address.street'

如果没有给出路径，则此方法检查当前控件上的错误。
