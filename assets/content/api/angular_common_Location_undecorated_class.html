描述

應用可以使用此服務與瀏覽器的 URL 進行互動。

根據所用的 LocationStrategy，Location 會持久化到 URL 的路徑或 URL 的雜湊片段。

用法說明

最好使用 Router.navigate() 服務來觸發路由更改。只有當需要在路由外部與規範化的 URL 互動或建立規範化的 URL 時，才使用 Location。

Location 負責根據應用的 base href 規範化 URL。 規範化的 URL 是相對於 URL 宿主絕對的，包含應用的 base href，並且沒有尾部斜槓：

  • /my/app/user/123 已規範化
  • my/app/user/123 未 規範化
  • /my/app/user/123/ 未 規範化
