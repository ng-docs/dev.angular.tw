ViewRef

類別
程式碼

表示一個 Angular 檢視。

變更檢測用法

API

  
    abstract class ViewRef extends ChangeDetectorRef {}

destroy

void

銷燬此檢視以及所有與之關聯的資料結構。

@returnsvoid

destroyed

boolean

報告此檢視是否已被銷燬。

onDestroy

void

一個生命週期鉤子，為檢視提供額外的開發者定義清理功能。

@paramcallbackFunction

一個處理函式，用於清理與檢視關聯的開發者定義資料。當呼叫 destroy() 方法時調用。

@returnsvoid

markForCheck

void

當檢視使用 ChangeDetectionStrategy#OnPush (checkOnce) 變更檢測策略時，顯式地將該檢視標記為已更改，以便可以再次檢查它。

當輸入屬性已更改或檢視中已觸發事件時，元件通常會被標記為髒（需要重新渲染）。呼叫此方法以確保即使這些觸發器未發生，也會檢查元件。

@returnsvoid

detach

void

將此檢視從變更檢測樹中分離。 分離的檢視在重新附著之前不會被檢查。 與 detectChanges() 結合使用以實現本地變更檢測檢查。

分離的檢視在變更檢測執行期間不會被檢查，直到它們被重新附著，即使它們被標記為髒也是如此。

@returnsvoid

detectChanges

void

檢查此檢視及其子檢視。與 ChangeDetectorRef#detach 結合使用以實現本地變更檢測檢查。

@returnsvoid

checkNoChanges

void

檢查變更檢測器及其子檢測器，如果檢測到任何更改，則丟擲錯誤。

在開發模式下使用，以驗證執行變更檢測不會引入其他更改。在生產模式下調用它不起作用。

@deprecated

這是一個僅用於測試的 API，在生產介面中沒有位置。當應用在開發模式下執行時，checkNoChanges 已經是 ApplicationRef 滴答的一部分。對於更精細的 checkNoChanges 驗證，請使用 ComponentFixture。

@returnsvoid

reattach

void

將先前分離的檢視重新附著到變更檢測樹。預設情況下，檢視會附著到樹上。

@returnsvoid
