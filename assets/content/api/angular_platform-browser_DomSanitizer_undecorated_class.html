DomSanitizer

類別
程式碼

DomSanitizer 透過對值進行無害化處理，使其在不同的 DOM 上下文中安全使用，從而幫助防止跨站指令碼安全漏洞 (XSS)。

API

  
    abstract class DomSanitizer implements Sanitizer {}

sanitize

string | null

從已知安全的值或安全性未知的值中獲取安全值。

如果給定值已經是 SafeValue，則此方法回傳解開後的值。 如果安全上下文是 HTML 並且給定值是純字串，則此方法會對字串進行無害化處理，刪除任何潛在的不安全內容。 對於任何其他安全上下文，如果提供純字串，此方法將丟擲錯誤。

@paramcontextSecurityContext
@paramvaluestring | SafeValue | null
@returnsstring | null

bypassSecurityTrustHtml

SafeHtml

繞過安全性並信任給定的值是安全的 HTML。僅當繫結的 HTML 不安全（例如，包含 <script> 標記）並且程式碼應該執行時才使用此方法。無害化處理程式將保持安全 HTML 完好無損，因此在大多數情況下不應使用此方法。

警告： 使用不受信任的使用者資料呼叫此方法會將你的應用暴露於 XSS 安全風險！

@paramvaluestring
@returnsSafeHtml

bypassSecurityTrustStyle

SafeStyle

警告： 使用不受信任的使用者資料呼叫此方法會將你的應用暴露於 XSS 安全風險！

@paramvaluestring
@returnsSafeStyle

bypassSecurityTrustScript

SafeScript

繞過安全性並信任給定的值是安全的 JavaScript。

警告： 使用不受信任的使用者資料呼叫此方法會將你的應用暴露於 XSS 安全風險！

@paramvaluestring

SafeUrl

繞過安全性並信任給定的值是安全的樣式 URL，即可在超連結或 <img src> 中使用的值。

警告： 使用不受信任的使用者資料呼叫此方法會將你的應用暴露於 XSS 安全風險！

@paramvaluestring
@returnsSafeUrl

SafeResourceUrl

繞過安全性並信任給定的值是安全的資源 URL，即可用於從 <script src> 或 <iframe src> 等位置載入可執行程式碼的位置。

警告： 使用不受信任的使用者資料呼叫此方法會將你的應用暴露於 XSS 安全風險！

@paramvaluestring
@returnsSafeResourceUrl

說明

DomSanitizer 透過對值進行無害化處理，使其在不同的 DOM 上下文中安全使用，從而幫助防止跨站指令碼安全漏洞 (XSS)。

例如，當在 <a [href]="someValue"> 超連結中繫結 URL 時，將對 someValue 進行無害化處理，以使攻擊者無法注入例如 javascript: URL，從而在網站上執行程式碼。

在特定情況下，可能有必要停用無害化處理，例如，如果應用確實需要生成帶有動態值的 javascript: 樣式連結。 使用者可以透過使用 bypassSecurityTrust... 方法之一構造一個值來繞過安全性，然後從範本繫結到該值。

這些情況應該非常少見，並且必須格外小心，以避免建立跨站指令碼 (XSS) 安全漏洞！

當使用 bypassSecurityTrust... 時，請確保儘早呼叫該方法，並儘可能靠近值的來源，以便輕鬆驗證其使用不會產生安全漏洞。

如果值是安全的，則不需要（也不建議）繞過安全性，例如，不以可疑協定開頭的 URL，或者不包含危險程式碼的 HTML 片段。無害化處理程式會保持安全值完好無損。
