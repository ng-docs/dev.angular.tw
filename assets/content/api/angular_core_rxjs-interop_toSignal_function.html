說明

獲取 Observable 的當前值，作為回應式的 Signal。

toSignal 回傳一個 Signal，它透過訂閱給定的 Observable，來提供對該 Observable 所產生值的同步回應式訪問。回傳的 Signal 將始終具有訂閱發出的最新值，並且如果 Observable 發生錯誤，則會丟擲錯誤。

當 requireSync 設定為 true 時，toSignal 將斷言 Observable 在訂閱後立即生成一個值。在這種情況下，不需要 initialValue，並且回傳的 signal 不包含 undefined 型別。

預設情況下，當當前的注入上下文被銷燬時，訂閱將自動清理。例如，當在元件的建構式函式中呼叫 toSignal 時，訂閱將在元件銷燬時清理。如果注入上下文不可用，則可以改為傳入顯式的 Injector。

如果訂閱應持續到 Observable 本身完成，則可以改為指定 manualCleanup 選項，這將停用自動訂閱拆除。在這種設定中，也不需要注入上下文。
