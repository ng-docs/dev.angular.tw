@angular/platform-browser

HammerGestureConfig

類別
code

一個可注入的 HammerJS Manager，用於手勢識別。設定特定的事件識別。

API

  
    class HammerGestureConfig {}

events

string[]

Angular 中用於手勢的一組受支援的事件名稱。Angular 支援 HammerJS 文件 中列出的所有內建識別器。

overrides

{ [key: string]: Object; }

將手勢事件名稱對映到一組設定選項，這些選項指定對特定屬性的預設值的重寫。

鍵是要設定的受支援的事件名稱，選項物件包含一組屬性，其中包含要應用於命名識別器事件的重寫值。例如，要停用 rotate 事件的識別，請指定 {"rotate": {"enable": false}}。

不存在的屬性採用 HammerJS 預設值。有關哪些屬性支援哪些事件，以及它們的允許值和預設值的訊息，請參閱 HammerJS 文件。

options

{ cssProps?: any; domEvents?: boolean | undefined; enable?: boolean | ((manager: any) => boolean) | undefined; preset?: any[] | undefined; touchAction?: string | undefined; recognizers?: any[] | undefined; inputClass?: any; inputTarget?: EventTarget | undefined; } | undefined

可以為給定事件重寫其預設值的屬性。不同的屬性集適用於不同的事件。有關哪些屬性支援哪些事件，以及它們的允許值和預設值的訊息，請參閱 HammerJS 文件。

HammerInstance

建立一個 HammerJS Manager 並將其附著到給定的 HTML 元素。

@paramelementHTMLElement

將識別手勢的元素。
