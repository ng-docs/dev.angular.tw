如果进行运行时翻译，加载供 $localize 使用的翻译。

如果 $localize 标记的字符串不会在编译时被替换，可以加载一组翻译，这些翻译将在运行时（在浏览器中）应用于 $localize 标记的字符串。

如果新翻译具有相同的 MessageId，加载新翻译将覆盖以前的翻译。

请注意， $localize 消息在第一次遇到标记字符串时只会处理一次，并且不提供不刷新浏览器的动态语言切换。在应用生命周期的后期加载新翻译不会更改已经翻译的消息文本。

消息 ID 和翻译的格式与提取消息时渲染为“简单 JSON”翻译文件的格式相同。特别是，消息中的占位符是使用 {$PLACEHOLDER_NAME} 语法渲染的。例如来自以下模板的消息：

          
            <div i18n>pre<span>inner-pre<b>bold</b>inner-post</span>post</div>

在 translations 映射表中将有以下形式：

          
            {  "2932901491976224757":     "pre{$START_TAG_SPAN}inner-pre{$START_BOLD_TEXT}bold{$CLOSE_BOLD_TEXT}inner-post{$CLOSE_TAG_SPAN}post"}
