withComponentInputBinding

ComponentInputBindingFeature

允許將來自 Router 狀態的訊息直接繫結到 Route 設定中元件的輸入屬性（Property）。

@returnsComponentInputBindingFeature

用法說明

如何啟用此屬性的基本範例：

          
const appRoutes: Routes = [];bootstrapApplication(AppComponent,  {    providers: [      provideRouter(appRoutes, withComponentInputBinding())    ]  });

來自以下任何來源的路由繫結訊息：

  • 查詢引數
  • 路徑和矩陣引數
  • 靜態路由資料
  • 來自求解器的資料

重複的鍵會按照從上到下的相同順序求解，從最低到最高，這意味著求解器具有最高的優先級，並將覆蓋來自路由的任何其他訊息。

重要的是，當輸入屬性（Property）在路由資料中沒有具有對應鍵的條目時，此輸入屬性（Property）將被設定為 undefined。這可以防止在資料從路由中刪除後（例如，如果刪除了查詢引數）保留先前的訊息。 可以使用路由上的求解器來提供預設值，以確保該值始終存在，或者在元件中使用輸入屬性（Property）和輸入轉換。
