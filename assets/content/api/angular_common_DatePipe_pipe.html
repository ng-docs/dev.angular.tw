根据区域设置规则格式化日期值。

DatePipe 仅在检测到输入值的纯变化时执行。纯变化是指对原语输入值（如 String、 Number、 Boolean 或 Symbol）的变化，或对象引用的变化（如 Date、 Array、 Function 或 Object）。

请注意，更改 Date 对象不会导致管道再次渲染。为确保管道被执行，你必须创建一个新的 Date 对象。

Angular 仅带有 en-US 语言环境数据。要在另一种语言中本地化日期，你必须导入相应的语言环境数据。有关更多信息，请参阅 I18n 指南。

格式化值的时区可以通过两种方式指定：作为管道的第二个参数传入，或者通过设置 DATE_PIPE_DEFAULT_OPTIONS 依赖注入令牌来指定默认值。传入作为第二个参数的值优先于使用依赖注入令牌定义的值。

当输入值发生变化时，该管道的结果并不会改变。如果不想在每个变更检测周期中都强制重新格式化该日期，请把日期看做一个不可变对象， 当需要让该管道重新运行时，请赋给它一个新的对象，以更改它的引用。

格式范例

下面这些例子会把日期转换成多种格式。 这里假设 dateObj 是个 JavaScript 的 Date 对象：2015 年 6 月 15 日 21 时 43 分 11 秒， 使用的是 en-US 区域的当地时间。

          
            {{ dateObj | date }}               // output is 'Jun 15, 2015'{{ dateObj | date:'medium' }}      // output is 'Jun 15, 2015, 9:43:11 PM'{{ dateObj | date:'shortTime' }}   // output is '9:43 PM'{{ dateObj | date:'mm:ss' }}       // output is '43:11'{{ dateObj | date:"MMM dd, yyyy 'at' hh:mm a" }}  // output is 'Jun 15, 2015 at 09:43 PM'

使用范例

下列组件借助一个日期管道来以不同的格式显示当前日期。

          
            @Component({ selector: 'date-pipe', template: `<div>   <p>Today is {{today | date}}</p>   <p>Or if you prefer, {{today | date:'fullDate'}}</p>   <p>The time is {{today | date:'h:mm a z'}}</p> </div>`})// Get the current date and time as a date-time value.export class DatePipeComponent {  today: number = Date.now();}

@paramvaluestring | number | Date

日期表达式：一个 Date 对象，一个数字（自 UTC 纪元以来的毫秒数），或一个 ISO 字符串（ https://www.w3.org/TR/NOTE-datetime）。

@paramformatstring

要包含的日期/时间组件，使用预定义选项或自定义格式字符串。如果未提供， DatePipe 会使用 DATE_PIPE_DEFAULT_OPTIONS 注入令牌查找值（并读取 dateFormat 属性）。如果未配置该令牌，则使用 mediumDate 作为值。

@paramtimezonestring

一个时区偏移量（例如 '+0430'），或标准的 UTC/GMT，或美国大陆时区缩写。如果未提供， DatePipe 会使用 DATE_PIPE_DEFAULT_OPTIONS 注入令牌查找值（并读取 timezone 属性）。如果未配置该令牌，则使用终端用户的本地系统时区作为值。

@paramlocalestring

用于语言环境格式规则的语言环境代码。如果未提供，使用 LOCALE_ID 的值，默认是 en-US。参见 设置你的应用语言环境。
