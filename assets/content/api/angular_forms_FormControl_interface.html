@angular/forms

FormControl

interface
程式碼

跟蹤單個表單控制元件的值和驗證狀態。

AbstractControl回應式表單指南用法說明

API

  
    interface FormControl<TValue = any> extends AbstractControl<TValue> {
}

defaultValue

TValue

每當控制元件在沒有顯式值的情況下重置時，將使用此 FormControl 的預設值。有關設定預設值的更多訊息，請參閱FormControlOptions#nonNullable。

setValue

void

為表單控制元件設定一個新值。

@paramvalueTValue

控制元件的新值。

@paramoptions{ onlySelf?: boolean | undefined; emitEvent?: boolean | undefined; emitModelToViewChange?: boolean | undefined; emitViewToModelChange?: boolean | undefined; } | undefined

設定選項，用於確定當值更改時控制元件如何傳播更改和發出事件。 設定選項將傳遞給 * updateValueAndValidity 方法。

  • onlySelf：為 true 時，每次更改僅影響此控制元件，而不影響其父控制元件。預設為 false。
  • emitEvent：當為 true 或未提供（預設值）時，當控制元件值更新時，statusChanges 和 valueChanges 可觀察物件都會發出包含最新狀態和值的事件。 當為 false 時，不發出任何事件。
  • emitModelToViewChange：當為 true 或未提供（預設值）時，每次更改都會觸發 onChange 事件以更新檢視。
  • emitViewToModelChange：當為 true 或未提供（預設值）時，每次更改都會觸發 ngModelChange 事件以更新模型。
@returnsvoid

reset

void

重置表單控制元件，將其標記為 pristine 和 untouched，並重置該值。新值將是所提供的值（如果已傳遞）、null 或初始值（如果在建構式函式中透過 FormControlOptions 設定了 nonNullable）。

          
// By default, the control will reset to null.const dog = new FormControl('spot');dog.reset(); // dog.value is null// If this flag is set, the control will instead reset to the initial value.const cat = new FormControl('tabby', {nonNullable: true});cat.reset(); // cat.value is "tabby"// A value passed to reset always takes precedence.const fish = new FormControl('finn', {nonNullable: true});fish.reset('bubble'); // fish.value is "bubble"
@paramformStateTValue | FormControlState<TValue> | undefined

使用初始值或定義初始值和停用狀態的物件來重置控制元件。

@paramoptions{ onlySelf?: boolean | undefined; emitEvent?: boolean | undefined; } | undefined

設定選項，用於確定控制元件在值更改後如何傳播更改和發出事件。

status

FormControlStatus

控制元件的驗證狀態。

valid

boolean

當控制元件的 status 為 VALID 時，該控制元件為 valid。

invalid

boolean

當控制元件的 status 為 INVALID 時，該控制元件為 invalid。

pending

boolean

當控制元件的 status 為 PENDING 時，該控制元件為 pending。

disabled

boolean

當控制元件的 status 為 DISABLED 時，該控制元件為 disabled。

停用的控制元件無需進行驗證檢查，並且不包含在其祖先控制元件的聚合值中。

enabled

boolean

只要控制元件的 status 不是 DISABLED，該控制元件就為 enabled。

errors

ValidationErrors | null

一個物件，其中包含因驗證失敗而生成的任何錯誤；如果沒有錯誤，則為 null。

pristine

boolean

如果使用者尚未更改 UI 中的值，則控制元件為 pristine。

dirty

boolean

如果使用者已更改 UI 中的值，則控制元件為 dirty。

touched

boolean

如果控制元件標記為 touched，則為 True。

一旦使用者在其上觸發了 blur 事件，控制元件就會被標記為 touched。

untouched

boolean

如果控制元件尚未標記為 touched，則為 True

如果使用者尚未在其上觸發 blur 事件，則控制元件為 untouched。

events

Observable<ControlEvent<TValue>>

一個多播可觀察物件，每次控制元件狀態更改時都會發出事件。 它會針對值、狀態、pristine 或 touched 更改發出。

注意：在值更改時，發出操作在此控制元件的值更新後立即發生。父控制元件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍後更新，因此從此事件的回呼中訪問父控制元件的值（使用 value 屬性）可能會導致獲取尚未更新的值。請改為訂閱父控制元件的 events。 對於其他事件型別，事件在父控制元件更新後發出。

valueChanges

Observable<TValue>

這是一個多播可觀察物件，每當控制元件的值在 UI 中或以程式設計方式更改時，它都會發出一個事件。 每次你呼叫 enable() 或 disable() 而不傳遞 {emitEvent: false} 作為函式引數時，它也會發出一個事件。

注意：發出事件發生在控制元件的值更新之後。父控制元件的值（例如，如果此 FormControl 是 FormGroup 的一部分）會稍後更新，因此從此事件的回呼中訪問父控制元件的值（使用 value 屬性）可能會導致獲得尚未更新的值。請改為訂閱父控制元件的 valueChanges 事件。

statusChanges

Observable<FormControlStatus>

這是一個多播可觀察物件，每當控制元件的驗證 status 重新計算時，它都會發出一個事件。

updateOn

FormHooks

報告 AbstractControl 的更新策略（意味著控制元件自身更新所依據的事件）。 可能的值：'change' | 'blur' | 'submit' 預設值：'change'

描述

跟蹤單個表單控制元件的值和驗證狀態。

這是 Angular 表單的四個基本元素之一，另外三個是 FormGroup、FormArray 和 FormRecord。它擴充套件了 AbstractControl 類別，該類別實現了用於訪問值、驗證狀態、使用者互動和事件的大多數基本功能。

FormControl 接受單個泛型引數，該引數描述其值的型別。此引數始終隱式包含 null，因為可以重置控制元件。要更改此行為，請設定 nonNullable 或檢視下面的用法說明。

請參閱下面的用法範例。

用法說明

初始化表單控制元件

實例化一個帶有初始值的 FormControl。

          
const control = new FormControl('some value');console.log(control.value);     // 'some value'

以下範例使用表單狀態物件初始化控制元件。在這種情況下，value 和 disabled 鍵是必要的。

          
const control = new FormControl({ value: 'n/a', disabled: true });console.log(control.value);     // 'n/a'console.log(control.status);    // 'DISABLED'

以下範例使用同步驗證器初始化控制元件。

          
const control = new FormControl('', Validators.required);console.log(control.value);      // ''console.log(control.status);     // 'INVALID'

以下範例使用選項物件初始化控制元件。

          
const control = new FormControl('', {   validators: Validators.required,   asyncValidators: myAsyncValidator});

單一型別引數

FormControl 接受一個泛型引數，該引數描述其值的型別。在大多數情況下，將推斷此引數。

如果要將控制元件初始化為 null，或者希望提供更廣泛的型別，則可以顯式指定引數：

          
let fc = new FormControl<string|null>(null);fc.setValue('foo');

您可能會注意到 null 始終新增到控制元件的型別中。這是因為如果您呼叫 reset，控制元件將變為 null。您可以透過設定 {nonNullable: true} 來更改此行為。

設定控制元件以在失焦事件時更新

將 updateOn 選項設定為 'blur' 以在失焦 event 時更新。

          
const control = new FormControl('', { updateOn: 'blur' });

設定控制元件以在送出事件時更新

將 updateOn 選項設定為 'submit' 以在送出 event 時更新。

          
const control = new FormControl('', { updateOn: 'submit' });

將控制元件重置為特定值

您可以透過傳遞獨立值或包含值和停用狀態的表單狀態物件來重置為特定的表單狀態（這些是僅有的兩個無法計算的屬性）。

          
const control = new FormControl('Nancy');console.log(control.value); // 'Nancy'control.reset('Drew');console.log(control.value); // 'Drew'

將控制元件重置為其初始值

如果您希望始終將控制元件重置為其初始值（而不是 null），則可以傳遞 nonNullable 選項：

          
const control = new FormControl('Nancy', {nonNullable: true});console.log(control.value); // 'Nancy'control.reset();console.log(control.value); // 'Nancy'

將控制元件重置為初始值和停用狀態 

          
const control = new FormControl('Nancy');console.log(control.value); // 'Nancy'console.log(control.status); // 'VALID'control.reset({ value: 'Drew', disabled: true });console.log(control.value); // 'Drew'console.log(control.status); // 'DISABLED'
