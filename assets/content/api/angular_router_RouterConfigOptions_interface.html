@angular/router

RouterConfigOptions

interface
程式碼

可與 withRouterConfig 函式一起使用的額外設定選項。

API

  
    interface RouterConfigOptions {}

canceledNavigationResolution

"replace" | "computed" | undefined

設定當導向取消時，路由器嘗試恢復狀態的方式。

'replace' - 始終使用 location.replaceState 將瀏覽器狀態設定為導向開始之前路由器的狀態。這意味著，如果在導向之前更新了瀏覽器的 URL，路由器將只替換歷史記錄中的條目，而不是嘗試恢復到會話歷史記錄中的先前位置。這種情況最常發生在 urlUpdateStrategy: 'eager' 和使用瀏覽器後退/前進按鈕的導向中。

'computed' - 將嘗試回傳到會話歷史記錄中與導向取消時 Angular 路由對應的相同索引。例如，如果單擊瀏覽器後退按鈕並且導向被取消，路由器將觸發前進導向，反之亦然。

注意：'computed' 選項與任何僅處理部分 URL 的 UrlHandlingStrategy 都不相容，因為歷史記錄恢復會導向到瀏覽器歷史記錄中的先前位置，而不是簡單地重置部分 URL。

如果未設定，則預設值為 replace。

onSameUrlNavigation

OnSameUrlNavigation | undefined

設定處理到當前 URL 的導向請求的預設行為。

如果未設定，則 Router 將使用 'ignore'。

paramsInheritanceStrategy

"emptyOnly" | "always" | undefined

定義路由器如何將來自父路由的引數、資料和已求解資料合併到子路由。

預設情況下（'emptyOnly'），當路由本身具有空路徑（意味著其設定了 path: ''）或當父路由沒有任何元件集時，路由會繼承父路由的引數。

設定為 'always' 以啟用父引數的無條件繼承。

請注意，當處理矩陣引數時，『父級』指的是父級 Route 設定，這不一定意味著『左側的 URL 片段』。當 Route path 包含多個片段時，矩陣引數必須出現在最後一個片段上。例如，{path: 'a/b', component: MyComp} 的矩陣引數應顯示為 a/b;foo=bar 而不是 a;foo=bar/b。

urlUpdateStrategy

"deferred" | "eager" | undefined

定義路由器何時更新瀏覽器 URL。預設情況下（'deferred'），在成功導向後更新。 如果希望在導向開始時更新 URL，請設定為 'eager'。 提前更新 URL 允許您透過顯示包含失敗 URL 的錯誤訊息來處理導向失敗。

defaultQueryParamsHandling

QueryParamsHandling | undefined

當未提供查詢引數處理策略時，在 Router.createUrlTree 中用於處理查詢引數的預設策略。

createUrlTree 方法在內部被 Router.navigate 和 RouterLink 使用。 請注意，QueryParamsHandling 不適用於 Router.navigateByUrl。

當在呼叫 createUrlTree 時，既未指定預設策略，也未指定 queryParamsHandling 選項時，當前引數將被新引數替換。

resolveNavigationPromiseOnError

boolean | undefined

當為 true 時，Promise 將會求解為 false，就像其他失敗的導向一樣（例如，當守衛被拒絕時）。

否則，如果發生錯誤，路由器導向回傳的 Promise 將被拒絕。
