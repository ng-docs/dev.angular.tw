verify

验证没有任何未匹配的请求正在等待。

如果有任何未完成的请求，则失败并显示一条错误消息，指出未处理哪些请求。

如果未设置过 ignoreCancelled（默认），且未明确匹配已取消的请求，则 verify() 就会失败。
