nonNullable

NonNullableFormBuilder

回傳一個 FormBuilder，其中自動建立的 FormControl 元素具有 {nonNullable: true} 並且是非 nullable 的。

建立非 nullable 控制元件

當建立控制元件時，它將是非 nullable 的，並將重置為其初始值。

          
let nnfb = new FormBuilder().nonNullable;let name = nnfb.control('Alex'); // FormControl<string>name.reset();console.log(name); // 'Alex'

建立非 nullable 的組或陣列

當建立組或陣列時，所有自動建立的內部控制元件都將是非 nullable 的，並將重置為其初始值。

          
let nnfb = new FormBuilder().nonNullable;let name = nnfb.group({who: 'Alex'}); // FormGroup<{who: FormControl<string>}>name.reset();console.log(name); // {who: 'Alex'}

在組或陣列上建立可為 null 的欄位

仍然可以擁有可為 null 的欄位。 特別是，任何已經建立的 FormControl 都不會被更改。 例如：

          
let nnfb = new FormBuilder().nonNullable;// FormGroup<{who: FormControl<string|null>}>let name = nnfb.group({who: new FormControl('Alex')});name.reset(); console.log(name); // {who: null}

因為內部控制元件是由呼叫者顯式建立的，所以建構式無法控制它的建立方式，也無法排除 null。

@paramcontrols{ [key: string]: T; }

子控制元件的集合。每個子控制元件的鍵是它註冊時使用的名稱。

@paramoptionsAbstractControlOptions | null

FormRecord 的設定選項物件。該物件應具有 AbstractControlOptions 型別，並且可能包含以下欄位：

  • validators：一個同步驗證器函式，或一個驗證器函式陣列。
  • asyncValidators：一個非同步驗證器，或非同步驗證器函式陣列。
  • updateOn：控制元件應該更新的事件（選項：'change' | 'blur' | submit'）。
