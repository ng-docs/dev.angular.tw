@angular/core

NgModule

decorator
code

把一个类标记为 NgModule，并提供配置元数据。

class NgModule {
}

providers

(Provider | EnvironmentProviders)[]

在当前模块的注入器中可用的一组可注入对象。

declarations

(any[] | Type<any>)[]

属于此模块的一组组件、指令和管道（可声明对象）。

imports

(any[] | Type<any> | ModuleWithProviders<{}>)[]

此模块中模板可用的一组导出的 NgModules 的可声明对象。

exports

(any[] | Type<any>)[]

此 NgModule 中声明的一组组件、指令和管道可以在导入了本模块的模块下任何组件的模板中使用。 导出的这些可声明对象就是该模块的公共 API。

导出具有传递性。ModuleA 可以导入 ModuleB 并将其导出，这会让所有 ModuleB 中的导出同样可用在导入了 ModuleA 的那些模块中。

bootstrap

(any[] | Type<any>)[]

引导此模块时引导的一组组件。

schemas

(any[] | SchemaMetadata)[]

该 NgModule 中允许使用的声明元素的 schema（HTML 架构）。 元素和属性（无论是 Angular 组件还是指令）都必须声明在 schema 中。

允许的值是 NO_ERRORS_SCHEMA 和 CUSTOM_ELEMENTS_SCHEMA。

id

string

在 getNgModuleById 中唯一标识此 NgModule 的名称或路径。如果留空 undefined，则不会在 getNgModuleById 中注册此 NgModule。

jit

true

如果存在，则该指令/组件将被 AOT 编译器忽略。它会保留在发布代码中，并且 JIT 编译器会尝试在运行时在浏览器中对其进行编译。为了确保其行为正确，该应用程序必须导入 @angular/compiler 。
