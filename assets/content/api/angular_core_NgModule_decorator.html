@angular/core

NgModule

decorator
程式碼

用來標記某個類為 NgModule 並提供設定元資料的裝飾器。

API

  
    @NgModule ({})

providers

(Provider | EnvironmentProviders)[] | undefined

本模組的注入器中可用的可注入物件集。

declarations

(any[] | Type<any>)[] | undefined

屬於本模組的元件、指令和通道（可宣告物件）的集合。

imports

(any[] | Type<any> | ModuleWithProviders<{}>)[] | undefined

本模組中的樣板可用的那些匯出的可宣告物件的 NgModule 集合。

exports

(any[] | Type<any>)[] | undefined

本 NgModule 中宣告的元件、指令和通道的集合，它們可以在匯入此 NgModule 的任何 NgModule 所屬的元件的樣板中使用。匯出的宣告是該模組的公共 API。

一個可宣告物件只屬於一個 NgModule。一個模組可以在其匯出清單中列出另一個模組，在這種情況下，該模組的所有公共宣告都會被匯出。

bootstrap

(any[] | Type<any>)[] | undefined

當此模組啟動時要引導啟動的元件集合。

schemas

(any[] | SchemaMetadata)[] | undefined

用於宣告 NgModule 中允許使用的元素的架構集合。既不是 Angular 元件也不是指令的元素和屬性必須在架構中宣告。

允許的值為 NO_ERRORS_SCHEMA 和 CUSTOM_ELEMENTS_SCHEMA。

id

string | undefined

在 getNgModuleById 中唯一標識此 NgModule 的名稱或路徑。 如果保留為 undefined，則 NgModule 不會註冊到 getNgModuleById 中。

jit

true | undefined

如果存在，預先編譯器（AOT）將忽略此模組。它會保留在分發後的程式碼中，即時編譯器（JIT）會嘗試在執行時在瀏覽器中對其進行編譯。為了確保行為正確，應用必須匯入 @angular/compiler。
