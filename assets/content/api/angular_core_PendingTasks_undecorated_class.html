程式碼

該服務跟蹤有助於 Angular 應用穩定性的待處理任務。雖然一些現有的 Angular 服務（例如：HttpClient）會在內部管理影響穩定性的任務，但此 API 將穩定性的控制權交給函式庫和應用開發者，以處理 Angular 內部機制未涵蓋的特定情況。

API

  
    class PendingTasks {}

add

() => void

新增一個新任務，該任務應阻止應用的穩定性。

@returns() => void

run

Promise<T>

執行一個非同步函式並阻止應用的穩定性，直到該函式完成。

          
pendingTasks.run(async () => {  const userData = await fetch('/api/user');  this.userData.set(userData);});

應用穩定性至少會延遲到 run 方法求解後的下一個tick，因此可以安全地對應用狀態進行額外的更新，而這些更新將需要 UI 同步：

          
const userData = await pendingTasks.run(() => fetch('/api/user'));this.userData.set(userData);

說明

該服務跟蹤有助於 Angular 應用穩定性的待處理任務。雖然一些現有的 Angular 服務（例如：HttpClient）會在內部管理影響穩定性的任務，但此 API 將穩定性的控制權交給函式庫和應用開發者，以處理 Angular 內部機制未涵蓋的特定情況。

穩定性的概念在幾個重要場景中發揮作用：

  • SSR 程序需要在序列化和傳送渲染後的 HTML 之前等待應用穩定性；
  • 測試可能希望延遲斷言，直到應用變得穩定；
