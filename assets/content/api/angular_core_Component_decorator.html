@angular/core

元件

decorator
程式碼

用來將類別標記為 Angular 元件的裝飾器，並提供設定元資料，以確定元件在執行時應如何處理、實例化和使用。

API

  
    @Component ({})

changeDetection

ChangeDetectionStrategy | undefined

要用於此元件的變更檢測策略。

當元件被實例化時，Angular 會建立一個變更檢測器，它負責傳播元件的繫結。策略之一是：

  • ChangeDetectionStrategy#OnPush 將策略設定為 CheckOnce（按需檢測）。
  • ChangeDetectionStrategy#Default 將策略設定為 CheckAlways。

viewProviders

Provider[] | undefined

定義了對其檢視 DOM 子級可見的可注入物件集合。請參閱範例。

moduleId

string | undefined
@deprecated

此選項沒有任何效果。將在 Angular v17 中移除。

包含該組件的模組的模組 ID。元件必須能夠求解範本和樣式的相對 URL。SystemJS 在每個模組中公開 __moduleName 變數。在 CommonJS 中，可以將其設定為 module.id。

templateUrl

string | undefined

Angular 元件的範本檔案的相對路徑或絕對 URL。如果已提供，請勿使用 template 提供行內範本。

template

string | undefined

Angular 元件的行內範本。如果已提供，請勿使用 templateUrl 提供範本檔案。

styleUrl

string | undefined

用於此元件的包含 CSS 樣式表的檔案的相對路徑或絕對 URL。

styleUrls

string[] | undefined

用於此元件的包含 CSS 樣式表的檔案的相對路徑或絕對 URL。

styles

string | string[] | undefined

在此元件中使用的一個或多個行內 CSS 樣式表。

animations

any[] | undefined

一個或多個動畫 trigger() 呼叫，其中包含 state() 和 transition() 定義。 請參閱動畫指南和動畫 API 文件。

encapsulation

ViewEncapsulation | undefined

元件樣式的封裝策略。 可能的值：

  • ViewEncapsulation.Emulated：應用修改後的元件樣式，以模擬原生的 Shadow DOM CSS 封裝行為。
  • ViewEncapsulation.None：全域應用元件樣式，沒有任何形式的封裝。
  • ViewEncapsulation.ShadowDom：使用瀏覽器的原生 Shadow DOM API 來封裝樣式。

如果未提供，則該值取自 CompilerOptions，後者預設為 ViewEncapsulation.Emulated。

如果策略是 ViewEncapsulation.Emulated 並且元件沒有 styles 或 styleUrls，則策略會自動切換為 ViewEncapsulation.None。

interpolation

[string, string] | undefined
@deprecated

請改為使用 Angular 的預設內嵌分隔符。

覆蓋預設的內嵌開始和結束分隔符（{{ 和 }}）。

preserveWhitespaces

boolean | undefined

true 表示保留，false 表示從已編譯的範本中刪除可能多餘的空格字元。 空格字元是那些與 JavaScript 正則表示式中的 \s 字元類別對應的字元。 預設為 false，除非在編譯器選項中被覆蓋。

standalone

boolean | undefined

標記為 standalone 的 Angular 元件不需要在 NgModule 中宣告。 這樣的元件透過 imports 屬性直接管理它們自己的範本依賴項（在範本中使用的元件、指令和通道）。

關於獨立元件、指令和通道的更多訊息，請參見本指南。

imports

(readonly any[] | Type<any>)[] | undefined

imports 屬性指定了獨立元件的範本依賴項——即可以在其範本中使用的那些指令、元件和通道。獨立元件可以匯入其他獨立元件、指令和通道，以及現有的 NgModule。

此屬性（Property）僅適用於獨立元件 - 為 NgModule 中宣告的元件指定它會生成編譯錯誤。

關於獨立元件、指令和通道的更多訊息，請參見本指南。

schemas

SchemaMetadata[] | undefined

用於宣告允許在獨立元件中使用的元素的架構集合。 既不是 Angular 元件也不是指令的元素和屬性（Property）必須在架構中宣告。

此屬性（Property）僅適用於獨立元件 - 為 NgModule 中宣告的元件指定它會生成編譯錯誤。

關於獨立元件、指令和通道的更多訊息，請參見本指南。

selector

string | undefined

CSS 選擇器，用於在範本中標識此指令並觸發指令的實例化。

宣告為以下之一：

  • element-name：按元素名稱選擇。
  • .class：按類別名稱選擇。
  • [attribute]：按屬性（Attribute）名選擇。
  • [attribute=value]：按屬性（Attribute）名和值選擇。
  • :not(sub_selector)：僅當元素與 sub_selector 不對應時才選擇。
  • selector1, selector2：如果 selector1 或 selector2 對應，則選擇。

Angular 只允許指令應用於不跨越元素邊界的 CSS 選擇器。

對於以下範本 HTML，帶有 input[type=text] 選擇器的指令將僅在 <input type="text"> 元素上實例化。

          
<form>  <input type="text">  <input type="radio"><form>

inputs

(string | { name: string; alias?: string | undefined; required?: boolean | undefined; transform?: ((value: any) => any) | undefined; })[] | undefined

列舉指令的資料繫結輸入屬性（Property）集

在變更檢測期間，Angular 會自動更新輸入屬性（Property）。inputs 屬性接受字串或物件字面量，它們會設定應該作為輸入公開的指令屬性（property）。

當傳入物件字面量時，name 屬性會指明輸入應該寫入到類別上的哪個屬性（property），而 alias 則決定了在範本繫結中可用的輸入名稱。required 屬性表示此輸入是必要的，如果在使用該指令時未傳入此輸入，則會觸發編譯時錯誤。

當將字串傳入 inputs 陣列時，它可以具有 'name' 或 'name: alias' 的格式，其中 name 是指令應寫入的類別上的屬性（property），而 alias 確定了在範本繫結中可用的輸入名稱。基於字串的輸入定義被假定為是可選的。

outputs

string[] | undefined

列舉事件繫結的輸出屬性（Property）的集合。

當輸出屬性（Property）發出事件時，將呼叫範本中附著到該事件的事件處理器。

outputs 屬性定義了一組從 directiveProperty 到 alias 的設定：

  • directiveProperty 指定了發出事件的元件屬性（property）。
  • alias 指定了事件處理器附著到的 DOM 屬性（Property）。

providers

Provider[] | undefined

使用一個令牌設定此指令或元件的注入器，該令牌對映到依賴項的提供者。

exportAs

string | undefined

定義可在範本中用於將此指令賦值給變數的名稱。

queries

{ [key: string]: any; } | undefined

設定將注入到指令中的查詢。

內容查詢在 ngAfterContentInit 回呼被呼叫之前設定。檢視查詢在 ngAfterViewInit 回呼被呼叫之前設定。

host

{ [key: string]: string; } | undefined

使用一組鍵值對，將類別屬性（property）對映到宿主元素繫結，以用於屬性（property）、Attribute 和事件。

在變更檢測期間，Angular 會自動檢查宿主屬性（Property）繫結。如果繫結發生更改，Angular 會更新指令的宿主元素。

當鍵是宿主元素的屬性（property）時，該屬性值將傳播到指定的 DOM 屬性（Property）。

當鍵是 DOM 中的靜態 Attribute 時，該 Attribute 值將傳播到宿主元素中的指定屬性（property）。

對於事件處理：

  • 鍵是指令監聽的 DOM 事件。要監聽全域事件，請將目標新增到事件名稱中。目標可以是 window、document 或 body。
  • 該值是事件發生時要執行的語句。如果語句求值結果為 false，則會對 DOM 事件應用 preventDefault。處理函式方法可以引用 $event 區域性變數。

jit

true | undefined

如果存在，則 AOT 編譯器會忽略此指令/元件。它會保留在分發程式碼中，而 JiT 編譯器會嘗試在執行時在瀏覽器中對其進行編譯。為了確保行為正確，應用必須匯入 @angular/compiler。

hostDirectives

(Type<unknown> | { directive: Type<unknown>; inputs?: string[] | undefined; outputs?: string[] | undefined; })[] | undefined

每當指令對應時，都應應用於宿主的獨立指令。預設情況下，宿主指令的任何輸入或輸出在宿主上都不可用，除非它們在 inputs 或 outputs 屬性（Property）中指定。

你還可以透過在原始輸入或輸出名稱後加上冒號和別名來為輸入和輸出設定別名。例如，如果透過 hostDirectives 應用的指令定義了一個名為 menuDisabled 的輸入，你可以透過將 'menuDisabled: disabled' 作為條目新增到 inputs 中，將其別名為 disabled。

描述

用來將類別標記為 Angular 元件的裝飾器，並提供設定元資料，以確定元件在執行時應如何處理、實例化和使用。

元件是 Angular 應用最基本的 UI 基礎元素。一個 Angular 應用套件含一棵 Angular 元件樹。

Angular 元件是指令的一個子集，始終與範本關聯。與其他指令不同，對於範本中的給定元素，只能實例化一個元件。

獨立元件可以直接匯入到任何其他獨立元件或 NgModule 中。另一方面，基於 NgModule 的應用要求元件屬於 NgModule，以便它們可用於另一個元件或應用。要使元件成為 NgModule 的成員，請在 NgModule 元資料的 declarations 欄位中列出它。

請注意，除了這些用於設定指令的選項之外，你還可以透過實現生命週期鉤子來控制組件的執行時行為。有關更多訊息，請參閱生命週期鉤子指南。

使用須知

設定元件輸入

以下範例建立了一個具有兩個資料繫結屬性（property）的元件，這些屬性由 inputs 值指定。

          
@Component({  selector: 'app-bank-account',  inputs: ['bankName', 'id: account-id'],  template: ` Bank Name: {{ bankName }} Account Id: {{ id }} `,  standalone: false,})export class BankAccountComponent {  bankName: string | null = null;  id: string | null = null;  // this property is not bound, and won't be automatically updated by Angular  normalizedBankName: string | null = null;}@Component({  selector: 'app-my-input',  template: ` <app-bank-account bankName="RBC" account-id="4747"> </app-bank-account>`,  standalone: false,})export class MyInputComponent {}

設定元件輸出

以下範例顯示了兩個事件發射器，它們按時間間隔發射事件。一個每秒發射一次輸出，而另一個每五秒發射一次。

          
@Directive({  selector: 'app-interval-dir',  outputs: ['everySecond', 'fiveSecs: everyFiveSeconds'],  standalone: false,})export class IntervalDirComponent {  everySecond = new EventEmitter<string>();  fiveSecs = new EventEmitter<string>();  constructor() {    setInterval(() => this.everySecond.emit('event'), 1000);    setInterval(() => this.fiveSecs.emit('event'), 5000);  }}@Component({  selector: 'app-my-output',  template: `    <app-interval-dir (everySecond)="onEverySecond()" (everyFiveSeconds)="onEveryFiveSeconds()">    </app-interval-dir>  `,  standalone: false,})export class MyOutputComponent {  onEverySecond() {    console.log('second');  }  onEveryFiveSeconds() {    console.log('five seconds');  }}

使用檢視提供者注入類別

以下簡單範例使用元件元資料中指定的檢視提供者將一個類別注入到元件中：

          
class Greeter {   greet(name:string) {     return 'Hello ' + name + '!';   }}@Directive({  selector: 'needs-greeter'})class NeedsGreeter {  greeter:Greeter;  constructor(greeter:Greeter) {    this.greeter = greeter;  }}@Component({  selector: 'greet',  viewProviders: [    Greeter  ],  template: `<needs-greeter></needs-greeter>`})class HelloWorld {}

保留空白符

刪除空白符可以大大減小預先編譯（AoT）生成的程式碼大小並加快檢視建立速度。從 Angular 6 開始，preserveWhitespaces 的預設值為 false（刪除空白符）。要更改應用中所有元件的預設設定，請設定預先編譯器（AoT）的 preserveWhitespaces 選項。

預設情況下，預先編譯器（AoT）按如下方式刪除空白字元：

  • 修剪範本開頭和結尾的所有空白符。
  • 刪除僅包含空白符的文字節點。例如，
          
<button>Action 1</button>  <button>Action 2</button>

變為：

          
<button>Action 1</button><button>Action 2</button>
  • 將文字節點中的一系列空白字元替換為單個空格。例如，<span>\n some text\n</span> 變為 <span> some text </span>。
  • 不會更改 HTML 標籤（如 <pre> 或 <textarea>）內的文字節點，在這些標籤中，空白字元是有意義的。

請注意，這些轉換可能會影響 DOM 節點佈局，儘管影響應該是最小的。

你可以重寫預設行為以在範本的某些片段中保留空白字元。例如，你可以使用 ngPreserveWhitespaces 屬性排除整個 DOM 子樹：

          
<div ngPreserveWhitespaces>    whitespaces are preserved here    <span>    and here </span></div>

你可以使用 &ngsp; 強制在文字節點中保留單個空格，Angular 的範本編譯器會將其替換為空格字元：

          
<a>Spaces</a>&ngsp;<a>between</a>&ngsp;<a>links.</a><!-- compiled to be equivalent to: <a>Spaces</a> <a>between</a> <a>links.</a>  -->

請注意，當 preserveWhitespaces 選項設定為 false 時，&ngsp; 序列仍然會摺疊為一個空格字元。

          
<a>before</a>&ngsp;&ngsp;&ngsp;<a>after</a><!-- compiled to be equivalent to: <a>before</a> <a>after</a> -->

要保留空白字元序列，請使用 ngPreserveWhitespaces 屬性。
