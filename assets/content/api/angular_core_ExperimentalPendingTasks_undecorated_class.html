实验性服务，用于跟踪影响 Angular 应用稳定性的待处理任务。虽然一些现有的 Angular 服务（如： HttpClient）将在内部管理影响稳定性的任务，这个 API 使库和应用开发人员能够在 Angular 内部未涵盖的特定情况下控制稳定性。

稳定性概念在几个重要场景中起作用：

  • SSR 过程需要等待应用稳定后再序列化和发送渲染的 HTML；
  • 测试可能希望在应用变得稳定后再进行断言；
          
            const pendingTasks = inject(ExperimentalPendingTasks);const taskCleanup = pendingTasks.add();// do work that should block application's stability and then:taskCleanup();

此 API 目前是试验性的。其结构和底层行为都不稳定，可能会在修补版本中发生变化。我们将基于反馈以及对问题和解决方案空间的理解，不断迭代精确的 API。
