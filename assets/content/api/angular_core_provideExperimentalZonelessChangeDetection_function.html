provideExperimentalZonelessChangeDetection

EnvironmentProviders

為使用 bootstrapApplication 引導的應用提供不帶 ZoneJS 的變更檢測。

此函式允許你設定應用，使其不使用 ZoneJS 的狀態/狀態更改來排程應用中的變更檢測。當頁面上根本沒有 ZoneJS，或者由於其他原因（另一個使用 ZoneJS 進行排程的 Angular 應用或某些依賴 ZoneJS 的其他函式庫）而存在 ZoneJS 時，此函式將起作用。

也可以將其新增到 TestBed 提供者中，以設定測試環境，使其更接近生產行為。這將有助於提高元件與不帶 Zone 的變更檢測相容的信心。

ZoneJS 使用瀏覽器事件來觸發變更檢測。當使用此提供者時，Angular 將改為使用 Angular API 來排程變更檢測。這些 API 包括：

  • ChangeDetectorRef.markForCheck
  • ComponentRef.setInput
  • 更新在範本中讀取的訊號
  • 當繫結的宿主或範本監聽器被觸發時
  • 附著一個被上述操作之一標記為髒的檢視
  • 移除檢視
  • 註冊渲染鉤子（僅當渲染鉤子執行上述操作之一時，範本才會重新整理）

用法說明

此 API 是實驗性的。無論是其形態還是底層行為都不穩定，並且可能會在補丁版本中發生更改。目前存在已知的功能差距和 API 工效學方面的考慮。我們將根據反饋以及我們對問題和解決方案空間的理解，迭代確切的 API。
