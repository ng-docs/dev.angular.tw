为使用 bootstrapApplication 启动的应用提供不依赖 ZoneJS 的变更检测。

此函数允许你配置应用不使用 ZoneJS 的状态/状态变更来调度应用中的变更检测。这将在页面上完全没有 ZoneJS 时工作，或如果它存在是因为其他某些东西在使用它（无论是另一个使用 ZoneJS 作调度的 Angular 应用，还是依赖于 ZoneJS 的其他库）。

这也可以添加到 TestBed 提供者中，以配置测试环境，使其更接近生产行为。这将有助于更有信心地确保组件与 ZonelessJS 的变更检测兼容。

ZoneJS 使用浏览器事件触发变更检测。当使用此提供者时，Angular 将改为使用 Angular API 来调度变更检测。这些 API 包括：

  • ChangeDetectorRef.markForCheck
  • ComponentRef.setInput
  • 更新在模板中读取的信号
  • 当绑定的宿主或模板监听器被触发时
  • 附着由上述之一标记为脏的视图
  • 移除视图
  • 注册渲染钩子（模板仅在渲染钩子执行上述之一时才会刷新）
          
            bootstrapApplication(MyApp, {providers: [  provideExperimentalZonelessChangeDetection(),]});

此 API 是实验性的。其形态和底层行为都不稳定，可能会在补丁版本中改变。存在已知的特性差距和 API 工效学方面的考虑。我们将根据反馈和对问题及解决方案空间的理解，迭代出确切的 API。
