@angular/core

BootstrapOptions

interface
程式碼

為引導啟動過程提供額外的選項。

API

  
    interface BootstrapOptions {}

ngZone

NgZone | "zone.js" | "noop" | undefined

可以選擇指定當提供者中未設定 NgZone 時，應該使用哪個 NgZone。

  • 提供你自己的 NgZone 實例。
  • zone.js - 使用預設的 NgZone，它需要 Zone.js。
  • noop - 使用 NoopNgZone，它什麼也不做。

ngZoneEventCoalescing

boolean | undefined

可以選擇指定是否合併事件變更檢測。 考慮以下情況。

          
<div (click)="doSomething()">  <button (click)="doSomethingElse()"></button></div>

當按鈕被點選時，由於事件冒泡，兩個事件處理程式都將被呼叫，並且會觸發 2 次變更檢測。我們可以合併這類事件，以便只觸發一次變更檢測。

預設情況下，此選項將為 false。因此，事件將不會被合併，並且變更檢測將被多次觸發。如果此選項設定為 true，則將透過排程動畫幀非同步觸發變更檢測。因此，在上述情況下，變更檢測將僅觸發一次。

ngZoneRunCoalescing

boolean | undefined

可以選擇指定是否應將 NgZone#run() 方法呼叫合併為單個變更檢測。

考慮以下情況。

          
for (let i = 0; i < 10; i ++) {  ngZone.run(() => {    // do something  });}

這種情況會多次觸發變更檢測。 使用 ngZoneRunCoalescing 選項，事件迴圈中的所有變更檢測僅觸發一次。 此外，變更檢測在 requestAnimation 中執行。

ignoreChangesOutsideZone

boolean | undefined
@deprecated

引入此選項是出於謹慎，旨在讓開發人員選擇退出 v18 中的新行為，即當上述事件發生在 Zone 之外時，會排程變更檢測。在監控發布後的結果後，我們已確定此屬性按預期工作，並且不認為應該透過將此選項設定為 true 來停用它。

當為 false 時，當 Angular 收到需要重新整理樣板的明確指示時，會排程變更檢測。這包括：

  • 呼叫 ChangeDetectorRef.markForCheck
  • 呼叫 ComponentRef.setInput
  • 更新在樣板中讀取的 Signal
  • 附著一個標記為髒的檢視
  • 移除檢視
  • 註冊渲染鉤子（僅當渲染鉤子執行上述操作之一時，才會重新整理樣板）
