interface HttpInterceptor {
}

拦截并处理 HttpRequest 或 HttpResponse。

大多数拦截器会在调用 next.handle(transformedReq) 之前，转换传出的请求，然后将其传递给链中的下一个拦截器。拦截器也可以通过在 next.handle() 返回的响应事件流上应用额外的 RxJS 操作符，来转换响应事件流。

极少数情况下，拦截器也可以自己完全处理一个请求，并且组合出新的事件流来而不必调用 next.handle()。这也是允许的，不过要时刻记住，这将会完全跳过所有后继拦截器。

另一种同样罕见但是有用的拦截器，会为单个请求在事件流上给出多个响应对象。

要在整个应用中使用相同的 HttpInterceptors 实例，只需在你的 AppModule 中导入 HttpClientModule，并将拦截器添加到根应用注入器中。 如果你在不同的模块（例如延迟加载模块）中多次导入 HttpClientModule，每次导入都会创建一个新的 HttpClientModule 副本，从而覆盖根模块中提供的拦截器。
