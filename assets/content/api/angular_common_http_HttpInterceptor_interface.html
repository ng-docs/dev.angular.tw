說明

攔截和處理 HttpRequest 或 HttpResponse。

大多數攔截器都會先轉換傳出的請求，然後再透過呼叫 next.handle(transformedReq) 將其傳遞給鏈中的下一個攔截器。攔截器也可以轉換回應事件流，方法是在 next.handle() 回傳的流上應用額外的 RxJS 運算子。

極少數情況下，攔截器可能會完全處理該請求，並組合一個新的事件流，而不是呼叫 next.handle()。這是一種可接受的行為，但請記住，後續的攔截器將被完全跳過。

攔截器在單個請求的事件流上回傳多個回應的情況也很少見，但也是有效的。

用法說明

要在整個應用中使用 HttpInterceptors 的相同實例，請僅在你的 AppModule 中匯入 HttpClientModule，並將攔截器新增到根應用注入器。如果你在不同的模組（例如，延遲載入模組）中多次匯入 HttpClientModule，則每次匯入都會建立 HttpClientModule 的新副本，這將覆蓋根模組中提供的攔截器。
