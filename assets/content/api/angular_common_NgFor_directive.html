指定一個自定義的 TrackByFunction 來計算可迭代物件中條目的標識。

如果未提供自定義的 TrackByFunction，則 NgForOf 將使用條目的物件標識作為鍵。

NgForOf 使用計算出的鍵將可迭代物件中的條目與其為這些條目生成的 DOM 元素相關聯。

在某些情況下，當使用 NgForOf 渲染的可迭代物件中的條目具有自然識別符號（例如，自定義 ID 或主鍵），並且此可迭代物件可以使用仍然表示相同底層實體的新物件實例進行更新時（例如，當從伺服器重新獲取資料，並且可迭代物件被重新建立和重新渲染，但大多數資料仍然相同時），自定義的 TrackByFunction 對於提供良好的使用者體驗非常有用。

描述

結構型指令，它為集合中的每個條目渲染一個範本。 該指令放在一個元素上，該元素將成為克隆範本的父元素。

ngForOf 指令通常在 簡寫形式 *ngFor 中使用。 在這種形式下，每次迭代要渲染的範本是包含該指令的錨元素的內容。

以下範例顯示了帶有某些選項的簡寫語法，包含在 <li> 元素中。

          
<li *ngFor="let item of items; index as i; trackBy: trackByFn">...</li>

簡寫形式會擴充套件為長形式，後者在 <ng-template> 元素上使用 ngForOf 選擇器。 <ng-template> 元素的內容是包含簡寫形式指令的 <li> 元素。

這是簡寫形式範例的展開版本。

          
<ng-template ngFor let-item [ngForOf]="items" let-i="index" [ngForTrackBy]="trackByFn">  <li>...</li></ng-template>

Angular 在編譯範本時會自動展開簡寫語法。 每個嵌入式檢視的上下文都根據其詞法位置邏輯合併到當前元件上下文中。

當使用簡寫語法時，Angular 只允許一個結構型指令位於一個元素上。 例如，如果要有條件地迭代，請將 *ngIf 放在包裝 *ngFor 元素的容器元素上。 有關更多討論，請參閱結構型指令。

用法說明

區域性變數

NgForOf 提供了可以別名給區域性變數的導出值。 例如：

          
<li *ngFor="let user of users; index as i; first as isFirst">  {{i}}/{{users.length}}. {{user}} <span *ngIf="isFirst">default</span></li>

以下匯出的值可以別名給區域性變數：

  • $implicit: T：可迭代物件（ngForOf）中各個條目的值。
  • ngForOf: NgIterable<T>：可迭代表示式的值。當表示式比屬性（Property）訪問更復雜時很有用，例如當使用 async 通道時 (userStreams | async)。
  • index: number：可迭代物件中當前條目的索引。
  • count: number：可迭代物件的長度。
  • first: boolean：當條目是可迭代物件中的第一個條目時為 True。
  • last: boolean：當條目是可迭代物件中的最後一個條目時為 True。
  • even: boolean：當條目在可迭代物件中具有偶數索引時為 True。
  • odd: boolean：當條目在可迭代物件中具有奇數索引時為 True。

變更傳播

當迭代器的內容更改時，NgForOf 會對 DOM 進行相應的更改：

  • 當新增條目時，範本的新實例會被新增到 DOM 中。
  • 當移除條目時，其範本實例會從 DOM 中移除。
  • 當條目被重新排序時，它們各自的範本也會在 DOM 中重新排序。

Angular 使用物件標識來跟蹤迭代器中的插入和刪除，並在 DOM 中重現這些更改。 這對動畫和任何存在的有狀態控制元件（例如接受使用者輸入的 <input> 元素）具有重要意義。 插入的行可以動畫進入，刪除的行可以動畫退出，未更改的行會保留任何未儲存的狀態，例如使用者輸入。 有關動畫的更多訊息，請參閱轉場和觸發器。

迭代器中元素的標識可能會在資料沒有變化的情況下發生變化。 例如，如果迭代器是透過到伺服器的 RPC 生成的，並且該 RPC 重新執行，則可能會發生這種情況。 即使資料沒有更改，第二個回應也會生成具有不同標識的物件，並且 Angular 必須拆除整個 DOM 並重建它（就像所有舊元素都被刪除，所有新元素都被插入一樣）。

為了避免這種昂貴的操作，你可以透過為 NgForOf 提供 trackBy 選項來自定義預設的跟蹤演算法。 trackBy 接受一個帶有兩個引數的函式：index 和 item。 如果給定了 trackBy，Angular 將透過函式的回傳值來跟蹤更改。
