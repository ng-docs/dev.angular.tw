@angular/common/http

HttpInterceptorFn

型別別名
程式碼

用於透過 HttpClient 發出的 HTTP 請求的攔截器。

HTTP 指南withInterceptors

API

  
    type HttpInterceptorFn = (  req: HttpRequest<unknown>,  next: HttpHandlerFn,) => Observable<HttpEvent<unknown>>

描述

用於透過 HttpClient 發出的 HTTP 請求的攔截器。

HttpInterceptorFn 是中間件函式，當發出請求時，HttpClient 會呼叫這些函式。 這些函式有機會修改發出的請求或回傳的任何回應，以及阻止、重導向或以其他方式更改請求或回應語義。

提供了一個 HttpHandlerFn，它表示下一個攔截器（或將發出真實 HTTP 請求的後端）。大多數攔截器將委託給此函式，但這不是必需的（有關更多詳細訊息，請參閱 HttpHandlerFn）。

HttpInterceptorFn 在依賴注入上下文中執行。 它們可以透過設定它們的 EnvironmentInjector 訪問 inject()。

用法說明

這是一個 noop 攔截器，它在不修改請求的情況下傳遞請求：

          
export const noopInterceptor: HttpInterceptorFn = (req: HttpRequest<unknown>, next:HttpHandlerFn) => {  return next(modifiedReq);};

如果你想更改請求，請先克隆它，然後在將其傳遞給 next() 處理函式之前修改克隆。

這是一個基本攔截器，它將 bearer 令牌新增到標頭

          
export const authenticationInterceptor: HttpInterceptorFn = (req: HttpRequest<unknown>, next:HttpHandlerFn) => {   const userToken = 'MY_TOKEN'; const modifiedReq = req.clone({     headers: req.headers.set('Authorization', `Bearer ${userToken}`),   });   return next(modifiedReq);};
