用法說明

狀態更改表示式

狀態更改表示式指示 Angular 何時執行轉場動畫，它可以是

  • 具有特定語法的字串
  • 或者是一個函式，用於比較先前的狀態和當前狀態（繫結到元素觸發器的表示式的值），如果應該發生轉場，則回傳 true，否則回傳 false

字串格式可以是：

  • fromState => toState，這表示當繫結到觸發器元素的表示式從 fromState 變為 toState 時，應發生轉場動畫

    範例：

              
      transition('open => closed', animate('.5s ease-out', style({ height: 0 }) ))

  • fromState <=> toState，這表示當繫結到觸發器元素的表示式從 fromState 變為 toState 或反之亦然時，應發生轉場動畫

    範例：

              
      transition('enabled <=> disabled', animate('1s cubic-bezier(0.8,0.3,0,1)'))

  • :enter/:leave，這表示當元素進入或離開 DOM 時，應發生轉場動畫

    範例：

              
      transition(':enter', [    style({ opacity: 0 }),    animate('500ms', style({ opacity: 1 }))  ])

  • :increment/:decrement，這表示當繫結到觸發器元素的數值表示式的值增加或減少時，應發生轉場動畫

    範例：

              
      transition(':increment', query('@counter', animateChild()))

  • 以上任何一項的序列（以逗號分隔），這表示每當狀態更改表示式之一對應時，都應發生轉場動畫

    範例：

              
      transition(':increment, * => enabled, :enter', animate('1s ease', keyframes([    style({ transform: 'scale(1)', offset: 0}),    style({ transform: 'scale(1.1)', offset: 0.7}),    style({ transform: 'scale(1)', offset: 1})  ]))),

另請注意，在這種上下文中：

  • void 可用於指示元素不存在
  • 星號可以用作對應任何狀態的萬用字元
  • （作為上述結果，void => * 等價於 :enter，而 * => void 則等價於 :leave）
  • true 和 false 也分別對應表示式值 1 和 0（但不對應真值和假值）

請注意進入和離開的元素，因為它們的轉場效果會給開發人員帶來常見的陷阱。

請注意，當帶有觸發器的元素進入 DOM 時，其 :enter 轉場效果始終會執行，但是如果該元素與其父元素一起被移除，則其 :leave 轉場效果將不會執行（因為它將在其轉場效果有機會執行之前『無警告』地被移除，這種轉場效果發生的唯一方式是元素自行退出 DOM）。

最終狀態動畫

如果轉場效果的最後一步是呼叫 animate()，且該呼叫使用了時序值但沒有 style 資料，則該步驟將自動被視為最終動畫弧，為了使元素達到最終狀態，在這種情況下，Angular 會自動新增或移除 CSS 樣式，以確保元素處於正確的最終狀態。

用例

  • 基於觸發器表示式值應用的轉場動畫
          
<div [@myAnimationTrigger]="myStatusExp"> ...</div>

        
          
trigger("myAnimationTrigger", [  ..., // states  transition("on => off, open => closed", animate(500)),  transition("* <=> error", query('.indicator', animateChild()))])

  • 基於自定義邏輯（取決於觸發器的表示式值和提供的引數）應用的轉場動畫

              
    <div [@myAnimationTrigger]="{ value: stepName, params: { target: currentTarget }}"> ...</div>
    
        
              
    trigger("myAnimationTrigger", [  ..., // states  transition(    (fromState, toState, _element, params) =>      ['firststep', 'laststep'].includes(fromState.toLowerCase())      && toState === params?.['target'],    animate('1s')  )])
