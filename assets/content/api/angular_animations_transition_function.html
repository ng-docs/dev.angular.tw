状态更改表达式

状态更改表达式会指示 Angular 何时运行转换的动画，它可以是

  • 具有特定语法的字符串
  • 或一个比较前一个状态和当前状态（绑定到元素触发器的表达式的值）的函数，如果应该发生转换，则返回 true，否则返回 false

字符串格式可以是：

  • fromState => toState，这表明应该发生转换的动画，然后绑定到触发器元素的表达式从 fromState 到 toState

    范例：

              
              transition('open => closed', animate('.5s ease-out', style({ height: 0 }) ))

  • fromState <=> toState ，表示当绑定到触发器元素的表达式从 fromState 变为 toState 或反之时，应该进行过渡动画。

    范例：

              
              transition('enabled <=> disabled', animate('1s cubic-bezier(0.8,0.3,0,1)'))

  • :enter / :leave，这表明转换的动画应该在元素进入或存在 DOM 时发生

    范例：

              
              transition(':enter', [    style({ opacity: 0 }),    animate('500ms', style({ opacity: 1 }))  ])

  • :increment / :decrement ，表示当绑定到触发器元素的数值表达式的值增加或减少时，应该发生转换的动画

    范例：

              
              transition(':increment', query('@counter', animateChild()))

  • 上述任何一项的序列除以逗号，这表明只要状态更改表达式之一匹配，就应该发生转换的动画

    范例：

              
              transition(':increment, * => enabled, :enter', animate('1s ease', keyframes([    style({ transform: 'scale(1)', offset: 0}),    style({ transform: 'scale(1.1)', offset: 0.7}),    style({ transform: 'scale(1)', offset: 1})  ]))),

另请注意，在这种情况下：

  • void 可用于表明此元素的不存在
  • 星号可以作为匹配任何状态的通配符
  • （由于上述原因，void => * 等效于 :enter，并且 * => void 等效于 :leave ）
  • true 和 false 也分别匹配表达式值 1 和 0 （但不匹配 truthy 和 falsy 值）

注意进入和离开元素，因为它们的过渡对开发者来说是一个常见陷阱。

注意，当带有触发器的元素进入 DOM 时，其 :enter 过渡总是会执行，但如果该元素与其父元素一起被移除，则其 :leave 过渡不会执行（因为它会在过渡有机会执行之前“无预警”地被移除，唯一能使这种过渡发生的情况是元素自行退出 DOM）。

把动画播放到最终状态

如果过渡的最后一步是对不带 style 数据的 animate() 调用，那么该步骤会自动被视为最终动画弧度，以使元素达到最终状态。在这种情况下，Angular 会自动添加或移除 CSS 样式，以确保元素处于正确的最终状态。

使用范例

  • 根据触发器的表达式值应用的过渡动画
          
            <div [@myAnimationTrigger]="myStatusExp"> ...</div>
          
        
          
            trigger("myAnimationTrigger", [  ..., // states  transition("on => off, open => closed", animate(500)),  transition("* <=> error", query('.indicator', animateChild()))])

  • 根据自定义逻辑应用的过渡动画，具体取决于触发器的表达式值和提供的参数

              
            <div [@myAnimationTrigger]="{ value: stepName, params: { target: currentTarget }}"> ...</div>
          
        
              
            trigger("myAnimationTrigger", [  ..., // states  transition(    (fromState, toState, _element, params) =>      ['firststep', 'laststep'].includes(fromState.toLowerCase())      && toState === params?.['target'],    animate('1s')  )])
