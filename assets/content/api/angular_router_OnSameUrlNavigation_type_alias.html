@angular/router

OnSameUrlNavigation

型別別名
程式碼

如何處理對當前 URL 的導向請求。選項之一：

RouteReuseStrategyRunGuardsAndResolversNavigationBehaviorOptionsRouterConfigOptions

API

  
    type OnSameUrlNavigation = 'reload' | 'ignore'

描述

如何處理對當前 URL 的導向請求。選項之一：

  • 'ignore'：如果請求與當前狀態相同，路由器將忽略該請求。
  • 'reload'：即使 URL 與當前狀態不同，路由器也會處理該 URL。 您可能想要使用此選項的一個範例是，如果 canMatch 守衛依賴於 應用狀態，並且最初拒絕導向到某個路由。修復狀態後，您希望 重新導向到同一 URL，以便具有 canMatch 守衛的路由可以啟用。

請注意，這僅設定 Route 是否重新處理 URL 並觸發相關的操作和事件，如重導向、守衛和求解器。預設情況下，當路由器重新導向到同一組件型別而沒有先訪問不同的元件時，它會複用元件實例。此行為由 RouteReuseStrategy 設定。為了在相同 URL 導向時重新載入路由元件，您需要將 onSameUrlNavigation 設定為 'reload'並且提供一個 RouteReuseStrategy，該策略為 shouldReuseRoute 回傳 false。此外，除非路徑或路徑引數已更改（由 runGuardsAndResolvers 設定），否則路由的求解器和大多數守衛不會執行。
