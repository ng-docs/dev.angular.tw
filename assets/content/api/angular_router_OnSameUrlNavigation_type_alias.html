type OnSameUrlNavigation = 'reload' | 'ignore'

定义当路由器收到一个导航到当前 URL 的请求时应该怎么做。可取下列值之一：

  • 'ignore' : 路由器忽略请求，与当前状态相同。
  • 'reload'：路由器处理 URL，即使它与当前状态没有区别。你可能需要此选项的一个示例是，如果 canMatch 守卫取决于应用程序状态并且最初拒绝导航到路由。修复状态后，你想要重新导航到相同的 URL，以便可以激活带有 canMatch 守卫的路由。

注意，这仅配置 Route 是否重新处理 URL 并触发相关操作和事件（如重定向、守卫和求解器）。默认情况下，Router 在重新导航到同一组件类型时复用组件实例，而不先访问不同组件。此行为由 RouteReuseStrategy 配置。为了在同一 URL 导航时重新加载路由组件，你需要将 onSameUrlNavigation 设置为 'reload' 并且 提供一个在 shouldReuseRoute 返回 false 的 RouteReuseStrategy。此外，除非路径或路径参数更改（由 runGuardsAndResolvers 配置），否则路由的大多数守卫和求解器不会运行。
