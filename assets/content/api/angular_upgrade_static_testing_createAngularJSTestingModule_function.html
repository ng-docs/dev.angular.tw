单元测试依赖于降级 Angular 服务的 AngularJS 服务时要使用的帮助器函数。

createAngularJSTestingModule(
  angularModules: any[]
): string;

单元测试依赖于降级 Angular 服务的 AngularJS 服务时要使用的帮助器函数。

此函数返回一个 AngularJS 模块，该模块被配置为连接 AngularJS 和 Angular 注入器，而无需实际引导混合应用程序。这使得单元测试服务变得更简单、更快。

在对angular.mocks.module的调用中使用返回的 AngularJS 模块，以将此模块包含在单元测试注入器中。

在以下代码片段中，我们使用两个模块配置 $injector：AngularJS ng1AppModule ，它是我们混合应用程序的 AngularJS 部分和 Ng2AppModule，它是 Angular 部分。

完成此操作后，我们可以像平常一样通过 AngularJS 的 $injector 获取服务。那些（或依赖于）降级的 Angular 服务的服务，将由 Angular 根 Injector 根据需要实例化。

在以下代码片段中，heroesService 是我们从 AngularJS 访问的降级 Angular 服务。

此辅助函数用于测试服务而不是组件。对于组件测试，你仍需引导混合应用。详见 UpgradeModule 或 downgradeModule。

生成的配置不会将 AngularJS 摘要连接到 Zone 钩子。测试作者有责任根据需要调用 $rootScope.$apply ，以触发来自 Angular 的异步事件的 AngularJS 处理程序。

帮助器设置全局变量来保存共享的 Angular 和 AngularJS 注入器。

  • 每个规范只调用一次此帮助器。
  • 不要在同一个规范中同时使用 createAngularJSTestingModule 和 createAngularTestingModule。

以下是使用 createAngularTestingModule 和 createAngularJSTestingModule 的示例应用及其单元测试。
