@angular/core

RendererType2

interface
程式碼

由 RendererFactory2 使用，以將自定義渲染資料和樣式與渲染實現關聯起來。

API

  
    interface RendererType2 {}

id

string

新渲染器的唯一標識字串，用於在為封裝建立唯一樣式時。

encapsulation

ViewEncapsulation

檢視封裝型別，它決定了樣式如何應用於 DOM 元素。它是以下型別之一

  • Emulated（預設值）：模擬樣式的原生作用域。
  • Native：使用渲染器的原生封裝機制。
  • ShadowDom：使用現代 Shadow DOM 併為元件的宿主元素建立一個 ShadowRoot。
  • None：不提供任何範本或樣式封裝。

styles

string[]

定義要儲存在渲染器實例上的 CSS 樣式。

data

{ [kind: string]: any; }

定義要儲存在渲染器實例上的任意開發者定義的資料。 這對於委託給其他渲染器的渲染器很有用。

getExternalStyles

((encapsulationId?: string | undefined) => string[]) | null | undefined

框架使用此函式來建立外部執行時樣式 URL 清單。
