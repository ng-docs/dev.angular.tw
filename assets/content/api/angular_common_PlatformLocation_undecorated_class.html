此类不应由应用开发者直接使用。相反，应使用 Location。

PlatformLocation 封装了所有对 DOM API 的调用，这使得路由器与平台无关。这意味着我们可以为 Angular 支持的不同平台提供不同的 PlatformLocation 实现。例如， @angular/platform-browser 提供了一个针对浏览器环境的实现，而 @angular/platform-server 提供了一个适合用于服务器端渲染的实现。

PlatformLocation 类由所有 {@link LocationStrategy} 的实现直接使用，当它们需要与 pushState、popState 等 DOM API 交互时。

{@link LocationStrategy} 反过来被 {@link Location} 服务使用，而 {@link Router} 直接使用该服务在路由之间导航。由于 {@link Router} / {@link Location} / {@link LocationStrategy} 和 DOM API 之间的所有交互都通过 PlatformLocation 类进行，所以它们都是平台无关的。
