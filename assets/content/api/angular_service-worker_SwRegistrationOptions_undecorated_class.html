可用于在 ServiceWorkerModule.register() 之外为 ServiceWorkerModule 提供选项的令牌。

你可以用此标记来定义一个在运行时生成注册选项的提供者，例如通过函数调用：

{@example service-worker/registration-options/module.ts region="registration-options" header="app.module.ts"}

enabled

boolean

ServiceWorker 是否会被注册以及相关服务（如 SwPush 和 SwUpdate）是否会尝试与其通信和交互。

默认值：true

scope

string

定义 ServiceWorker 的注册范围的 URL；也就是说，它可以控制的 URL 范围。调用ServiceWorkerContainer#register()时将使用它。

registrationStrategy

string | (() => Observable<unknown>)

定义 ServiceWorker 注册策略，该策略确定它将何时注册到浏览器。

应用程序稳定后注册的默认行为（即一旦没有挂起的微任务和宏任务）旨在尽快注册 ServiceWorker，但不影响应用程序的首次加载。

不过，在某些情况下，你可能希望对 ServiceWorker 的注册时间进行更多控制（例如，可能会有长时间运行的超时或轮询间隔，以防止应用程序稳定）。可用的选项是：

  • registerWhenStable:<timeout>：在应用稳定后（没有未完成的微任务/宏任务）立即注册，但不超过 <timeout> 毫秒。如果应用在 <timeout> 毫秒后仍未稳定（例如，由于反复的异步任务），ServiceWorker 仍会被注册。如果省略 <timeout>，ServiceWorker 仅在应用稳定后注册。
  • registerImmediately：立即注册。
  • registerWithDelay:<timeout>：延迟 <timeout> 毫秒注册。例如，使用 registerWithDelay:5000 在 5 秒后注册 ServiceWorker。如果省略 <timeout>，默认为 0，即在所有未完成的微任务完成后尽快异步注册 ServiceWorker。
  • 一个可观察者工厂函数：一个返回 Observable 的函数。该函数将在运行时用于获取并订阅 Observable，并且 ServiceWorker 会在首次发出值时注册。

默认值：“registerWhenStable:30000”
