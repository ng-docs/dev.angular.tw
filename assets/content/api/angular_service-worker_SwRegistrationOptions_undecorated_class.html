@angular/service-worker

SwRegistrationOptions

類別
程式碼

令牌，可用於為 ServiceWorkerModule 提供選項，而無需在 ServiceWorkerModule.register() 中提供。

API

  
    abstract class SwRegistrationOptions {}

enabled

boolean | undefined

ServiceWorker 是否將被註冊，以及相關的服務（例如 SwPush 和 SwUpdate）是否會嘗試與之通訊和互動。

預設值：true

scope

string | undefined

一個 URL，用於定義 ServiceWorker 的註冊作用域；即，它可以控制的 URL 範圍。它將在呼叫 ServiceWorkerContainer#register() 時使用。

registrationStrategy

string | (() => Observable<unknown>) | undefined

定義 ServiceWorker 註冊策略，該策略決定了它何時在瀏覽器中註冊。

預設行為是在應用穩定後（即，一旦沒有待處理的微任務和巨集任務）立即註冊，其目的是儘快註冊 ServiceWorker，但又不影響應用的首次載入。

不過，在某些情況下，你可能希望更好地控制 ServiceWorker 的註冊時間（例如，可能存在長時間執行的超時或輪詢間隔，從而阻止應用穩定）。可用的選項包括：

  • registerWhenStable:<timeout>：只要應用穩定（沒有待處理的微任務/巨集任務）就立即註冊，但不得晚於 <timeout> 毫秒。如果應用在 <timeout> 毫秒後仍未穩定（例如，由於週期性的非同步任務），ServiceWorker 仍將被註冊。 如果省略 <timeout>，則僅在應用穩定後才註冊 ServiceWorker。
  • registerImmediately：立即註冊。
  • registerWithDelay:<timeout>：延遲 <timeout> 毫秒後註冊。例如，使用 registerWithDelay:5000 在 5 秒後註冊 ServiceWorker。如果省略 <timeout>，則預設為 0，這將盡快註冊 ServiceWorker，但仍然是非同步的，一旦所有待處理的微任務完成。
  • 一個 Observable 工廠函式：一個回傳 Observable 的函式。 該函式將在執行時用於獲取和訂閱 Observable，並且 ServiceWorker 將在發出第一個值後立即註冊。

預設值：'registerWhenStable:30000'
