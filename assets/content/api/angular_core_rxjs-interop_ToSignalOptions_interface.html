toSignal 的选项。

interface ToSignalOptions {
}

initialValue

unknown

由 toSignal 生成的信号的初始值。

这将是信号的值，直到 observable 发出它的第一个值。

requireSync

boolean

是否要求在 toSignal 订阅时，可观察者同步发出。

如果此选项为 true， toSignal 将断言可观察者在订阅时立即产生一个值。设置此选项可以省去在信号类型中处理 undefined 或提供 initialValue 的需要，但如果不满足此要求，将导致运行时错误。

injector

Injector

将提供用于清理可观察者订阅的 DestroyRef 的 Injector。

如果未提供此项，除非请求手动清理，否则将从当前的 注入上下文中获取一个 DestroyRef。

manualCleanup

boolean

当 toSignal的创建上下文被销毁时，订阅是否应通过 DestroyRef自动清理。

如果启用了手动清理，则不使用 DestroyRef，订阅将持续到 Observable本身完成。

rejectErrors

boolean

是否应将 toSignal从Observable错误通道抛出的错误返回给RxJS，在那里它们将被处理为未捕获的异常。

实际上，这意味着由 toSignal 返回的信号将永远返回最后的有效值，因为产生错误的可观察者不会产生进一步的值。此选项模拟了 async 管道的行为。
