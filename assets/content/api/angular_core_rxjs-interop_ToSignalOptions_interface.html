@angular/core/rxjs-interop

ToSignalOptions

interface
程式碼

toSignal 的選項。

API

  
    interface ToSignalOptions<T> {}

initialValue

unknown

由 toSignal 生成的訊號的初始值。

在可觀察者發出它的第一個值之前，這將是訊號的值。

requireSync

boolean | undefined

是否要求當 toSignal 訂閱時，可觀察者同步發出值。

如果為 true，則 toSignal 將斷言可觀察者在訂閱時立即生成一個值。設定此選項消除了在訊號型別中處理 undefined 或提供 initialValue 的需要，但如果未滿足此要求，則會產生執行時錯誤。

injector

Injector | undefined

Injector，它將提供用於清理 Observable 訂閱的 DestroyRef。

如果未提供，則將從當前的 注入上下文 中檢索 DestroyRef，除非請求手動清理。

manualCleanup

boolean | undefined

當 toSignal 的建立上下文被銷燬時，是否應自動清理訂閱（透過 DestroyRef）。

如果啟用了手動清理，則不使用 DestroyRef，並且訂閱將持續存在，直到 Observable 本身完成。

rejectErrors

boolean | undefined

toSignal 是否應將來自 Observable 錯誤通道的錯誤拋回 RxJS，在那裡它們將被視為未捕獲的異常。

在實踐中，這意味著由 toSignal 回傳的訊號將永遠保持回傳最後一個好值，因為發生錯誤的可觀察者不會產生進一步的值。此選項模擬了 async 通道的行為。

equal

ValueEqualityFn<T> | undefined

一個比較函式，它定義了可觀察者發出的值的相等性。

如果提供了初始值，則針對該初始值執行相等性比較。
