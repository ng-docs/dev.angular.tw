程式碼

路由器事件，使你可以跟蹤路由器的生命週期。

API

  
    type Event = | NavigationStart  | NavigationEnd  | NavigationCancel  | NavigationError  | RoutesRecognized  | GuardsCheckStart  | GuardsCheckEnd  | RouteConfigLoadStart  | RouteConfigLoadEnd  | ChildActivationStart  | ChildActivationEnd  | ActivationStart  | ActivationEnd  | Scroll  | ResolveStart  | ResolveEnd  | NavigationSkipped

說明

路由器事件，使你可以跟蹤路由器的生命週期。

這些事件按以下順序發生：

  • NavigationStart：導向開始。
  • RouteConfigLoadStart：在路由器延遲載入路由設定之前。
  • RouteConfigLoadEnd：在路由已被延遲載入之後。
  • RoutesRecognized：當路由器解析 URL 並且路由被識別時。
  • GuardsCheckStart：當路由器開始路由的守衛階段時。
  • ChildActivationStart：當路由器開始啟用路由的子路由時。
  • ActivationStart：當路由器開始啟用路由時。
  • GuardsCheckEnd：當路由器成功完成路由的守衛階段時。
  • ResolveStart：當路由器開始路由的 resolve 階段時。
  • ResolveEnd：當路由器成功完成路由的 resolve 階段時。
  • ChildActivationEnd：當路由器完成啟用路由的子路由時。
  • ActivationEnd：當路由器完成啟用路由時。
  • NavigationEnd：當導向成功結束時。
  • NavigationCancel：當導向被取消時。
  • NavigationError：當導向由於意外錯誤而失敗時。
  • Scroll：當用戶捲動時。
