@angular/core

ng-template

element
程式碼

Angular 的 <ng-template> 元素定義了一個預設情況下不會渲染的樣板。

使用 <ng-template>，你可以定義樣板內容，這些內容只有在你直接或間接明確指示 Angular 執行渲染時才會被渲染，從而使你可以完全控制內容的顯示方式和時間。

請注意，如果你將內容包裹在 <ng-template> 中，而不指示 Angular 渲染它，則此類內容將不會顯示在頁面上。例如，請看下面的 HTML 程式碼，當處理它時，Angular 不會渲染短語 "Hip! Hip! Hooray!" 中的中間的 "Hip!"，因為周圍有 <ng-template>。

          
  <p>Hip!</p>  <ng-template>    <p>Hip!</p>  </ng-template>  <p>Hooray!</p>

用法說明

結構型指令

<ng-template> 的主要用途之一是儲存將由結構型指令使用的樣板內容。這些指令可以根據它們自己的邏輯新增和刪除樣板內容的副本。

當使用結構型指令簡寫形式時，Angular 會在幕後建立一個 <ng-template> 元素。

TemplateRef

<ng-template> 元素表示為 TemplateRef 類別的實例。

要將樣板的副本新增到 DOM，請將此物件傳遞給 ViewContainerRef 的 createEmbeddedView() 方法。

樣板變數

<ng-template> 元素可以在樣板中被引用，可以使用標準樣板變數。

這就是 <ng-template> 元素如何用作 ngIf 的 else 子句。

這樣的樣板變數可以與 ngTemplateOutlet 指令結合使用，以渲染 <ng-template> 標籤內定義的內容。

查詢

Query（例如 ViewChild）可以找到與 <ng-template> 元素關聯的 TemplateRef，以便可以以程式設計方式使用它；例如，將其傳遞給 ViewContainerRef 的 createEmbeddedView() 方法。

上下文

在 <ng-template> 標籤內，你可以引用周圍外部樣板中存在的變數。 此外，上下文物件可以與 <ng-template> 元素關聯。 這樣的物件包含可以透過樣板（let 和 as）宣告從樣板內容中訪問的變數。
