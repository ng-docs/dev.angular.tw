Angular 的 <ng-template> 元素定义了一个默认不渲染的模板。

使用 <ng-template>，你可以定义模板内容，只有当你（无论是直接还是间接）明确指示 Angular 渲染时，该内容才会被渲染，从而使你可以完全控制内容的显示方式和时机。

注意，如果你将内容包裹在 <ng-template> 中而没有指示 Angular 渲染它，则该内容不会出现在页面上。例如，参见以下 HTML 代码，处理它时，Angular 不会渲染短语 "Hip! Hip! Hooray!" 中间的 "Hip!"，因为它被 <ng-template> 包围。

          
              <p>Hip!</p>  <ng-template>    <p>Hip!</p>  </ng-template>  <p>Hooray!</p>

使用说明

结构型指令

<ng-template> 的主要用途之一是保存将由 结构型指令 使用的模板内容。这些指令可以根据它们自己的逻辑添加和删除模板内容的副本。

使用 结构型指令简写时，Angular 会在后台创建一个 <ng-template> 元素。

TemplateRef

<ng-template> 元素表示为 TemplateRef 类的实例。

要将模板的副本添加到 DOM 中，请将此对象传递给 ViewContainerRef 方法 createEmbeddedView()。

模板变量

<ng-template> 元素可以在模板中使用 标准模板变量引用。

这是 <ng-template> 元素作为 ngIf else 子句使用的方式。

这些模板变量可以与 ngTemplateOutlet 指令结合使用来渲染 <ng-template> 标签内定义的内容。

查询

某个 查询（例如 ViewChild）可以找到与 <ng-template> 元素关联的 TemplateRef，以便可以在程序中使用它；例如，将其传递给 ViewContainerRef 方法 createEmbeddedView()。

上下文

在 <ng-template> 标签内，你可以引用外部模板中存在的变量。此外，可以将一个上下文对象与 <ng-template> 元素关联。这样的对象包含的变量可以通过模板（ let 和 as）声明从模板内容中访问。
