interface Route {
}

定义单一路由的配置对象。 一组路由被收集在一个 Routes 数组中，以定义一个 Router 配置。 路由器尝试使用此对象中定义的配置选项，将给定 URL 的片段与每个路由匹配。

支持静态、参数化、重定向和通配符路由，以及自定义路由数据和求解方法。

有关详细的使用信息，请参阅 路由指南。

简单配置

以下路由指定在导航到 /team/11/user/bob 时，路由器会在其中创建带有 'User' 子组件的 'Team' 组件。

          
            [{  path: 'team/:id', component: Team,  children: [{    path: 'user/:name',    component: User  }]}]

多重出口

以下路由创建具有多个出口的兄弟组件。当导航到 /team/11(aux:chat/jim) 时，路由器会在 'Chat' 组件旁边创建 'Team' 组件。'Chat' 组件放置在 'aux' 出口中。

          
            [{  path: 'team/:id',  component: Team}, {  path: 'chat/:user',  component: Chat  outlet: 'aux'}]

通配符

以下路由使用通配符表示法来指定始终实例化的组件，无论你导航到何处。

          
            [{  path: '**',  component: WildcardComponent}]

重定向

以下路由在查找子路径时使用 redirectTo 属性忽略给定 URL 的一部分。

当导航到“/team/11/legacy/user/jim”时，路由器将 URL 段“/team/11/legacy/user/jim”更改为“/team/11/user/jim”，然后实例化包含 User 子组件的团队组件。

          
            [{  path: 'team/:id',  component: Team,  children: [{    path: 'legacy/user/:name',    redirectTo: 'user/:name'  }, {    path: 'user/:name',    component: User  }]}]

重定向路径可以是相对的（如本示例所示），也可以是绝对的。redirectTo 值更改为绝对 URL 段 '/user/:name'，则结果 URL 也是绝对 URL，'/user/jim'。

空路径

空路径路由配置可用于实例化不『消耗』任何 URL 段的组件。

在以下配置中，导航到 /team/11 时，路由器会实例化 'AllUsers' 组件。

          
            [{  path: 'team/:id',  component: Team,  children: [{    path: '',    component: AllUsers  }, {    path: 'user/:name',    component: User  }]}]

空路径路由可以有子路由。在下面的例子中，导航到 /team/11/user/jim 时，路由器会实例化包含用户组件的包装组件。

请注意，空路径路由会继承其父级的参数和数据。

          
            [{  path: 'team/:id',  component: Team,  children: [{    path: '',    component: WrapperCmp,    children: [{      path: 'user/:name',      component: User    }]  }]}]

匹配策略

默认的路径匹配策略是 'prefix'，这意味着路由器从左开始检查 URL 元素以查看 URL 是否与指定的路径匹配。比如，'/team/11/user' 与 'team/:id' 匹配。

          
            [{  path: '',  pathMatch: 'prefix', //default  redirectTo: 'main'}, {  path: 'main',  component: Main}]

你可以指定路径匹配策略 'full' 以确保路径覆盖整个未消耗的 URL。当重定向空路径路由时，这一点非常重要。否则，由于空路径是任何 URL 的前缀，路由器在导航到重定向目的地时会应用重定向，从而创建一个无限循环。

在以下示例中，提供 'full' pathMatch 策略可确保路由器当且仅当导航到 '/' 时应用重定向。

          
            [{  path: '',  pathMatch: 'full',  redirectTo: 'main'}, {  path: 'main',  component: Main}]

无组件路由

在以下示例中，'MainChild' 和 'AuxChild' 是同级。当导航到 'parent/10/(a//aux:b)' 时，该路由会实例化彼此相邻的主要子组件和 aux 子组件。为此，应用程序组件必须定义主要和辅助出口。

在以下示例中，“MainChild”和“AuxChild”是同级。当导航到 'parent/10/(a//aux:b)' 时，该路由会实例化彼此相邻的主子组件和辅助子组件。为此，应用程序组件必须定义主要的出口和辅助出口。

          
            [{   path: 'parent/:id',   children: [     { path: 'a', component: MainChild },     { path: 'b', component: AuxChild, outlet: 'aux' }   ]}]

路由器将无组件父路由的参数、数据和求解合并到子路由的参数、数据和求解中。

当使用空路径字符串定义子组件时，这特别有用，如下例所示。使用此配置，导航到 '/parent/10' 会创建主子组件和 aux 组件。

          
            [{   path: 'parent/:id',   children: [     { path: '', component: MainChild },     { path: '', component: AuxChild, outlet: 'aux' }   ]}]

惰性加载

延迟加载通过将应用拆分为多个捆绑包并按需加载它们来加快应用加载时间。 要使用延迟加载，请在 Route 对象中提供 loadChildren 属性，而不是 children 属性。

对于以下示例路由，路由器将使用浏览器原生导入系统按需惰性加载关联的模块。

          
            [{  path: 'lazy',  loadChildren: () => import('./lazy-route/lazy.module').then(mod => mod.LazyModule),}];

title

string | Type<Resolve<string>> | ResolveFn<string>

用于定义路由的页面标题。这个标题可以是一个静态字符串或实现了 Resolve 的 Injectable。

path

string

匹配的路径。不能与自定义 matcher 功能一起使用。使用路由器匹配表示法的 URL 字符串。可以是与任何 URL 匹配的通配符（**）（请参阅下面的使用说明）。默认值为 “/”（根路径）。

pathMatch

"prefix" | "full"

路径匹配策略，为 “prefix” 或 “full” 之一。默认为“prefix”。

默认情况下，路由器会从左侧检查 URL 元素以查看 URL 是否与给定路径匹配，并在匹配上某个配置时停止。重要的是，URL 的每个段仍然必须与某个配置匹配。例如，如果路由的子项（children）之一与段“user”匹配，则认为“/team/11/user”与前缀“team/:id”匹配。也就是说，URL '/team/11/user' 可以匹配 {path: 'team/:id', children: [{path: ':user', component: User}]} 配置项，但当 {path: 'team/:id', component: Team} 中没有子项（children 属性）时则不匹配。

路径匹配策略 'full' 匹配整个 URL。重定向空路径路由时，这样做很重要。否则，由于空路径是任何 URL 的前缀，因此路由器即使导航到重定向目标时也会应用重定向，从而创建一个无限循环。

matcher

UrlMatcher

自定义 URL 匹配功能。不能与 path 一起使用。

component

Type<any>

路径匹配时实例化的组件。如果要由其子路由指定组件，则可以为空。

loadComponent

() => Type<unknown> | Observable<Type<unknown> | DefaultExport<Type<unknown>>> | Promise<Type<unknown> | DefaultExport<Type<unknown>>>

指定惰性加载组件的对象。

redirectTo

string | RedirectFunction

一个 URL 或返回 URL 的函数，当路径匹配时重定向到该 URL。

如果 URL 以斜杠（/）开头或函数返回一个 UrlTree，则为绝对路径，否则相对于路径 URL。

RedirectFunction 在注入上下文中运行，因此它可以调用 inject 以获取任何必要的依赖项。

当不存在时，路由器不会重定向。

outlet

string

一个 RouterOutlet 对象的名称，当路径匹配时可以放置组件。

canActivate

any[]

一个 CanActivateFn 或 DI 令牌数组，用于查找 CanActivate() 处理程序，以确定当前用户是否被允许激活组件。默认情况下，任何用户都可以激活。

当使用函数而不是 DI 标记时，函数可以调用 inject 来获取任何所需的依赖项。此 inject 调用必须在同步上下文中完成。

canMatch

any[]

一个 CanMatchFn 或 DI 令牌数组，用于查找 CanMatch() 处理程序，以确定当前用户是否被允许匹配 Route。默认情况下，任何路由都可以匹配。

当使用函数而不是 DI 标记时，函数可以调用 inject 来获取任何所需的依赖项。此 inject 调用必须在同步上下文中完成。

canActivateChild

any[]

一个 CanActivateChildFn 或 DI 令牌数组，用于查找 CanActivateChild() 处理程序，以确定当前用户是否被允许激活组件的子组件。默认情况下，任何用户都可以激活子组件。

当使用函数而不是 DI 标记时，函数可以调用 inject 来获取任何所需的依赖项。此 inject 调用必须在同步上下文中完成。

canDeactivate

any[]

一个 CanDeactivateFn 或 DI 令牌数组，用于查找 CanDeactivate() 处理程序，以确定当前用户是否被允许停用组件。默认情况下，任何用户都可以停用。

当使用函数而不是 DI 标记时，函数可以调用 inject 来获取任何所需的依赖项。此 inject 调用必须在同步上下文中完成。

canLoad

any[]
@deprecated

请改用 canMatch

一个 CanLoadFn 或 DI 令牌数组，用于查找 CanLoad() 处理程序，以确定当前用户是否被允许加载组件。默认情况下，任何用户都可以加载。

当使用函数而不是 DI 标记时，函数可以调用 inject 来获取任何所需的依赖项。此 inject 调用必须在同步上下文中完成。

data

Data

通过 ActivatedRoute 提供给组件的附加开发者定义的数据。默认情况下，不会传递任何附加数据。

resolve

ResolveData

用于查找数据解析器的 DI 令牌映射表。参见 Resolve。

children

Routes

指定嵌套路由配置的一组子 Route 对象。

loadChildren

LoadChildrenCallback

一个对象，指定要惰性加载的子路由。

runGuardsAndResolvers

RunGuardsAndResolvers

何时在路由上运行警卫和解析器的策略。

激活或停用路由时，守卫和/或解析器将始终运行。当路由未更改时，默认行为与 paramsChange 相同。

paramsChange：当路径或路径参数更改时重新运行警卫和解析器。这不包括查询参数。此选项是默认值。

  • always：在每次执行时运行。
  • pathParamsChange：路径参数更改时重新运行保护器和解析器。这不会比较矩阵或查询参数。
  • paramsOrQueryParamsChange：在路径、矩阵或查询参数更改时运行。
  • pathParamsOrQueryParamsChange：当路径参数更改或查询参数更改时，重新运行保护器和解析器。这不包括矩阵参数。

providers

(EnvironmentProviders | Provider)[]

用于此 Route 及其 children 的 Provider 数组。

Router 将为此 Route 创建一个新的 EnvironmentInjector，并将其用于此 Route 及其 children。如果此路由还具有返回 NgModuleRef 的 loadChildren 函数，这个注入器将作为延迟加载模块的父级使用。
