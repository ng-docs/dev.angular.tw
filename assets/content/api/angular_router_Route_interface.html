@angular/router

Route

interface
code

一個設定物件，用於定義單個路由。一組路由收集在 Routes 陣列中，以定義 Router 設定。路由器嘗試將給定 URL 的片段與每個路由進行對應，使用此物件中定義的設定選項。

API

  
    interface Route {}

title

string | Type<Resolve<string>> | ResolveFn<string> | undefined

用於為路由定義頁面標題。它可以是靜態字串，也可以是實現了 Resolve 的 Injectable。

path

string | undefined

要對應的路徑。不能與自定義的 matcher 函式一起使用。 一個使用路由器對應語法的 URL 字串。 可以是一個萬用字元 (**)，以對應任何 URL（見下面的『用法說明』）。 預設為 "/"（根路徑）。

pathMatch

"prefix" | "full" | undefined

路徑對應策略，為 'prefix' 或 'full' 之一。 預設為 'prefix'。

預設情況下，路由器從左到右檢查 URL 元素，以檢視 URL 是否與給定的路徑對應，並在設定對應時停止。重要的是，URL 的每個段仍然必須有設定對應。例如，如果路由的子路由之一與段 'user' 對應，則 '/team/11/user' 對應字首 'team/:id'。也就是說，URL '/team/11/user' 對應設定 {path: 'team/:id', children: [{path: ':user', component: User}]}，但當沒有子路由時（如 {path: 'team/:id', component: Team}）則不對應。

路徑對應策略 'full' 針對整個 URL 進行對應。 當重導向空路徑路由時，這樣做很重要。 否則，由於空路徑是任何 URL 的字首，因此即使導向到重導向目標，路由器也會應用重導向，從而建立無限迴圈。

matcher

UrlMatcher | undefined

自定義 URL 對應函式。不能與 path 一起使用。

component

Type<any> | undefined

當路徑對應時要實例化的元件。 如果子路由指定了元件，則可以為空。

loadComponent

(() => Type<unknown> | Observable<Type<unknown> | DefaultExport<Type<unknown>>> | Promise<Type<unknown> | DefaultExport<Type<unknown>>>) | undefined

一個指定延遲載入元件的物件。

redirectTo

string | RedirectFunction | undefined

當路徑對應時，要重導向到的 URL 或回傳 URL 的函式。

如果 URL 以斜槓 (/) 開頭，或者函式回傳 UrlTree，則為絕對路徑，否則相對於路徑 URL。

RedirectFunction 在注入上下文中執行，因此它可以呼叫 inject 以獲取任何必要的依賴項。

如果不存在，路由器不會重導向。

outlet

string | undefined

RouterOutlet 物件的名稱，當路徑對應時，元件可以放置在該物件中。

canActivate

any[] | undefined

CanActivateFn 或 DI 令牌的陣列，用於查詢 CanActivate() 處理程式，以確定當前使用者是否被允許啟用元件。預設情況下，任何使用者都可以啟用。

當使用函式而不是 DI 令牌時，該函式可以呼叫 inject 以獲取任何必需的依賴項。這個 inject 呼叫必須在同步上下文中完成。

canMatch

any[] | undefined

CanMatchFn 或 DI 令牌的陣列，用於查詢 CanMatch() 處理程式，以確定當前使用者是否被允許對應 Route。預設情況下，任何路由都可以對應。

當使用函式而不是 DI 令牌時，該函式可以呼叫 inject 以獲取任何必需的依賴項。這個 inject 呼叫必須在同步上下文中完成。

canActivateChild

any[] | undefined

CanActivateChildFn 或 DI 令牌的陣列，用於查詢 CanActivateChild() 處理程式，以確定當前使用者是否被允許啟用元件的子元件。預設情況下，任何使用者都可以啟用子元件。

當使用函式而不是 DI 令牌時，該函式可以呼叫 inject 以獲取任何必需的依賴項。這個 inject 呼叫必須在同步上下文中完成。

canDeactivate

any[] | undefined

CanDeactivateFn 或 DI 令牌的陣列，用於查詢 CanDeactivate() 處理程式，以確定當前使用者是否被允許停用元件。預設情況下，任何使用者都可以停用。

當使用函式而不是 DI 令牌時，該函式可以呼叫 inject 以獲取任何必需的依賴項。這個 inject 呼叫必須在同步上下文中完成。

canLoad

any[] | undefined

CanLoadFn 或 DI 令牌的陣列，用於查詢 CanLoad() 處理程式，以確定當前使用者是否被允許載入元件。預設情況下，任何使用者都可以載入。

當使用函式而不是 DI 令牌時，該函式可以呼叫 inject 以獲取任何必需的依賴項。這個 inject 呼叫必須在同步上下文中完成。

data

Data | undefined

透過 ActivatedRoute 提供給元件的、由開發人員定義的額外資料。預設情況下，不傳遞任何額外資料。

resolve

ResolveData | undefined

用於查詢資料求解器的 DI 令牌的對映表。請參閱 Resolve。

children

Routes | undefined

一個 Route 子物件陣列，用於指定巢狀路由設定。

loadChildren

LoadChildrenCallback | undefined

一個用於指定延遲載入的子路由的物件。

runGuardsAndResolvers

RunGuardsAndResolvers | undefined

一個策略，用於指定何時在路由上執行守衛和求解器。

當路由被啟用或停用時，守衛和/或求解器將始終執行。當路由未更改時，預設行為與 paramsChange 相同。

paramsChange：當路徑或路徑引數更改時，重新執行守衛和求解器。這不包括查詢引數。此選項為預設選項。

  • always：每次執行時都執行。
  • pathParamsChange：當路徑引數更改時，重新執行守衛和求解器。這不會比較矩陣引數或查詢引數。
  • paramsOrQueryParamsChange：當路徑、矩陣或查詢引數更改時運行。
  • pathParamsOrQueryParamsChange：當路徑引數更改或查詢引數已更改時，重新執行守衛和求解器。這不包括矩陣引數。

providers

(Provider | EnvironmentProviders)[] | undefined

一個 Provider 陣列，用於此 Route 及其 children。

Router 將為此 Route 建立一個新的 EnvironmentInjector，並將其用於此 Route 及其 children。如果此路由還具有回傳 NgModuleRef 的 loadChildren 函式，則此注入器將用作延遲載入模組的父注入器。

描述

一個設定物件，用於定義單個路由。一組路由收集在 Routes 陣列中，以定義 Router 設定。路由器嘗試將給定 URL 的片段與每個路由進行對應，使用此物件中定義的設定選項。

支援靜態路由、引數化路由、重導向路由和萬用字元路由，以及自定義路由資料和求解方法。

有關詳細用法訊息，請參閱路由指南。

用法說明

簡單設定

以下路由指定，例如，當導向到 /team/11/user/bob 時，路由器將建立 'Team' 元件，並在其中包含 'User' 子元件。

          
[{  path: 'team/:id', component: Team,  children: [{    path: 'user/:name',    component: User  }]}]

多出口

以下路由使用多個出口建立同級元件。當導向到 /team/11(aux:chat/jim) 時，路由器將建立 'Team' 元件，並在其旁邊建立 'Chat' 元件。'Chat' 元件被放置到 'aux' 出口中。

          
[{  path: 'team/:id',  component: Team}, {  path: 'chat/:user',  component: Chat  outlet: 'aux'}]

萬用字元

以下路由使用萬用字元表示法來指定一個元件，無論導向到何處，該組件始終會被實例化。

          
[{  path: '**',  component: WildcardComponent}]

重導向

以下路由使用 redirectTo 屬性在查詢子路徑時忽略給定 URL 的片段。

當導向到 '/team/11/legacy/user/jim' 時，路由器將 URL 片段 '/team/11/legacy/user/jim' 更改為 '/team/11/user/jim'，然後實例化 Team 元件，並在其中包含 User 子元件。

          
[{  path: 'team/:id',  component: Team,  children: [{    path: 'legacy/user/:name',    redirectTo: 'user/:name'  }, {    path: 'user/:name',    component: User  }]}]

重導向路徑可以是相對路徑（如本例所示），也可以是絕對路徑。如果我們將範例中的 redirectTo 值更改為絕對 URL 片段 '/user/:name'，則結果 URL 也是絕對路徑 '/user/jim'。

空路徑

空路徑路由設定可用於實例化不『消耗』任何 URL 片段的元件。

在以下設定中，當導向到 /team/11 時，路由器將實例化 'AllUsers' 元件。

          
[{  path: 'team/:id',  component: Team,  children: [{    path: '',    component: AllUsers  }, {    path: 'user/:name',    component: User  }]}]

空路徑路由可以有子路由。在以下範例中，當導向到 /team/11/user/jim 時，路由器將實例化套件裝器元件，並在其中包含使用者元件。

請注意，空路徑路由會繼承其父路由的引數和數據。

          
[{  path: 'team/:id',  component: Team,  children: [{    path: '',    component: WrapperCmp,    children: [{      path: 'user/:name',      component: User    }]  }]}]

對應策略

預設的路徑對應策略是 'prefix'，這意味著路由器從左側檢查 URL 元素，以檢視 URL 是否與指定的路徑對應。例如，'/team/11/user' 對應 'team/:id'。

          
[{  path: '',  pathMatch: 'prefix', //default  redirectTo: 'main'}, {  path: 'main',  component: Main}]

你可以指定路徑對應策略 'full'，以確保路徑覆蓋整個未消耗的 URL。在重導向空路徑路由時，這樣做很重要。否則，由於空路徑是任何 URL 的字首，因此即使導向到重導向目標，路由器也會應用重導向，從而建立無限迴圈。

在以下範例中，提供 'full' pathMatch 策略可確保路由器僅在導向到 '/' 時才應用重導向。

          
[{  path: '',  pathMatch: 'full',  redirectTo: 'main'}, {  path: 'main',  component: Main}]

無元件路由

你可以在同級元件之間共享引數。例如，假設兩個同級元件應該彼此相鄰，並且它們都需要一個 ID 引數。你可以使用一個未在頂層指定元件的路由來實現此目的。

在以下範例中，'MainChild' 和 'AuxChild' 是同級元件。當導向到 'parent/10/(a//aux:b)' 時，路由會實例化主子元件和輔助子元件，使它們彼此相鄰。為了使此方法有效，應用元件必須定義主出口和輔助出口。

          
[{   path: 'parent/:id',   children: [     { path: 'a', component: MainChild },     { path: 'b', component: AuxChild, outlet: 'aux' }   ]}]

路由器將無元件父路由的引數、資料和求解合併到子路由的引數、資料和求解中。

當子元件使用空路徑字串定義時，這尤其有用，如以下範例所示。使用此設定，導向到 '/parent/10' 將建立主子元件和輔助元件。

          
[{   path: 'parent/:id',   children: [     { path: '', component: MainChild },     { path: '', component: AuxChild, outlet: 'aux' }   ]}]

延遲載入

延遲載入透過將應用拆分為多個捆綁包並按需載入它們來加快應用載入時間。要使用延遲載入，請在 Route 物件中提供 loadChildren 屬性，而不是 children 屬性。

給定以下範例路由，路由器將使用瀏覽器原生匯入系統按需延遲載入關聯的模組。

          
[{  path: 'lazy',  loadChildren: () => import('./lazy-route/lazy.module').then(mod => mod.LazyModule),}];
