withRequestsMadeViaParent

HttpFeature<HttpFeatureKind.RequestsMadeViaParent>

設定當前的 HttpClient 實例，使其透過父注入器的 HttpClient 而不是直接發起請求。

預設情況下，provideHttpClient 將其注入器中的 HttpClient 設定為獨立實例。例如，即使 HttpClient 在父注入器中設定了一個或多個攔截器，它們也不會攔截透過此實例發出的請求。

啟用此選項後，一旦請求透過當前注入器的攔截器，它將被委託給父注入器的 HttpClient 鏈，而不是直接排程，並且父設定中的攔截器將應用於該請求。

如果在注入器層級結構中有多個 HttpClient 實例，則可以在多個級別使用 withRequestsMadeViaParent，這將導致請求『冒泡』上升，直到到達根級別或未設定此選項的 HttpClient。
