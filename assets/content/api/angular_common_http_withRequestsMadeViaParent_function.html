将当前 HttpClient 实例配置为通过父注入器的 HttpClient 发出请求，而不是直接发出。

默认情况下， provideHttpClient 在其注入器中配置 HttpClient 为独立实例。例如，即使在父注入器中配置了一个或多个拦截器，它们也不会拦截通过此实例发出的请求。

启用此选项后，一旦请求通过当前注入器的拦截器处理完毕，它将被委托给父注入器的 HttpClient 链，而不是直接分发，并且父配置中的拦截器将应用于该请求。

如果注入器层次结构中有多个 HttpClient 实例，可以在多个层级使用 withRequestsMadeViaParent，这将导致请求“冒泡”直到到达根层级或一个未配置此选项的 HttpClient。
