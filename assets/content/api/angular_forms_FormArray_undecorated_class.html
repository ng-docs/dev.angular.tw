events

Observable<ControlEvent<TValue>>

一個多播可觀察物件，每當控制元件的狀態更改時都會發出事件。 它會在值、狀態、pristine 或 touched 更改時發出。

注意：在值更改時，發出操作在此控制元件的值更新後立即發生。父控制元件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍後更新，因此從此事件的回呼中訪問父控制元件的值（使用 value 屬性）可能會導致獲得尚未更新的值。請改為訂閱父控制元件的 events。 對於其他事件型別，事件在父控制元件更新後發出。

valueChanges

Observable<TValue>

一個多播可觀察物件，每當控制元件的值在 UI 中或以程式設計方式更改時，都會發出一個事件。 每次您呼叫 enable() 或 disable() 而沒有傳遞 {emitEvent: false} 作為函式引數時，它也會發出一個事件。

注意：發出操作在此控制元件的值更新後立即發生。父控制元件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍後更新，因此從此事件的回呼中訪問父控制元件的值（使用 value 屬性）可能會導致獲得尚未更新的值。請改為訂閱父控制元件的 valueChanges 事件。

statusChanges

AbstractControl<any, any>

檢索此控制元件的頂層祖先。

描述

跟蹤 FormControl、FormGroup 或 FormArray 實例陣列的值和有效性狀態。

FormArray 將每個子 FormControl 的值聚合到一個陣列中。 它透過縮減其子級的狀態值來計算自身的狀態。例如，如果 FormArray 中的一個控制元件無效，則整個陣列都將變為無效。

FormArray 接受一個泛型引數，它是內部控制元件的型別。 如果需要異構陣列，請使用 UntypedFormArray。

FormArray 是用於在 Angular 中定義表單的四個基本元素之一，另外三個是 FormControl、FormGroup 和 FormRecord。

用法說明

建立一個表單控制元件陣列 

          
const arr = new FormArray([  new FormControl('Nancy', Validators.minLength(2)),  new FormControl('Drew'),]);console.log(arr.value);   // ['Nancy', 'Drew']console.log(arr.status);  // 'VALID'

建立一個帶有陣列級驗證器的表單陣列

你可以包含陣列級驗證器和非同步驗證器。當你想執行驗證，且此驗證需要考慮多個子控制元件的值時，這些驗證器會派上用場。

這兩種型別的驗證器可以分別作為第二和第三個引數單獨傳入，也可以作為選項物件的一部分一起傳入。

          
const arr = new FormArray([  new FormControl('Nancy'),  new FormControl('Drew')], {validators: myValidator, asyncValidators: myAsyncValidator});

為表單陣列中的所有控制元件設定 updateOn 屬性（Property）。

選項物件用於為每個子控制元件的 updateOn 屬性（Property）設定預設值。如果在陣列級別將 updateOn 設定為 'blur'，則所有子控制元件都預設為 'blur'，除非子控制元件已明確指定了不同的 updateOn 值。

          
const arr = new FormArray([   new FormControl()], {updateOn: 'blur'});

在表單陣列中新增或移除控制元件

要更改陣列中的控制元件，請使用 FormArray 本身中的 push、insert、removeAt 或 clear 方法。這些方法確保在表單的分層結構中正確跟蹤這些控制元件。不要直接修改用於實例化 FormArray 的 AbstractControl 陣列，因為這會導致奇怪和意外的行為，例如變更檢測中斷。
