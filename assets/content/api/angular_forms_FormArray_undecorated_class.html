跟踪一组 FormControl、 FormGroup或 FormArray实例的值和有效状态。

FormArray将每个子 FormControl的值聚合到数组中。它通过减少其子项的状态值来计算其状态。例如，如果 FormArray中的一个控件无效，则整个数组也将无效。

FormArray接受一个泛型参数，即内部控件的类型。如果需要一个异构数组，请使用 UntypedFormArray 。

FormArray是Angular中用于定义表单的四个基本构建块之一，其他三个是 FormControl、 FormGroup和 FormRecord。

创建一个表单控件数组

          
            const arr = new FormArray([  new FormControl('Nancy', Validators.minLength(2)),  new FormControl('Drew'),]);console.log(arr.value);   // ['Nancy', 'Drew']console.log(arr.status);  // 'VALID'

创建带有数组级验证器的表单数组

你包括数组级验证器和异步验证器。当你要执行考虑多个子控件的值的验证时，它们会派上用场。

这两种类型的验证器分别作为第二个和第三个 arg 单独传入，或作为 options 对象的一部分一起传入。

          
            const arr = new FormArray([  new FormControl('Nancy'),  new FormControl('Drew')], {validators: myValidator, asyncValidators: myAsyncValidator});

为表单数组中的所有控件设置 updateOn 属性

options 对象用于为每个子控件的 updateOn 属性设置默认值。如果你在数组级别将 updateOn 设置为 'blur'，则所有子控件默认为 'blur'，除非子项显式指定不同的 updateOn 值。

          
            const arr = new FormArray([   new FormControl()], {updateOn: 'blur'});

在表单数组中添加或删除控件

要更改数组中的控件，请在 FormArray本身中使用 push、 insert、 removeAt或 clear方法。这些方法确保正确跟踪表单层次结构中的控件。不要直接修改用于实例化 FormArray的 AbstractControl数组，否则可能导致诸如破损的变更检测等奇怪和意外的行为。

events

Observable<ControlEvent<TValue>>

一个多播的可观察者，每当控件的状态发生变化时发出一个事件。它发出值、状态、原始或已触碰变化的事件。

注意：在值更改时，发出事件的时间会紧跟在此控件值更新后。父控件的值（例如，如果此FormControl是FormGroup的一部分）稍后更新，因此在此事件的回调中访问父控件的值（使用 value属性）可能导致获取尚未更新的值。请改为订阅父控件的 events。对于其他事件类型，事件会在父控件更新后发出。

valueChanges

Observable<TValue>

一个多播的可观察者，每次控件的值在 UI 或程序中发生变化时都会发出事件。每次你调用 enable() 或 disable() 而不传递 {emitEvent: false} 作为函数参数时，它也会发出事件。

注意：发射发生在更新此控件的值之后。父控件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍后更新，因此从该事件的回调中访问父控件的值（使用 value 属性）可能会导致获得一个值尚未更新。改为订阅父控件的 valueChanges 事件。

禁用此控件。这意味着该控件在表单验证检查时会被豁免，并且从其父控件的聚合值中排除它的值。它的状态是 DISABLED。

如果该控件有子控件，则所有子控件也会被禁用。

@paramopts{ onlySelf?: boolean; emitEvent?: boolean; }

在该控件被禁用之后，该配置项决定如何传播更改以及发出事件。

  • onlySelf：当为 true 时，只标记此控件。当为 false 或未提供时，标记所有直接祖先。默认值为 false。
  • emitEvent：当为true或未提供（默认值）时， statusChanges、 valueChanges和 events可观察者会在控件被禁用时发出带有最新状态和值的事件。当为false时，不会发出事件。
@returnsvoid

enable

void

启用该控件。这意味着该控件包含在有效性检查中，并会出现在其父控件的聚合值中。它的状态会根据它的值和验证器而重新计算。

默认情况下，如果该控件具有子控件，则所有子控件都会被启用。

Usage notes

例如，对于以下 FormGroup：

          
            form = new FormGroup({  address: new FormGroup({ street: new FormControl() })});

从根表单到 'street' 控件的路径是 'address' -> 'street'。

调用此方法有两种形式：

  1. 控件名称的字符串数组，如 ['address', 'street']
  2. 以一个字符串表示的句号分隔的控件名称列表，如 'address.street'
