interface RouterOutletContract {
}

定义用于 Router 的组件入口开发契约的接口。

出口充当占位符，Angular 会根据当前的路由器状态动态填充。

路由器插座应该通过 ChildrenOutletContexts#onChildOutletCreated 在 Router 中注册自己，并在 ChildrenOutletContexts#onChildOutletDestroyed 中注销。当 Router 识别到匹配的 Route 时，它会在 ChildrenOutletContexts 中查找已注册的插座并激活它。

isActivated

boolean

给定的插座是否已激活。

如果插座有活动组件，则认为它是“激活的”。

component

Object

已激活组件的实例；如果未激活插座，则为 null。

activatedRouteData

Data

ActivatedRoute 快照的 Data。

activatedRoute

ActivatedRoute

插座的 ActivatedRoute，如果插座未激活则为 null。

activateWith

void

当插座需要激活（创建组件）时由 Router 调用。

@returnsvoid

deactivate

void

销毁当前激活的组件的请求。

当 RouteReuseStrategy 指示某个 ActivatedRoute 应该被移除但存储以便以后复用而不是销毁时， Router 将调用 detach。

@returnsvoid

detach

ComponentRef<unknown>

当 RouteReuseStrategy 指示分离子树时调用。

这类似于 deactivate，但激活的组件 不应 被销毁。相反，它会被返回，以便稍后可以通过 attach 方法重新附加它。

实例化新组件时发出 activate 事件

EventEmitter<unknown>

当组件被销毁时发出 deactivate 事件。

attachEvents

EventEmitter<unknown>

当 RouteReuseStrategy 指示重新附着先前分离的子树时，发出一个附着的组件实例。

EventEmitter<unknown>

当 RouteReuseStrategy 指示分离子树时，发出一个分离的组件实例。

supportsBindingToComponentInputs

true

用于指示插座能够将 Router 的数据绑定到插座组件的输入中。

当此值为 undefined 或 false 并且开发者使用 withComponentInputBinding 选择加入该特性时，如果在应用中使用此插座，开发模式下会记录一个警告。
