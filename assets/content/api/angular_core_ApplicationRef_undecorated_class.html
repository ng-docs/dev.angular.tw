使用說明

啟動過程

當啟動元件時，Angular 會將其掛載到目標 DOM 元素上，並啟動自動變更檢測。可以使用 rootSelectorOrNode 引數提供目標 DOM 元素。

如果未提供目標 DOM 元素，Angular 將嘗試在頁面上使用正在啟動的元件的 selector 查詢一個（使用第一個對應的元素）。

tick

void

呼叫此方法以顯式處理變更檢測及其副作用。

在開發模式下，tick() 還會執行第二個變更檢測週期，以確保未檢測到更多更改。如果在第二個週期中檢測到其他更改，則應用中的繫結具有無法在單個變更檢測過程中求解的副作用。 在這種情況下，Angular 會丟擲一個錯誤，因為 Angular 應用在其中只能有一個變更檢測過程，所有變更檢測都必須在此過程中完成。

使用須知

isStable 範例和注意事項

請注意關於 isStable 的兩個要點，如下麵的範例所示：

  • 如果在應用啟動時啟動任何型別的週期性非同步任務（例如，輪詢過程，使用 setInterval、setTimeout 或使用 RxJS 運算子（如 interval）啟動），則應用將永遠不會穩定；
  • isStable 可觀察者在 Angular Zone 之外執行。

假設你啟動了一個週期性任務（此處使用 RxJS interval 遞增計數器），同時訂閱了 isStable。

          
constructor(appRef: ApplicationRef) {  appRef.isStable.pipe(     filter(stable => stable)  ).subscribe(() => console.log('App is stable now');  interval(1000).subscribe(counter => console.log(counter));}

在此範例中，isStable 將永遠不會發出 true，並且永遠不會記錄跟蹤訊息『App is stable now』。

如果要在應用穩定時執行某些操作，則必須等待應用穩定後再啟動輪詢過程。

          
constructor(appRef: ApplicationRef) {  appRef.isStable.pipe(    first(stable => stable),    tap(stable => console.log('App is stable now')),    switchMap(() => interval(1000))  ).subscribe(counter => console.log(counter));}

在此範例中，將記錄跟蹤訊息『App is stable now』，然後計數器開始每秒遞增。

另請注意，此可觀察者在 Angular Zone 之外執行，這意味著訂閱此可觀察者的程式碼不會觸發變更檢測。

假設你不是記錄計數器值，而是更新元件的欄位並在其範本中顯示它。

          
constructor(appRef: ApplicationRef) {  appRef.isStable.pipe(    first(stable => stable),    switchMap(() => interval(1000))  ).subscribe(counter => this.value = counter);}

由於 isStable 可觀察者在 Zone 之外執行，因此 value 欄位將正確更新，但範本不會重新整理！

你必須手動觸發變更檢測以更新範本。

          
constructor(appRef: ApplicationRef, cd: ChangeDetectorRef) {  appRef.isStable.pipe(    first(stable => stable),    switchMap(() => interval(1000))  ).subscribe(counter => {    this.value = counter;    cd.detectChanges();  });}

或者使訂閱回呼在 Zone 內部執行。

          
constructor(appRef: ApplicationRef, zone: NgZone) {  appRef.isStable.pipe(    first(stable => stable),    switchMap(() => interval(1000))  ).subscribe(counter => zone.run(() => this.value = counter));}
