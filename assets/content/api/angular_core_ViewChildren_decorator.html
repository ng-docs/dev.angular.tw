說明

Property 裝飾器，用於設定檢視查詢。

用於從檢視 DOM 中獲取元素或指令的 QueryList。 每當新增、刪除或移動子元素時，查詢清單都會更新，並且查詢清單的 changes 可觀察者將發出一個新值。

檢視查詢是在 ngAfterViewInit 回呼被呼叫之前設定的。

元資料 Property：

  • selector - 用於查詢的指令類型或名稱。
  • read - 用於從查詢到的元素中讀取不同的令牌。
  • emitDistinctChangesOnly - 僅當 QueryList 結果已更改時， QueryList#changes 可觀察者才會發出新值。當 false 時，即使 QueryList 沒有更改，changes 可觀察者也可能會發出。 ** 注意：*** 此設定選項已棄用，它將被永久設定為 true 並在 Angular 的未來版本中刪除。

支援以下選擇器。

  • 任何帶有 @Component 或 @Directive 裝飾器的類別
  • 範本引用變數（字串形式），例如，查詢 <my-component #cmp></my-component>，使用 @ViewChildren('cmp')
  • 當前元件的子元件樹中定義的任何提供者（例如，@ViewChildren(SomeService) someService!: SomeService）
  • 透過字串令牌定義的任何提供者（例如，@ViewChildren('someToken') someTokenVal!: any）
  • 一個 TemplateRef（例如，查詢 <ng-template></ng-template>，使用 @ViewChildren(TemplateRef) template;）

此外，多個字串選擇器可以用逗號分隔（例如，@ViewChildren('cmp1,cmp2')）

read 支援以下值：

  • 任何帶有 @Component 或 @Directive 裝飾器的類別
  • 由本查詢的 selector 對應的元件的注入器上定義的任何提供者
  • 透過字串令牌定義的任何提供者（例如，{provide: 'token', useValue: 'val'}）
  • TemplateRef、ElementRef 和 ViewContainerRef

import {AfterViewInit, Component, Directive, QueryList, ViewChildren} from '@angular/core';@Directive({  selector: 'child-directive',  standalone: false,})class ChildDirective {}@Component({  selector: 'someCmp',  templateUrl: 'someCmp.html',  standalone: false,})class SomeCmp implements AfterViewInit {  @ViewChildren(ChildDirective) viewChildren!: QueryList<ChildDirective>;  ngAfterViewInit() {    // viewChildren is set  }}
