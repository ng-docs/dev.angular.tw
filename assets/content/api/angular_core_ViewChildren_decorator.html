用于配置视图查询的参数装饰器。

用于从视图 DOM 中获取元素或指令的 QueryList。 每当一个子元素被添加、移除或移动时，查询列表将会更新， 并且查询列表的变化可观察者将会发出新值。

在调用 ngAfterViewInit 回调之前设置视图查询。

元数据属性：

  • selector - 要查询的指令类型或名称。
  • read - 用于从查询到的元素中读取不同的令牌。
  • emitDistinctChangesOnly - 此 QueryList#changes 可观察者只有在 QueryList 结果发生变化时才会发出新值。当 false 时，即使 QueryList 未发生变化， changes 可观察者也可能会发出新值。 ** 注意：*** 此配置选项已 弃用，在未来的 Angular 版本中将永久设置为 true 并移除。

支持下列选择器：。

  • 任何带有 @Component 或 @Directive 装饰器的类
  • 作为字符串的模板引用变量（例如，查询 <my-component #cmp></my-component> 与 @ViewChildren('cmp')）
  • 当前组件的子组件树中定义的任何提供者（例如 @ViewChildren(SomeService) someService!: SomeService）
  • 通过字符串令牌定义的任何提供者（例如 @ViewChildren('someToken') someTokenVal!: any）
  • 一个 TemplateRef（例如查询 <ng-template></ng-template> 与 @ViewChildren(TemplateRef) template;）

另外，可以用逗号分隔多个字符串选择器（例如 @ViewChildren('cmp1,cmp2')）

read 支持以下值：

  • 任何带有 @Component 或 @Directive 装饰器的类
  • 在此查询的 selector 匹配的组件注入器上定义的任何提供程序
  • 通过字符串标记定义的任何提供程序（例如 {provide: 'token', useValue: 'val'}）
  • TemplateRef、 ElementRef 和 ViewContainerRef
