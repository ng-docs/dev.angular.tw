AfterRenderPhase

列舉
已棄用
程式碼

在此階段執行 afterRender 或 afterNextRender 回呼。

API

  
    enum AfterRenderPhase {}

EarlyRead

對於只需要在後續的 AfterRenderPhase.Write 回呼之前從 DOM 中讀取的回呼，請使用 AfterRenderPhase.EarlyRead，例如，執行瀏覽器本身不支援的自定義佈局。如果讀取可以等到寫入階段之後，則首選 AfterRenderPhase.Read 階段。絕不要在此階段寫入 DOM。

使用此值可能會降低效能。 相反，如果可能，請首選使用內建瀏覽器功能。

寫入

對於僅寫入 DOM 的回呼，請使用 AfterRenderPhase.Write。絕不要在此階段從 DOM 中讀取。

MixedReadWrite

對於從 DOM 讀取或寫入 DOM，但尚未重構為使用不同階段的回呼，請使用 AfterRenderPhase.MixedReadWrite。如果可以將工作分配到其他階段，則絕不要使用此階段。

使用此值會顯著降低效能。 相反，請首選將工作分配到適當的階段回呼中。

讀取

對於僅從 DOM 中讀取的回呼，請使用 AfterRenderPhase.Read。絕不要在此階段寫入 DOM。

描述

在此階段執行 afterRender 或 afterNextRender 回呼。

同一階段中的回呼按照註冊順序執行。各階段在每次渲染後按以下順序執行：

  1. AfterRenderPhase.EarlyRead
  2. AfterRenderPhase.Write
  3. AfterRenderPhase.MixedReadWrite
  4. AfterRenderPhase.Read

Angular 無法驗證或強制執行階段是否被正確使用，而是依賴於每位開發者來遵循為每個值記錄的指南，並仔細選擇合適的階段，並在必要時重構其程式碼。透過這樣做，Angular 能夠更好地最大限度地減少與手動 DOM 訪問相關的效能下降，從而確保為你的應用或函式庫的最終使用者提供最佳體驗。
