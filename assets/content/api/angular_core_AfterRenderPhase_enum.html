}

运行 afterRender 或 afterNextRender 回调的阶段。

同一阶段中的回调按注册顺序运行。每次渲染后，阶段按以下顺序运行：

  1. AfterRenderPhase.EarlyRead
  2. AfterRenderPhase.Write
  3. AfterRenderPhase.MixedReadWrite
  4. AfterRenderPhase.Read

Angular 无法验证或强制要求正确使用阶段，而是依赖每个开发者遵循为每个值记录的指南，并仔细选择适当的阶段，必要时重构自己的代码。通过这样做，Angular 能够更好地减少与手动 DOM 访问相关的性能下降，确保你的应用或库的最终用户获得最佳体验。

EarlyRead

使用 AfterRenderPhase.EarlyRead 进行仅需在后续 AfterRenderPhase.Write 回调之前 读取 DOM 的回调，例如执行浏览器本身不支持的自定义布局。 绝不要在可以写入 DOM 的回调中使用该阶段，或在 AfterRenderPhase.Read 足够时使用。

使用此值会降低性能。相反，尽可能使用内置的浏览器功能。

写入

使用 AfterRenderPhase.Write 进行仅 写入 DOM 的回调。 绝不要在可以读取 DOM 的回调中使用该阶段。

MixedReadWrite

使用 AfterRenderPhase.MixedReadWrite 进行读取或写入 DOM 的回调，这些回调还没有重构为使用不同的阶段。 绝不要在可以使用不同阶段的回调中使用该阶段。

使用此值会 显著降低性能。相反，优先重构为使用更具体阶段的多个回调。

读取

使用 AfterRenderPhase.Read 进行仅 读取 DOM 的回调。 绝不要在可以写入 DOM 的回调中使用该阶段。
