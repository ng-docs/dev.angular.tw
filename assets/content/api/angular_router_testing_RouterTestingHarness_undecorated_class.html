触发 Router 导航并等待其完成。

为使用工具创建的根组件带有 RouterOutlet，用于渲染 Route 组件。根组件在同一测试中后续调用 navigateForTest 时被复用。

当测试具有拒绝导航的守卫的 Routes 时， RouterOutlet 可能未激活， activatedComponent 可能为 null。

触发路由器导航并等待它完成。

为测试工具创建的带有 RouterOutlet 的根组件用于渲染 Route 组件。

{@example router/testing/test/router_testing_harness_examples.spec.ts region='RoutedComponent'}

在对 navigateByUrl 后续调用中，根组件在同一测试中重复使用。

此函数还使得测试依赖 ActivatedRoute 数据的组件变得更容易。

{@example router/testing/test/router_testing_harness_examples.spec.ts region='ActivatedRoute'}

@paramurlstring

导航的目标。传递给 Router.navigateByUrl。

@paramrequiredRoutedComponentTypeType<T>

导航完成后， RouterOutlet 的已激活组件所需的类型。如果 outlet 未激活或激活了不同的组件，此函数将抛出错误。

@returnsPromise<T>

create

Promise<RouterTestingHarness>

创建一个 RouterTestingHarness 实例。

RouterTestingHarness 还会创建其自己的根组件，带有一个 RouterOutlet，用于渲染路由组件。

如果实例已被创建，则抛出错误。使用此测试工具还需要在 ModuleTeardownOptions 中设置 destroyAfterEach: true。

@paraminitialUrlstring

在返回工具之前触发导航的目标。
