navigateByUrl

2 個過載

觸發 Router 導向並等待其完成。

帶有為該工具建立的 RouterOutlet 的根元件用於渲染 Route 元件。在後續對 navigateForTest 的呼叫中，根元件將在同一測試中複用。

當測試帶有拒絕導向的守衛的 Routes 時，RouterOutlet 可能未啟用，並且 activatedComponent 可能為 null。

觸發路由器導向並等待其完成。

帶有為該工具建立的 RouterOutlet 的根元件用於渲染 Route 元件。

{@example router/testing/test/router_testing_harness_examples.spec.ts region='RoutedComponent'}

在同一測試中後續呼叫 navigateByUrl 時，根元件會被複用。

此函式還使測試依賴於 ActivatedRoute 資料的元件變得更容易。

{@example router/testing/test/router_testing_harness_examples.spec.ts region='ActivatedRoute'}

@paramurlstring

導向的目標。傳遞給 Router.navigateByUrl。

@paramrequiredRoutedComponentTypeType<T>

導向完成後，RouterOutlet 的已啟用元件的必要型別。如果出口未啟用或啟用了不同的元件，此函式將丟擲一個錯誤。

@returnsPromise<T>

create

Promise<RouterTestingHarness>

建立一個 RouterTestingHarness 實例。

RouterTestingHarness 還會建立自己的帶有 RouterOutlet 的根元件，以便渲染路由元件。

如果已建立實例，則丟擲錯誤。 使用此工具還需要在 ModuleTeardownOptions 中設定 destroyAfterEach: true
