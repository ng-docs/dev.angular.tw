表示可以将一个或多个视图附着到组件中的容器。

可以包含 宿主视图（通过调用 createComponent() 方法实例化组件创建），以及 内嵌视图（通过调用 createEmbeddedView() 方法实例化 TemplateRef 创建）。

一个视图容器实例可以包含其他视图容器，创建一个视图层次结构。

下面的示例演示了如何使用 createComponent 函数动态创建一个 ComponentRef 实例并将其附着到 ApplicationRef 上，使其被包含在变更检测周期中。

注：该示例使用了独立组件，但该函数也可用于非独立组件（在 NgModule 中声明）。

This is the main content.</main>`,})class AppComponent {  vcr = inject(ViewContainerRef);  ngAfterViewInit() {    const compRef = this.vcr.createComponent(DynamicComponent);    compRef.changeDetectorRef.detectChanges();  }}

element

ElementRef<any>

锚点元素用来指定本容器在父容器视图中的位置。 每个视图容器都只能有一个锚点元素，每个锚点元素也只能属于一个视图容器。

附着到此容器的视图的根元素在渲染视图中成为锚元素的兄弟元素。

通过在元素上放置一个注入了 ViewContainerRef 的 Directive，或者使用 ViewChild 查询，访问元素的 ViewContainerRef。

createComponent

2 overloads

实例化一个组件并将其宿主视图插入到本容器中。

@paramcomponentTypeType<C>

要使用的组件类型。

@paramoptions{ index?: number; injector?: Injector; ngModuleRef?: NgModuleRef<unknown>; environmentInjector?: EnvironmentInjector | NgModuleRef<unknown>; projectableNodes?: Node[][]; }

包含额外参数的对象：

  • index：将新组件的宿主视图插入此容器的索引。如果未指定，则将新视图作为最后一个条目附加。
  • 注入器：用作新组件的父级的注入器。
  • ngModuleRef：组件的 NgModule 的 NgModuleRef，你几乎总是应该提供此参数，以确保组件实例化时所有预期的提供者都可用。
  • environmentInjector：一个 EnvironmentInjector，将提供组件的环境。你几乎总是应该提供此参数，以确保组件实例化时所有预期的提供者都可用。此选项旨在替代 ngModuleRef 参数。
  • projectableNodes：应该通过新组件实例的<ng-content>投影的 DOM 节点列表。
@returnsComponentRef<C>
