描述

屬性裝飾器，用於設定檢視查詢。 變更檢測器會在檢視 DOM 中查詢與選擇器對應的首個元素或指令。如果檢視 DOM 發生變化，並且新的子級與選擇器對應，則該屬性就會更新。

元資料屬性:

  • selector - 用於查詢的指令類型或名稱。
  • read - 用於從查詢到的元素中讀取不同的令牌。
  • static - true 在變更檢測執行之前求解查詢結果，false 在變更檢測之後求解。預設為 false。

支援以下選擇器。

  • 任何帶有 @Component 或 @Directive 裝飾器的類別
  • 範本引用變數（字串形式）（例如，使用 @ViewChild('cmp') 查詢 <my-component #cmp></my-component>）
  • 當前元件的子元件樹中定義的任何提供者（例如 @ViewChild(SomeService) someService: SomeService）
  • 透過字串令牌定義的任何提供者（例如 @ViewChild('someToken') someTokenVal: any）
  • TemplateRef（例如，使用 @ViewChild(TemplateRef) template; 查詢 <ng-template></ng-template>）

read 支援以下值：

  • 任何帶有 @Component 或 @Directive 裝飾器的類別
  • 在此查詢的 selector 對應的元件的注入器上定義的任何提供者
  • 透過字串令牌定義的任何提供者（例如 {provide: 'token', useValue: 'val'}）
  • TemplateRef、ElementRef 和 ViewContainerRef

動態查詢和靜態查詢之間的區別：

  • 動態查詢 (static: false) - 查詢會在 ngAfterViewInit() 回呼被呼叫之前求解。結果會隨著檢視的變化而更新，例如 ngIf 和 ngFor 控制區塊的變化。
  • 靜態查詢 (static: true) - 查詢在檢視建立後，但在變更檢測執行之前（在 ngOnInit() 回呼被呼叫之前）求解。但是，結果永遠不會更新以反映檢視的變化，例如 ngIf 和 ngFor 控制區塊的變化。

用法說明

範例 1 

          
import {Component, Directive, Input, ViewChild} from '@angular/core';@Directive({  selector: 'pane',  standalone: false,})export class Pane {  @Input() id!: string;}@Component({  selector: 'example-app',  template: `    <pane id="1" *ngIf="shouldShow"></pane>    <pane id="2" *ngIf="!shouldShow"></pane>    <button (click)="toggle()">Toggle</button>    <div>Selected: {{ selectedPane }}</div>  `,  standalone: false,})export class ViewChildComp {  @ViewChild(Pane)  set pane(v: Pane) {    setTimeout(() => {      this.selectedPane = v.id;    }, 0);  }  selectedPane: string = '';  shouldShow = true;  toggle() {    this.shouldShow = !this.shouldShow;  }}
