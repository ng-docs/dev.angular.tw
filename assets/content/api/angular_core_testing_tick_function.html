tick

void

模擬 fakeAsync Zone 中計時器的非同步時間流逝。

微任務佇列在本函式開始時以及任何計時器回呼 執行完畢後都會被清空。

@parammillisnumber

虛擬計時器提前的毫秒數。

@paramtickOptions{ processNewMacroTasksSynchronously: boolean; }

要傳給 tick() 函式的選項。

@returnsvoid

描述

模擬 fakeAsync Zone 中計時器的非同步時間流逝。

微任務佇列在本函式開始時以及任何計時器回呼 執行完畢後都會被清空。

用法說明

tick() 選項是一個名為 processNewMacroTasksSynchronously 的標誌， 它決定是否呼叫新的巨集任務。

如果提供 tickOptions 物件，但未指定 processNewMacroTasksSynchronously 屬性 (tick(100, {}))， 則 processNewMacroTasksSynchronously 預設為 true。

如果省略 tickOptions 引數 (tick(100)))，則 tickOptions 預設為 {processNewMacroTasksSynchronously: true}。

範例 

          
describe('this test', () => {  it(    'looks async but is synchronous',    <any>fakeAsync((): void => {      let flag = false;      setTimeout(() => {        flag = true;      }, 100);      expect(flag).toBe(false);      tick(50);      expect(flag).toBe(false);      tick(50);      expect(flag).toBe(true);    }),  );});

以下範例包含一個巢狀的超時（新的巨集任務），並且 tickOptions 引數允許使用預設值。在這種情況下，processNewMacroTasksSynchronously 預設為 true，並且巢狀函式在每次 tick 時都會執行。

          
it ('test with nested setTimeout', fakeAsync(() => {  let nestedTimeoutInvoked = false;  function funcWithNestedTimeout() {    setTimeout(() => {      nestedTimeoutInvoked = true;    });  };  setTimeout(funcWithNestedTimeout);  tick();  expect(nestedTimeoutInvoked).toBe(true);}));

在以下情況下，processNewMacroTasksSynchronously 被顯式設定為 false，因此不會呼叫巢狀的超時函式。

          
it ('test with nested setTimeout', fakeAsync(() => {  let nestedTimeoutInvoked = false;  function funcWithNestedTimeout() {    setTimeout(() => {      nestedTimeoutInvoked = true;    });  };  setTimeout(funcWithNestedTimeout);  tick(0, {processNewMacroTasksSynchronously: false});  expect(nestedTimeoutInvoked).toBe(false);}));
