当应用于模板中的元素时，使该元素成为一个链接，启动导航到某个路由。导航在页面上的一个或多个 <router-outlet> 位置打开一个或多个路由组件。

给定一个路由配置 [{ path: 'user/:name', component: UserCmp }]， 以下创建了一个到该路由的静态链接： <a routerLink="/user/bob">链接到用户组件</a>

你也可以使用动态值来生成链接。对于动态链接，请传递路径段数组，然后传递每个段的参数。比如， ['/team', teamId, 'user', userName, {details: true}] 生成到 /team/11/user/bob;details=true。

多个静态段可以合并为一个词，并与动态段组合。比如，['/team/11/user', userName, {details: true}]

你提供给链接的输入被视为当前 URL 的一个增量。 例如，假设当前 URL 是 /user/(box//aux:team)。 链接 <a [routerLink]="['/user/jim']">Jim</a> 创建 URL /user/(jim//aux:team)。 更多信息请参见 RoutercreateUrlTree。

你可以在链接中使用绝对或相对路径、设置查询参数、控制如何处理参数以及保留导航状态的历史记录。

相对链接路径

第一段名称可以用 /、./ 或 ../ 开头。

  • 如果第一个片段用 / 开头，则路由器会从应用的根路由开始查找。
  • 如果第一个片段用 ./ 开头或者没有用斜杠开头，路由器就会从当前激活路由开始查找。
  • 如果第一段以 ../ 开头，则路由器将去往路由树中的上一层。

设置和处理查询参数和片段

以下链接将查询参数和一个片段添加到所生成的 URL：

          
            <a [routerLink]="['/user/bob']" [queryParams]="{debug: true}" fragment="education">  link to user component</a>

默认情况下，该指令使用给定的查询参数构造新的 URL。该示例生成链接：/user/bob?debug=true#education。

你可以通过在链接中使用 queryParamsHandling 选项，来指示该指令以不同的方式处理查询参数。允许的值为：

  • 'merge'：将给定的 queryParams 合并到当前查询参数中。
  • 'preserve'：保留当前的查询参数。

比如：

          
            <a [routerLink]="['/user/bob']" [queryParams]="{debug: true}" queryParamsHandling="merge">  link to user component</a>

参见 {@link UrlCreationOptionsqueryParamsHandling}。

保留导航历史

你可以提供要持久到浏览器的 History.state 属性中的 state 值。比如：

          
            <a [routerLink]="['/user/bob']" [state]="{tracingId: 123}">  link to user component</a>

使用 {@link RoutergetCurrentNavigation} 来检索保存的导航状态值。例如，在 NavigationStart 事件期间捕获 tracingId：

          
            // Get NavigationStart eventsrouter.events.pipe(filter(e => e instanceof NavigationStart)).subscribe(e => {  const navigation = router.getCurrentNavigation();  tracingService.trace({id: navigation.extras.state.tracingId});});
