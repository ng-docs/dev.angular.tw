用于根据 URL 匹配路由的函数。当 path 和 pathMatch 的组合不够表达时，为 Route.matcher 实现自定义 URL 匹配器。不能与 path 和 pathMatch 一起使用。

type UrlMatcher = (
  segments: UrlSegment[],
  group: UrlSegmentGroup,
  route: Route,
) => UrlMatchResult | null

用于根据 URL 匹配路由的函数。当 path 和 pathMatch 的组合不够表达时，为 Route.matcher 实现自定义 URL 匹配器。不能与 path 和 pathMatch 一起使用。

该函数接受以下参数并返回一个 UrlMatchResult 对象。

  • segment ：URL 段的数组。
  • group ：段组。
  • route ：要匹配的路由。

下列例子中实现的匹配器会匹配 HTML 文件。

          
            export function htmlFiles(url: UrlSegment[]) {  return url.length === 1 && url[0].path.endsWith('.html') ? ({consumed: url}) : null;}export const routes = [{ matcher: htmlFiles, component: AnyComponent }];
