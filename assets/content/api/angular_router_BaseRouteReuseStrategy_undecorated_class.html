}

此基本路由复用策略仅在匹配的路由器配置完全相同时复用路由。这样可以防止仅在片段或查询参数发生更改时销毁和重新创建组件（也就是复用现有组件）。

此策略不存储任何路由以供以后复用。

Angular 默认使用此策略。

它可以作为自定义路由复用策略的基类，即你可以创建自己的类来扩展 BaseRouteReuseStrategy。

是否应分离给定的路由以供以后复用。对于 BaseRouteReuseStrategy 始终返回 false。
