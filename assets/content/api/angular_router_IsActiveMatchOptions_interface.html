@angular/router

IsActiveMatchOptions

interface
程式碼

一組選項，用於指定在給定當前路由器狀態的 UrlTree 的情況下，如何確定 UrlTree 是否處於啟用狀態。

Router#isActive

API

  
    interface IsActiveMatchOptions {}

matrixParams

"exact" | "subset" | "ignored"

定義了比較兩個 UrlTree 的矩陣引數的策略。

矩陣引數對應取決於對應段的策略。也就是說，如果 paths 選項設定為 'subset'，則只會比較對應段的矩陣引數。

  • 'exact'：要求對應的段也必須具有精確的矩陣引數對應。
  • 'subset'：路由器啟用的 UrlTree 中的對應段可能包含額外的矩陣引數，但有問題的 UrlTree 中存在的矩陣引數必須對應。
  • 'ignored'：當比較 UrlTree 時，矩陣引數將被忽略。

queryParams

"exact" | "subset" | "ignored"

定義了比較兩個 UrlTree 的查詢引數的策略。

  • 'exact'：查詢引數必須精確對應。
  • 'subset'：啟用的 UrlTree 可能包含額外的引數，但必須對應有問題的 UrlTree 中存在的任何引數的鍵和值。
  • 'ignored'：當比較 UrlTree 時，查詢引數將被忽略。

paths

"exact" | "subset"

定義了比較 UrlTree 的 UrlSegment 的策略。

  • 'exact'：每個 UrlTree 中的所有段都必須對應。
  • 'subset'：如果 UrlTree 是活動路由的子樹，則它將被確定為啟用狀態。也就是說，活動路由可能包含額外的段，但必須至少包含有問題的 UrlTree 的所有段。

fragment

"exact" | "ignored"
  • 'exact'：表明 UrlTree 片段必須相等。
  • 'ignored'：在確定 UrlTree 是否處於啟用狀態時，將不會比較片段。
