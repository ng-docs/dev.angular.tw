IsActiveMatchOptions

interface
code

一组选项，用于指定在给定当前路由器状态的 UrlTree 时，如何确定 UrlTree 是否处于活动状态。

interface IsActiveMatchOptions {
}

matrixParams

"exact" | "subset" | "ignored"

定义比较两个 UrlTree 的矩阵参数的策略。

矩阵参数匹配取决于匹配段的策略。也就是说，如果 paths 选项设置为 'subset'，则只会比较匹配段的矩阵参数。

  • 'exact'：要求匹配的段也具有精确的矩阵参数匹配。
  • 'subset'：路由器的活动 UrlTree 中的匹配段可能包含额外的矩阵参数，但所讨论的 UrlTree 中存在的参数必须匹配。
  • 'ignored'：在比较 UrlTree 时，矩阵参数将被忽略。

queryParams

"exact" | "subset" | "ignored"

定义比较两个 UrlTree 的查询参数的策略。

  • 'exact'：查询参数必须完全匹配。
  • 'subset'：活动的 UrlTree 可能包含额外的参数，但必须匹配所讨论的 UrlTree 中存在的键和值。
  • 'ignored'：在比较 UrlTree 时，查询参数将被忽略。

paths

"exact" | "subset"

定义比较 UrlTree 的 UrlSegment 的策略。

  • 'exact'：每个 UrlTree 中的所有段必须匹配。
  • 'subset'：如果某个 UrlTree 是活动路由的子树，它将被视为活动的。也就是说，活动路由可能包含额外的段，但必须至少包含所讨论的 UrlTree 的所有段。

fragment

"exact" | "ignored"
  • 'exact'：表示 UrlTree 片段必须相等。
  • 'ignored'：在确定 UrlTree 是否活动时，片段将不被比较。
