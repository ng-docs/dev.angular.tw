API

  
    interface NgZoneOptions {}

eventCoalescing

boolean | undefined

（可選）指定是否合併事件變更檢測。 考慮以下情況。

          
<div (click)="doSomething()">  <button (click)="doSomethingElse()"></button></div>

當按鈕被點選時，由於事件冒泡，兩個事件處理程式都將被呼叫，並且會觸發 2 次變更檢測。我們可以合併此類別事件，以便僅觸發一次變更檢測。

預設情況下，此選項設定為 false，這意味著事件不會被合併，並且變更檢測將被多次觸發。如果此選項設定為 true，則在上述場景中，變更檢測將只觸發一次。

runCoalescing

boolean | undefined

（可選）指定是否應將 NgZone#run() 方法呼叫合併為單次變更檢測。

考慮以下情況。

          
for (let i = 0; i < 10; i ++) {  ngZone.run(() => {    // do something  });}

這種情況會多次觸發變更檢測。使用 ngZoneRunCoalescing 選項，事件迴圈中的所有變更檢測都只觸發一次。此外，變更檢測在 requestAnimation 中執行。

ignoreChangesOutsideZone

boolean | undefined
@deprecated

引入此選項是出於謹慎，旨在讓開發人員選擇退出 v18 中的新行為，即當上述事件發生在 Zone 外部時，會排程變更檢測。在發布後監控結果後，我們已確定此屬性執行正常，並且認為不應透過將此選項設定為 true 來停用它。

當為 false 時，當 Angular 收到範本需要重新整理的明確指示時，會排程變更檢測。這包括：

  • 呼叫 ChangeDetectorRef.markForCheck
  • 呼叫 ComponentRef.setInput
  • 更新在範本中讀取的 Signal
  • 附著一個標記為髒檢視
  • 移除檢視
  • 註冊渲染鉤子（僅當渲染鉤子執行上述操作之一時，範本才會重新整理）
