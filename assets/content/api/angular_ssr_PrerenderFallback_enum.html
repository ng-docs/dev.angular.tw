開發者預覽

定義了當預渲染路徑不可用時，靜態站點生成 (SSG) 路由的回退策略。 這對於帶有引數化 URL 的路由尤其重要，因為某些路徑可能在建立時未進行預渲染。

ServerRoutePrerenderWithParams

API

  
    enum PrerenderFallback {}

伺服器端

如果預渲染路徑不可用，則回退到伺服器端渲染 (SSR)。 此策略在請求時於伺服器上動態生成頁面。

用戶端

如果預渲染路徑不可用，則回退到用戶端渲染 (CSR)。 此策略允許在用戶端渲染頁面。

無

沒有回退；如果路徑未預渲染，伺服器將不會處理該請求。 這意味著對於未預渲染的路徑，應用將不會提供任何回應。
