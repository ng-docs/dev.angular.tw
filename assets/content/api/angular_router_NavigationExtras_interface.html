@angular/router

NavigationExtras

interface
程式碼

用於修改 Router 導向策略的選項。 向 Router 導向函式提供一個包含這些屬性中任意屬性的物件，以控制目標 URL 的構造或解釋方式。

API

  
    interface NavigationExtras extends UrlCreationOptions {}

relativeTo

ActivatedRoute | null | undefined

指定用於相對導向的根 URI。

例如，考慮以下路由設定，其中父路由有兩個子路由。

          
[{  path: 'parent',  component: ParentComponent,  children: [{    path: 'list',    component: ListComponent  },{    path: 'child',    component: ChildComponent  }]}]

以下 go() 函式透過將目標 URI 解釋為相對於已啟用的 child 路由來導向到 list 路由

          
 @Component({...}) class ChildComponent {   constructor(private router: Router, private route: ActivatedRoute) {}   go() {     router.navigate(['../list'], { relativeTo: this.route });   } }

null 或 undefined 值表示導向指令應相對於根路徑應用。

queryParams

Params | null | undefined

設定 URL 的查詢引數。

          
// Navigate to /results?page=1router.navigate(['/results'], { queryParams: { page: 1 } });

fragment

string | undefined

設定 URL 的雜湊片段。

          
// Navigate to /results#toprouter.navigate(['/results'], { fragment: 'top' });

queryParamsHandling

QueryParamsHandling | null | undefined

如何在路由器連結中處理查詢引數以進行下一次導向。 選項之一：

  • preserve：保留當前引數。
  • merge：將新引數與當前引數合併。

『preserve』選項會丟棄任何新的查詢引數：

          
// from /view1?page=1 to/view2?page=1router.navigate(['/view2'], { queryParams: { page: 2 },  queryParamsHandling: "preserve"});

『merge』選項將新的查詢引數附加到當前 URL 中的引數：

          
// from /view1?page=1 to/view2?page=1&otherKey=2router.navigate(['/view2'], { queryParams: { otherKey: 2 },  queryParamsHandling: "merge"});

如果當前引數與 queryParams 物件中的引數之間發生鍵衝突，則使用新值。

preserveFragment

boolean | undefined

為 true 時，為下一次導向保留 URL 片段

          
// Preserve fragment from /results#top to /view#toprouter.navigate(['/view'], { preserveFragment: true });

onSameUrlNavigation

OnSameUrlNavigation | undefined

如何處理對當前 URL 的導向請求。

此值是 OnSameUrlNavigation 中可用選項的子集，並將優先於為 Router 設定的預設值。

skipLocationChange

boolean | undefined

為 true 時，導向時不將新狀態推入歷史記錄。

          
// Navigate silently to /viewthis.router.navigate(['/view'], { skipLocationChange: true });

replaceUrl

boolean | undefined

為 true 時，導向時替換歷史記錄中的當前狀態。

          
// Navigate to /viewthis.router.navigate(['/view'], { replaceUrl: true });

state

{ [k: string]: any; } | undefined

開發者定義的狀態，可以傳遞給任何導向。 在執行導向時，透過 Router.getCurrentNavigation() 方法 回傳的 Navigation.extras 物件訪問此值。

導向完成後，路由器會將一個包含此值的物件以及 navigationId 寫入 history.state。 在啟用此路由之前呼叫 location.go() 或 location.replaceState() 時，會寫入該值。

請注意，history.state 未透過物件相等性測試，因為路由器在每次導向時都會新增 navigationId。

info

unknown

使用它來傳遞關於此特定導向的瞬態訊息，例如導向是如何發生的。 這樣，它與將設定為 history.state 的持久值 state 不同。 此物件被直接賦值給路由器當前的 Navigation（它不會被複制或克隆），因此應該謹慎地修改。

一個關於如何使用它的例子是，根據到達特定路由的方式，觸發不同的單頁導向動畫。 例如，考慮一個照片函式庫應用，你可以透過各種路由訪問相同的照片 URL 和狀態：

  • 在畫廊檢視中點擊它
  • 點選
  • 在相簿中檢視另一張照片時點選『下一張』或『上一張』
  • 等等。

它們中的每一個都希望在導向時有不同的動畫。 此訊息不適合儲存在持久 URL 或歷史記錄條目狀態中，但從應用的其餘部分傳遞到路由器仍然很重要。

此訊息可以與檢視轉場屬性和 onViewTransitionCreated 回呼結合使用。 該訊息可以在回呼中使用，以便在文件上設定類別，從而控制轉場動畫，並在轉場動畫完成後刪除這些類別。

browserUrl

string | UrlTree | undefined

設定後，路由器將更新瀏覽器的地址欄，以對應給定的 UrlTree，而不是用於路由對應的地址。
