status

FormControlStatus

控制元件的驗證狀態。

valid

boolean

當控制元件的 status 為 VALID 時，控制元件為 valid。

invalid

boolean

當控制元件的 status 為 INVALID 時，控制元件為 invalid。

pending

boolean

當控制元件的 status 為 PENDING 時，控制元件為 pending。

disabled

boolean

當控制元件的 status 為 DISABLED 時，控制元件為 disabled。

停用的控制元件免於驗證檢查，並且不包含在其祖先控制元件的聚合值中。

enabled

boolean

只要控制元件的 status 不是 DISABLED，控制元件就為 enabled。

errors

ValidationErrors | null

一個物件，其中包含驗證失敗產生的任何錯誤；如果沒有錯誤，則為 null。

pristine

boolean

如果使用者尚未更改 UI 中的值，則控制元件為 pristine。

dirty

boolean

如果使用者在 UI 中更改了值，則控制元件為 dirty。

touched

boolean

如果控制元件被標記為 touched，則為 True。

一旦使用者在其上觸發了 blur 事件，控制元件就會被標記為 touched。

untouched

boolean

如果控制元件未被標記為 touched，則為 True

如果使用者尚未在其上觸發 blur 事件，則控制元件為 untouched。

events

Observable<ControlEvent<TValue>>

一個多播的可觀察物件，每當控制元件的狀態更改時都會發出事件。 它會針對值、狀態、pristine 或 touched 更改發出。

注意：在值更改時，發出操作在此控制元件的值更新後立即發生。父控制元件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍後更新，因此從此事件的回呼中訪問父控制元件的值（使用 value 屬性）可能會導致獲取尚未更新的值。請改為訂閱父控制元件的 events。 對於其他事件型別，事件在父控制元件更新後發出。

valueChanges

Observable<TValue>

一個多播的可觀察物件，每當控制元件的值在 UI 中或以程式設計方式更改時，都會發出一個事件。 每次呼叫 enable() 或 disable() 而不傳遞 {emitEvent: false} 作為函式引數時，它也會發出一個事件。

注意：發出操作在此控制元件的值更新後立即發生。父控制元件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍後更新，因此從此事件的回呼中訪問父控制元件的值（使用 value 屬性）可能會導致獲取尚未更新的值。請改為訂閱父控制元件的 valueChanges 事件。

statusChanges

Observable<FormControlStatus>

一個多播的可觀察物件，每當控制元件的驗證 status 重新計算時，都會發出一個事件。

updateOn

FormHooks

報告 AbstractControl 的更新策略（指的是控制元件自身更新所依據的事件）。 可能的值：'change' | 'blur' | 'submit' 預設值：'change'

markAsTouched

2 個過載

將控制元件標記為 touched。控制元件會被不更改值的焦點和失焦事件觸碰。

@paramopts{ onlySelf?: boolean | undefined; emitEvent?: boolean | undefined; } | undefined

設定選項，用於確定在應用標記後控制元件如何傳播更改和發出事件。

  • onlySelf：當為 true 時，僅標記此控制元件。當為 false 或不提供時，標記所有直系祖先控制元件。預設為 false。
  • emitEvent：當為 true 或不提供（預設值）時，events 可觀察物件會發出一個 TouchedChangeEvent，且 touched 屬性為 true。當為 false 時，不發出任何事件。
@returnsvoid
@paramopts{ onlySelf?: boolean | undefined; emitEvent?: boolean | undefined; sourceControl?: AbstractControl<any, any> | undefined; } | undefined
@returnsvoid

disable

2 個過載

停用控制元件。這意味著該控制元件免於驗證檢查，並從任何父控制元件的聚合值中排除。其狀態為 DISABLED。

如果控制元件有子控制元件，則所有子控制元件也會被停用。

@paramopts{ onlySelf?: boolean | undefined; emitEvent?: boolean | undefined; } | undefined

設定選項，用於確定在控制元件停用後，控制元件如何傳播變更和發出事件。

  • emitEvent：當為 true 或不提供（預設值）時，當控制元件被停用時，statusChanges、valueChanges 和 events 可觀察者會發出包含最新狀態和值的事件。當為 false 時，不發出任何事件。

從根表單到 'street' 控制元件的路徑將是 'address' -> 'street'。

可以透過以下兩種格式之一提供給此方法：

  1. 字串形式的控制元件名稱陣列，例如：['address', 'street']
  2. 用句點分隔的控制元件名稱清單，放在一個字串中，例如：'address.street'

如果未提供路徑，則此方法檢查當前控制元件上的錯誤。

root

AbstractControl<any, any>

檢索此控制元件的頂層祖先。

描述

這是 FormControl、FormGroup 和 FormArray 的基底類別。

它提供所有控制元件和控制元件群組共有的一些行為，例如執行驗證器、計算狀態和重置狀態。它還定義了所有子類別之間共享的屬性，例如 value、valid 和 dirty。不應直接實例化它。

第一個型別引數 TValue 表示控制元件的值型別 (control.value)。 可選的型別引數 TRawValue 表示原始值型別 (control.getRawValue())。
