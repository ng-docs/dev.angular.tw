@angular/router

ExtraOptions

interface
程式碼

路由器模組的一組設定選項，在 forRoot() 方法中提供。

forRoot

API

  
    interface ExtraOptions extends InMemoryScrollingOptions {}

enableTracing

boolean | undefined

為 true 時，將所有內部導向事件記錄到控制台。 用於除錯。

useHash

boolean | undefined

為 true 時，啟用使用 URL 片段而不是 history API 的位置策略。

initialNavigation

InitialNavigation | undefined

enabled、enabledBlocking、enabledNonBlocking 或 disabled 之一。 當設定為 enabled 或 enabledBlocking 時，初始導向在根元件建立之前開始。引導程式會被阻塞，直到初始導向完成。如果你使用伺服器端渲染，但沒有為你的應用啟用水合，則應設定此值。當設定為 enabledNonBlocking 時，初始導向在根元件建立之後開始。 引導程式不會因初始導向完成而被阻塞。當設定為 disabled 時，不執行初始導向。位置監聽器在根元件建立之前設定。如果由於某些複雜的初始化邏輯而需要更好地控制路由器何時啟動其初始導向，請使用此選項。

bindToComponentInputs

boolean | undefined

為 true 時，允許將來自 Router 狀態的訊息直接繫結到 Route 設定中元件的輸入屬性（Property）。

enableViewTransitions

boolean | undefined

為 true 時，透過在 document.startViewTransition 內部執行路由啟用和停用，在路由器中啟用檢視轉場。

errorHandler

((error: any) => any) | undefined

用於失敗導向的自定義錯誤處理程式。 如果處理程式回傳值，則導向 Promise 將以此值求解。 如果處理程式丟擲異常，則導向 Promise 將以該異常被拒絕。

preloadingStrategy

any

設定預載入策略。 PreloadAllModules 或 NoPreloading（預設值）之一。

scrollOffset

[number, number] | (() => [number, number]) | undefined

設定路由器在捲動到元素時將使用的捲動偏移量。

當給定一個帶有 x 和 y 位置值的元組時， 路由器每次捲動時都會使用該偏移量。 當給定一個函式時，路由器每次都會呼叫該函式 它恢復捲動位置。

anchorScrolling

"disabled" | "enabled" | undefined

當設定為『enabled』時，當 URL 具有片段時，捲動到錨點元素。 預設情況下，錨點捲動處於停用狀態。

錨點捲動不會在『popstate』上發生。相反，我們會恢復我們儲存的位置或捲動到頂部。

scrollPositionRestoration

"disabled" | "enabled" | "top" | undefined

設定在後退導向時是否需要恢復捲動位置。

  • 『disabled』 -（預設）不執行任何操作。捲動位置在導向時保持不變。
  • 『top』 - 在所有導向上將捲動位置設定為 x = 0，y = 0。
  • 『enabled』 - 在後退導向時恢復之前的捲動位置，否則，如果提供了錨點，則將位置設定為錨點，或者將捲動位置設定為 [0, 0]（前進導向）。此選項將在未來成為預設選項。

你可以透過調整啟用行為來實現自定義捲動恢復行為，如下例所示。

          
class AppComponent {  movieData: any;  constructor(private router: Router, private viewportScroller: ViewportScroller,changeDetectorRef: ChangeDetectorRef) {  router.events.pipe(filter((event: Event): event is Scroll => event instanceof Scroll)    ).subscribe(e => {      fetch('http://example.com/movies.json').then(response => {        this.movieData = response.json();        // update the template with the data before restoring scroll        changeDetectorRef.detectChanges();        if (e.position) {          viewportScroller.scrollToPosition(e.position);        }      });    });  }}

canceledNavigationResolution

"replace" | "computed" | undefined

設定當導向取消時，路由器嘗試恢復狀態的方式。

'replace' - 始終使用 location.replaceState 將瀏覽器狀態設定為導向開始前路由器的狀態。這意味著，如果瀏覽器的 URL 在導向取消之前更新，路由器將只替換歷史記錄中的條目，而不是嘗試恢復到會話歷史記錄中的先前位置。這種情況最常發生在 urlUpdateStrategy: 'eager' 和使用瀏覽器後退/前進按鈕的導向中。

'computed' - 將嘗試回傳到會話歷史記錄中與導向取消時 Angular 路由對應的相同索引。例如，如果單擊瀏覽器後退按鈕且導向被取消，路由器將觸發前進導向，反之亦然。

注意：'computed' 選項與任何僅處理部分 URL 的 UrlHandlingStrategy 都不相容，因為歷史記錄恢復會導向到瀏覽器歷史記錄中的先前位置，而不是簡單地重置部分 URL。

如果未設定，則預設值為 replace。

onSameUrlNavigation

OnSameUrlNavigation | undefined

設定處理到當前 URL 的導向請求的預設行為。

如果未設定，Router 將使用 'ignore'。

paramsInheritanceStrategy

"emptyOnly" | "always" | undefined

定義路由器如何將來自父路由的引數、資料和已求解資料合併到子路由。

預設情況下（'emptyOnly'），當路由本身具有空路徑（意味著其設定為 path: ''）或當父路由沒有任何元件設定時，路由會繼承父路由的引數。

設定為 'always' 以啟用父引數的無條件繼承。

請注意，當處理矩陣引數時，『parent』指的是父 Route 設定，這不一定意味著『左側的 URL 片段』。當 Route path 包含多個片段時，矩陣引數必須出現在最後一個片段上。例如，{path: 'a/b', component: MyComp} 的矩陣引數應顯示為 a/b;foo=bar 而不是 a;foo=bar/b。

urlUpdateStrategy

"deferred" | "eager" | undefined

定義路由器何時更新瀏覽器 URL。預設情況下（'deferred'），在成功導向後更新。 如果希望在導向開始時更新 URL，請設定為 'eager'。 提前更新 URL 允許你透過顯示包含失敗 URL 的錯誤訊息來處理導向失敗。

defaultQueryParamsHandling

QueryParamsHandling | undefined

當未提供時，用於處理 Router.createUrlTree 中查詢引數的預設策略。

createUrlTree 方法在內部被 Router.navigate 和 RouterLink 使用。 請注意，QueryParamsHandling 不適用於 Router.navigateByUrl。

當在呼叫 createUrlTree 時既未指定預設值也未指定 queryParamsHandling 選項時，當前引數將被新引數替換。

resolveNavigationPromiseOnError

boolean | undefined

當為 true 時，Promise 將會求解為 false，就像其他失敗的導向一樣（例如，當守衛被拒絕時）。

否則，如果發生錯誤，路由器導向回傳的 Promise 將被拒絕。
