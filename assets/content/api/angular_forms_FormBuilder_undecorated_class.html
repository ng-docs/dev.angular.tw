从用户指定的配置创建一个 AbstractControl。

FormBuilder 提供了简化创建 FormControl、 FormGroup 或 FormArray 实例的语法糖。它减少了构建复杂表单所需的样板代码量。

nonNullable

NonNullableFormBuilder

返回一个 FormBuilder，其中自动构造的 FormControl 元素具有 {nonNullable: true} 且不可为 null。

构建不可为空的控件

构造控件时，它将是不可为空的，并将重置为其初始值。

          
            let nnfb = new FormBuilder().nonNullable;let name = nnfb.control('Alex'); // FormControl<string>name.reset();console.log(name); // 'Alex'

构建不可为空的组或数组

构造组或数组时，所有自动创建的内部控件都将不可为空，并将重置为它们的初始值。

          
            let nnfb = new FormBuilder().nonNullable;let name = nnfb.group({who: 'Alex'}); // FormGroup<{who: FormControl<string>}>name.reset();console.log(name); // {who: 'Alex'}

构建组或数组中的 可为空字段

仍然可以有可为 null 的字段。特别是，任何 已经构造的 FormControl 都不会被改变。例如：

          
            let nnfb = new FormBuilder().nonNullable;// FormGroup<{who: FormControl<string|null>}>let name = nnfb.group({who: new FormControl('Alex')});name.reset(); console.log(name); // {who: null}

因为内部控件是由调用者显式构造的，所以构建器无法控制它的创建方式，并且不能排除 null。

构建一个新的 FormGroup 实例。接受一个泛型参数，该参数是一个包含所有键和相应内部控件类型的对象。

@paramcontrolsT

一组子控件。每个子控件的名字就是它注册时用的 key。

@paramoptionsAbstractControlOptions

用于 FormGroup 的配置选项对象。该对象应具有 AbstractControlOptions 类型，并可能包含以下字段：

  • validators：一个同步验证器函数或其数组。
  • asyncValidators：一个异步验证器函数或其数组。
  • updateOn：控件应更新的事件（选项：'change' | 'blur' | submit'）。
@returnsFormGroup<{ [K in keyof T]: ɵElement<T[K], null>; }>
