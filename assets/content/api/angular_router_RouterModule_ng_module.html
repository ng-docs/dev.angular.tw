forChild

ModuleWithProviders<RouterModule>

建立一個模組，其中包含所有路由器指令和一個註冊路由的提供者， 而無需建立新的 Router 服務。 當為子模組和延遲載入的子模組註冊時，按如下方式建立 NgModule：

          
@NgModule({  imports: [RouterModule.forChild(ROUTES)]})class MyNgModule {}
@paramroutesRoutes

一個 Route 物件陣列，用於定義子模組的導向路徑。

@returnsModuleWithProviders<RouterModule>

描述

為應用中定義的檢視之間的應用內導向新增指令和提供者。 使用 Angular 的 Router 服務以宣告方式指定應用狀態並管理狀態轉換。

你可以多次匯入此 NgModule，每個延遲載入的捆綁包匯入一次。 但是，只能有一個 Router 服務處於活動狀態。 為了確保這一點，匯入此模組時有兩種註冊路由的方法：

  • forRoot() 方法建立一個 NgModule，其中包含所有指令、給定的路由以及 Router 服務本身。
  • forChild() 方法建立一個 NgModule，其中包含所有指令和給定的路由，但不包括 Router 服務。
