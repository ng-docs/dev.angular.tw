class RouterModule {
}

添加用于应用内不同视图间导航的指令和提供者。 使用 Angular 的 Router 服务声明式地指定应用状态并管理状态过渡。

你可以多次导入这个 NgModule，每个延迟加载的捆绑包一次。 然而，只有一个 Router 服务可以是活跃的。 为了确保这一点，有两种方式在导入此模块时注册路由：

  • forRoot() 方法创建一个包含所有指令、给定路由和 Router 服务本身的 NgModule。
  • forChild() 方法创建一个包含所有指令和给定路由但不包括 Router 服务的 NgModule。

forRoot

ModuleWithProviders<RouterModule>

带着所有路由器提供者和指令创建和配置模块。（可选）设置应用程序监听器以执行初始导航。

在根目录下注册 NgModule 时，请按以下方式导入：

          
            @NgModule({  imports: [RouterModule.forRoot(ROUTES)]})class MyNgModule {}
@paramroutesRoutes

定义应用导航路径的 Route 对象数组。

@paramconfigExtraOptions

控制导航如何执行的 ExtraOptions 配置对象。

@returnsModuleWithProviders<RouterModule>

forChild

ModuleWithProviders<RouterModule>

创建带有所有路由器指令和提供者注册的路由的模块，而无需创建新的路由器服务。注册子模块和惰性加载的子模块时，像这样创建 NgModule：

          
            @NgModule({  imports: [RouterModule.forChild(ROUTES)]})class MyNgModule {}
@paramroutesRoutes

定义子模块导航路径的 Route 对象数组。

@returnsModuleWithProviders<RouterModule>
