provideExperimentalCheckNoChangesForDebug

EnvironmentProviders

用于周期性地验证在检查后没有表达式发生变化。

@paramoptions{ interval?: number; useNgZoneOnStable?: boolean; exhaustive?: boolean; }

用于配置检查何时执行。

  • interval 将周期性地对应用视图运行全面的 checkNoChanges
  • useNgZoneOnStable 将使用 ZoneJS 来确定何时在使用 ZoneJS 驱动变更检测的应用中可能运行了变更检测。当 NgZone.onStable 会发出事件时，所有附着到 ApplicationRef 的视图都会检查变更。
  • 「全面」的意思是所有附着到 ApplicationRef 的视图及其所有后代视图都会检查变更（排除那些通过 ChangeDetectorRef.detach() 分离的子树）。这是有用的，因为在常规变更检测后运行的检查对使用 ChangeDetectionStrategy.OnPush 的组件不起作用。此检查将揭示任何被 OnPush 组件隐藏的现有错误。默认情况下，此检查是全面的，并且会始终检查所有视图，无论其「脏」状态和 ChangeDetectionStrategy 如何。

当 useNgZoneOnStable 选项为 true 时，此函数将提供其自己的 NgZone 实现，并需要在任何其他 NgZone 提供者之后，包括 provideZoneChangeDetection() 和 provideExperimentalZonelessChangeDetection()。

@returnsEnvironmentProviders
