@angular/common/http

HTTP_TRANSFER_CACHE_ORIGIN_MAP

常數
程式碼

如果你的應用在服務器端和用戶端使用不同的 HTTP 源發起 API 呼叫（透過 HttpClient），則 HTTP_TRANSFER_CACHE_ORIGIN_MAP 令牌允許你在這些源之間建立對映表，以便 HttpTransferCache 屬性可以將這些請求識別為相同的請求，並在用戶端水合期間複用在服務器端快取的資料。

API

  
    const HTTP_TRANSFER_CACHE_ORIGIN_MAP: InjectionToken<Record<string, string>>;

描述

如果你的應用在服務器端和用戶端使用不同的 HTTP 源發起 API 呼叫（透過 HttpClient），則 HTTP_TRANSFER_CACHE_ORIGIN_MAP 令牌允許你在這些源之間建立對映表，以便 HttpTransferCache 屬性可以將這些請求識別為相同的請求，並在用戶端水合期間複用在服務器端快取的資料。

重要提示：HTTP_TRANSFER_CACHE_ORIGIN_MAP 令牌只能在你的應用的伺服器程式碼中提供（通常在 app.server.config.ts 指令碼中）。如果 Angular 檢測到在用戶端執行時定義了該令牌，則會丟擲錯誤。

用法說明

當透過服務器上的 http://internal-domain.com:8080 和用戶端上的 https://external-domain.com 訪問同一 API 端點時，你可以使用以下設定：

          
// in app.server.config.ts{    provide: HTTP_TRANSFER_CACHE_ORIGIN_MAP,    useValue: {        'http://internal-domain.com:8080': 'https://external-domain.com'    }}
