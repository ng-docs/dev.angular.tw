@angular/core

@for

控制區塊
程式碼

@for 控制區塊為集合中的每個條目重複渲染一個控制區塊的內容。

語法 

          
@for (item of items; track item.name) {<li>{{ item.name }}</li>} @empty {<li>There are no items.</li>}

描述

@for 控制區塊會渲染其內容，以回應集合中的更改。集合可以是任何 JavaScript 可迭代者，但使用常規 Array 具有效能優勢。

你可以選擇在 @for 控制區塊內容之後立即包含一個 @empty 區塊。當沒有條目時，將顯示 @empty 控制區塊的內容。

Angular 的 @for 控制區塊不支援像 JavaScript 的 continue 或 break 這樣的流程修改語句。

track 和物件標識

track 表示式的值決定了一個鍵，該鍵用於將陣列條目與 DOM 中的檢視相關聯。清晰地指示條目標識，使 Angular 可以在集合中新增、刪除或移動條目時執行最少的 DOM 操作集。

為了最佳化效能，尤其是在遍歷不可變資料的迴圈中，請確保有效地使用 track 表示式來唯一標識每個條目。由於可能導致效能不佳，因此 @for 迴圈需要 track 表示式。

對於保持靜態的集合，track $index 提供了一種直接的跟蹤機制。對於經歷新增、刪除或重新排序的動態集合，請選擇每個條目的唯一屬性作為跟蹤鍵。

$index 和其他上下文變數

在 @for 內容內部，始終有幾個隱式變數可用：

變數 含義
$count 迭代集合中的條目數
$index 當前行的索引
$first 當前行是否為第一行
$last 當前行是否為最後一行
$even 當前行索引是否為偶數
$odd 當前行索引是否為奇數

這些變數始終以這些名稱提供，但可以透過 let 段進行別名：

          
@for (item of items; track item.id; let idx = $index, e = $even) {Item #{{ idx }}: {{ item.name }}}

在使用巢狀的 @for 控制區塊（其中上下文變數名稱可能衝突）的情況下，別名特別有用。
