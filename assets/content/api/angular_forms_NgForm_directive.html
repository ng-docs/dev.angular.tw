创建一个顶级 FormGroup 实例，并将其绑定到表单以跟踪表单的整体值和验证状态。

一旦你导入了 FormsModule，此指令会默认在所有 <form> 标签上生效。你不需要添加特殊的选择器。

你可以选择使用 ngForm 作为键将指令导出到一个本地模板变量中（例如： #myForm="ngForm"）。这是可选的，但很有用。许多来自底层 FormGroup 实例的属性会在指令本身上重复出现，因此引用它可以让你访问该表单的汇总值和有效性状态，以及用户交互属性（如 dirty 和 touched）。

要注册子控件到此表单中，使用带 name 属性的 NgModel。你可以使用 NgModelGroup 在表单内创建子组。

如果需要，还可以监听该指令的 ngSubmit 事件，以便当用户触发了一次表单提交时得到通知。发出 ngSubmit 事件时，会携带原始的 DOM 表单提交事件。

在模板驱动表单中，所有 <form> 标签都会自动标记为 NgForm。要导入 FormsModule 但在某些表单中跳过它的使用，例如，要使用原生的 HTML5 验证，添加 ngNoForm，此 <form> 标签将不会创建 NgForm 指令。在响应式表单中，使用 ngNoForm 是不必要的，因为 <form> 标签是惰性的。在这种情况下，你只需避免使用 formGroup 指令。

设置更新选项

以下示例向你展示了如何使用 ngFormOptions 更改“updateOn”选项的默认值。

          
            <form [ngFormOptions]="{updateOn: 'blur'}">   <input name="one" ngModel>  <!-- this ngModel will update on blur --></form>

原生 DOM 验证 UI

为了防止原生 DOM 表单验证 UI 干扰 Angular 的表单验证，每当将 FormModule 或 ReactiveFormModule 导入应用程序时，Angular 都会自动在任何 <form> 上添加 novalidate 属性。如果你想显式启用 Angular 表单的原生 DOM 验证 UI，你可以将 ngNativeValidate 属性添加到 <form> 元素：

          
            <form ngNativeValidate>  ...</form>

@returnsboolean
Usage notes

例如，对于以下 FormGroup：

          
            form = new FormGroup({  address: new FormGroup({ street: new FormControl() })});

从根表单到 'street' 控件的路径是 'address' -> 'street'。

调用此方法有两种形式：

  1. 控件名称的字符串数组，如 ['address', 'street']
  2. 以一个字符串表示的句号分隔的控件名称列表，如 'address.street'

如果没有给出路径，则此方法检查当前控件上的错误。
