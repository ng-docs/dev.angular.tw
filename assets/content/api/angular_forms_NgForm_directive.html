描述

建立一個頂層 FormGroup 實例，並將其繫結到一個表單，以跟蹤聚合表單值和驗證狀態。

一旦你匯入了 FormsModule，這個指令就會預設在所有 <form> 標籤上變為啟用狀態。你不需要新增特殊的選擇器。

你可以選擇性地使用 ngForm 作為鍵（例如：#myForm="ngForm"）將該指令匯出到一個本地範本變數中。這是可選的，但很有用。來自底層 FormGroup 實例的許多屬性在該指令本身上被複制，因此對它的引用使你可以訪問表單的聚合值和有效性狀態，以及諸如 dirty 和 touched 等使用者互動屬性。

要向表單註冊子控制元件，請使用帶有 name 屬性的 NgModel。你可以使用 NgModelGroup 在表單內建立子組。

如有必要，請監聽該指令的 ngSubmit 事件，以便在使用者觸發表單送出時收到通知。ngSubmit 事件會發出原始的表單送出事件。

在範本驅動表單中，所有 <form> 標籤都會被自動標記為 NgForm。要匯入 FormsModule 但在某些表單中跳過其使用，例如，要使用原生 HTML5 驗證，新增 ngNoForm，<form> 標籤將不會建立 NgForm 指令。在回應式表單中，使用 ngNoForm 是不必要的，因為 <form> 標籤是惰性的。在這種情況下，你應該避免使用 formGroup 指令。

用法說明

監聽表單送出

以下範例展示了如何從 『ngSubmit』 事件中捕獲表單值。

          
import {Component} from '@angular/core';import {NgForm} from '@angular/forms';@Component({  selector: 'example-app',  template: `    <form #f="ngForm" (ngSubmit)="onSubmit(f)" novalidate>      <input name="first" ngModel required #first="ngModel" />      <input name="last" ngModel />      <button>Submit</button>    </form>    <p>First name value: {{ first.value }}</p>    <p>First name valid: {{ first.valid }}</p>    <p>Form value: {{ f.value | json }}</p>    <p>Form valid: {{ f.valid }}</p>  `,  standalone: false,})export class SimpleFormComp {  onSubmit(f: NgForm) {    console.log(f.value); // { first: '', last: '' }    console.log(f.valid); // false  }}

設定更新選項

以下範例向你展示瞭如何使用 ngFormOptions 從預設值更改 『updateOn』 選項。

          
<form [ngFormOptions]="{updateOn: 'blur'}">   <input name="one" ngModel><!-- this ngModel will update on blur --></form>

原生 DOM 驗證 UI

為了防止原生 DOM 表單驗證 UI 干擾 Angular 的表單驗證，每當 FormModule 或 ReactiveFormModule 匯入到應用中時，Angular 都會自動在任何 <form> 上新增 novalidate 屬性。如果你想使用 Angular 表單顯式啟用原生 DOM 驗證 UI，你可以將 ngNativeValidate 屬性新增到 <form> 元素：

          
<form ngNativeValidate>  ...</form>
