用於設定如何確定路由器連結是否處於活動狀態的選項。

可能的值：'page' | 'step' | 'location' | 'date' | 'time' | true | false。

說明

跟蹤元素的連結路由當前是否處於活動狀態，並允許你指定在連結路由處於活動狀態時要新增到元素的一個或多個 CSS 類。

使用此指令為與活動路由關聯的元素建立視覺區分。 例如，以下程式碼在路由器啟用關聯路由時突出顯示單詞『Bob』：

          
<a routerLink="/user/bob" routerLinkActive="active-link">Bob</a>

每當 URL 為『/user』或『/user/bob』時，就會將『active-link』類新增到 標籤。如果 URL 更改，則會刪除該類別。

你可以使用空格分隔的字串或陣列來設定多個類。 例如：

          
<a routerLink="/user/bob" routerLinkActive="class1 class2">Bob</a><a routerLink="/user/bob" [routerLinkActive]="['class1', 'class2']">Bob</a>

要僅在 URL 與連結完全對應時才新增類，請新增選項 exact: true：

          
<a routerLink="/user/bob" routerLinkActive="active-link" [routerLinkActiveOptions]="{exact:true}">Bob</a>

要直接檢查連結的 isActive 狀態，請將 RouterLinkActive 實例賦值給樣板變數。 例如，以下程式碼檢查狀態，而不分配任何 CSS 類：

          
<a routerLink="/user/bob" routerLinkActive #rla="routerLinkActive">  Bob {{ rla.isActive ? '(already open)' : ''}}</a>

你可以將 RouterLinkActive 指令應用於連結元素的祖先。 例如，當 URL 為 '/user/jim' 或 '/user/bob' 時，以下程式碼在 <div> 父標籤上設定 active-link 類。

          
<div routerLinkActive="active-link" [routerLinkActiveOptions]="{exact: true}">  <a routerLink="/user/jim">Jim</a>  <a routerLink="/user/bob">Bob</a></div>

RouterLinkActive 指令還可以用於設定 aria-current 屬性，以便為有視覺障礙的使用者提供活動元素的其他區分方式。

例如，以下程式碼在『首頁』連結確實處於活動狀態時，為其新增 'active' 類，並在這種情況下，還將其 aria-current 屬性設定為 'page'：

          
<a routerLink="/" routerLinkActive="active" ariaCurrentWhenActive="page">Home Page</a>
