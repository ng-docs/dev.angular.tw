跟踪元素上的链接路由当前是否处于活动状态，并允许你指定一个或多个 CSS 类，以便在链接路由处于活动状态时添加到该元素。

使用此指令为与活动路径关联的元素创建视觉差异。比如，以下代码会在路由器激活关联的路由时突出显示单词 “Bob”：

          
            <a routerLink="/user/bob" routerLinkActive="active-link">Bob</a>

每当 URL 是 '/user' 或 '/user/bob' 时，会将 "active-link" 类添加到锚标签中。如果 URL 发生变化，该类会被移除。

你可以用空格分隔的字符串或用数组来设置一个或多个类，比如：

          
            <a routerLink="/user/bob" routerLinkActive="class1 class2">Bob</a><a routerLink="/user/bob" [routerLinkActive]="['class1', 'class2']">Bob</a>

你可以通过传入 exact: true 来配置 RouterLinkActive。这样，只有当 url 和此链接精确匹配时才会添加这些类。

          
            <a routerLink="/user/bob" routerLinkActive="active-link" [routerLinkActiveOptions]="{exact:true}">Bob</a>

要直接检查链接的 isActive 状态，请将 RouterLinkActive 实例赋予模板变量。例如，以下示例无需分配任何 CSS 类即可检查状态：

          
            <a routerLink="/user/bob" routerLinkActive #rla="routerLinkActive">  Bob {{ rla.isActive ? '(already open)' : ''}}</a>

你可以将 RouterLinkActive 指令应用于链接元素的祖先。例如，当 URL 是 '/user/jim' 或 '/user/bob' 时，以下示例将 active-link 类设置到 <div> 父标签上。

          
            <div routerLinkActive="active-link" [routerLinkActiveOptions]="{exact: true}">  <a routerLink="/user/jim">Jim</a>  <a routerLink="/user/bob">Bob</a></div>

RouterLinkActive 指令还可用于设置 aria-current 属性，为视力障碍用户提供活动元素的替代区分方式。

例如，以下代码在确实处于活动状态时将 'active' 类添加到主页链接，并在这种情况下将其 aria-current 属性设置为 'page'：

          
            <a routerLink="/" routerLinkActive="active" ariaCurrentWhenActive="page">Home Page</a>

links

QueryList<RouterLink>

isActive

boolean

routerLinkActiveOptions

IsActiveMatchOptions | { exact: boolean; }

用于配置如何确定路由器链接是否处于活动状态的选项。

这些选项会传递给 Router.isActive() 函数。

ariaCurrentWhenActive

boolean | "page" | "step" | "location" | "date" | "time"

路由器链接处于活动状态时要应用的 Aria-current 属性。

可能的值： 'page' | 'step' | 'location' | 'date' | 'time' | true | false。

isActiveChange

EventEmitter<boolean>

你可以用输出 isActiveChange 来在每次链接变为活动或非活动时获取通知。

发出：true -> 路由处于活动状态 false -> 路由处于非活动状态

          
            <a routerLink="/user/bob" routerLinkActive="active-link" (isActiveChange)="this.onRouterLinkActive($event)">Bob</a>
