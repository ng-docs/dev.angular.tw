用于配置内容查询的参数装饰器。

用于从内容 DOM 中获取第一个与选择器匹配的元素或指令。如果内容 DOM 改变，并且有新的子元素匹配选择器，该属性将会更新。

不检索其他组件模板中的元素或指令，因为组件模板对其祖先来说始终是黑匣子。

元数据属性：

  • selector - 要查询的指令类型或名称。
  • 后代- 如果为 true （默认），则包括元素的所有后代。如果为 false，则仅查询元素的直接子项。
  • read - 用于从查询到的元素读取不同的令牌。
  • static - 如果为 true，则在变更检测运行之前解析查询结果，如果为 false，则在变更检测之后解析。默认为 false。

支持下列选择器：。

  • 任何带有 @Component 或 @Directive 装饰器的类
  • 一个作为字符串的模板引用变量（例如，使用 @ContentChild('cmp') 查询 <my-component #cmp></my-component>）
  • 在当前组件的子组件树中定义的任何提供者（例如 @ContentChild(SomeService) someService: SomeService）
  • 通过字符串令牌定义的任何提供者（例如 @ContentChild('someToken') someTokenVal: any）
  • 一个 TemplateRef（例如，使用 @ContentChild(TemplateRef) template; 查询 <ng-template></ng-template>）

read 支持以下值：

  • 任何带有 @Component 或 @Directive 装饰器的类
  • 在此查询的 selector 匹配的组件注入器上定义的任何提供程序
  • 通过字符串标记定义的任何提供程序（例如 {provide: 'token', useValue: 'val'}）
  • TemplateRef、 ElementRef 和 ViewContainerRef

动态查询和静态查询之间的区别：

查询 详情
动态查询（ static: false） 查询在 ngAfterContentInit() 回调之前求解。
回调被调用。结果将随你的视图更改而更新，例如对
ngIf 与 ngFor 代码块。
视图已创建，但在变更检测运行之前（在调用 ngOnInit() 回调之前）。
结果永远不会更新以反映视图的更改，例如
ngIf 和 ngFor 控制块的更改。
