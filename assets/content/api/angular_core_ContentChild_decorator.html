描述

屬性裝飾器，用於設定內容查詢。

用於從內容 DOM 中獲取與選擇器對應的首個元素或指令。 如果內容 DOM 發生變化，並且新的子條目與選擇器對應，則該屬性（Property）將會更新。

不會檢索其他元件範本中的元素或指令，因為對於元件的祖先元件來說，元件的範本始終是一個黑盒。

元資料屬性（Property）：

  • selector - 用於查詢的指令類型或名稱。
  • descendants - 如果為 true (預設值)，則包括該元素的所有後代。如果為 false，則僅查詢該元素的直接子條目。
  • read - 用於從查詢到的元素中讀取不同的令牌。
  • static - 如果為 true，則在變更檢測執行之前求解查詢結果；如果為 false，則在變更檢測之後求解。預設為 false。

支援以下選擇器。

  • 任何帶有 @Component 或 @Directive 裝飾器的類別
  • 範本引用變數（字串形式），例如，使用 @ContentChild('cmp') 查詢 <my-component #cmp></my-component>
  • 當前元件的子元件樹中定義的任何提供者 (例如 @ContentChild(SomeService) someService: SomeService)
  • 透過字串令牌定義的任何提供者 (例如 @ContentChild('someToken') someTokenVal: any)
  • TemplateRef (例如，使用 @ContentChild(TemplateRef) template; 查詢 <ng-template></ng-template>)

read 支援以下值：

  • 任何帶有 @Component 或 @Directive 裝飾器的類別
  • 此查詢的 selector 所對應的元件的注入器上定義的任何提供者
  • 透過字串令牌定義的任何提供者 (例如 {provide: 'token', useValue: 'val'})
  • TemplateRef、ElementRef 和 ViewContainerRef

動態查詢和靜態查詢之間的區別：

查詢 詳情
動態查詢 (static: false) 查詢在 ngAfterContentInit() 之前求解
回呼被呼叫。結果會隨著檢視的變化而更新，例如
ngIf 和 ngFor 控制區塊的變化。
檢視已建立之後，但在變更檢測執行之前（在 ngOnInit() 回呼之前）
被呼叫）。但是，結果永遠不會更新以反映檢視的變化，例如
ngIf 和 ngFor 控制區塊的變化。

使用須知

          
import {AfterContentInit, ContentChild, Directive} from '@angular/core';@Directive({  selector: 'child-directive',  standalone: false,})class ChildDirective {}@Directive({  selector: 'someDir',  standalone: false,})class SomeDir implements AfterContentInit {  @ContentChild(ChildDirective) contentChild!: ChildDirective;  ngAfterContentInit() {    // contentChild is set  }}

範例 

          
import {Component, ContentChild, Directive, Input} from '@angular/core';@Directive({  selector: 'pane',  standalone: false,})export class Pane {  @Input() id!: string;}@Component({  selector: 'tab',  template: ` <div>pane: {{ pane?.id }}</div>`,  standalone: false,})export class Tab {  @ContentChild(Pane) pane!: Pane;}@Component({  selector: 'example-app',  template: `    <tab>      <pane id="1" *ngIf="shouldShow"></pane>      <pane id="2" *ngIf="!shouldShow"></pane>    </tab>    <button (click)="toggle()">Toggle</button>  `,  standalone: false,})export class ContentChildComp {  shouldShow = true;  toggle() {    this.shouldShow = !this.shouldShow;  }}
