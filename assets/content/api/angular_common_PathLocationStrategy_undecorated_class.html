}

用于配置 {@link Location} 服务以在浏览器 URL 的 路径中表示其状态的 LocationStrategy。

如果你在使用 PathLocationStrategy，可以提供一个 {@link APP_BASE_HREF} 或在文档中添加 <base href> 元素以覆盖默认值。

例如，如果你提供一个 APP_BASE_HREF 为 '/my/app/' 并调用 location.go('/foo')，浏览器的 URL 将变为 example.com/my/app/foo。为确保所有相对 URI 正确解析， <base href> 和/或 APP_BASE_HREF 应以 / 结尾。

同样，如果将 <base href='/my/app/'/> 添加到文档中并调用 location.go('/foo')，则浏览器的 URL 将变为 example.com/my/app/foo。

注意，当使用 PathLocationStrategy 时， <base href> 中的查询和片段都不会被保留，如 RFC 所述。
