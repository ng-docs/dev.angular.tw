@angular/router

NavigationBehaviorOptions

interface
程式碼

用於修改 Router 導向策略的選項。 向 Router 導向函式提供一個包含任何這些屬性的物件，以控制應如何處理導向。

Router#navigateRouter#navigateByUrl路由與導向指南

API

  
    interface NavigationBehaviorOptions {}

onSameUrlNavigation

OnSameUrlNavigation | undefined

如何處理對當前 URL 的導向請求。

此值是 OnSameUrlNavigation 中可用選項的子集，並將優先於為 Router 設定的預設值。

skipLocationChange

boolean | undefined

為 true 時，導向時不向歷史記錄推送新狀態。

          
// Navigate silently to /viewthis.router.navigate(['/view'], { skipLocationChange: true });

replaceUrl

boolean | undefined

為 true 時，導向時替換歷史記錄中的當前狀態。

          
// Navigate to /viewthis.router.navigate(['/view'], { replaceUrl: true });

state

{ [k: string]: any; } | undefined

開發者定義的狀態，可以傳遞給任何導向。 在導向執行時，透過 Router.getCurrentNavigation() 方法 回傳的 Navigation.extras 物件訪問此值。

導向完成後，路由器會將一個包含此值的物件以及 navigationId 寫入 history.state。 當在啟用此路由之前呼叫 location.go() 或 location.replaceState() 時，會寫入該值。

請注意，history.state 未透過物件相等性測試，因為路由器在每次導向時都會新增 navigationId。

info

unknown

使用此選項來傳遞關於此特定導向的瞬態訊息，例如它是如何發生的。這樣，它就不同於將設定為 history.state 的持久值 state。此物件直接賦值給 Router 當前的 Navigation（它不會被複制或克隆），因此應該謹慎地修改它。

一個關於如何使用它的範例是，根據到達特定路由的方式觸發不同的單頁導向動畫。例如，考慮一個照片函式庫應用，你可以透過各種路由訪問相同的照片 URL 和狀態：

  • 在畫廊檢視中單擊它
  • 單擊
  • 在相簿中檢視另一張照片時單擊『下一張』或『上一張』
  • 等等。

它們中的每一個都希望在導向時使用不同的動畫。此訊息儲存在持久 URL 或歷史記錄條目狀態中沒有意義，但從應用的其餘部分傳遞到路由器仍然很重要。

此訊息可以與 View Transitions 屬性和 onViewTransitionCreated 回呼結合使用。該訊息可能在回呼中用於在文件上設定類，以便控制轉場動畫，並在轉場動畫完成時刪除這些類。

browserUrl

string | UrlTree | undefined

設定後，路由器將更新瀏覽器的地址欄，以對應給定的 UrlTree，而不是用於路由對應的那個。
