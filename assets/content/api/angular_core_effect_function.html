effect

EffectRef

註冊一個『副作用』，每當它讀取的訊號發生變化時，該副作用將被排程和執行。

Angular 有兩種不同型別的副作用：元件副作用和根副作用。當從元件、指令或元件/指令的服務中呼叫 effect() 時，會建立元件副作用。當從元件樹外部（例如在根服務中）呼叫 effect() 或提供 forceRoot 選項時，會建立根副作用。

這兩種副作用型別在時序上有所不同。元件副作用在 Angular 的同步（變更檢測）過程中作為元件生命週期事件執行，並且可以安全地讀取輸入訊號或建立/銷燬依賴於元件狀態的檢視。根副作用作為微任務執行，並且與元件樹或變更檢測無關。

effect() 必須在注入上下文中執行，除非手動指定了 injector 選項。
