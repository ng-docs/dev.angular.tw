驗證器

類別
程式碼

提供一組內建的驗證器，這些驗證器可用於表單控制元件。

表單驗證

API

}

驗證器，要求控制元件的值大於或等於所提供的數字。

@paramminnumber
@returnsValidatorFn
用法說明

email

ValidationErrors | null

驗證器，要求控制元件的值透過電子郵件驗證測試。

使用正則表示式模式測試該值，此模式適用於常見用例。此模式基於 WHATWG HTML 規範中有效電子郵件地址的定義，並進行了一些增強，以納入更多 RFC 規則（例如與網域名稱和地址不同部分的長度相關的規則）。

與 WHATWG 版本的差異包括：

  • 不允許 local-part（@ 符號之前的部分）以句點 (.) 開頭或結尾。
  • 不允許 local-part 超過 64 個字元。
  • 不允許整個地址超過 254 個字元。

如果此模式不能滿足你的業務需求，你可以使用 Validators.pattern() 來根據不同的模式驗證值。

@paramcontrolAbstractControl<any, any>
@returnsValidationErrors | null
用法說明

驗證該欄位是否符合有效的電子郵件模式 

          
const control = new FormControl('bad@', Validators.email);console.log(control.errors); // {email: true}

minLength

ValidatorFn

驗證器，它要求控制元件值的條目數大於或等於所提供的最小長度。如果你使用 HTML5 minlength 屬性，預設也會提供此驗證器。請注意，minLength 驗證器旨在僅用於具有數字 length 或 size 屬性的型別，例如字串、陣列或集合。當值的 length 或 size 屬性為 0 時（例如，在空字串或空陣列的情況下），也不會呼叫 minLength 驗證器邏輯，以支援可選控制元件。如果空值不應被視為有效值，則可以使用標準 required 驗證器。

@paramminLengthnumber
@returnsValidatorFn
用法說明

驗證該欄位至少包含 3 個字元 

          
const control = new FormControl('ng', Validators.minLength(3));console.log(control.errors); // {minlength: {requiredLength: 3, actualLength: 2}}

        
          
<input minlength="5">

pattern

ValidatorFn

驗證器，它要求控制元件的值與正則表示式模式對應。如果你使用 HTML5 pattern 屬性，預設也會提供此驗證器。

@parampatternstring | RegExp

一個正則表示式，將按原樣用於測試值，或者一個字串。如果傳遞的是字串，則在提供的字串前面加上 ^ 字元，並在後面加上 $ 字元（如果尚不存在），並且生成的正則表示式用於測試值。

@returnsValidatorFn
用法說明

驗證該欄位是否僅包含字母或空格 

          
const control = new FormControl('1', Validators.pattern('[a-zA-Z ]*'));console.log(control.errors); // {pattern: {requiredPattern: '^[a-zA-Z ]*$', actualValue: '1'}}

        
          
<input pattern="[a-zA-Z ]*">

使用 global 或 sticky 標誌進行模式對應

使用 g 或 y 標誌建立的 RegExp 物件，當傳遞到 Validators.pattern 時，在連續執行驗證時，可能會在相同的輸入上產生不同的結果。這是因為 RegExp.prototype.test 的行為在 ECMA-262 中是如何規定的（當使用 global 或 sticky 標誌時，RegExp 會保留上次對應的索引）。由於這種行為，建議在使用 Validators.pattern 時，不要傳入啟用了 global 或 sticky 標誌的 RegExp 物件。

          
// Not recommended (since the `g` flag is used)const controlOne = new FormControl('1', Validators.pattern(/foo/g));// Goodconst controlTwo = new FormControl('1', Validators.pattern(/foo/));

描述

提供一組內建的驗證器，這些驗證器可用於表單控制元件。

驗證器是一個函式，它處理 FormControl 或控制元件的集合，並回傳一個錯誤對映表或 null。null 對映表意味著驗證已透過。
