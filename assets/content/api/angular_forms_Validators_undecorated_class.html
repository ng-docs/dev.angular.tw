提供一组内置验证器，可用于各种表单控件。

验证器是一个处理 FormControl 或控件集合的函数，并返回一个错误映射表或 null。null 映射表意味着验证通过。

email

ValidationErrors

此验证器要求控件的值能通过 email 格式验证。

使用适合普通用例的正则表达式模式测试值。该模式基于 WHATWG HTML 规范中有效电子邮件地址的定义，并进行了一些增强以支持更多的 RFC 规则（比如与域名相关的规则以及地址不同部分的长度）。

与 WHATWG 版本的区别包括：

  • 禁止 local-part（@ 符号前面的部分）以句点（.）开头或结尾。
  • 不允许 local-part 超过 64 个字符。
  • 不允许整个地址超过 254 个字符。

如果此模式不满足你的业务需求，可以使用 Validators.pattern() 以针对不同模式验证值。

@paramcontrolAbstractControl<any, any>
@returnsValidationErrors
Usage notes

验证该字段匹配有效的 email 格式。

          
            const control = new FormControl('bad@', Validators.email);console.log(control.errors); // {email: true}

minLength

ValidatorFn

验证器要求控件的值长度大于或等于提供的最小长度。如果使用 HTML5 minlength 属性，此验证器也会默认提供。请注意， minLength 验证器仅用于具有数值 length 属性的类型，例如字符串或数组。当值的 length 属性为 0 时（例如空字符串或空数组）， minLength 验证器逻辑也不会被调用，以支持可选控件。如果不应将空值视为有效，可以使用标准 required 验证器。

@paramminLengthnumber
@returnsValidatorFn
Usage notes

验证该字段至少有 3 个字符

          
            const control = new FormControl('ng', Validators.minLength(3));console.log(control.errors); // {minlength: {requiredLength: 3, actualLength: 2}}
          
        
          
            <input minlength="5">

pattern

ValidatorFn

验证器要求控件的值匹配正则表达式模式。如果使用 HTML5 pattern 属性，此验证器也会默认提供。

@parampatternstring | RegExp

用于测试值的正则表达式或字符串。如果传递了字符串，会在它前面追加 ^ 字符，并在后面追加 $ 字符（如果尚不存在），然后使用所得的正则表达式测试这些值。

@returnsValidatorFn
Usage notes

验证该字段只包含字母或空格

          
            const control = new FormControl('1', Validators.pattern('[a-zA-Z ]*'));console.log(control.errors); // {pattern: {requiredPattern: '^[a-zA-Z ]*$', actualValue: '1'}}
          
        
          
            <input pattern="[a-zA-Z ]*">

带有全局或粘性（sticky）标志的匹配模式

当要连续运行验证时，使用传递给 Validators.pattern 的 g 或 y 标志创建的 RegExp 对象可以在同一输入上产生不同的结果。这是由于在 ECMA-262 中为 RegExp.prototype.test 定义的行为（RegExp 保留了最后一个匹配项的索引）。由于这种现象，建议你使用 Validators.pattern 时不要传入启用了全局或粘性标志的 RegExp。

          
            // Not recommended (since the `g` flag is used)const controlOne = new FormControl('1', Validators.pattern(/foo/g));// Goodconst controlTwo = new FormControl('1', Validators.pattern(/foo/));
