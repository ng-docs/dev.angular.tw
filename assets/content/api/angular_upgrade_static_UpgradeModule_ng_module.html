描述

一個 NgModule，你可以匯入它以提供 AngularJS 核心服務，並且它有一個實例方法，用於啟動混合升級應用。

支援預先編譯的混合升級應用的 upgrade/static 函式庫的一部分

upgrade/static 套件中包含一些輔助程式，它們允許 AngularJS 和 Angular 元件在支援預先編譯的混合升級應用中一起使用。

具體來說，upgrade/static 模組中的類別和函式允許以下操作：

  1. 建立一個 Angular 指令，它可以包裝和公開一個 AngularJS 元件，以便它可以在 Angular 範本中使用。請參閱 UpgradeComponent。
  2. 建立一個 AngularJS 指令，它可以包裝和公開一個 Angular 元件，以便它可以在 AngularJS 範本中使用。請參閱 downgradeComponent。
  3. 建立一個 Angular 根注入器提供者，它可以包裝和公開一個 AngularJS 服務，以便它可以注入到 Angular 上下文中。請參閱下面的升級 AngularJS 服務。
  4. 建立一個 AngularJS 服務，它可以包裝和公開一個 Angular 可注入物件，以便它可以注入到 AngularJS 上下文中。請參閱 downgradeInjectable。
  5. 啟動一個混合 Angular 應用，其中包含在單個應用中共存的兩個框架。

用法說明

          
import {UpgradeModule} from '@angular/upgrade/static';

心智模型

當推理混合應用如何工作時，擁有一個心智模型會很有用，該模型描述了正在發生的事情，並解釋了在最低級別上正在發生的事情。

  1. 在單個應用中執行著兩個獨立的框架，每個框架都將另一個框架視為黑盒。
  2. 頁面上的每個 DOM 元素都恰好歸一個框架所有。無論哪個框架實例化了該元素，該框架就是所有者。每個框架僅更新/與其自己的 DOM 元素互動，而忽略其他元素。
  3. 無論 AngularJS 指令在何處實例化，它們始終在 AngularJS 框架程式碼庫內部執行。
  4. 無論 Angular 元件在何處實例化，它們始終在 Angular 框架程式碼庫內部執行。
  5. AngularJS 元件可以『升級』為 Angular 元件。這是透過定義一個 Angular 指令來實現的，該指令在其在 DOM 中的位置啟動 AngularJS 元件。請參閱 UpgradeComponent。
  6. Angular 元件可以『降級』為 AngularJS 元件。這是透過定義一個 AngularJS 指令來實現的，該指令在其在 DOM 中的位置啟動 Angular 元件。請參閱 downgradeComponent。
  7. 每當實例化『升級』/『降級』元件時，宿主元素都歸執行實例化的框架所有。然後，另一個框架實例化並擁有該組件的檢視。
    1. 這意味著元件繫結將始終遵循實例化框架的語義。
    2. DOM Attribute 由擁有當前範本的框架解析。因此，AngularJS 範本中的 Attribute 必須使用 kebab-case，而 AngularJS 範本必須使用 camelCase。
    3. 但是，範本繫結語法將始終使用 Angular 風格，例如，方括號 ([...]) 用於屬性（Property）繫結。
  8. Angular 首先啟動；AngularJS 其次啟動。AngularJS 始終擁有應用的根元件。
  9. 新的應用執行在 Angular Zone 中，因此不再需要呼叫 $apply()。

UpgradeModule 類別

這個類別是一個 NgModule，你可以匯入它以提供 AngularJS 核心服務，並且它有一個實例方法用於啟動混合升級應用。

  • AngularJS 核心服務
    匯入此 NgModule 將會把核心 AngularJS 服務的提供者新增到根注入器中。

  • 啟動引導
    這個類別的執行時實例包含一個 bootstrap() 方法，你可以使用它把頂層的 AngularJS 模組啟動引導到混合升級應用的 DOM 元素上。

    它還包含一些屬性，可以用來訪問 root injector、啟動引導 NgZone 和 AngularJS $injector。

範例

將 UpgradeModule 匯入到你的頂層 Angular NgModule 中。

          
// This NgModule represents the Angular pieces of the application@NgModule({  declarations: [Ng2HeroesComponent, Ng1HeroComponentWrapper],  providers: [    HeroesService,    // Register an Angular provider whose value is the "upgraded" AngularJS service    {provide: TextFormatter, useFactory: (i: any) => i.get('textFormatter'), deps: ['$injector']},  ],  // We must import `UpgradeModule` to get access to the AngularJS core services  imports: [BrowserModule, UpgradeModule],})export class Ng2AppModule {}

然後將 UpgradeModule 注入到你的 Angular NgModule 中，並在 ngDoBootstrap() 方法中使用它來啟動引導頂層的 AngularJS 模組。

          
export class Ng2AppModule {  constructor(private upgrade: UpgradeModule) {}  ngDoBootstrap() {    // We bootstrap the AngularJS app.    this.upgrade.bootstrap(document.body, [ng1AppModule.name]);  }}

最後，透過啟動引導你的頂層 Angular NgModule 來啟動整個過程。

          
// We bootstrap the Angular module as we would do in a normal Angular app.// (We are using the dynamic browser platform as this example has not been compiled AOT.)platformBrowserDynamic().bootstrapModule(Ng2AppModule);

升級 AngularJS 服務

沒有用於升級 AngularJS 服務的特定 API。相反，你應該遵循以下指南：

假設你有一個 AngularJS 服務：

          
export class TextFormatter {  titleCase(value: string) {    return value.replace(/((^|\s)[a-z])/g, (_, c) => c.toUpperCase());  }}// This AngularJS service will be "upgraded" to be used in Angularng1AppModule.service('textFormatter', [TextFormatter]);

那麼你應該定義一個 Angular 提供者，以便包含在你的 NgModule 的 providers 屬性中。

          
// Register an Angular provider whose value is the "upgraded" AngularJS service    {provide: TextFormatter, useFactory: (i: any) => i.get('textFormatter'), deps: ['$injector']},

然後你可以透過將『升級後』的 AngularJS 服務注入到 Angular 元件或服務中使用它。

          
constructor(textFormatter: TextFormatter) {    // Change all the hero names to title case, using the "upgraded" AngularJS service    this.heroes.forEach((hero: Hero) => (hero.name = textFormatter.titleCase(hero.name)));  }
