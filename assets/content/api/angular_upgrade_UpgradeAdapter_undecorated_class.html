downgradeNg2Component

Function

允許從 AngularJS 中使用 Angular 元件。

使用 downgradeNg2Component 從 Angular 元件建立一個 AngularJS 指令定義工廠。配接器將從 AngularJS 範本中引導 Angular 元件。

@paramcomponentType<any>
@returnsFunction
用法說明

心智模型

  1. 該組件透過在 AngularJS 範本中列出而被實例化。這意味著宿主元素由 AngularJS 控制，但元件的檢視將由 Angular 控制。
  2. 即使元件是在 AngularJS 中實例化的，它也將使用 Angular 語法。 必須這樣做，因為我們必須遵循 Angular 元件不宣告應如何解釋屬性（Attribute）。
  3. ng-model 由 AngularJS 控制，並透過來自 ControlValueAccessor 介面的方式與降級的 Angular 元件進行通訊

@paramnamestring
@returnsType<any>
用法說明

心智模型

  1. 該組件透過在 Angular 範本中列出而被實例化。這意味著宿主元素由 Angular 控制，但元件的檢視將由 AngularJS 控制。

支援的屬性

  • 繫結（Bindings）：
    • 屬性（Attribute）：<comp name="World">
    • 內嵌： <comp greeting="Hello {{name}}!">
    • 表示式： <comp [name]="username">
    • 事件： <comp (close)="doSomething()">
  • 內容嵌入：是
  • 僅支援 指令定義物件 的部分屬性：
    • compile：不受支援，因為宿主元素歸 Angular 所有，Angular 不允許在編譯期間修改 DOM 結構。
    • controller：受支援。（注意：不支援注入 $attrs 和 $transclude。）
    • controllerAs：受支援。
    • bindToController：受支援。
    • link：受支援。（注意：僅支援 pre-link 函式。）
    • name：受支援。
    • priority：已忽略。
    • replace：不受支援。
    • require：受支援。
    • restrict：必須設定為 'E'。
    • scope：受支援。
    • template：受支援。
    • templateUrl：受支援。
    • terminal：已忽略。
    • transclude：受支援。

說明

使用 UpgradeAdapter 以允許 AngularJS 和 Angular 在單個應用中共存。

UpgradeAdapter 允許：

  1. 從 AngularJS 元件指令建立 Angular 元件 （參見 UpgradeAdapter#upgradeNg1Component）
  2. 從 Angular 元件建立 AngularJS 指令。 （參見 UpgradeAdapter#downgradeNg2Component）
  3. 引導啟動一個混合的 Angular 應用，其中包含在單個應用中共存的兩個框架。

用法說明

心智模型

當思考混合應用如何工作時，擁有一個心智模型會很有用，它可以描述正在發生的事情，並解釋在最低級別上發生的事情。

  1. 在單個應用中執行著兩個獨立的框架，每個框架都將另一個框架視為黑盒。
  2. 頁面上的每個 DOM 元素都恰好歸一個框架所有。無論哪個框架實例化了該元素，該框架就是所有者。每個框架僅更新/與其自己的 DOM 元素互動，而忽略其他元素。
  3. 無論 AngularJS 指令在何處實例化，它們始終在 AngularJS 框架程式碼庫內部執行。
  4. 無論 Angular 元件在何處實例化，它們始終在 Angular 框架程式碼庫內部執行。
  5. AngularJS 元件可以升級為 Angular 元件。這將建立一個 Angular 指令，該指令在該位置引導啟動 AngularJS 元件指令。
  6. Angular 元件可以降級為 AngularJS 元件指令。這將建立一個 AngularJS 指令，該指令在該位置引導啟動 Angular 元件。
  7. 每當實例化配接器元件時，宿主元素都歸執行實例化的框架所有。然後，另一個框架實例化並擁有該組件的檢視。這意味著元件繫結將始終遵循實例化框架的語義。語法始終是 Angular 語法。
  8. AngularJS 始終首先被引導啟動，並擁有最底層的檢視。
  9. 新應用正在 Angular Zone 中執行，因此不再需要呼叫 $apply()。
