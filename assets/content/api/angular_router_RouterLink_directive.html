描述

當應用於範本中的元素時，使該元素成為一個連結，該連結會啟動到某個路由的導向。導向會在頁面上的一個或多個 <router-outlet> 位置開啟一個或多個路由元件。

給定路由設定 [{ path: 'user/:name', component: UserCmp }]， 以下程式碼建立到該路由的靜態連結： <a routerLink="/user/bob">連結到使用者元件</a>

你可以使用動態值來生成連結。 對於動態連結，請傳遞一個路徑段陣列， 後跟每個段的引數。 例如，['/team', teamId, 'user', userName, {details: true}] 會生成一個指向 /team/11/user/bob;details=true 的連結。

多個靜態段可以合併為一個術語，並與動態段組合。 例如，['/team/11/user', userName, {details: true}]

你提供給連結的輸入被視為當前 URL 的增量。 例如，假設當前 URL 是 /user/(box//aux:team)。 連結 <a [routerLink]="['/user/jim']">Jim</a> 會建立 URL /user/(jim//aux:team)。 更多訊息請參閱 Router#createUrlTree。

用法說明

你可以在連結中使用絕對路徑或相對路徑，設定查詢引數， 控制引數的處理方式，並保留導向狀態的歷史記錄。

相對連結路徑

第一個段名稱可以加上字首 /、./ 或 ../。

  • 如果第一個段以 / 開頭，則路由器會從應用的根路徑查詢路由。
  • 如果第一個段以 ./ 開頭，或者不以斜槓開頭，則路由器會在當前啟用路由的子路由中查詢。
  • 如果第一個段以 ../ 開頭，則路由器會在路由樹中向上移動一級。

設定和處理查詢引數與片段

以下連結會將查詢引數和片段新增到生成的 URL：

          
<a [routerLink]="['/user/bob']" [queryParams]="{debug: true}" fragment="education">  link to user component</a>

預設情況下，該指令使用給定的查詢引數構造新的 URL。此範例會生成連結：/user/bob?debug=true#education。

你可以指示該指令透過在連結中指定 queryParamsHandling 選項來以不同的方式處理查詢引數。 允許的值為：

  • 'merge'：將給定的 queryParams 合併到當前的查詢引數中。
  • 'preserve'：保留當前的查詢引數。

例如：

          
<a [routerLink]="['/user/bob']" [queryParams]="{debug: true}" queryParamsHandling="merge">  link to user component</a>

當 routerLink 輸入為 UrlTree 時，不能使用 queryParams、fragment、queryParamsHandling、preserveFragment 和 relativeTo。

請參閱 UrlCreationOptions#queryParamsHandling。

保留導向歷史記錄

你可以提供一個 state 值，該值將持久儲存到瀏覽器的 History.state 屬性中。 例如：

          
<a [routerLink]="['/user/bob']" [state]="{tracingId: 123}">  link to user component</a>

使用 Router#getCurrentNavigation 來檢索已儲存的導向狀態值。例如，要在 NavigationStart 事件期間捕獲 tracingId：

          
// Get NavigationStart eventsrouter.events.pipe(filter(e => e instanceof NavigationStart)).subscribe(e => {  const navigation = router.getCurrentNavigation();  tracingService.trace({id: navigation.extras.state.tracingId});});
