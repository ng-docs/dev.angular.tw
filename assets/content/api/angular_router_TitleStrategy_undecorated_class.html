}

提供一种在路由器导航后设置页面标题的策略。

内置实现会遍历路由器状态快照，并找到具有 title 属性的最深层主 outlet。根据以下 Routes，导航到 /base/child(popup:aux) 将导致文档标题被设置为 "child"。

          
            [  {path: 'base', title: 'base', children: [    {path: 'child', title: 'child'},  ],  {path: 'aux', outlet: 'popup', title: 'popupTitle'}]

此类可以用作自定义标题策略的基类。也就是说，你可以创建自己的类来扩展 TitleStrategy。请注意，在上述示例中，命名 outlet 的 title 从未使用。然而，可以实现自定义策略以结合命名 outlet 中的标。
