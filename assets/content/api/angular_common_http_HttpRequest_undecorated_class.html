body

T | null

請求體；如果未設定，則為 null。

請求體並非強制不可變的，因為它們可以包含對任何使用者定義資料型別的引用。但是，攔截器應注意透過將它們視為不可變來保持冪等性。

headers

HttpHeaders

此請求的傳出標頭。

context

HttpContext

攔截器可以使用的共享且可變上下文

reportProgress

boolean

是否應以公開進度事件的方式發出此請求。

進度事件開銷很大（變更檢測會在每個事件上執行），因此只有當使用者打算監視它們時才應請求它們。

注意：FetchBackend 不支援上傳的進度報告。

withCredentials

boolean

是否應隨傳出憑據（Cookie）一起傳送此請求。

responseType

"arraybuffer" | "blob" | "text" | "json"

伺服器的預期回應型別。

這用於在將回應回傳給請求者之前對其進行適當的解析。

method

string

傳出的 HTTP 請求方法。

引數

HttpParams

發出的 URL 引數。

要在 URL 查詢字串格式中傳遞 HTTP 引數的字串表示形式，可以使用 HttpParamsOptions 的 fromString。例如：

          
new HttpParams({fromString: 'angular=awesome'})

urlWithParams

string

設定了所有 URL 引數的傳出 URL。

transferCache

boolean | { includeHeaders?: string[] | undefined; } | undefined

請求的 HttpTransferCache 選項

serializeBody

string | ArrayBuffer | Blob | FormData | URLSearchParams | null

將自由格式的主體轉換為適合傳輸到伺服器的序列化格式。

描述

一個發出的 HTTP 請求，帶有可選的型別化主體。

HttpRequest 表示一個發出的請求，包括 URL、方法、標頭、主體和其他請求設定選項。實例應被假定為不可變的。要修改 HttpRequest，應使用 clone 方法。
