一个包含可选类型化请求体的传出 HTTP 请求。

HttpRequest 代表一个外发请求，包括 URL、方法、标头、主体和其他请求配置选项。应假定其实例是不可变的。要修改 HttpRequest，应使用 clone 方法。

body

T

请求体，如果没有则为 null。

请求体无法确保自己是不可变的，因为它们可以包含指向任何自定义数据类型的引用。 不过，在拦截器中，要小心维护其幂等性 —— 把它们当做不可变对象。

headers

HttpHeaders

本请求的外发请求头。

context

HttpContext

拦截器可以使用的共享和可变上下文

reportProgress

boolean

该请求是否应该暴露出进度事件。

进度事件很昂贵（在每个事件中都会执行一次变更检测），所以只有当消费者关心这些事件时才应该请求这些进度事件。

注意： FetchBackend 不支持上传的进度报告。

withCredentials

boolean

此请求是否应该带着凭证（Cookie）一起外发。

responseType

"arraybuffer" | "blob" | "text" | "json"

所期待的服务器响应类型。

它用来在把响应对象返回给被请求者之前以恰当的方式解析它。

method

string

外发 HTTP 请求的方法。

params

HttpParams

外发的 URL 参数。

要在 URL 查询字符串格式中传递 HTTP 参数的字符串表示，可以使用 HttpParamsOptions 的 fromString。例如：

          
            new HttpParams({fromString: 'angular=awesome'})

urlWithParams

string

外发的 URL，及其所有 URL 参数。

transferCache

boolean | { includeHeaders?: string[]; }

请求的 HttpTransferCache 选项

serializeBody

string | ArrayBuffer | Blob | FormData | URLSearchParams

把无格式的请求体转换成适合传给服务器的序列化格式。

@returnsstring | ArrayBuffer | Blob | FormData | URLSearchParams

detectContentTypeHeader

string

检测请求体，并尝试给它推断出一个合适的 MIME 类型。

如果没有合适的 MIME 类型，该方法就会返回 null。

@returnsstring
