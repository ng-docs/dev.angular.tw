versionUpdates

Observable<VersionEvent>

每當在服務器上檢測到新版本時，就會發出 VersionDetectedEvent 事件。

每當檢查或下載新版本失敗時，就會發出 VersionInstallationFailedEvent 事件。

每當新版本下載完成並準備好啟用時，就會發出 VersionReadyEvent 事件。

unrecoverable

Observable<UnrecoverableStateEvent>

每當 Service Worker 用來為當前用戶端提供服務的應用版本處於損壞狀態，且不完全重新載入頁面就無法恢復時，就會發出 UnrecoverableStateEvent 事件。

isEnabled

boolean

如果 Service Worker 已啟用（瀏覽器支援並透過 ServiceWorkerModule 啟用），則為 True。

checkForUpdate

Promise<boolean>

檢查更新，並等待直到從伺服器下載新版本並準備好啟用。

@returnsPromise<boolean>

activateUpdate

Promise<boolean>

將當前用戶端（即瀏覽器頁籤）更新到準備好啟用的最新版本。

在大多數情況下，你不應該使用此方法，而應該透過重新載入頁面來更新用戶端。

在不重新載入的情況下更新用戶端很容易導致應用崩潰，這是因為應用外殼程式與其他頁面資源（例如延遲載入的程式碼區塊）之間存在版本不對應，這些程式碼區塊的檔案名可能在不同版本之間發生更改。

僅當您確定此方法對您的特定用例是安全時，才使用此方法。

@returnsPromise<boolean>
