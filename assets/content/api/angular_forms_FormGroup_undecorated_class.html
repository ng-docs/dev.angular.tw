setValue

void

設定 FormGroup 的值。 它接受一個與組結構對應的物件，其中控制元件名稱作為鍵。

@paramvalueɵIsAny<TControl, { [key: string]: any; }, { [K in keyof TControl]: ɵRawValue<TControl[K]>; }>

與組結構對應的控制元件的新值。

@paramoptions{ onlySelf?: boolean | undefined; emitEvent?: boolean | undefined; }

設定選項，用於確定控制元件在值更改後如何傳播更改和發出事件。 設定選項會傳遞給 * updateValueAndValidity 方法。

  • onlySelf：為 true 時，每次更改僅影響此控制元件，而不影響其父控制元件。預設為 false。
  • emitEvent：當為 true 或未提供（預設值）時，當控制元件值更新時，statusChanges 和 valueChanges 可觀察物件都會發出包含最新狀態和值的事件。當為 false 時，不發出任何事件。
@returnsvoid
用例說明

為表單組設定完整的值 

          
const form = new FormGroup({  first: new FormControl(),  last: new FormControl()});console.log(form.value);   // {first: null, last: null}form.setValue({first: 'Nancy', last: 'Drew'});console.log(form.value);   // {first: 'Nancy', last: 'Drew'}

reset

void

重置 FormGroup，將所有後代標記為 pristine 和 untouched，並將所有後代的值設定為其預設值；如果未提供預設值，則設定為 null。

你可以透過傳入一個狀態對映表來重置為特定的表單狀態，該對映表與你的表單結構對應，其中控制元件名稱作為鍵。 該狀態可以是獨立的值，也可以是同時包含值和停用狀態的表單狀態物件。

@paramvalueɵTypedOrUntyped<TControl, ɵIsAny<TControl, { [key: string]: any; }, Partial<{ [K in keyof TControl]: ɵValue<TControl[K]>; }>>, any>

使用初始值或定義初始值和停用狀態的物件來重置控制元件。

@paramoptions{ onlySelf?: boolean | undefined; emitEvent?: boolean | undefined; }

設定選項，用於確定當組重置時，控制元件如何傳播更改和發出事件。

  • onlySelf：為 true 時，每次更改僅影響此控制元件，而不影響其父控制元件。預設為 false。
  • emitEvent：當為 true 或未提供（預設值）時，當控制元件重置時，statusChanges 和 valueChanges 可觀察物件都會發出包含最新狀態和值的事件。當為 false 時，不發出任何事件。設定選項會傳遞給 * updateValueAndValidity 方法。
@returnsvoid
用例說明

重置表單組的值 

          
const form = new FormGroup({  first: new FormControl('first name'),  last: new FormControl('last name')});console.log(form.value);  // {first: 'first name', last: 'last name'}form.reset({ first: 'name', last: 'last name' });console.log(form.value);  // {first: 'name', last: 'last name'}

重置表單組的值和停用狀態 

          
const form = new FormGroup({  first: new FormControl('first name'),  last: new FormControl('last name')});form.reset({  first: {value: 'name', disabled: true},  last: 'last'});console.log(form.value);  // {last: 'last'}console.log(form.get('first').status);  // 'DISABLED'

TValue

控制元件的當前值。

  • 對於 FormControl，為當前值。
  • 對於啟用的 FormGroup，啟用控制元件的值是一個物件，其中包含該組每個成員的鍵值對。

enabled

只要控制元件的 status 不是 DISABLED，該控制元件就是 enabled 的。

errors

ValidationErrors | null

一個物件，其中包含驗證失敗產生的任何錯誤；如果沒有錯誤，則為 null。

pristine

boolean

如果使用者尚未更改 UI 中的值，則控制元件是 pristine 的。

dirty

boolean

如果使用者已更改 UI 中的值，則控制元件是 dirty 的。

touched

boolean

如果控制元件被標記為 touched，則為 True。

一旦使用者在其上觸發了 blur 事件，控制元件就會被標記為 touched。

untouched

boolean

如果控制元件尚未被標記為已觸碰，則為 True

如果使用者尚未在其上觸發 blur 事件，則控制元件為 untouched。

events

Observable<ControlEvent<TValue>>

一個多播可觀察物件，每當控制元件的狀態更改時都會發出事件。 它會在值、狀態、pristine 或 touched 更改時發出。

注意：在值更改時，發出操作在此控制元件的值更新後立即發生。父控制元件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍後更新，因此從此事件的回呼中訪問父控制元件的值（使用 value 屬性）可能會導致獲取尚未更新的值。請改為訂閱父控制元件的 events。 對於其他事件型別，事件在父控制元件更新後發出。

valueChanges

Observable<TValue>

一個多播可觀察物件，每當控制元件的值在 UI 中或以程式設計方式更改時，都會發出一個事件。每次呼叫 enable() 或 disable() 而不傳遞 {emitEvent: false} 作為函式引數時，它也會發出一個事件。

注意：發出操作在此控制元件的值更新後立即發生。父控制元件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍後更新，因此從此事件的回呼中訪問父控制元件的值（使用 value 屬性）可能會導致獲取尚未更新的值。請改為訂閱父控制元件的 valueChanges 事件。

statusChanges

Observable<FormControlStatus>

一個多播可觀察物件，每當控制元件的驗證 status 重新計算時，都會發出一個事件。

updateOn

FormHooks

報告 AbstractControl 的更新策略（意味著控制元件自身更新所依據的事件）。 可能的值：'change' | 'blur' | 'submit' 預設值：'change'

描述

跟蹤一組 FormControl 實例的值和有效性狀態。

FormGroup 將每個子 FormControl 的值聚合到一個物件中，其中每個控制元件名稱都作為鍵。它透過縮減其子級的狀態值來計算其狀態。例如，如果組中的一個控制元件無效，則整個組都將變為無效。

FormGroup 是用於在 Angular 中定義表單的四個基本元素之一，另外三個是 FormControl、FormArray 和 FormRecord。

當實例化 FormGroup 時，傳入子控制元件的集合作為第一個引數。每個子級的鍵註冊控制元件的名稱。

FormGroup 旨在用於預先知道鍵的用例。如果需要動態新增和刪除控制元件，請改用 FormRecord。

FormGroup 接受一個可選的型別引數 TControl，它是一個物件型別，其中內部控制元件型別作為值。

用法說明

建立一個包含 2 個控制元件的表單組 

          
const form = new FormGroup({  first: new FormControl('Nancy', Validators.minLength(2)),  last: new FormControl('Drew'),});console.log(form.value);   // {first: 'Nancy', last; 'Drew'}console.log(form.status);  // 'VALID'

型別引數和可選控制元件

FormGroup 接受一個泛型引數，它是一個包含其內部控制元件的物件。此類型通常會自動推斷，但如果需要，您可以始終顯式指定它。

如果您有可選的控制元件（即，可以刪除它們，您可以在型別中使用 ?）：

          
const form = new FormGroup<{  first: FormControl<string|null>,  middle?: FormControl<string|null>, // Middle name is optional.  last: FormControl<string|null>,}>({  first: new FormControl('Nancy'),  last: new FormControl('Drew'),});

建立一個帶有組級別驗證器的表單組

您可以將組級別驗證器作為第二個引數包含在內，或將組級別非同步驗證器作為第三個引數包含在內。當您要執行考慮多個子控制元件的值的驗證時，這些會派上用場。

          
const form = new FormGroup({  password: new FormControl('', Validators.minLength(2)),  passwordConfirm: new FormControl('', Validators.minLength(2)),}, passwordMatchValidator);function passwordMatchValidator(g: FormGroup) {   return g.get('password').value === g.get('passwordConfirm').value      ? null : {'mismatch': true};}

與 FormControl 實例一樣，您可以選擇將驗證器和非同步驗證器作為選項物件的一部分傳入。

          
const form = new FormGroup({  password: new FormControl('')  passwordConfirm: new FormControl('')}, { validators: passwordMatchValidator, asyncValidators: otherValidator });

為表單組中的所有控制元件設定 updateOn 屬性（Property）

選項物件用於為每個子控制元件的 updateOn 屬性（Property）設定預設值。如果在組級別將 updateOn 設定為 'blur'，則所有子控制元件都預設為 'blur'，除非子控制元件已明確指定了不同的 updateOn 值。

          
const c = new FormGroup({  one: new FormControl()}, { updateOn: 'blur' });

將 FormGroup 與可選控制元件一起使用

在 FormGroup 中可以包含可選控制元件。可以使用 removeControl 稍後刪除可選控制元件，並且在呼叫 reset 時可以省略可選控制元件。必須在組的型別中將可選控制元件宣告為可選。

          
const c = new FormGroup<{one?: FormControl<string>}>({  one: new FormControl('')});

請注意，c.value.one 的型別為 string|null|undefined。這是因為呼叫 c.reset({}) 而不提供可選鍵 one 將導致它變為 null。
