}

跟踪一组 FormControl 实例的值和有效状态。

FormGroup 将每个子 FormControl 的值聚合到一个对象中，每个控件名称作为键。 通过减少其子控件的状态值来计算其状态。例如，如果组中的一个控件无效，则整个组将变为无效。

FormGroup 是用于在 Angular 中定义表单的四个基本构建块之一，另外还有 FormControl、 FormArray 和 FormRecord。

实例化 FormGroup 时，将子控件的集合作为第一个参数传入。 每个子控件的键注册控件的名称。

FormGroup 旨在用于已知键的情况。 如果需要动态添加和移除控件，请使用 FormRecord 。

FormGroup 接受一个可选的类型参数 TControl，该参数是一个对象类型，其中包含内部控件类型作为值。

创建一个带有两个控件的表单组

          
            const form = new FormGroup({  first: new FormControl('Nancy', Validators.minLength(2)),  last: new FormControl('Drew'),});console.log(form.value);   // {first: 'Nancy', last; 'Drew'}console.log(form.status);  // 'VALID'

类型参数和可选控件

FormGroup 接受一个泛型参数，该参数是一个包含其内部控件的对象。 这种类型通常会自动推断，但如果你希望的话，你也可以明确指定它。

如果你有可选的控件（即它们可以被删除，你可以在类型中使用 ?）：

          
            const form = new FormGroup<{  first: FormControl<string|null>,  middle?: FormControl<string|null>, // Middle name is optional.  last: FormControl<string|null>,}>({  first: new FormControl('Nancy'),  last: new FormControl('Drew'),});

创建一个具有组级验证器的表单组

你可以用第二个参数传入一些组级验证器或用第三个参数传入一些组级异步验证器。 当你要根据一个以上子控件的值来决定有效性时，这很好用。

          
            const form = new FormGroup({  password: new FormControl('', Validators.minLength(2)),  passwordConfirm: new FormControl('', Validators.minLength(2)),}, passwordMatchValidator);function passwordMatchValidator(g: FormGroup) {   return g.get('password').value === g.get('passwordConfirm').value      ? null : {'mismatch': true};}

与 FormControl 实例一样，你可以选择将验证器和异步验证器作为选项对象的一部分传入。

          
            const form = new FormGroup({  password: new FormControl('')  passwordConfirm: new FormControl('')}, { validators: passwordMatchValidator, asyncValidators: otherValidator });

设置表单组中所有控件的 updateOn 属性

选项对象用于为每个子控件的 updateOn 属性设置默认值。如果你在组级别设置 updateOn 为 'blur'，则所有子控件默认设置为 'blur'，除非子控件明确指定了不同的 updateOn 值。

          
            const c = new FormGroup({  one: new FormControl()}, { updateOn: 'blur' });

使用带有可选控件的 FormGroup

FormGroup 中可以有可选控件。可选控件可以在以后使用 removeControl 删除，并且在调用 reset 时可以省略。可选控件必须在组的类型中声明为 optional。

          
            const c = new FormGroup<{one?: FormControl<string>}>({  one: new FormControl('')});

请注意，c.value.one 的类型为 string|null|undefined。这是因为在不提供可选键 one 的情况下调用 c.reset({}) 将导致它变为 null。

events

Observable<ControlEvent<TValue>>

一个多播的可观察者，每当控件的状态发生变化时发出一个事件。它发出值、状态、原始或已触碰变化的事件。

注意：在值更改时，发出事件的时间会紧跟在此控件值更新后。父控件的值（例如，如果此FormControl是FormGroup的一部分）稍后更新，因此在此事件的回调中访问父控件的值（使用 value属性）可能导致获取尚未更新的值。请改为订阅父控件的 events。对于其他事件类型，事件会在父控件更新后发出。

valueChanges

Observable<TValue>

一个多播的可观察者，每次控件的值在 UI 或程序中发生变化时都会发出事件。每次你调用 enable() 或 disable() 而不传递 {emitEvent: false} 作为函数参数时，它也会发出事件。

注意：发射发生在更新此控件的值之后。父控件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍后更新，因此从该事件的回调中访问父控件的值（使用 value 属性）可能会导致获得一个值尚未更新。改为订阅父控件的 valueChanges 事件。
