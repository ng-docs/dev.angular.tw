getLocaleNumberFormat

string

檢索給定語言環境的數字格式。

數字使用模式格式化，例如 #,###.00。例如，模式 #,###.00 當用于格式化數字 12345.678 時，可能會產生 "12'345,678"。如果你的語言的分組分隔符是撇號，而小數分隔符是逗號，則會發生這種情況。

重要提示：字元 . , 0 #（以及下面的其他字元）是特殊佔位符，它們代表小數分隔符等等，而不是實際字元。 你絕不能『翻譯』佔位符。例如，即使在你的語言中，小數點是用逗號書寫的，也不要將 . 更改為 ,。這些符號應替換為本地等效項，併為你使用的語言使用適當的 NumberSymbol。

以下是數字模式中使用的特殊字元：

符號 含義
. 由用於小數點的字元自動替換。
, 由『分組』（千位）分隔符替換。
0 由數字替換（如果數字不足，則為零）。
# 由數字替換（如果數字不足，則不替換）。
¤ 由貨幣符號替換，例如 $ 或 USD。
% 標記百分比格式。% 符號可能會更改位置，但必須保留。
E 標記科學計數法格式。E 符號可能會更改位置，但必須保留。
' 用作字面量字元的特殊字元用 ASCII 單引號引起來。
