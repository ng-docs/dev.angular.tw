设置启用应用水合功能所需的提供者。

默认情况下，此功能启用推荐的一组特性，以实现大多数应用的最佳性能。它包括以下特性：

  • 协调 DOM 水合。了解更多信息请点击 这里。
  • HttpClient 在服务器上运行时的响应缓存，并将此缓存传输到客户端以避免额外的 HTTP 请求。了解更多关于数据缓存的信息 请点击这里。

这些函数允许你禁用某些默认特性或配置特性。

  • withNoHttpTransferCache 来禁用 HTTP 传输缓存
  • {@link withHttpTransferCacheOptions } 来配置某些 HTTP 传输缓存选项

当使用 bootstrapApplication 函数时，启用应用水合的基本示例：

          
            bootstrapApplication(AppComponent, {  providers: [provideClientHydration()]});

或者，如果你正在使用 NgModules，可以将 provideClientHydration 添加到根应用模块的提供者列表中。

          
            @NgModule({  declarations: [RootCmp],  bootstrap: [RootCmp],  providers: [provideClientHydration()],})export class AppModule {}
