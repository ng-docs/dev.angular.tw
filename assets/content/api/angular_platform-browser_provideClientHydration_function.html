provideClientHydration

EnvironmentProviders

建立必要的提供者，以啟用應用的 hydration 功能。

預設情況下，此函式為大多數應用啟用推薦的一組屬性，以獲得最佳效能。它包括以下屬性：

  • 協調 DOM 水合。要了解更多訊息，請點選這裡。
  • 在服務器上執行時對 HttpClient 回應進行快取，並將此快取傳輸到用戶端，以避免額外的 HTTP 請求。要了解關於資料快取的更多訊息，請點選這裡。

這些函式允許你停用一些預設屬性或啟用新的屬性：

  • withNoHttpTransferCache 以停用 HTTP 傳輸快取
  • withHttpTransferCacheOptions 以設定一些 HTTP 傳輸快取選項
  • withI18nSupport 以啟用對 i18n 控制區塊的水合支援
  • withEventReplay 以啟用對重放使用者事件的支援

用法說明

當使用 bootstrapApplication 函式時，如何在你的應用中啟用 hydration 的基本範例：

          
bootstrapApplication(AppComponent, {  providers: [provideClientHydration()]});

或者，如果你正在使用 NgModules，你可以將 provideClientHydration 新增到你的根應用模組的提供者清單中。

          
@NgModule({  declarations: [RootCmp],  bootstrap: [RootCmp],  providers: [provideClientHydration()],})export class AppModule {}
