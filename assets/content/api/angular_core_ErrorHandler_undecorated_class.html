說明

為集中式異常處理提供了一個鉤子。

ErrorHandler 的預設實現會把錯誤訊息列印到 console 中。要攔截錯誤處理，請編寫一個自定義的異常處理器，並在你的應用中根據需要替換掉這個預設實現。

使用須知

範例 

          
class MyErrorHandler implements ErrorHandler {  handleError(error) {    // do something with the exception  }}// Provide in standalone appsbootstrapApplication(AppComponent, {  providers: [{provide: ErrorHandler, useClass: MyErrorHandler}]})// Provide in module-based apps@NgModule({  providers: [{provide: ErrorHandler, useClass: MyErrorHandler}]})class MyModule {}
