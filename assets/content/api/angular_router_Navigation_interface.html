interface Navigation {
}

有关导航操作的信息。使用Router.getCurrentNavigation() 方法检索最新的导航对象。

  • id ：当前导航的唯一标识符。
  • initialUrl ：在导航之前传递给 Router#navigateByUrl() 调用的目标 URL。这个值是在 Router 解析或应用重定向之前的值。
  • extractedUrl：使用 UrlSerializer.extract() 解析后的初始目标 URL。
  • finalUrl ：应用重定向后的提取 URL。该 URL 可能无法立即获得，因此此属性可以是 undefined。在 RoutesRecognized 事件触发后保证被设置。
  • trigger：表明这次导航是如何触发的。 -- 'imperative'-- 由 router.navigateByUrl 或 router.navigate 触发。 -- 'popstate'-- 由 popstate 事件触发。 -- 'hashchange'-- 由 hashchange 事件触发。
  • extras ：一个 NavigationExtras 选项对象，控制此导航使用的策略。
  • previousNavigation ：先前成功的 Navigation 对象。仅有一个先前的导航可用，因此此先前的 Navigation 对象的 previousNavigation 属性为 null。

trigger

"imperative" | "popstate" | "hashchange"

标识此导航的触发方式。

  • “命令式” - 由 router.navigateByUrl 或 router.navigate 触发。
  • “popstate”--由 popstate 事件触发。
  • “hashchange”--由 hashchange 事件触发。
