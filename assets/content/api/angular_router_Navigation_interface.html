@angular/router

Navigation

interface
程式碼

關於導向操作的訊息。 使用 Router.getCurrentNavigation() 方法檢索最新的導向物件。

API

  
    interface Navigation {}

id

number

當前導向的唯一識別符號。

initialUrl

UrlTree

在導向之前，傳遞給 Router#navigateByUrl() 呼叫的目標 URL。這是路由器解析或應用重導向之前的值。

extractedUrl

UrlTree

使用 UrlHandlingStrategy.extract() 解析後的初始目標 URL。

finalUrl

UrlTree | undefined

應用重導向後提取的 URL。 此 URL 可能不會立即提供，因此此屬性可以是 undefined。 保證在 RoutesRecognized 事件觸發後設置。

trigger

"imperative" | "popstate" | "hashchange"

標識此導向是如何觸發的。

  • 『imperative』——由 router.navigateByUrl 或 router.navigate 觸發。
  • 『popstate』——由 popstate 事件觸發。
  • 『hashchange』——由 hashchange 事件觸發。

extras

NavigationExtras

控制用於此導向的策略的選項。 請參閱 NavigationExtras。

previousNavigation

Navigation | null

先前成功的 Navigation 物件。只有一個先前的導向可用，因此，此先前的 Navigation 物件的 previousNavigation 屬性值為 null。

說明

關於導向操作的訊息。 使用 Router.getCurrentNavigation() 方法檢索最新的導向物件。

  • id ：當前導向的唯一識別符號。
  • initialUrl ：在導向之前，傳遞給 Router#navigateByUrl() 呼叫的目標 URL。 這是路由器解析或應用重導向之前的值。
  • extractedUrl ：使用 UrlSerializer.extract() 解析後的初始目標 URL。
  • finalUrl ：應用重導向後提取的 URL。 此 URL 可能不會立即提供，因此此屬性可以是 undefined。 保證在 RoutesRecognized 事件觸發後設置。
  • trigger ：標識此導向是如何觸發的。 ——『imperative』——由 router.navigateByUrl 或 router.navigate 觸發。 ——『popstate』——由 popstate 事件觸發。 ——『hashchange』——由 hashchange 事件觸發。
  • extras ：一個 NavigationExtras 選項物件，用於控制此導向所用的策略。
  • previousNavigation ：先前成功的 Navigation 物件。只有一個先前的導向可用，因此，此先前的 Navigation 物件的 previousNavigation 屬性值為 null。
