允许从 AngularJS 使用 Angular 组件的帮助器函数。

属于 upgrade/static 库的一部分，用于支持 AOT 编译的混合升级应用。

此帮助器函数返回一个工厂函数，用于注册 AngularJS 包装器指令以“降级” Angular 组件。

例子

假设你有一个名为 ng2Heroes 的 Angular 组件，需要在 AngularJS 模板中提供。

我们必须创建一个 AngularJS 指令，将此 Angular 组件在 AngularJS 模板中可用。 downgradeComponent() 函数返回一个工厂函数，我们可以使用它来定义包装「降级」组件的 AngularJS 指令。

有关将 Angular 组件降级为 AngularJS 组件的更多细节和示例，请访问 升级指南。

downgradeComponent

any

允许从 AngularJS 使用 Angular 组件的帮助器函数。

属于 upgrade/static 库的一部分，用于支持 AOT 编译的混合升级应用。

此帮助器函数返回一个工厂函数，用于注册 AngularJS 包装器指令以“降级” Angular 组件。

@paraminfo{ component: Type<any>; downgradedModule?: string; propagateDigest?: boolean; inputs?: string[]; outputs?: string[]; selectors?: string[]; }

包含有关要降级的组件的信息：

  • component: Type<any>：将要降级的组件类型
  • downgradedModule?: string：降级模块的名称（如果有），即组件所属的模块，调用 downgradeModule() 时返回。此模块将在需要实例化组件时引导相应的 Angular 模块。
    （仅在使用 downgradeModule() 降级多个 Angular 模块时需要此选项。）
  • propagateDigest?: boolean：是否在每次 ChangeDetectorRefdetectChanges 时对组件执行 $digest。如果设置为 false，当组件的任何输入发生变化时仍会执行变更检测。（默认值：true）
@returnsany
