单元测试依赖于升级后的 AngularJS 服务的 Angular 服务时要使用的帮助器函数。

createAngularTestingModule(
  angularJSModules: string[],
  strictDi?: boolean
): Type<any>;

单元测试依赖于升级后的 AngularJS 服务的 Angular 服务时要使用的帮助器函数。

该函数返回一个 NgModule 装饰类，该类配置为连接 Angular 和 AngularJS 注入器，而无需实际引导混合应用。这使得单元测试服务更加简单和快速。

在配置 TestBed 时，将返回的类作为“import”使用。

在以下代码片段中，我们使用两个导入配置 TestBed。Ng2AppModule 是我们混合应用程序的 Angular 部分，而 ng1AppModule 是 AngularJS 部分。

完成此操作后，我们可以像平常一样通过 Angular 的 Injector 获取服务。那些（或依赖于）升级的 AngularJS 服务的服务，将由 AngularJS 的 $injector 根据需要实例化。

在以下代码片段中，HeroesService 是一项依赖于 AngularJS 服务 titleCase 的 Angular 服务。

此辅助函数用于测试服务而不是组件。对于组件测试，你仍需引导混合应用。详见 UpgradeModule 或 downgradeModule。

生成的配置不会将 AngularJS 摘要连接到 Zone 钩子。测试作者有责任根据需要调用 $rootScope.$apply ，以触发来自 Angular 的异步事件的 AngularJS 处理程序。

帮助器设置全局变量来保存共享的 Angular 和 AngularJS 注入器。

  • 每个规范只调用一次此帮助器。
  • 不要在同一个规范中同时使用 createAngularTestingModule 和 createAngularJSTestingModule。

以下是使用 createAngularTestingModule 和 createAngularJSTestingModule 的示例应用及其单元测试。
