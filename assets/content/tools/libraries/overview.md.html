Angular 库开发概览

edit

许多应用都需要解决一些同样的常见问题，比如提供统一的用户界面、渲染数据，以及允许数据输入。开发人员可以为特定的领域创建一些通用解决方案，以便在不同的应用中重复使用。像这样的解决方案就可以构建成 Angular 库，这些库可以作为 npm 包进行发布和共享。

Angular 库是一个与应用不同的 Angular 项目，它无法独立运行。 一个库必须在应用中导入和使用。

库扩展了 Angular 的基本特性。 例如，要向应用添加 响应式表单，可以使用 ng add @angular/forms 添加库包，然后在你的应用代码中从 @angular/forms 库中导入 ReactiveFormsModule。 类似地，将 服务工作线程 库添加到 Angular 应用是将应用转变为 渐进式 Web 应用（PWA）的一步。 Angular Material 是一个大型通用库的示例，它提供了复杂、可复用和可适应的 UI 组件。

任何应用开发者都可以使用这些由 Angular 团队或第三方发布为 npm 包的库。 参见 使用已发布的库。

提示： 库旨在供 Angular 应用使用。要向非 Angular Web 应用添加 Angular 特性，请使用 Angular 自定义元素。

创建库

如果你开发了适合复用的特性，可以创建你自己的库。 这些库可以在你的工作区本地使用，或者你可以将它们作为 npm 包 发布，与其他项目或其他 Angular 开发者共享。 这些包可以发布到 npm 注册表、私有 npm 企业注册表或支持 npm 包的私有包管理系统。 参见 创建库。

是否把一些功能打包成库是一种架构决策。做出比较以决定一个功能应该做成组件还是服务，或决定一个组件的范围该有多大。

把功能打包成库会强迫库中的工件与应用的业务逻辑分离。这有助于避免各种不良实践或架构失误，这些失误会导致将来很难解耦和复用代码。

把代码放到一个单独的库中比简单地把所有内容都放在一个应用中要复杂得多。它需要更多的时间投入，并且需要管理、维护和更新这个库。不过，当把该库用在多个应用中时，这种复杂性就会得到回报。
