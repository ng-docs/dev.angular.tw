Angular 函式庫概覽

edit

許多應用都需要解決相同的通用問題，例如呈現統一的使用者介面、呈現資料以及允許資料錄入。 開發人員可以為特定領域建立通用解決方案，這些方案可以調整後在不同的應用中複用。 這樣的解決方案可以建立為 Angular 函式庫，並且這些函式庫可以作為 npm 套件 發布和共享。

Angular 函式庫是一個 Angular 專案，它與應用的不同之處在於它不能獨立執行。 函式庫必須在應用中匯入和使用。

函式庫擴充套件了 Angular 的基本屬性。 例如，要嚮應用新增 回應式表單，請使用 ng add @angular/forms 新增函式庫套件，然後在你的應用程式碼中從 @angular/forms 函式庫匯入 ReactiveFormsModule。 同樣，將 Service Worker 函式庫新增到 Angular 應用是將應用轉變為 Progressive Web App (PWA) 的步驟之一。 Angular Material 是一個大型通用函式庫的範例，它提供了複雜、可複用且可適配的 UI 元件。

任何應用開發者都可以使用這些函式庫以及 Angular 團隊或第三方發布的其他函式庫。 請參閱 使用已發布的函式庫。

提示： 函式庫旨在供 Angular 應用使用。要將 Angular 屬性新增到非 Angular Web 應用，請使用 Angular 自定義元素。

建立函式庫

如果你開發了適合複用的屬性，則可以建立自己的函式庫。 這些函式庫可以在你的工作區本地使用，或者你可以將它們作為 npm 套件 發布，以便與其他專案或其他 Angular 開發者共享。 這些套件可以發布到 npm 倉庫、私有 npm 企業倉庫或支援 npm 套件的私有套件管理系統。 請參閱 建立函式庫。

決定將屬性打包為函式庫是一項架構決策。這類似於決定屬性是元件還是服務，或者決定元件的範圍。

將屬性打包為函式庫會強制庫中的工件與應用的業務邏輯解耦。 這可以幫助避免各種不良實踐或架構錯誤，這些錯誤可能會使將來難以解耦和複用程式碼。

將程式碼放入單獨的庫中比僅僅將所有內容放在一個應用中更復雜。 它需要在管理、維護和更新函式庫方面投入更多的時間和思考。 當函式庫在多個應用中使用時，這種複雜性會得到回報。
