Angular CLI

edit

Angular CLI 是一種指令行介面工具，使你可以直接從命令 Shell 搭建、開發、測試、部署和維護 Angular 應用。

Angular CLI 在 npm 上以 @angular/cli 套件的形式發布，幷包含一個名為 ng 的二進位檔案。呼叫 ng 指令時，就會使用 Angular CLI。

無需本地安裝即可試用 Angular

如果你是 Angular 新手，不妨從立即試用！開始，它在現成的基本線上商店應用的上下文中介紹了 Angular 的要點，供你檢查和修改。 這個獨立的教程利用了用於在線開發的互動式 StackBlitz 環境。 在準備就緒之前，你無需建立本地環境。

CLI 指令行語言語法

Angular CLI 在選項語法方面大致遵循 Unix/POSIX 約定。

布林選項

布林選項有兩種形式：--this-option 將標誌設定為 true，--no-this-option 將其設定為 false。 你也可以使用 --this-option=false 或 --this-option=true。 如果未提供任何選項，則該標誌將保持其預設狀態，如參考文件中所列。

陣列選項

陣列選項可以透過兩種形式提供：--option value1 value2 或 --option value1 --option value2。

鍵/值選項

某些選項（如 --define）需要 key=value 對的陣列作為其值。 與陣列選項一樣，鍵/值選項可以透過兩種形式提供： --define 'KEY_1="value1"' KEY_2=true 或 --define 'KEY_1="value1"' --define KEY_2=true。

相對路徑

指定檔案的選項可以作為絕對路徑給出，也可以作為相對於當前工作目錄的路徑給出，當前工作目錄通常是工作區或專案根目錄。
