端到端测试

edit

端到端测试（E2E）是一种测试形式，用于验证整个应用从开始到结束（ "端到端"）是否按预期工作。E2E 测试不同于单元测试，因为它完全与代码的底层实现细节解耦。通常用于以模拟用户交互的方式验证应用。本页面作为指南，介绍如何使用 Angular CLI 在 Angular 中开始端到端测试。

设置端到端测试

Angular CLI 会下载和安装你运行端到端测试所需的一切。

      
ng e2e

ng e2e 命令会首先检查你的项目是否有“e2e”目标。如果找不到，CLI 会提示你选择要使用的 e2e 包，并引导你完成设置。

      
Cannot find "e2e" target for the specified project.
You can add a package that implements these capabilities.
For example:
Cypress: ng add @cypress/schematic
Nightwatch: ng add @nightwatch/schematics
WebdriverIO: ng add @wdio/schematics
Puppeteer: ng add @puppeteer/ng-schematics
Would you like to add a package with "e2e" capabilities now?
No
❯ Cypress
Nightwatch
WebdriverIO
Puppeteer

如果你在上面的列表中没有找到你想要使用的测试运行器，可以使用 ng add 手动添加一个包。

运行端到端测试

现在你的应用已配置好用于端到端测试，我们可以再次运行相同的命令来执行你的测试。

      
ng e2e

注意，使用任何集成的 e2e 包运行测试都没有什么「特别」之处。 ng e2e 命令实际上只是运行 e2e 构建器。你可以随时 创建你自己的自定义构建器，命名为 e2e，并使用 ng e2e 运行它。
