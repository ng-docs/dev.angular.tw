端到端測試

edit

端到端 (E2E) 測試是一種用於斷言你的整個應用從開始到結束或『端到端』都能按預期工作的測試形式。E2E 測試與單元測試的不同之處在於，它與程式碼的底層實現細節完全解耦。它通常用於以模擬使用者與應用互動的方式來驗證應用。本頁是關於如何使用 Angular CLI 開始在 Angular 中進行端到端測試的指南。

建立 E2E 測試

Angular CLI 下載並安裝執行 Angular 應用的端到端測試所需的一切。

      
ng e2e

ng e2e 指令將首先檢查你的專案以查詢 "e2e" 目標。如果找不到，CLI 將提示你想要使用哪個 e2e 套件，並引導你完成建立過程。

      
Cannot find "e2e" target for the specified project.You can add a package that implements these capabilities.For example:Cypress: ng add @cypress/schematicNightwatch: ng add @nightwatch/schematicsWebdriverIO: ng add @wdio/schematicsPlaywright: ng add playwright-ng-schematicsPuppeteer: ng add @puppeteer/ng-schematicsWould you like to add a package with "e2e" capabilities now?No❯ CypressNightwatchWebdriverIOPlaywrightPuppeteer

如果你在上面的清單中沒有找到想要使用的測試執行器，你可以使用 ng add 手動新增一個套件。

執行 E2E 測試

既然你的應用已經設定了端到端測試，我們現在可以執行相同的指令來執行你的測試。

      
ng e2e

請注意，使用任何整合的 e2e 套件執行測試都沒有什麼『特別』之處。ng e2e 指令實際上只是在底層執行 e2e 建構式。你始終可以建立你自己的自定義建構式，命名為 e2e，並使用 ng e2e 執行它。
