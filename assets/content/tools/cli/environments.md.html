設定應用環境

edit

你可以為你的專案定義不同的命名建立設定，例如 development 和 staging，它們具有不同的預設值。

每個命名的設定都可以為適用於各種建構式目標的任何選項設定預設值，例如 build、serve 和 test。 Angular CLI 的 build、serve 和 test 指令隨後可以使用適用於你的目標環境的相應版本來替換檔案。

Angular CLI 設定

Angular CLI 建構式支援 configurations 物件，該物件允許基於指令行上提供的設定來覆蓋建構式的特定選項。

      
{  "projects": {    "my-app": {      "architect": {        "build": {          "builder": "@angular-devkit/build-angular:browser",          "options": {            // By default, disable source map generation.            "sourceMap": false          },          "configurations": {            // For the `debug` configuration, enable source maps.            "debug": {              "sourceMap": true            }          }        },        …      }    }  }}

你可以使用 --configuration 選項選擇要使用的設定。

      
ng build --configuration debug

設定可以應用於任何 Angular CLI 建構式。可以使用逗號分隔符指定多個設定。設定按順序應用，衝突的選項使用最後一個設定中的值。

      
ng build --configuration debug,production,customer-facing

設定特定於環境的預設值

@angular-devkit/build-angular:browser 支援檔案替換，這是一種在執行建立之前替換源檔案的選項。 將此選項與 --configuration 結合使用，可以為你的應用設定特定於環境的資料。

首先生成環境以建立 src/environments/ 目錄，並將專案設定為使用檔案替換。

      
ng generate environments

專案的 src/environments/ 目錄包含基本設定檔案 environment.ts，它為生產環境提供預設設定。 你可以在特定於目標的設定檔案中，為其他環境（如 development 和 staging）覆蓋預設值。

例如：

      
my-app/src/environments├── environment.development.ts├── environment.staging.ts└── environment.ts

基本檔案 environment.ts 包含預設環境設定。 例如：

      
export const environment = {  production: true};

當未指定環境時，build 指令將此用作建立目標。 你可以新增更多變數，可以作為環境物件上的附加屬性，也可以作為單獨的物件。 例如，以下程式碼將變數的預設值新增到預設環境中：

      
export const environment = {  production: true,  apiUrl: 'http://my-prod-url'};

你可以新增特定於目標的設定檔案，例如 environment.development.ts。 以下內容為開發建立目標設定預設值：

      
export const environment = {  production: false,  apiUrl: 'http://my-dev-url'};

在你的應用中使用特定於環境的變數

要使用你定義的環境設定，你的元件必須匯入原始環境檔案：

      
import { environment } from './environments/environment';

這確保了 build 和 serve 指令可以找到特定建立目標的設定。

元件檔案 (app.component.ts) 中的以下程式碼使用在設定檔案中定義的環境變數。

      
import { environment } from './../environments/environment';// Fetches from `http://my-prod-url` in production, `http://my-dev-url` in development.fetch(environment.apiUrl);

主要的 CLI 設定檔案 angular.json 在每個建立目標的設定中都包含一個 fileReplacements 部分，它允許你將 TypeScript 程式中的任何檔案替換為該檔案的特定於目標的版本。 這對於在以特定環境（如生產或暫存環境）為目標的建立中包含特定於目標的程式碼或變數非常有用。

預設情況下，不替換任何檔案，但是 ng generate environments 會自動建立此設定。 你可以透過直接編輯 angular.json 設定來更改或新增特定建立目標的檔案替換。

      
"configurations": {    "development": {      "fileReplacements": [          {            "replace": "src/environments/environment.ts",            "with": "src/environments/environment.development.ts"          }        ],        …

這意味著，當你使用 ng build --configuration development 建立開發設定時，src/environments/environment.ts 檔案將被替換為該檔案的特定於目標的版本 src/environments/environment.development.ts。

要新增暫存環境，請建立 src/environments/environment.ts 的副本，名為 src/environments/environment.staging.ts，然後將 staging 設定新增到 angular.json 中：

      
"configurations": {    "development": { … },    "production": { … },    "staging": {      "fileReplacements": [        {          "replace": "src/environments/environment.ts",          "with": "src/environments/environment.staging.ts"        }      ]    }  }

你也可以向此目標環境新增更多設定選項。 你的建立支援的任何選項都可以在建立目標設定中被覆蓋。

要使用暫存設定進行建立，請執行以下指令：

      
ng build --configuration staging

預設情況下，build 目標包括 production 和 development 設定，並且 ng serve 使用應用的開發版本建立。 如果設定 buildTarget 選項，你還可以設定 ng serve 以使用目標建立設定：

      
"serve": {    "builder": "@angular-devkit/build-angular:dev-server",    "options": { … },    "configurations": {      "development": {        // Use the `development` configuration of the `build` target.        "buildTarget": "my-app:build:development"      },      "production": {        // Use the `production` configuration of the `build` target.        "buildTarget": "my-app:build:production"      }    },    "defaultConfiguration": "development"  },

defaultConfiguration 選項指定預設情況下使用哪個設定。 當未設定 defaultConfiguration 時，將直接使用 options 而不進行修改。
