建立本地環境和工作區

edit

本指南介紹如何使用 Angular CLI 為 Angular 開發建立你的環境。 它包括有關安裝 CLI、建立初始工作區和入門應用以及在本地執行該應用以驗證你的設定的訊息。

無需本地設定即可試用 Angular

如果你是 Angular 新手，不妨從 立即試用！ 開始，它會在你的瀏覽器中介紹 Angular 的要點。 這個獨立的教程利用了用於在線開發的互動式 StackBlitz 環境。 在你準備好之前，你無需建立本地環境。

開始之前

要使用 Angular CLI，你應該熟悉以下內容：

JavaScriptopen_in_new HTMLopen_in_new CSSopen_in_new

你應該熟悉指令行介面 (CLI) 工具的用法，並對指令 Shell 有一個大致的瞭解。 瞭解 TypeScript 會有所幫助，但不是必需的。

依賴

要在你的本地系統上安裝 Angular CLI，你需要安裝 Node.js。 Angular CLI 使用 Node 及其相關的套件管理器 npm，在瀏覽器之外安裝和執行 JavaScript 工具。

下載並安裝 Node.js，其中也將包含 npm CLI。 Angular 需要 active LTS 或 maintenance LTS 版本的 Node.js。 有關更多訊息，請參閱 Angular 的版本相容性 指南。

安裝 Angular CLI

要安裝 Angular CLI，請開啟終端視窗並執行以下指令：

npm

      
npm install -g @angular/cli

pnpm

      
pnpm install -g @angular/cli

yarn

      
yarn global add @angular/cli

bun

      
bun install -g @angular/cli

Powershell 執行策略

在 Windows 用戶端電腦上，預設情況下停用 PowerShell 指令碼的執行，因此上述指令可能會失敗並出現錯誤。 要允許執行 PowerShell 指令碼（npm 全域二進位檔案需要這樣做），你必須設定以下執行策略：

      
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

仔細閱讀執行指令後顯示的訊息，並按照說明進行操作。確保你瞭解設定執行策略的含義。

Unix 許可權

在某些類 Unix 的設定中，全域指令碼可能歸 root 使用者所有，因此上述指令可能會失敗並出現許可權錯誤。 使用 sudo 以 root 使用者身份執行指令，並在出現提示時輸入你的密碼：

npm

      
sudo npm install -g @angular/cli

pnpm

      
sudo pnpm install -g @angular/cli

yarn

      
sudo yarn global add @angular/cli

bun

      
sudo bun install -g @angular/cli

確保你瞭解以 root 身份執行指令的含義。

建立一個工作區和初始應用

你在 Angular 工作區的上下文中開發應用。

要建立一個新的工作區和初始入門應用，請執行 CLI 指令 ng new 並提供名稱 my-app，如此處所示，然後回答有關要包含的功能的提示：

      
ng new my-app

Angular CLI 安裝必要的 Angular npm 套件和其他依賴項。 這可能需要幾分鐘。

CLI 在一個與工作區同名的新目錄中建立一個新的工作區和一個小的歡迎應用，隨時可以執行。 導向到新目錄，以便後續指令使用此工作區。

      
cd my-app

執行應用

Angular CLI 包含一個開發伺服器，供你在本地建立和啟動你的應用。執行以下指令：

      
ng serve --open

ng serve 指令啟動伺服器，監視你的檔案，並在你更改這些檔案時重建應用並重新載入瀏覽器。

--open（或僅 -o）選項會自動開啟你的瀏覽器並訪問 http://localhost:4200/ 以檢視生成的應用。

工作區和專案檔案

ng new 指令建立一個 Angular 工作區 資料夾，並在其中生成一個新的應用。 一個工作區可以包含多個應用和函式庫。 ng new 指令建立的初始應用位於工作區的根目錄下。 當你在現有工作區中生成額外的應用或函式庫時，預設情況下它會進入 projects/ 子資料夾。

新生成的應用套件含根元件和樣板的源檔案。 每個應用都有一個 src 資料夾，其中包含其元件、資料和資產。

你可以直接編輯生成的檔案，或者使用 CLI 指令新增和修改它們。 使用 ng generate 指令為其他元件、指令、通道、服務等新增新檔案。 諸如 ng add 和 ng generate 之類的指令（用於建立或操作應用和函式庫）必須從工作區內執行。 相比之下，諸如 ng new 之類的指令必須在工作區外部執行，因為它們將建立一個新的工作區。

後續步驟

  • 瞭解有關生成的工作區的 檔案結構 和 設定 的更多訊息。

  • 使用 ng test 測試你的新應用。

  • 使用 ng generate 生成樣板程式碼，例如元件、指令和通道。

  • 使用 ng deploy 部署你的新應用，使其對真實使用者可用。

  • 使用 ng e2e 建立並執行你的應用的端到端測試。
