搭建本地开发环境和工作区

edit

本指南解释了如何使用 Angular CLI 配置你的 Angular 开发环境。 其中包括有关安装 CLI、创建初始工作空间和起始应用程序，以及在本地运行该应用程序以验证你的设置的信息。

无需本地环境，学习 Angular

如果你是 Angular 新手，你可能想从 现在开始尝试！ 开始，这会在浏览器中向你介绍 Angular 的基础知识。 这个独立教程利用了在线开发环境 StackBlitz。在准备好之前，你无需设置本地环境。

开始之前

要使用 Angular CLI，你应该熟悉以下内容：

JavaScript open_in_new HTML open_in_new CSS open_in_new

你还应该熟悉命令行界面（CLI）工具的使用，对命令行有一般的了解。 了解 TypeScript 会有帮助，但不是必需的。

依赖项

要在本地系统上安装 Angular CLI，你需要安装 Node.js。 Angular CLI 使用 Node 及其相关的包管理器 npm 来安装和运行浏览器外的 JavaScript 工具。

下载并安装 Node.js，这将包括 npm CLI。 Angular 需要一个 Node.js 的 活跃 LTS 或维护 LTS 版本。 查看 Angular 版本兼容性 指南获取更多信息。

安装 Angular CLI

要安装 Angular CLI，请打开终端/控制台窗口，并运行如下命令：

      
npm install -g @angular/cli

Powershell 执行策略

在 Windows 客户端计算机上，默认情况下禁用了 PowerShell 脚本的执行，因此上述命令可能会因错误而失败。 为了允许执行 PowerShell 脚本（这对于 npm 全局二进制文件是必需的），你必须设置以下 执行策略：

      
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

仔细阅读执行命令后显示的消息并按照说明进行操作。确保你了解设置执行策略的含义。

Unix 权限

在某些类 Unix 设置中，全局 npm 脚本可能由 root 用户拥有，因此上述命令可能因权限错误而失败。 使用 sudo 运行以将命令作为 root 用户执行，并在提示时输入密码：

      
sudo npm install -g @angular/cli

确保了解以 root 身份运行命令的影响。

创建工作区和初始应用

你在 Angular 工作空间 的上下文中开发应用。

要创建一个新的工作空间和初始起始应用程序，请运行 CLI 命令 ng new，并提供名称 my-app，如下所示，然后回答有关要包含的功能的提示：

      
ng new my-app

Angular CLI 会安装必要的 Angular npm 包和其它依赖包。这可能要花几分钟的时间。

CLI 会在一个与工作空间同名的新目录中创建一个新的工作空间和一个小的欢迎应用程序，准备运行。 转到新目录，以便后续命令使用该工作空间。

      
cd my-app

运行应用

Angular CLI 包含一个开发服务器，供你在本地构建和提供应用。运行以下命令：

      
ng serve --open

ng serve 命令启动服务器，监视你的文件，并在你对这些文件进行更改时重新构建应用并重新加载浏览器。

--open（或 -o）选项会自动打开你的浏览器，以查看生成的应用程序在 http://localhost:4200/ 上的情况。

工作区与项目文件

ng new 命令会创建一个 Angular 工作区 文件夹，并在其中生成一个新应用程序。工作区可以包含多个应用程序和库。通过 ng new 命令创建的初始应用程序位于工作区的根目录。当你在现有工作区中生成额外的应用程序或库时，默认情况下会放入 projects/ 子文件夹中。

新生成的应用程序包含用于根组件和模板的源文件。每个应用程序都有一个包含其组件、数据和资产的 src 文件夹。

你可以直接编辑生成的文件，或使用 CLI 命令添加和修改它们。使用 ng generate 命令为额外的组件、指令、管道、服务等添加新文件。 ng add 和 ng generate 等命令创建或操作应用和库，必须在工作区内执行。相比之下， ng new 等命令必须在工作区外执行，因为它们将创建一个新的工作区。

后续步骤

  • 了解有关生成工作区的 文件结构 和 配置 的更多信息。

  • 使用 ng test 测试你的新应用程序。

  • 使用 ng generate 生成诸如组件、指令和管道之类的样板代码。

  • 使用 ng deploy 部署你的新应用程序，并使其对真实用户可用。

  • 使用 ng e2e 设置和运行应用程序的端到端测试。
