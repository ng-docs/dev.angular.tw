函式庫的原理圖

edit

當你建立一個 Angular 函式庫時，你可以提供原理圖並將其與原理圖一起打包，以便將其與 Angular CLI 整合。 藉助你的原理圖，你的使用者可以使用 ng add 來安裝你的函式庫的初始版本，使用 ng generate 來建立你的庫中定義的工件，並使用 ng update 來調整他們的專案以適應引入重大更改的函式庫的新版本。

所有這三種型別的原理圖都可以成為你與函式庫一起打包的集合的一部分。

建立原理圖集合

要啟動一個集合，你需要建立原理圖檔案。 以下步驟向你展示如何在不修改任何專案檔案的情況下新增初始支援。

  1. 在你的函式庫的根資料夾中，建立一個 schematics 資料夾。

  2. 在 schematics/ 資料夾中，為你的第一個原理圖建立一個 ng-add 資料夾。

  3. 在 schematics 資料夾的根級別，建立一個 collection.json 檔案。

  4. 編輯 collection.json 檔案以定義你的集合的初始模式。

    projects/my-lib/schematics/collection.json（原理圖集合）

          
    {  "$schema": "../../../node_modules/@angular-devkit/schematics/collection-schema.json",  "schematics": {    "ng-add": {      "description": "Add my library to the project.",      "factory": "./ng-add/index#ngAdd",      "schema": "./ng-add/schema.json"    }  }}
    • $schema 路徑相對於 Angular Devkit 集合模式。
    • schematics 物件描述了作為此集合一部分的命名原理圖。
    • 第一個條目是針對名為 ng-add 的原理圖。 它包含描述，並指向在執行原理圖時調用的工廠函式。

  5. 在你的庫專案的 package.json 檔案中，新增一個 "schematics" 條目，其中包含指向你的模式檔案的路徑。 當 Angular CLI 執行指令時，它會使用此條目在你的集合中查詢命名的原理圖。

    projects/my-lib/package.json（原理圖集合參考）

          
    {  "name": "my-lib",  "version": "0.0.1",  "scripts": {    "build": "tsc -p tsconfig.schematics.json",    "postbuild": "copyfiles schematics/*/schema.json schematics/*/files/** schematics/collection.json ../../dist/my-lib/"  },  "peerDependencies": {    "@angular/common": "^19.0.0",    "@angular/core": "^19.0.0"  },  "schematics": "./schematics/collection.json",  "ng-add": {    "save": "devDependencies"  },  "devDependencies": {    "copyfiles": "file:../../node_modules/copyfiles",    "typescript": "file:../../node_modules/typescript"  }}

你建立的初始模式告訴 CLI 在哪裡可以找到支援 ng add 指令的原理圖。 現在你已準備好建立該原理圖。

提供安裝支援

用於 ng add 指令的原理圖可以增強使用者的初始安裝過程。 以下步驟定義了這種型別的原理圖。

  1. 轉到 <lib-root>/schematics/ng-add 資料夾。
  2. 建立主檔案 index.ts。
  3. 開啟 index.ts 併為你的原理圖工廠函式新增原始碼。

    projects/my-lib/schematics/ng-add/index.ts（ng-add 規則工廠）

          
    import {Rule} from '@angular-devkit/schematics';import {addRootImport} from '@schematics/angular/utility';import {Schema} from './schema';export function ngAdd(options: Schema): Rule {  // Add an import `MyLibModule` from `my-lib` to the root of the user's project.  return addRootImport(    options.project,    ({code, external}) => code`${external('MyLibModule', 'my-lib')}`,  );}

Angular CLI 將自動安裝函式庫的最新版本，並且此範例透過將 MyLibModule 新增到應用的根目錄更進一步。addRootImport 函式接受一個回呼，該回調需要回傳一個程式碼控制區塊。你可以在用 code 函式標記的字串內編寫任何程式碼，並且任何外部符號都必須用 external 函式包裝，以確保生成適當的匯入語句。

定義依賴型別

使用 ng-add 的 save 選項來設定是否應將函式庫新增到專案的 package.json 設定檔案中的 dependencies、devDependencies，或者根本不儲存。

projects/my-lib/package.json（ng-add 參考）

      
{  "name": "my-lib",  "version": "0.0.1",  "scripts": {    "build": "tsc -p tsconfig.schematics.json",    "postbuild": "copyfiles schematics/*/schema.json schematics/*/files/** schematics/collection.json ../../dist/my-lib/"  },  "peerDependencies": {    "@angular/common": "^19.0.0",    "@angular/core": "^19.0.0"  },  "schematics": "./schematics/collection.json",  "ng-add": {    "save": "devDependencies"  },  "devDependencies": {    "copyfiles": "file:../../node_modules/copyfiles",    "typescript": "file:../../node_modules/typescript"  }}

可能的值為：

值 詳情
false 不要將軟體套件新增到 package.json
true 將套件新增到 dependencies
"dependencies" 將套件新增到 dependencies
"devDependencies" 將套件新增到 devDependencies

建立你的原理圖

要將你的原理圖與你的函式庫捆綁在一起，你必須設定函式庫以分別建立原理圖，然後將它們新增到捆綁包中。 你必須在建立庫之後建立你的原理圖，以便將它們放置在正確的目錄中。

  • 你的函式庫需要一個自定義的 Typescript 設定檔案，其中包含有關如何將你的原理圖編譯到你的已分發庫中的說明
  • 要將原理圖新增到函式庫捆綁包，請將指令碼新增到函式庫的 package.json 檔案中

假設你的 Angular 工作區中有一個庫專案 my-lib。 要告知函式庫如何建立原理圖，請在設定函式庫建立的已生成的 tsconfig.lib.json 檔案旁邊新增一個 tsconfig.schematics.json 檔案。

  1. 編輯 tsconfig.schematics.json 檔案以新增以下內容。

    projects/my-lib/tsconfig.schematics.json（TypeScript 設定）

          
    {  "compilerOptions": {    "baseUrl": ".",    "lib": [      "es2018",      "dom"    ],    "declaration": true,    "module": "commonjs",    "moduleResolution": "node",    "noEmitOnError": true,    "noFallthroughCasesInSwitch": true,    "noImplicitAny": true,    "noImplicitThis": true,    "noUnusedParameters": true,    "noUnusedLocals": true,    "rootDir": "schematics",    "outDir": "../../dist/my-lib/schematics",    "skipDefaultLibCheck": true,    "skipLibCheck": true,    "sourceMap": true,    "strictNullChecks": true,    "target": "es6",    "types": [      "jasmine",      "node"    ]  },  "include": [    "schematics/**/*"  ],  "exclude": [    "schematics/*/files/**/*"  ]}
    選項 詳情
    rootDir 指定你的 schematics 資料夾包含要編譯的輸入檔案。
    outDir 對映到函式庫的輸出資料夾。預設情況下，這是你的工作區根目錄下的 dist/my-lib 資料夾。

  2. 為了確保你的原理圖源檔案被編譯到函式庫捆綁包中，請將以下指令碼新增到你的庫專案根資料夾 (projects/my-lib) 中的 package.json 檔案中。

    projects/my-lib/package.json（建立指令碼）

          
    {  "name": "my-lib",  "version": "0.0.1",  "scripts": {    "build": "tsc -p tsconfig.schematics.json",    "postbuild": "copyfiles schematics/*/schema.json schematics/*/files/** schematics/collection.json ../../dist/my-lib/"  },  "peerDependencies": {    "@angular/common": "^19.0.0",    "@angular/core": "^19.0.0"  },  "schematics": "./schematics/collection.json",  "ng-add": {    "save": "devDependencies"  },  "devDependencies": {    "copyfiles": "file:../../node_modules/copyfiles",    "typescript": "file:../../node_modules/typescript"  }}
    • build 指令碼使用自定義的 tsconfig.schematics.json 檔案編譯你的原理圖
    • postbuild 指令碼在 build 指令碼完成後複製原理圖檔案
    • build 和 postbuild 指令碼都需要 copyfiles 和 typescript 依賴項。 要安裝依賴項，請導向到 devDependencies 中定義的路徑，並在執行指令碼之前執行 npm install。

提供生成支援

你可以向你的集合新增一個命名的原理圖，讓你的使用者可以使用 ng generate 指令來建立你的庫中定義的工件。

我們假設你的函式庫定義了一個服務 my-service，它需要一些設定。 你希望你的使用者能夠使用以下 CLI 指令來生成它。

      
ng generate my-lib:my-service

首先，在 schematics 資料夾中建立一個新的子資料夾 my-service。

設定新的原理圖

當你向集合新增原理圖時，你必須在集合的模式中指向它，並提供設定檔案來定義使用者可以傳遞給指令的選項。

  1. 編輯 schematics/collection.json 檔案以指向新的原理圖子資料夾，幷包含指向模式檔案的指標，該模式檔案指定新原理圖的輸入。

    projects/my-lib/schematics/collection.json（原理圖集合）

          
    {  "$schema": "../../../node_modules/@angular-devkit/schematics/collection-schema.json",  "schematics": {    "ng-add": {      "description": "Add my library to the project.",      "factory": "./ng-add/index#ngAdd",      "schema": "./ng-add/schema.json"    },    "my-service": {      "description": "Generate a service in the project.",      "factory": "./my-service/index#myService",      "schema": "./my-service/schema.json"    }  }}

  2. 轉到 <lib-root>/schematics/my-service 資料夾。

  3. 建立一個 schema.json 檔案，併為原理圖定義可用的選項。

    projects/my-lib/schematics/my-service/schema.json（原理圖 JSON 模式）

          
    {  "$schema": "http://json-schema.org/schema",  "$id": "SchematicsMyService",  "title": "My Service Schema",  "type": "object",  "properties": {    "name": {      "description": "The name of the service.",      "type": "string"    },    "path": {      "type": "string",      "format": "path",      "description": "The path to create the service.",      "visible": false,      "$default": {        "$source": "workingDirectory"      }    },    "project": {      "type": "string",      "description": "The name of the project.",      "$default": {        "$source": "projectName"      }    }   },  "required": [    "name"  ]}
    • id：集合中模式的唯一 ID。
    • title：模式的人類可讀的描述。
    • type：屬性提供的型別的描述符。
    • properties：一個物件，用於定義原理圖的可用選項。

    每個選項都將鍵與型別、描述和可選別名關聯起來。 型別定義了你期望的值的形狀，當用戶請求你的原理圖的用法幫助時，將顯示該描述。

    有關原理圖選項的其他自定義項，請參閱工作區模式。

  4. 建立一個 schema.ts 檔案，並定義一個介面，用於儲存在 schema.json 檔案中定義的選項的值。

    projects/my-lib/schematics/my-service/schema.ts（原理圖介面）

          
    export interface Schema {  // The name of the service.  name: string;  // The path to create the service.  path?: string;  // The name of the project.  project?: string;}
    選項 詳情
    name 你想為建立的服務提供的名稱。
    path 覆蓋提供給原理圖的路徑。預設路徑值基於當前工作目錄。
    project 提供一個特定的專案來執行原理圖。在原理圖中，如果使用者未提供該選項，你可以提供一個預設值。

新增樣板檔案

要向專案新增工件，你的原理圖需要自己的樣板檔案。 原理圖樣板支援特殊的語法來執行程式碼和變數替換。

  1. 在 schematics/my-service/ 資料夾內建立一個 files/ 資料夾。

  2. 建立一個名為 __name@dasherize__.service.ts.template 的檔案，用於定義生成檔案時要使用的樣板。 此樣板將生成一個服務，該服務已將 Angular 的 HttpClient 注入到 http 屬性中。

    projects/my-lib/schematics/my-service/files/__name@dasherize__.service.ts.template（原理圖樣板）

          
    import { Injectable } from '@angular/core'; import { HttpClient } from '@angular/common/http'; @Injectable({   providedIn: 'root' }) export class <%= classify(name) %>Service {   private http = inject(HttpClient); }

    • classify 和 dasherize 方法是實用工具函式，你的原理圖使用它們來轉換你的源樣板和檔案名。

    • name 是從你的工廠函式中作為屬性提供的。 它與你在模式中定義的 name 相同。

新增工廠函式

既然你已經有了基礎設施，你就可以定義主函式，該函式執行你在使用者的專案中需要的修改。

原理圖框架提供了一個檔案樣板系統，該系統同時支援路徑和內容樣板。 該系統對輸入 Tree 中載入的檔案或路徑內定義的佔位符進行操作。 它使用傳遞到 Rule 中的值填充這些佔位符。

有關這些資料結構和語法的詳細訊息，請參閱 原理圖 README。

  1. 建立主檔案 index.ts 併為你的原理圖工廠函式新增原始碼。

  2. 首先，匯入你將需要的原理圖定義。 原理圖框架提供了許多實用工具函式，用於在執行原理圖時建立和使用規則。

    projects/my-lib/schematics/my-service/index.ts（匯入）

          
    import {  Rule,  Tree,  SchematicsException,  apply,  url,  applyTemplates,  move,  chain,  mergeWith,} from '@angular-devkit/schematics';import {strings, normalize, virtualFs, workspaces} from '@angular-devkit/core';import {Schema as MyServiceSchema} from './schema';function createHost(tree: Tree): workspaces.WorkspaceHost {  return {    async readFile(path: string): Promise<string> {      const data = tree.read(path);      if (!data) {        throw new SchematicsException('File not found.');      }      return virtualFs.fileBufferToString(data);    },    async writeFile(path: string, data: string): Promise<void> {      return tree.overwrite(path, data);    },    async isDirectory(path: string): Promise<boolean> {      return !tree.exists(path) && tree.getDir(path).subfiles.length > 0;    },    async isFile(path: string): Promise<boolean> {      return tree.exists(path);    },  };}export function myService(options: MyServiceSchema): Rule {  return async (tree: Tree) => {    const host = createHost(tree);    const {workspace} = await workspaces.readWorkspace('/', host);    const project = options.project != null ? workspace.projects.get(options.project) : null;    if (!project) {      throw new SchematicsException(`Invalid project name: ${options.project}`);    }    const projectType = project.extensions.projectType === 'application' ? 'app' : 'lib';    if (options.path === undefined) {      options.path = `${project.sourceRoot}/${projectType}`;    }    const templateSource = apply(url('./files'), [      applyTemplates({        classify: strings.classify,        dasherize: strings.dasherize,        name: options.name,      }),      move(normalize(options.path as string)),    ]);    return chain([mergeWith(templateSource)]);  };}

  3. 匯入已定義的模式介面，該介面為你的原理圖選項提供型別訊息。

    projects/my-lib/schematics/my-service/index.ts（模式匯入）

'app' : 'lib';    if (options.path === undefined) {      options.path = `${project.sourceRoot}/${projectType}`;    }    const templateSource = apply(url('./files'), [      applyTemplates({        classify: strings.classify,        dasherize: strings.dasherize,        name: options.name,      }),      move(normalize(options.path as string)),    ]);    return chain([mergeWith(templateSource)]);  };}

  4. 要建立生成原理圖，請從一個空的規則工廠開始。

    projects/my-lib/schematics/my-service/index.ts（初始規則）

          
    import {Rule, Tree} from '@angular-devkit/schematics';import {Schema as MyServiceSchema} from './schema';export function myService(options: MyServiceSchema): Rule {  return (tree: Tree) => tree;}

此規則工廠回傳未經修改的樹。 這些選項是從 ng generate 指令傳遞過來的選項值。

定義生成規則

你現在已經有了框架，可以建立實際修改使用者應用的程式碼，以便為你的庫中定義的服務進行設定。

使用者安裝你的函式庫的 Angular 工作區包含多個專案（應用和函式庫）。 使用者可以在指令行上指定專案，也可以使用預設專案。 在任何一種情況下，你的程式碼都需要識別出此原理圖正在應用到的特定專案，以便你可以從專案設定中檢索訊息。

使用傳遞到工廠函式中的 Tree 物件來執行此操作。 Tree 方法使你可以訪問工作區中的完整檔案樹，從而允許你在原理圖執行期間讀取和寫入檔案。

獲取專案設定

  1. 要確定目標專案，請使用 workspaces.readWorkspace 方法讀取工作區設定檔案 angular.json 的內容。 要使用 workspaces.readWorkspace，你需要從 Tree 建立一個 workspaces.WorkspaceHost。 將以下程式碼新增到你的工廠函式中。

    projects/my-lib/schematics/my-service/index.ts（模式匯入）

'app' : 'lib';    if (options.path === undefined) {      options.path = `${project.sourceRoot}/${projectType}`;    }    const templateSource = apply(url('./files'), [      applyTemplates({        classify: strings.classify,        dasherize: strings.dasherize,        name: options.name,      }),      move(normalize(options.path as string)),    ]);    return chain([mergeWith(templateSource)]);  };}

    請務必檢查上下文是否存在並丟擲相應的錯誤。

  2. 現在你有了專案名稱，使用它來檢索特定於專案的設定訊息。

    projects/my-lib/schematics/my-service/index.ts（專案）

          
    import {  Rule,  Tree,  SchematicsException,  apply,  url,  applyTemplates,  move,  chain,  mergeWith,} from '@angular-devkit/schematics';import {strings, normalize, virtualFs, workspaces} from '@angular-devkit/core';import {Schema as MyServiceSchema} from './schema';function createHost(tree: Tree): workspaces.WorkspaceHost {  return {    async readFile(path: string): Promise<string> {      const data = tree.read(path);      if (!data) {        throw new SchematicsException('File not found.');      }      return virtualFs.fileBufferToString(data);    },    async writeFile(path: string, data: string): Promise<void> {      return tree.overwrite(path, data);    },    async isDirectory(path: string): Promise<boolean> {      return !tree.exists(path) && tree.getDir(path).subfiles.length > 0;    },    async isFile(path: string): Promise<boolean> {      return tree.exists(path);    },  };}export function myService(options: MyServiceSchema): Rule {  return async (tree: Tree) => {    const host = createHost(tree);    const {workspace} = await workspaces.readWorkspace('/', host);    const project = options.project != null ? workspace.projects.get(options.project) : null;    if (!project) {      throw new SchematicsException(`Invalid project name: ${options.project}`);    }    const projectType = project.extensions.projectType === 'application' ? 'app' : 'lib';    if (options.path === undefined) {      options.path = `${project.sourceRoot}/${projectType}`;    }    const templateSource = apply(url('./files'), [      applyTemplates({        classify: strings.classify,        dasherize: strings.dasherize,        name: options.name,      }),      move(normalize(options.path as string)),    ]);    return chain([mergeWith(templateSource)]);  };}

    workspace.projects 物件包含所有特定於專案的設定訊息。

  3. options.path 確定在應用原理圖後，原理圖樣板檔案移動到哪裡。

    原理圖模式中的 path 選項預設替換為當前工作目錄。 如果未定義 path，請使用專案設定中的 sourceRoot 以及 projectType。

    projects/my-lib/schematics/my-service/index.ts（專案訊息）

定義規則

Rule 可以使用外部樣板檔案，轉換它們，並回傳另一個帶有轉換後樣板的 Rule 物件。 使用樣板來生成原理圖所需的任何自定義檔案。

  1. 將以下程式碼新增到你的工廠函式中。

    projects/my-lib/schematics/my-service/index.ts（樣板轉換）

          
    import {  Rule,  Tree,  SchematicsException,  apply,  url,  applyTemplates,  move,  chain,  mergeWith,} from '@angular-devkit/schematics';import {strings, normalize, virtualFs, workspaces} from '@angular-devkit/core';import {Schema as MyServiceSchema} from './schema';function createHost(tree: Tree): workspaces.WorkspaceHost {  return {    async readFile(path: string): Promise<string> {      const data = tree.read(path);      if (!data) {        throw new SchematicsException('File not found.');      }      return virtualFs.fileBufferToString(data);    },    async writeFile(path: string, data: string): Promise<void> {      return tree.overwrite(path, data);    },    async isDirectory(path: string): Promise<boolean> {      return !tree.exists(path) && tree.getDir(path).subfiles.length > 0;    },    async isFile(path: string): Promise<boolean> {      return tree.exists(path);    },  };}export function myService(options: MyServiceSchema): Rule {  return async (tree: Tree) => {    const host = createHost(tree);    const {workspace} = await workspaces.readWorkspace('/', host);    const project = options.project != null ? workspace.projects.get(options.project) : null;    if (!project) {      throw new SchematicsException(`Invalid project name: ${options.project}`);    }    const projectType = project.extensions.projectType === 'application' ? 'app' : 'lib';    if (options.path === undefined) {      options.path = `${project.sourceRoot}/${projectType}`;    }    const templateSource = apply(url('./files'), [      applyTemplates({        classify: strings.classify,        dasherize: strings.dasherize,        name: options.name,      }),      move(normalize(options.path as string)),    ]);    return chain([mergeWith(templateSource)]);  };}
    方法 詳情
    apply() 將多個規則應用於源，並回傳轉換後的源。它接受 2 個引數：一個源和一個規則陣列。
    url() 從你的檔案系統中讀取源檔案，相對於原理圖。
    applyTemplates() 接收你希望提供給原理圖樣板和原理圖檔案名的方法和屬性的引數。它回傳一個 Rule。你可以在這裡定義 classify() 和 dasherize() 方法，以及 name 屬性。
    classify() 接受一個值並以首字母大寫形式回傳該值。例如，如果提供的名稱是 my service，則回傳為 MyService。
    dasherize() 接受一個值，並回傳以短劃線分隔的小寫形式的值。例如，如果提供的名稱是 MyService，則回傳 my-service。
    move() 當原理圖應用時，將提供的源檔案移動到其目標位置。

  2. 最後，規則工廠必須回傳一個規則。

執行你的庫原理圖

在建立庫和原理圖之後，你可以安裝原理圖集合以針對你的專案執行。 以下步驟向你展示如何使用你先前建立的原理圖生成一個服務。

建立你的函式庫和原理圖

從你的工作區根目錄，為你的函式庫執行 ng build 指令。

      
ng build my-lib

然後，你進入你的函式庫目錄以建立原理圖

      
cd projects/my-libnpm run build

連結該函式庫

你的函式庫和原理圖被打包並放置在你的工作區根目錄的 dist/my-lib 資料夾中。 為了執行原理圖，你需要將該函式庫連結到你的 node_modules 資料夾中。 從你的工作區根目錄，使用你的可分發函式庫的路徑執行 npm link 指令。

      
npm link dist/my-lib

執行原理圖

現在你的函式庫已安裝，使用 ng generate 指令執行原理圖。

      
ng generate my-lib:my-service --name my-data

在控制台中，你看到原理圖已執行，並且 my-data.service.ts 檔案已在你的應用文件夾中建立。

      
CREATE src/app/my-data.service.ts (208 bytes)
