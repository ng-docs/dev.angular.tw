DevTools 概览

edit

Angular DevTools是一个浏览器扩展，为Angular应用程序提供调试和性能分析功能。

从 Chrome网络应用商店或从 Firefox插件安装Angular DevTools。

你可以通过按下 F12或 Ctrl+Shift+I（Windows或Linux）和 Fn+F12或 Cmd+Option+I（Mac）在任何网页上打开Chrome或Firefox DevTools。 一旦浏览器DevTools打开并安装了Angular DevTools，你可以在“Angular”选项卡下找到它。

提示： Chrome的新标签页不运行已安装的扩展程序，因此Angular选项卡不会出现在DevTools中。请访问任何其他页面以查看它。

打开你的应用

打开扩展程序时，你还会看到另外两个选项卡：

标签页 详情
组件 允许你探索应用中的组件和指令，并预览或编辑它们的状态。
分析器 允许你分析应用并了解在变更检测执行过程中性能瓶颈是什么。

在 Angular DevTools 的右上角，你将找到页面上正在运行哪个版本的 Angular 以及该扩展的最后一次提交的哈希串。

未检测到 Angular 应用

如果在打开 Angular DevTools 时看到错误消息"未检测到 Angular 应用"，这意味着它无法与页面上的 Angular 应用进行通信。 这种情况最常见的原因是因为你检查的网页中不包含 Angular 应用。 请仔细检查你是否在检查正确的网页，以及 Angular 应用是否正在运行。

我们检测到使用生产配置构建的应用

如果看到错误消息"我们检测到使用生产配置构建的应用。Angular DevTools 仅支持开发构建。"，这意味着在页面上找到了一个 Angular 应用，但它是使用生产优化编译的。 在编译为生产环境时，Angular CLI 会删除各种调试功能，以减少页面上的 JavaScript 量，以提高性能。这包括与 DevTools 通信所需的功能。

要运行 DevTools，你需要禁用优化编译应用。默认情况下， ng serve 会执行此操作。 如果需要调试部署的应用，可以在构建中使用 optimization 配置选项 来禁用优化编译（ {"optimization": false}）。

调试你的应用

组件选项卡允许你探索应用的结构。 你可以在 DOM 中可视化组件和指令实例，并检查或修改它们的状态。

探索应用结构

组件树显示了应用中 组件和指令 的分层关系。

在组件资源管理器中单击各个组件或指令以选择它们并预览其属性。 Angular DevTools 会在组件树的右侧显示属性和元数据。

要通过名称查找组件或指令，请使用组件树上方的搜索框。

导航到宿主节点

要转到特定组件或指令的宿主元素，请在组件资源管理器中双击它。 Angular DevTools 将在 Chrome 中打开「元素」选项卡或在 Firefox 中打开「检查器」选项卡，并选择相关的 DOM 节点。

导航到源码

对于组件，Angular DevTools 允许你在「Sources」选项卡（Chrome）和「Debugger」选项卡（Firefox）中导航到组件定义。 选择特定组件后，点击属性视图右上角的图标：

修改属性值

类似于浏览器的 DevTools，属性视图允许你编辑输入、输出或其他属性的值。右键单击属性值，如果此值类型支持编辑功能，则会出现文本输入。输入新值并按 Enter 键将此值应用到属性。

在控制台中访问选定的组件或指令

在控制台中，Angular DevTools 提供了访问最近选定组件或指令实例的快捷方式。输入 $ng0 获取当前选定组件或指令实例的引用，输入 $ng1 获取之前选定的实例，输入 $ng2 获取再之前选定的实例，依此类推。

选择指令或组件

类似于浏览器的 DevTools，你可以检查页面以选择特定的组件或指令。在 Angular DevTools 的左上角点击 检查元素 图标，悬停在页面上的一个 DOM 元素上。扩展会识别相关的指令和/或组件，并允许你在组件树中选择相应的元素。

剖析你的应用程序

在 Profiler 选项卡中，你可以可视化 Angular 变更检测的执行情况。这对于确定变更检测如何影响应用程序性能很有帮助。

Profiler 选项卡允许你开始对当前应用程序进行性能分析，或导入前次运行的现有性能分析。要开始对应用程序进行性能分析，在 Profiler 选项卡的左上角悬停在圆圈上，然后点击 开始记录。

在进行性能分析时，Angular DevTools 捕获执行事件，比如变更检测和生命周期钩子执行。与应用程序交互以触发变更检测并生成 Angular DevTools 可用的数据。要结束记录，再次点击圆圈并选择 停止记录。

你也可以导入现有的记录。在 导入记录 部分了解更多关于此功能的信息。

了解应用程序的执行过程

在记录或导入性能分析后，Angular DevTools 将其变更检测周期可视化显示。

序列中的每个条形代表你的应用中的一次变更检测周期。条形越高，应用在此周期中运行变更检测的时间就越长。当你选择一个条形时，DevTools 会显示关于它的有用信息（包括以下内容）：

  • 一个条形图，显示此周期中捕获的所有组件和指令
  • Angular 在此周期中运行变更检测所花费的时间。
  • 用户体验到的估计帧率。
  • 触发变更检测的源码。

分层视图

你还可以以类似火焰图的方式可视化变更检测的执行情况。

图中每个瓷砖代表屏幕上特定位置的元素在渲染树中的位置。 例如，考虑一个变更检测周期，其中一个 LoggedOutUserComponent被移除，并且在其位置上 Angular 渲染了一个 LoggedInUserComponent。在这种情况下，这两个组件将显示在同一个瓷砖中。

横轴表示渲染该变更检测周期所需的全部时间。 纵轴表示元素层次结构。对一个元素运行变更检测需要渲染它的指令和子组件。 总之，这个图像显示了哪些组件渲染时间最长，以及这些时间花在了哪里。

每个瓷砖的颜色取决于 Angular 在该位置花费的时间。 Angular DevTools通过相对于渲染时间最长的瓷砖花费的时间来确定颜色的强度。

当你单击特定瓷砖时，你将在右侧面板中看到有关它的详细信息。 双击瓷砖会放大它，这样你可以更轻松地查看其嵌套子元素。

调试变更检测和 OnPush 组件

通常情况下，该图可视化了应用在任意给定变更检测帧中渲染所需的时间。然而，某些组件（如 OnPush 组件）只有在输入属性更改时才会重新渲染。在特定帧中不包含这些组件来可视化火焰图可能会很有用。

要仅可视化经历了变更检测过程的变更检测帧中的组件，请在火焰图顶部选择 变更检测复选框。

此视图突出显示了所有经历了变更检测的组件，并以灰色显示那些没有重新渲染的组件，例如未重新渲染的 OnPush组件。

导入和导出录制

单击记录的分析会话右上角的 保存配置文件按钮，将其导出为 JSON 文件并保存到磁盘上。 稍后，通过单击 选择文件输入框，在分析器的初始视图中导入文件。

检查你的注入器

注意：注入器树适用于使用版本 17 或更高版本构建的 Angular 应用程序。

查看你的应用程序的注入器层次结构

注入器树选项卡让你探索应用程序中配置的注入器的结构。在这里，你将看到代表应用程序的 注入器层次结构的两棵树。一棵树是你的环境层次结构，另一棵树是你的元素层次结构。

可视化解决路径

当选择特定的注入器时，会突出显示 Angular 依赖注入算法从该注入器到根的路径。对于元素注入器，这包括突出显示依赖注入算法在元素层次结构中无法解析依赖时跳转到的环境注入器。

有关 Angular 如何解析解决路径的更多详细信息，请参阅 解决规则。

查看注入器提供者

点击已配置提供者的注入器将在注入器树视图右侧显示这些提供者的列表。在这里你可以查看提供的令牌及其类型。
