DevTools 概覽

edit

Angular DevTools 是一款瀏覽器擴充套件程式，為 Angular 應用提供除錯和效能分析功能。

從 Chrome Web Store 或 Firefox Addons 安裝 Angular DevTools。

你可以透過按 F12 或 Ctrl+Shift+I (Windows 或 Linux) 以及 Fn+F12 或 Cmd+Option+I (Mac) 在任何網頁上開啟 Chrome 或 Firefox DevTools。 一旦瀏覽器 DevTools 開啟並且 Angular DevTools 安裝完畢，你就可以在 『Angular』 標籤頁下找到它。

提示： Chrome 的新標籤頁不會執行已安裝的擴充套件程式，因此 Angular 標籤頁不會出現在 DevTools 中。請訪問任何其他頁面以檢視它。

開啟你的應用

當你開啟擴充套件程式時，你將看到兩個額外的標籤頁：

標籤頁 詳情
元件 讓你探索應用中的元件和指令，並預覽或編輯它們的狀態。
效能分析器 讓你分析你的應用，並瞭解在變更檢測執行期間的效能瓶頸是什麼。

在 Angular DevTools 的右上角，你將找到頁面上正在執行的 Angular 版本以及擴充套件程式的最新送出雜湊值。

未檢測到 Angular 應用

如果在開啟 Angular DevTools 時看到錯誤訊息 『未檢測到 Angular 應用』，這意味著它無法與頁面上的 Angular 應用通訊。 最常見的原因是你正在檢查的網頁不包含 Angular 應用。 請仔細檢查你是否正在檢查正確的網頁以及 Angular 應用是否正在執行。

我們檢測到一個使用生產設定建立的應用

如果你看到錯誤訊息 『我們檢測到一個使用生產設定建立的應用。Angular DevTools 僅支援開發版本。』，這意味著在該頁面上找到了一個 Angular 應用，但它是使用生產最佳化編譯的。 當為生產環境編譯時，Angular CLI 會刪除各種除錯功能，以最大限度地減少頁面上的 JavaScript 數量，從而提高效能。這包括與 DevTools 通訊所需的功能。

要執行 DevTools，你需要編譯你的應用時停用最佳化。ng serve 預設會這樣做。 如果你需要除錯已部署的應用，請使用 optimization 設定選項 ({"optimization": false}) 在你的建立中停用最佳化。

除錯你的應用

元件 標籤頁讓你探索應用的結構。 你可以視覺化 DOM 中的元件和指令實例，並檢查或修改它們的狀態。

探索應用結構

元件樹顯示了你的應用中元件和指令的分層關係。

單擊元件瀏覽器中的各個元件或指令以選擇它們並預覽其屬性。 Angular DevTools 在元件樹的右側顯示屬性和元資料。

要按名稱查詢元件或指令，請使用元件樹上方的搜尋框。

導向到宿主節點

要轉到特定元件或指令的宿主元素，請在元件瀏覽器中雙擊它。 Angular DevTools 將在 Chrome 中開啟 Elements 標籤頁或在 Firefox 中開啟 Inspector 標籤頁，並選擇關聯的 DOM 節點。

導向到原始碼

對於元件，Angular DevTools 允許你導向到 Sources 標籤頁 (Chrome) 和 Debugger 標籤頁 (Firefox) 中的元件定義。 選擇特定元件後，單擊屬性檢視右上角的圖示：

更新屬性值

與瀏覽器的 DevTools 類似，屬性檢視允許你編輯輸入屬性、輸出屬性或其他屬性的值。 右鍵單擊屬性值，如果此值型別可以使用編輯功能，則會出現一個文字輸入框。 鍵入新值並按 Enter 以將此值應用於該屬性。

在控制台中訪問所選元件或指令

作為控制台中的快捷方式，Angular DevTools 提供了對最近選擇的元件或指令實例的訪問。 鍵入 $ng0 以獲取對當前所選元件或指令實例的引用，鍵入 $ng1 獲取對先前選擇的實例的引用，$ng2 獲取對之前選擇的實例的引用，依此類別推。

選擇一個指令或元件

與瀏覽器的 DevTools 類似，你可以檢查頁面以選擇特定的元件或指令。 單擊 Angular DevTools 左上角的檢查元素圖示，並將滑鼠懸停在頁面上的 DOM 元素上。 該擴充套件程式會識別關聯的指令和/或元件，並允許你在元件樹中選擇相應的元素。

分析你的應用效能

Profiler 標籤頁可以讓你視覺化 Angular 變更檢測的執行過程。 這對於確定變更檢測何時以及如何影響你的應用效能非常有用。

Profiler 標籤頁允許你開始分析當前應用，或從之前的執行中匯入現有的分析檔案。 要開始分析你的應用，請將滑鼠懸停在 Profiler 標籤頁左上角的圓圈上，然後點選 Start recording（開始錄製）。

在分析期間，Angular DevTools 會捕獲執行事件，例如變更檢測和生命週期鉤子的執行。 與你的應用互動以觸發變更檢測並生成 Angular DevTools 可以使用的資料。 要完成錄製，再次點選圓圈以 Stop recording（停止錄製）。

你也可以匯入現有的錄製檔案。 在 匯入錄製檔案 部分閱讀關於此屬性的更多訊息。

瞭解你的應用的執行過程

在錄製或匯入分析檔案後，Angular DevTools 會顯示變更檢測週期的視覺化檢視。

序列中的每個條形都代表你應用中的一個變更檢測週期。 條形越高，表示應用在此週期中執行變更檢測所花費的時間越長。 當你選擇一個條形時，DevTools 會顯示關於它的有用訊息，包括：

  • 一個條形圖，其中包含在此週期中捕獲的所有元件和指令
  • Angular 在此週期中執行變更檢測所花費的時間。
  • 使用者體驗到的估計幀率。
  • 觸發變更檢測的來源。

瞭解元件執行過程

點選變更檢測週期後顯示的條形圖，會顯示關於你的應用在該特定元件或指令中執行變更檢測所花費時間的詳細檢視。

此範例展示了 NgForOf 指令所花費的總時間以及在其上呼叫的方法。

分層檢視

你也可以在類似火焰圖的檢視中視覺化變更檢測的執行過程。

圖中的每個瓦片代表螢幕上渲染樹中特定位置的元素。 例如，考慮一個變更檢測週期，其中 LoggedOutUserComponent 被移除，並且 Angular 在其位置渲染了 LoggedInUserComponent。在這種情況下，兩個元件都將顯示在同一個瓦片中。

x 軸表示渲染此變更檢測週期所花費的總時間。 y 軸表示元素分層結構。為元素執行變更檢測需要渲染其指令和子元件。 總而言之，此圖視覺化了哪些元件的渲染時間最長以及時間都花在了哪裡。

每個瓦片的顏色都取決於 Angular 在那裡花費的時間。 Angular DevTools 根據相對於渲染時間最長的瓦片所花費的時間來確定顏色的強度。

當你點選某個瓦片時，你會在右側面板中看到關於它的詳細訊息。 雙擊瓦片會放大它，以便你可以更輕鬆地檢視其巢狀的子級。

除錯變更檢測和 OnPush 元件

通常，對於任何給定的變更檢測幀，該圖都會視覺化渲染應用所花費的時間。但是，某些元件（例如 OnPush 元件）僅在其輸入屬性更改時才會重新渲染。對於特定幀，在沒有這些元件的情況下視覺化火焰圖可能很有用。

要僅視覺化變更檢測幀中經歷了變更檢測過程的元件，請選中火焰圖上方頂部的 Change detection（變更檢測）複選框。

此檢視突出顯示了所有經歷了變更檢測的元件，並將未經歷變更檢測的元件顯示為灰色，例如未重新渲染的 OnPush 元件。

匯入和匯出錄製檔案

點選已錄製的分析會話右上角的 Save Profile（儲存分析檔案）按鈕，將其匯出為 JSON 檔案並儲存到磁碟。 稍後，透過點選 Choose file（選擇檔案）輸入框，在分析器的初始檢視中匯入該檔案。

檢查你的注入器

注意：Injector Tree 適用於使用 v17 或更高版本建立的 Angular 應用。

檢視你應用的注入器分層結構

Injector Tree（注入器樹）標籤頁允許你探索在你的應用中設定的注入器的結構。在這裡，你將看到兩棵樹，它們代表你應用的 注入器分層結構。一棵樹是你的環境分層結構，另一棵樹是你的元素分層結構。

視覺化求解路徑

當選擇特定的注入器時，從該注入器到根注入器 Angular 依賴注入演算法遍歷的路徑將被高亮顯示。對於元素注入器，這包括高亮顯示當依賴項無法在元素分層結構中被求解時，依賴注入演算法跳轉到的環境注入器。

要了解關於 Angular 如何求解解析路徑的更多詳情，請參閱解析規則。

檢視注入器提供者

單擊設定了提供者的注入器，將在注入器樹狀檢視的右側清單中顯示這些提供者。在這裡，你可以檢視所提供的令牌及其型別。
