遷移

edit

瞭解如何將現有的 Angular 專案逐步遷移到最新屬性。

獨立

獨立元件提供了一種建立 Angular 應用的簡化方式。獨立元件直接指定它們的依賴，而不是透過 NgModule 獲取。

立即遷移

控制流語法

內建的控制流語法允許你使用更符合工效學的語法，這種語法更接近 JavaScript，並且具有更好的型別檢查。它取代了匯入 CommonModule 以使用 *ngFor、*ngIf 和 *ngSwitch 等功能的必要性。

立即遷移

inject() 函式

與基於建構式函式的注入相比，Angular 的 inject 函式提供了更準確的型別以及與標準裝飾器更好的相容性。

立即遷移

延遲載入路由

將急性載入的元件路由轉換為延遲載入的路由。這允許建立過程將生產捆綁包拆分成更小的區塊，從而在初始頁面載入時載入更少的 JavaScript。

立即遷移

新的 `input()` API

將現有的 @Input 欄位轉換為現在已可用於生產環境的新的訊號輸入 API。

立即遷移

新的 `output()` 函式

將現有的 @Output 自定義事件轉換為現在已可用於生產環境的新的 output 函式。

立即遷移

作為訊號的查詢

將現有的裝飾器查詢欄位轉換為改進後的訊號查詢 API。該 API 現在已可用於生產環境。

立即遷移

清理未使用的匯入

清理專案中未使用的匯入。

立即嘗試

自閉合標籤

儘可能將元件樣板轉換為使用自閉合標籤。

立即遷移
