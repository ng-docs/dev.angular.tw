Npm 包

edit

Angular 框架、Angular CLI 和 Angular 应用使用的组件被打包为 npm 包 并通过 npm 注册表 分发。

你可以使用 npm CLI 客户端 下载并安装这些 npm 包。默认情况下，Angular CLI 使用 npm 客户端。

提示：请参阅 本地环境设置 了解 Node.js 和 npm 的必要版本和安装信息。

如果你已经有一些在你的机器上运行的项目使用了其他版本的 Node.js 和 npm，可以考虑使用 nvm 管理多个版本的 Node.js 和 npm。

package.json

npm 安装 package.json 文件中标识的包。

CLI 命令 ng new 在创建新工作区时创建一个 package.json 文件。这个 package.json 文件被工作区中的所有项目使用，包括 CLI 在创建工作区时创建的初始应用项目。使用 ng generate library 创建的库将包含它们自己的 package.json 文件。

最初，这个 package.json 包含 一个入门包集合，其中一些是 Angular 所必需的，另一些支持常见的应用场景。随着你的应用发展，你可以向 package.json 中添加包。

默认依赖项

以下 Angular 包被包含为新 Angular 工作区的默认 package.json 文件中的依赖项。有关 Angular 包的完整列表，请参阅 API 参考。

包名称 详情
@angular/animations Angular 的动画库使定义和应用动画效果（如页面和列表过渡）变得容易。有关更多信息，请参阅 动画指南。
@angular/common 由 Angular 团队提供的一些常用服务、管道和指令。
@angular/compiler Angular 的模板编译器。它理解 Angular 模板并可以将其转换为使应用运行的代码。
@angular/compiler-cli 由 Angular CLI 的 ng build 和 ng serve 命令调用的 Angular 编译器。它在标准 TypeScript 编译中使用 @angular/compiler 处理 Angular 模板。
@angular/core 每个应用都需要的框架关键运行时部分。包括所有元数据装饰器（如 @Component）、依赖注入和组件生命周期钩子。
@angular/forms 支持 模板驱动表单 和 响应式表单。参见 表单简介。
@angular/platform-browser 所有与 DOM 和浏览器相关的内容，特别是有助于渲染到 DOM 的部分。
@angular/platform-browser-dynamic 包含 提供者 和在客户端使用 JIT 编译器 编译并运行应用的方法。
@angular/router 当浏览器 URL 变化时，路由器模块在应用页面之间导航。更多信息，参见 路由和导航。
@angular/cli 包含用于运行 ng 命令的 Angular CLI 二进制文件。
@angular-devkit/build-angular 包含用于打捆、测试和启动 Angular 应用和库的默认 CLI 构建器。
rxjs 用于响应式编程的库，使用 Observables。
zone.js Angular 依赖于 `zone.js` 在原生 JavaScript 操作引发事件时运行 Angular 的变更检测过程。
typescript TypeScript 编译器、语言服务器和内置类型定义。
