工作區 npm 依賴

edit

Angular 框架、Angular CLI 以及 Angular 應用所使用的元件都打包為 npm 軟體套件，並使用 npm 儲存庫 進行分發。

你可以使用 npm CLI 用戶端 下載和安裝這些 npm 軟體套件。 預設情況下，Angular CLI 使用 npm 用戶端。

提示： 有關 Node.js 和 npm 的必要版本和安裝訊息，請參閱 本地環境建立。

如果你的機器上已經有正在執行的專案使用了其他版本的 Node.js 和 npm，請考慮使用 nvm 來管理多個版本的 Node.js 和 npm。

package.json

npm 安裝在 package.json 檔案中標識的套件。

當 CLI 指令 ng new 建立新的工作區時，會建立一個 package.json 檔案。 工作區中的所有專案（包括 CLI 在建立工作區時建立的初始應用專案）都使用此 package.json。 使用 ng generate library 建立的函式庫將包含它們自己的 package.json 檔案。

最初，此 package.json 包含一組入門軟體套件，其中一些是 Angular 所必需的，另一些則支援常見的應用場景。 隨著應用的演進，你可以向 package.json 中新增軟體套件。

預設依賴

以下 Angular 軟體套件作為依賴項包含在新的 Angular 工作區的預設 package.json 檔案中。 有關 Angular 軟體套件的完整清單，請參閱 API 參考。

軟體套件名稱 詳情
@angular/animations Angular 的動畫庫可以輕鬆定義和應用動畫效果，例如頁面和清單轉場。有關更多訊息，請參閱動畫指南。
@angular/common Angular 團隊提供的常用服務、通道和指令。
@angular/compiler Angular 的範本編譯器。它理解 Angular 範本，並可以將它們轉換為使應用執行的程式碼。
@angular/compiler-cli Angular 的編譯器，由 Angular CLI 的 ng build 和 ng serve 指令呼叫。它在標準的 TypeScript 編譯中，使用 @angular/compiler 處理 Angular 範本。
@angular/core 每個應用都需要的框架的關鍵執行時部分。包括所有元資料裝飾器，例如 @Component、依賴注入和元件生命週期鉤子。
@angular/forms 同時支援範本驅動表單和回應式表單。請參閱表單簡介。
@angular/platform-browser 所有與 DOM 和瀏覽器相關的內容，尤其是那些有助於渲染到 DOM 中的部分。
@angular/platform-browser-dynamic 包括 提供者 以及使用 即時編譯器 在用戶端編譯和執行應用的方法。
@angular/router 當瀏覽器 URL 更改時，路由器模組會在你的應用頁面之間導向。有關更多訊息，請參閱路由與導向。
@angular/cli 包含用於執行 ng 指令的 Angular CLI 二進位檔案。
@angular-devkit/build-angular 包含用於捆綁、測試和為 Angular 應用和函式庫啟動伺服器的預設 CLI 建構式。
rxjs 一個使用 Observables 進行回應式程式設計的函式庫。
zone.js 當原生 JavaScript 操作引發事件時，Angular 依賴 zone.js 執行 Angular 的變更檢測過程。
typescript TypeScript 編譯器、語言服務器和內建型別定義。
