Angular 工作區設定

edit

位於 Angular 工作區根目錄下的 angular.json 檔案提供了工作區範圍和專案特定的設定預設值。這些預設值用於 Angular CLI 提供的建立和開發工具。 設定中給定的路徑值是相對於根工作區目錄的。

通用 JSON 結構

在 angular.json 的頂層，一些屬性設定了工作區，而 projects 部分包含其餘的每個專案的設定選項。 你可以透過在專案級別設定的預設值來覆蓋在工作區級別設定的 Angular CLI 預設值。 你還可以使用指令行覆蓋在專案級別設定的預設值。

檔案頂層的以下屬性設定了工作區。

屬性 詳情
version 設定檔案版本。
newProjectRoot 透過諸如 ng generate application 或 ng generate library 之類的工具建立新專案的路徑。路徑可以是絕對路徑，也可以是相對於工作區目錄的相對路徑。預設為 projects
cli 一組用於自定義 Angular CLI 的選項。請參閱下面的Angular CLI 設定選項。
schematics 一組原理圖，用於自定義此工作區的 ng generate 子指令選項預設值。請參閱下面的原理圖。
projects 包含工作區中每個應用或函式庫的子部分，其中包含特定於專案的設定選項。

你使用 ng new app-name 建立的初始應用在 "projects" 下列出：

當你使用 ng generate library 建立一個庫專案時，該庫專案也會被新增到 projects 部分。

注意： 設定檔案中的 projects 部分與工作區檔案結構並不完全對應。

  • 由 ng new 建立的初始應用位於工作區檔案結構的頂層。
  • 預設情況下，其他應用和函式庫位於 projects 目錄下。

有關更多訊息，請參閱工作區和專案檔案結構。

專案設定選項

以下頂層設定屬性可用於每個專案，位於 projects['project-name'] 下。

屬性（Property） 詳情 值型別 預設值
root 此項目檔案的根目錄，相對於工作區目錄。對於位於工作區頂層的初始應用，此項為空。 string 無（必要）
projectType "application" 或 "library" 之一。應用可以在瀏覽器中獨立執行，而函式庫不能。 application | library 無（必要）
sourceRoot 此項目源檔案的根目錄。 string ''
prefix 當使用 ng generate 生成新的元件、指令和通道時，Angular 會預先新增到選擇器中的字串。可以自定義此字串以標識應用或屬性區域。 string 'app'
schematics 一組原理圖，用於自定義此項目的 ng generate 子指令選項預設值。請參閱生成原理圖部分。 參閱原理圖 {}
architect 此項目的 Architect 建構式目標的設定預設值。 參閱設定建構式目標 {}

原理圖

Angular 原理圖是透過新增新檔案或修改現有檔案來修改專案的指令。 可以透過將原理圖名稱對映到一組預設選項來設定它們。

原理圖的 "name" 格式為：<schematic-package>:<schematic-name>。 用於預設 Angular CLI ng generate 子指令的原理圖收集在 @schematics/angular 套件中。 例如，使用 ng generate component 生成元件的原理圖是 @schematics/angular:component。

原理圖的 schema 中給出的欄位對應於 Angular CLI 子指令選項的允許的指令行引數值和預設值。 你可以更新你的工作區 schema 檔案，為子指令選項設定不同的預設值。例如，預設停用 ng generate component 中的 standalone：

      
{  "projects": {    "my-app": {      "schematics": {        "@schematics/angular:component": {          "standalone": false        }      }    }  }}

設定 CLI 建構式

Architect 是 Angular CLI 用來執行複雜任務（如編譯和執行測試）的工具。 Architect 是一個外殼，它執行指定的建構式以根據目標設定執行給定的任務。 你可以定義和設定新的建構式和目標，以擴充套件 Angular CLI。 請參閱 Angular CLI 建構式。

預設 Architect 建構式和目標

Angular 為特定指令或通用的 ng run 指令定義了預設的建構式。 定義每個建構式的選項和預設值的 JSON 模式都收集在 @angular-devkit/build-angular 套件中。 這些模式為以下建構式設定選項。

設定建構式目標

angular.json 的 architect 部分包含一組 Architect 目標。 許多目標都對應於執行它們的 Angular CLI 指令。 其他目標可以使用 ng run 指令執行，你也可以定義自己的目標。

每個目標物件都為該目標指定了 builder，它是 Architect 執行的工具的 npm 套件。 每個目標還具有一個 options 部分，用於設定目標的預設選項；以及一個 configurations 部分，用於命名和指定目標的可選設定。 請參閱下面的建立目標中的範例。

屬性（Property） 詳情
build 為 ng build 指令的選項設定預設值。有關更多訊息，請參閱建立目標部分。
serve 為 ng serve 指令覆蓋建立預設值並提供額外的 serve 預設值。除了 ng build 指令可用的選項外，它還新增了與為應用啟動伺服器相關的選項。
e2e 使用 ng e2e 指令覆蓋用於建立端到端測試應用的建立預設值。
test 為測試建立覆蓋建立預設值，併為 ng test 指令提供額外的測試執行預設值。
lint 為 ng lint 指令的選項設定預設值，該指令對專案源檔案執行靜態程式碼分析。
extract-i18n 為 ng extract-i18n 指令的選項設定預設值，該指令從原始碼中提取本地化的訊息字串，併為國際化輸出翻譯檔案。

提示： 設定檔案中的所有選項都必須使用 camelCase，而不是像指令行中使用的 dash-case。

建立目標

architect 下的每個目標都具有以下屬性：

屬性（Property） 詳情
builder 用於建立此目標的 CLI 建構式，格式為 <package-name>:<builder-name>。
options 建立目標預設選項。
configurations 用於執行目標的可選設定。每個設定都為目標環境設定預設選項，從而覆蓋 options 下的關聯值。請參閱下面的可選建立設定。

例如，要設定停用最佳化的建立，請執行以下操作：

      
{  "projects": {    "my-app": {      "architect": {        "build": {          "builder": "@angular-devkit/build-angular:application",          "options": {            "optimization": false          }        }      }    }  }}

可選建立設定

Angular CLI 帶有兩個建立設定：production 和 development。 預設情況下，ng build 指令使用 production 設定，該設定應用了多項建立最佳化，包括：

  • 打捆檔案
  • 最小化多餘的空格
  • 刪除註解和死程式碼
  • 最小化程式碼以使用簡短、混亂的名稱

你可以根據你的開發過程定義和命名額外的可選設定（例如 staging）。 你可以透過將其名稱傳遞給 --configuration 指令行標誌來選擇可選設定。

例如，要設定僅為生產版本啟用最佳化的建立 (ng build --configuration production)：

      
{  "projects": {    "my-app": {      "architect": {        "build": {          "builder": "@angular-devkit/build-angular:application",          "options": {            "optimization": false          },          "configurations": {            "production": {              "optimization": true            }          }        }      }    }  }}

你還可以傳入多個設定名稱作為逗號分隔的清單。 例如，要同時應用 staging 和 french 建立設定，請使用指令 ng build --configuration staging,french。 在這種情況下，該指令從左到右解析命名的設定。 如果多個設定更改了同一設定，則最後設定的值是最終值。 在此範例中，如果 staging 和 french 設定都設定了輸出路徑，則將使用 french 中的值。

額外的建立和測試選項

預設或目標建立的可設定選項通常對應於 ng build、ng serve 和 ng test 指令可用的選項。 有關這些選項及其可能值的詳細訊息，請參閱Angular CLI 參考文獻。

選項屬性（Property） 詳情
assets 一個物件，其中包含要與應用一起提供的靜態資產的路徑。預設路徑指向專案的 public 目錄。有關更多訊息，請參閱資產設定部分。
styles 一個 CSS 檔案數組，用於新增到專案的全域上下文中。Angular CLI 支援 CSS 匯入和所有主要的 CSS 預處理器。有關更多訊息，請參閱樣式和指令碼設定部分。
stylePreprocessorOptions 一個物件，其中包含要傳遞給樣式預處理器的選項-值對。有關更多訊息，請參閱樣式和指令碼設定部分。
scripts 一個物件，其中包含要新增到應用中的 JavaScript 檔案。載入指令碼的方式與你在 index.html 內的 <script> 標籤中新增它們的方式完全相同。有關更多訊息，請參閱樣式和指令碼設定部分。
budgets 應用的所有部分或某些部分的預設大小預算型別和閾值。你可以設定建構式，以便在輸出達到或超過閾值大小時報告警告或錯誤。請參閱設定大小預算。
fileReplacements 一個物件，其中包含檔案及其編譯時替換項。有關更多訊息，請參閱設定目標特定的檔案替換。
index 一個載入應用的基礎 HTML 文件。有關更多訊息，請參閱索引設定。

複雜設定值

assets、index、outputPath、styles 和 scripts 選項可以具有簡單的路徑字串值，也可以是具有特定欄位的物件值。 sourceMap 和 optimization 選項可以設定為簡單的布林值。也可以使用設定檔案為它們賦予複雜的值。

以下各節提供了有關如何在每種情況下使用這些複雜值的更多詳細訊息。

資產設定

每個 build 目標設定都可以包含一個 assets 陣列，其中列出了在建立專案時要按原樣複製的檔案或資料夾。 預設情況下，public/ 目錄的內容會被複制過來。

要排除資產，你可以從資產設定中將其刪除。

你可以透過將資產指定為物件，而不是相對於工作區根目錄的簡單路徑，來進一步設定要複製的資產。 資產規範物件可以具有以下欄位。

欄位 詳情
glob 一個 node-glob，使用 input 作為基本目錄。
input 相對於工作區根目錄的路徑。
output 相對於 outDir 的路徑。由於安全隱患，Angular CLI 永遠不會將檔案寫入專案輸出路徑之外。
ignore 要排除的 glob 清單。
followSymlinks 允許 glob 模式跟蹤符號連結目錄。這允許搜尋符號連結的子目錄。預設為 false。

例如，可以使用以下物件更詳細地表示預設的資產路徑。

      
{  "projects": {    "my-app": {      "architect": {        "build": {          "builder": "@angular-devkit/build-angular:application",          "options": {            "assets": [              {                "glob": "**/*",                "input": "src/assets/",                "output": "/assets/"              },              {                "glob": "favicon.ico",                "input": "src/",                "output": "/"              }            ]          }        }      }    }  }}

以下範例使用 ignore 欄位來排除 assets 目錄中的某些檔案，使其不被複製到建立中：

      
{  "projects": {    "my-app": {      "architect": {        "build": {          "builder": "@angular-devkit/build-angular:application",          "options": {            "assets": [              {                "glob": "**/*",                "input": "src/assets/",                "ignore": ["**/*.svg"],                "output": "/assets/"              }            ]          }        }      }    }  }}

樣式和指令碼設定

styles 和 scripts 選項的陣列條目可以是一個簡單的路徑字串，也可以是一個指向額外入口點檔案的物件。 相關的建構式會在建立期間將該檔案及其依賴項載入為單獨的捆綁包。 使用設定物件，你可以選擇使用 bundleName 欄位為入口點捆綁包命名。

預設情況下會注入捆綁包，但你可以將 inject 設定為 false 以排除捆綁包的注入。 例如，以下物件值建立並命名一個包含樣式和指令碼的捆綁包，並將其排除在注入之外：

      
{  "projects": {    "my-app": {      "architect": {        "build": {          "builder": "@angular-devkit/build-angular:application",          "options": {            "styles": [              {                "input": "src/external-module/styles.scss",                "inject": false,                "bundleName": "external-module"              }            ],            "scripts": [              {                "input": "src/external-module/main.js",                "inject": false,                "bundleName": "external-module"              }            ]          }        }      }    }  }}

樣式預處理器選項

在 Sass 中，你可以為元件和全域樣式使用 includePaths 屬性。這允許你新增額外的基礎路徑，以便在匯入時進行檢查。

要新增路徑，請使用 stylePreprocessorOptions 選項：

      
{  "projects": {    "my-app": {      "architect": {        "build": {          "builder": "@angular-devkit/build-angular:application",          "options": {            "stylePreprocessorOptions": {              "includePaths": [                "src/style-paths"              ]            }          }        }      }    }  }}

該目錄中的檔案（例如 src/style-paths/_variables.scss）可以從專案中的任何位置匯入，而無需相對路徑：

      
// src/app/app.component.scss// A relative path works@import '../style-paths/variables';// But now this works as well@import 'variables';

提示： 如果單元測試需要任何樣式或指令碼，你還需要將它們新增到 test 建構式中。 另請參閱 在你的應用中使用執行時全域函式庫。

最佳化設定

optimization 選項可以是布林值或物件，以進行更精細的設定。 此選項啟用建立輸出的各種最佳化，包括：

  • 指令碼和樣式的最小化
  • 搖樹最佳化
  • 死程式碼消除
  • 關鍵 CSS 的行內
  • 字型行內

可以使用多個選項來微調應用的最佳化。

選項 詳情 值型別 預設值
scripts 啟用指令碼輸出的最佳化。 boolean true
styles 啟用樣式輸出的最佳化。 boolean | 樣式最佳化選項 true
fonts 啟用字型最佳化。這需要網際網路連線。 boolean | 字型最佳化選項 true

樣式最佳化選項

選項 詳情 值型別 預設值
minify 透過刪除多餘的空格和註解、合併識別符號和最小化值來最小化 CSS 定義。 boolean true
inlineCritical 提取並行內關鍵 CSS 定義，以改進 首次內容繪製。 boolean true
removeSpecialComments 刪除全域 CSS 中包含 @license 或 @preserve 或以 //! 或 /*! 開頭的註解。 boolean true

字型最佳化選項

選項 詳情 值型別 預設值
inline 透過在應用的 HTML 索引檔案中行內外部 Google Fonts 和 Adobe Fonts CSS 定義，減少渲染阻塞請求。這需要網際網路連線。 boolean true

你可以提供如下值，以便將最佳化應用於其中一項：

      
{  "projects": {    "my-app": {      "architect": {        "build": {          "builder": "@angular-devkit/build-angular:application",          "options": {            "optimization": {              "scripts": true,              "styles": {                "minify": true,                "inlineCritical": true              },              "fonts": true            }          }        }      }    }  }}

原始碼對映設定

sourceMap 建構式選項可以是布林值或物件，用於更精細的設定，以控制應用的原始碼對映。

選項 詳情 值型別 預設值
scripts 為所有指令碼輸出原始碼對映。 boolean true
styles 為所有樣式輸出原始碼對映。 boolean true
vendor 求解供應商套件原始碼對映。 boolean false
hidden 從輸出 JavaScript 中省略指向原始碼對映的連結。 boolean false

以下範例展示了如何切換一個或多個值來設定原始碼對映輸出：

      
{  "projects": {    "my-app": {      "architect": {        "build": {          "builder": "@angular-devkit/build-angular:application",          "options": {            "sourceMap": {              "scripts": true,              "styles": false,              "hidden": true,              "vendor": true            }          }        }      }    }  }}

提示： 當使用隱藏原始碼對映時，原始碼對映不會在捆綁包中被引用。 如果你只想讓原始碼對映在錯誤報告工具中對映堆疊跟蹤，而不希望在瀏覽器開發者工具中顯示，這將非常有用。 請注意，即使 hidden 阻止了原始碼對映在輸出捆綁包中被連結，你的部署過程也必須注意不要在生產環境中提供生成的原始碼對映，否則訊息仍然會洩露。

索引設定

設定應用的 HTML 索引的生成。

index 選項可以是字串或物件，用於更精細的設定。

當以字串形式提供值時，指定路徑的檔案名將用於生成的檔案，並將在應用的設定輸出路徑的根目錄中建立。

索引選項

選項 詳情 值型別 預設值
input 用於應用的生成的 HTML 索引的檔案的路徑。 string 無（必要）
output 應用的生成的 HTML 索引檔案的輸出路徑。將使用提供的完整路徑，並將其視為相對於應用的設定輸出路徑。 string index.html

輸出路徑設定

outputPath 選項可以是 String（將用作 base 值）或 Object，用於更精細的設定。

可以使用多個選項來微調應用的輸出結構。

選項 詳情 值型別 預設值
base 指定相對於工作區根目錄的輸出路徑。 string
browser 你的瀏覽器建立的輸出目錄名位於基本輸出路徑內。可以安全地為使用者啟動伺服器。 string browser
server 你的伺服器建立的輸出目錄名位於輸出路徑基本路徑內。 string server
media 你的媒體檔案輸出目錄名，位於瀏覽器輸出目錄內。這些媒體檔案通常在 CSS 檔案中被稱為資源。 string media
