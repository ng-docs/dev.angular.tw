Angular 工作区配置

edit

位于 Angular 工作区根级别的 angular.json 文件提供了工作区范围和项目特定的配置默认值。这些默认值用于由 Angular CLI 提供的构建和开发工具。配置中给出的路径值是相对于根工作区目录的。

JSON 的总体结构

在 angular.json 的顶层，一些属性用于配置工作区，其中的 projects 区则包含其余的针对每个项目的配置项。CLI 在工作区级的默认设置可以被项目级的设置所覆盖，而项目级的设置可以被命令行中的设置所覆盖。

下列属性位于文件的顶层，用于配置工作区。

属性 详情
version 配置文件版本。
newProjectRoot 通过 ng generate application 或 ng generate library 等工具创建新项目的路径。路径可以是绝对路径或相对于工作区目录的相对路径。默认值为 projects
cli 一组用于自定义 Angular CLI的选项。请参阅下文的 Angular CLI 配置选项。
schematics 一组 原理图，用于自定义此工作区中 ng generate子命令选项的默认值。请参阅下文的 原理图。
projects 包含此工作区中每个应用或库的子部分，具有项目特定的配置选项。

你使用 ng new app-name创建的初始应用列在 "projects" 下：

当你使用 ng generate library 创建库项目时，库项目也会添加到 projects 节。

有用提示：配置文件的 projects部分并不完全对应工作区的文件结构。

  • 由 ng new创建的初始应用位于工作区文件结构的顶层。
  • 其他应用和库默认位于 projects目录下。

更多信息，请参阅 工作区和项目文件结构。

以下属性是一组用于自定义 Angular CLI 的选项。

属性 详情 值类型 默认值
analytics 与 Angular 团队共享匿名使用数据。布尔值表示是否共享数据，而 UUID 字符串使用伪匿名标识符共享数据。 boolean string

项目配置选项

以下顶级配置属性可用于每个项目，在 projects['project-name'] 下。

属性 详情 值类型 默认值
root 此项目文件的根目录，相对于工作区目录。对于初始应用，该目录为空，位于工作区的顶级目录。 string 无（必要）
projectType "application" 或 "library" 之一。应用可以在浏览器中独立运行，而库不能。 application library
sourceRoot 此项目源文件的根目录。 string ''
prefix Angular 在使用 ng generate 生成新组件、指令和管道时，预先添加到选择器的字符串。可以自定义以识别某个应用或特性区域。 string 'app'
schematics 自定义此项目的 ng generate 子命令选项默认值的一组原理图。参见 生成原理图部分。 参见 原理图 {}
architect 此项目的 Architect 构建目标的配置默认值。 参见 配置构建器目标 {}

原理图

Angular 原理图是通过添加新文件或修改现有文件来修改项目的指令。这些可以通过将原理图名称映射到一组默认选项进行配置。

原理图的 "name" 格式为： <schematic-package>:<schematic-name>。默认 Angular CLI ng generate 子命令的原理图收集在包 @schematics/angular中。例如，使用 ng generate component 生成组件的原理图是 @schematics/angular:component。

原理图模式中给出的字段对应于 Angular CLI 子命令选项允许的命令行参数值和默认值。你可以更新你的工作区模式文件来为子命令选项设置不同的默认值。例如，要默认禁用 ng generate component 中的 standalone：

      
{
  "projects": {
    "my-app": {
      "schematics": {
        "@schematics/angular:component": {
          "standalone": false
        }
      }
    }
  }
}

配置 CLI 构建器

Architect 是 Angular CLI 用来执行复杂任务（如编译和测试运行）的工具。Architect 是一个外壳，根据目标配置运行指定的构建器来执行特定任务。你可以定义和配置新的构建器和目标来扩展 Angular CLI。参见 Angular CLI 构建器。

默认的建筑师构建器和目标

Angular 定义了默认的构建器以用于特定命令，或用于通用的 ng run 命令。定义每个构建器的选项和默认值的 JSON 模式收集在 @angular-devkit/build-angular 包中。这些模式配置了以下构建器的选项。

配置构建器目标

angular.json 文件中的 architect 部分包含一组 Architect 目标。 许多目标对应运行它们的 Angular CLI 命令。 其他目标可以使用 ng run 命令执行，并且你可以定义自己的目标。

每个目标对象指定该目标的 builder，即 Architect 运行的工具的 npm 包。 每个目标还具有一个 options 部分，该部分配置目标的默认选项，还有一个 configurations 部分，用于命名和指定目标的替代配置。 请参见下面的 构建目标 示例。

属性 详情
build 配置 ng build 命令选项的默认值。有关更多信息，请参见 构建目标 部分。
serve 覆盖构建默认值并为 ng serve 命令提供额外的启动服务器默认值。除了 ng build 命令可用的选项外，还添加了与启动应用相关的选项。
e2e 覆盖使用 ng e2e 命令构建端到端测试应用的构建默认值。
test 覆盖测试构建的默认值，并为 ng test 命令提供额外的测试运行默认值。
lint 配置 ng lint 命令的选项默认值，该命令对项目源文件执行静态代码分析。
extract-i18n 配置 ng extract-i18n 命令的选项默认值，该命令从源代码中提取本地化消息字符串并输出国际化翻译文件。

有用提示： 配置文件中的所有选项必须使用 camelCase，而不是命令行中使用的 dash-case。

构建目标

architect 下的每个目标具有以下属性：

属性 详情
builder 用于创建此目标的 CLI 构建器，形式为 <package-name>:<builder-name>。
options 构建目标的默认选项。
configurations 执行目标的替代配置。每个配置设置预期环境的默认选项，覆盖 options 下的相关值。请参见下面的 替代构建配置。

例如，要配置禁用优化的构建：

      
{
  "projects": {
    "my-app": {
      "architect": {
        "build": {
          "builder": "@angular-devkit/build-angular:application",
          "options": {
            "optimization": false
          }
        }
      }
    }
  }
}

备用的构建配置

Angular CLI 具有两种构建配置：production 和 development。默认情况下，ng build 命令使用 production 配置，该配置将应用许多构建优化，包括：

  • 打包文件
  • 最小化多余的空白
  • 删除注释和无效代码
  • 最小化代码以使用简短、混淆的名称

你可以根据开发过程定义和命名额外的替代配置（例如 staging）。 你可以通过将其名称传递给 --configuration 命令行标志来选择替代配置。

例如，要配置仅为生产构建启用优化的构建（ ng build --configuration production）：

      
{
  "projects": {
    "my-app": {
      "architect": {
        "build": {
          "builder": "@angular-devkit/build-angular:application",
          "options": {
            "optimization": false
          },
          "configurations": {
            "production": {
              "optimization": true
            }
          }
        }
      }
    }
  }
}

你还可以传入多个配置名称，作为逗号分隔的列表。 例如，要应用 staging 和 french 构建配置，使用命令 ng build --configuration staging,french。 在这种情况下，命令从左到右解析命名配置。 如果多个配置更改了相同的设置，则最后设置的值为最终值。 在此示例中，如果 staging 和 french 配置都设置了输出路径，则使用 french 中的值。

额外的构建和测试选项

默认或目标构建的可配置选项通常对应于 ng build、 ng serve 和 ng test 命令可用的选项。有关这些选项及其可能值的详细信息，请参见 Angular CLI 参考。

选项属性 详情
assets 一个包含静态资产路径的对象，这些资产将与应用一起提供。默认路径指向项目的图标文件及其 assets 目录。更多信息请参见 资产配置 部分。
styles 一个要添加到项目全局上下文的 CSS 文件数组。Angular CLI 支持 CSS 导入和所有主要的 CSS 预处理器。更多信息请参见 样式和脚本配置 部分。
stylePreprocessorOptions 一个包含传递给样式预处理器的选项值对的对象。更多信息请参见 样式和脚本配置 部分。
scripts 一个包含要添加到应用的 JavaScript 文件的对象。这些脚本的加载方式与将它们添加到 index.html 中的 <script> 标签内完全一样。更多信息请参见 样式和脚本配置 部分。
budgets 应用所有或部分内容的默认大小预算类型和阈值。你可以配置构建器在输出达到或超过阈值时报告警告或错误。请参见 配置大小预算。
fileReplacements 一个包含文件及其编译时替代品的对象。更多信息请参见 配置目标特定的文件替代品。
index 加载应用的基础 HTML 文档。更多信息请参见 索引配置。

复杂配置的值

assets、 index、 outputPath、 styles 和 scripts 选项可以有简单的路径字符串值或具有特定字段的对象值。 sourceMap 和 optimization 选项可以设置为简单的布尔值。它们也可以使用配置文件赋予复杂值。

以下部分提供了这些复杂值在每种情况下如何使用的更多详细信息。

项目资产（asset）配置

每个 build 目标配置都可以包含一个 assets 数组，它列出了当你构建项目时要复制的文件或文件夹。默认情况下，会复制 src/assets/ 文件夹和 src/favicon.ico。

要排除某个资产，可以从这份资产配置中删除它。

你可以通过把资产指定为对象的形式来进一步配置要复制的资产，而不仅是相对于工作区根目录的路径。一个资产对象可以包含如下字段。

字段 详情
glob 使用 input 作为基目录的 node-glob。
input 相对于工作区根目录的路径。
output 相对于 outDir 的路径。由于安全原因，Angular CLI 从不在项目输出路径之外写入文件。
ignore 要排除的一些 glob 模式。
followSymlinks 允许 glob 模式跟随符号链接目录。这允许搜索符号链接的子目录。默认为 false。

比如，可以使用如下对象来更详细地表达默认的资产路径。

      
{
  "projects": {
    "my-app": {
      "architect": {
        "build": {
          "builder": "@angular-devkit/build-angular:application",
          "options": {
            "assets": [
              {
                "glob": "**/*",
                "input": "src/assets/",
                "output": "/assets/"
              },
              {
                "glob": "favicon.ico",
                "input": "src/",
                "output": "/"
              }
            ]
          }
        }
      }
    }
  }
}

下面的例子使用 ignore 字段排除了 assets 文件夹中的某些特定文件，防止它们被复制到 build 中：

      
{
  "projects": {
    "my-app": {
      "architect": {
        "build": {
          "builder": "@angular-devkit/build-angular:application",
          "options": {
            "assets": [
              {
                "glob": "**/*",
                "input": "src/assets/",
                "ignore": ["**/*.svg"],
                "output": "/assets/"
              }
            ]
          }
        }
      }
    }
  }
}

样式和脚本配置

styles 和 scripts 选项的数组型条目可以是简单的路径字符串，也可以是指向额外入口点文件的对象。其关联的构建器将在构建过程中将该文件及其依赖项作为单独的捆绑包进行加载。对于配置对象，你可以选择使用 bundleName 字段为该入口点命名捆绑包。

此捆绑包默认被注入，但你可以将 inject 设置为 false 来排除该捆绑包的注入。例如，以下对象值创建并命名了一个包含样式和脚本的捆绑包，并将其排除在注入之外：

      
{
  "projects": {
    "my-app": {
      "architect": {
        "build": {
          "builder": "@angular-devkit/build-angular:application",
          "options": {
            "styles": [
              {
                "input": "src/external-module/styles.scss",
                "inject": false,
                "bundleName": "external-module"
              }
            ],
            "scripts": [
              {
                "input": "src/external-module/main.js",
                "inject": false,
                "bundleName": "external-module"
              }
            ]
          }
        }
      }
    }
  }
}

样式预处理器选项

在 Sass 中，你可以在组件和全局样式中使用 includePaths 特性。这允许你添加一些额外的基路径来检查导入。

要添加路径，请使用 stylePreprocessorOptions 选项：

      
{
  "projects": {
    "my-app": {
      "architect": {
        "build": {
          "builder": "@angular-devkit/build-angular:application",
          "options": {
            "stylePreprocessorOptions": {
              "includePaths": [
                "src/style-paths"
              ]
            }
          }
        }
      }
    }
  }
}

该文件夹中的文件，比如 src/style-paths/_variables.scss，可以从项目中的任何位置导入，而无需相对路径：

      
// src/app/app.component.scss
// A relative path works
@import '../style-paths/variables';
// But now this works as well
@import 'variables';

提示：如果需要在单元测试中使用某些样式或脚本，你还需要将它们添加到 test 构建器中。 另见 在应用中使用运行时全局库。

优化配置

optimization 选项可以是布尔值或对象，用于更精细的配置。 此选项启用对构建输出的各种优化，包括：

  • 脚本和样式的最小化
  • 摇树优化
  • 消除死代码
  • 内联关键 CSS
  • 字体内联

有几个选项可用于微调应用程序的优化方式。

选项 详情 值类型 默认值
scripts 启用脚本输出的优化。 boolean true
styles 启用样式输出的优化。 boolean 样式优化选项
fonts 启用字体的优化。这需要互联网访问。 boolean 字体优化选项

样式优化选项

选项 详情 值类型 默认值
minify 通过删除多余的空白和注释，合并标识符和最小化值来最小化 CSS 定义。 boolean true
inlineCritical 提取并内联关键 CSS 定义，以改进 首次内容绘制。 boolean true
removeSpecialComments 删除全局 CSS 中包含 @license 或 @preserve 的注释，或以 //! 或 /*! 开头的注释。 boolean true

字体优化选项

选项 详情 值类型 默认值
inline 通过在应用的 HTML 索引文件中内联外部 Google Fonts 和 Adobe Fonts 的 CSS 定义，减少 渲染阻塞请求。这需要互联网访问。 boolean true

你可以提供如下值，以便应用其中的一项或几项优化：

      
{
  "projects": {
    "my-app": {
      "architect": {
        "build": {
          "builder": "@angular-devkit/build-angular:application",
          "options": {
            "optimization": {
              "scripts": true,
              "styles": {
                "minify": true,
                "inlineCritical": true
              },
              "fonts": true
            }
          }
        }
      }
    }
  }
}

Source map（源码映射）配置

sourceMap 构建器选项可以是布尔值或对象，以便对应用的源码映射进行更细致的配置。

选项 详情 值类型 默认值
scripts 为所有脚本输出源码映射。 boolean true
styles 为所有样式输出源码映射。 boolean true
vendor 求解供应商包的源码映射。 boolean false
hidden 从输出的 JavaScript 中省略到源码映射的链接。 boolean false

以下示例展示了如何切换一个或多个值以配置源码映射输出：

      
{
  "projects": {
    "my-app": {
      "architect": {
        "build": {
          "builder": "@angular-devkit/build-angular:application",
          "options": {
            "sourceMap": {
              "scripts": true,
              "styles": false,
              "hidden": true,
              "vendor": true
            }
          }
        }
      }
    }
  }
}

提示： 使用隐藏源码映射时，源码映射不会在捆绑包中引用。如果你只想在错误报告工具中将源码映射到堆栈跟踪中而不在浏览器开发工具中显示，这些是很有用的。注意，尽管 hidden 阻止了在输出捆绑包中链接源码映射，但你的部署过程必须确保不在生产环境中提供生成的源码映射，否则信息仍会泄露。

索引配置

配置生成应用的 HTML 索引。

index 选项可以是字符串或对象，以便进行更细致的配置。

当提供值为字符串时，将使用指定路径的文件名作为生成文件，并在应用配置的输出路径的根目录中创建。

索引选项

选项 详情 值类型 默认值
input 用于应用生成的 HTML 索引的文件路径。 string 无（必要）
output 应用生成的 HTML 索引文件的输出路径。将使用提供的完整路径，并将相对于应用配置的输出路径进行考虑。 string index.html

输出路径配置

outputPath 选项可以是作为 base 值的字符串或用于更细致配置的对象。

可以使用多个选项来细化应用的输出结构。

选项 详情 值类型 默认值
base 指定相对于工作区根目录的输出路径。 string
browser 你的浏览器构建的输出目录名称位于基础输出路径中。这可以安全地提供给用户。 string browser
server 你的服务器构建的输出目录名称位于输出路径基础中。 string server
media 输出浏览器目录中媒体文件的输出目录名。这些媒体文件在 CSS 文件中通常称为资源。 string media
