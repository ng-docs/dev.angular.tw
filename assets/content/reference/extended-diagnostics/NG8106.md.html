字尾不支援

edit

此診斷會在屬性（Attribute）繫結中使用了 .px、.% 和 .em 字尾時進行檢測。

      
<img [attr.width.px]="5">

那有什麼問題？

這些字尾僅在樣式繫結中可用。當繫結到屬性（Attribute）時，它們沒有任何意義。

我應該怎麼做？

與其使用僅在樣式繫結中受支援的 .px、.% 或 .em 字尾，不如將其移至繫結的值賦值中。

      
<img [attr.width]="'5px'">

設定要求

strictTemplates 必須啟用才能發出任何擴充套件診斷。suffixNotSupported 除了 strictTemplates 之外沒有其他要求。

如果我無法避免這種情況怎麼辦？

可以透過編輯專案的 tsconfig.json 檔案來停用此診斷：

      
{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "suffixNotSupported": "suppress"      }    }  }}

有關更多訊息，請參閱擴充套件診斷設定。
