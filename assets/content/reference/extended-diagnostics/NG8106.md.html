后缀不支持

edit

此诊断检测在属性绑定中使用 .px、 .% 和 .em 后缀的情况。

      
<img [attr.width.px]="5">

那有什么问题？

这些后缀仅适用于样式绑定。在绑定到属性时，它们没有任何意义。

我该怎么办？

与其使用只能在样式绑定中受支持的 .px 、 .% 或 .em 后缀，不如将其移动到对绑定进行赋值的表达式中。

      
<img [attr.width]="'5px'">

配置要求

strictTemplates 必须启用，才能发出任何扩展诊断。 invalidBananaInBox 除了 strictTemplates 之外没有其他要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "suffixNotSupported": "suppress"
      }
    }
  }
}

有关更多信息，请参阅扩展诊断配置。
