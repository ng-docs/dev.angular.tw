*ngFor 表達式中缺少 let 關鍵字

edit

當在 *ngFor 中使用的表達式遺漏 let 關鍵字時，會發出此診斷。

      
import {Component} from '@angular/core';
@Component({
  // The `let` keyword is missing in the `*ngFor` expression.
  template: `<div *ngFor="item of items">{{ item }}</div>`,
})
class MyComponent {
  items = [1, 2, 3];
}

有什麼問題？

遺失的 let 表示 *ngFor 字串中的語法錯誤。這也表示 item 無法正確地拉入範圍，而 {{ item }} 將無法正確解析。

我該怎麼做？

增加遺漏的 let 關鍵字。

      
import {Component} from '@angular/core';
@Component({
  // The `let` keyword is now present in the `*ngFor` expression,
  // no diagnostic messages are emitted in this case.
  template: `<div *ngFor="let item of items">{{ item }}</div>`,
})
class MyComponent {
  items = [1, 2, 3];
}

組態需求

必須啟用 [strictTemplates](tools/cli/template-typecheck#strict-mode) 才能發出任何延伸診斷。 missingNgForOfLet 除了 strictTemplates 外沒有其他額外需求。

如果我無法避免這件事怎麼辦？

這個診斷可以透過編輯專案的 tsconfig.json 檔案停用：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "missingNgForOfLet": "suppress"
      }
    }
  }
}

參閱 進階診斷設定 以取得更多資訊。
