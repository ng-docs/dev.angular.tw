*ngFor 表达式中缺少 let 关键字

edit

当 *ngFor 中使用的表达式缺少 let 关键字时，会发出此诊断。

      
import {Component} from '@angular/core';
@Component({
  // The `let` keyword is missing in the `*ngFor` expression.
  template: `<div *ngFor="item of items">{{ item }}</div>`,
})
class MyComponent {
  items = [1, 2, 3];
}

那有什么问题？

缺少 let 表示 *ngFor 字符串中存在语法错误。这也意味着 item 不会被正确地引入作用域， {{ item }} 也不会正确求解。

我该怎么办？

添加缺失的 let 关键字。

      
import {Component} from '@angular/core';
@Component({
  // The `let` keyword is now present in the `*ngFor` expression,
  // no diagnostic messages are emitted in this case.
  template: `<div *ngFor="let item of items">{{ item }}</div>`,
})
class MyComponent {
  items = [1, 2, 3];
}

配置要求

strictTemplates 必须启用，任何扩展诊断才会发出。 missingNgForOfLet 除了 strictTemplates 之外没有其他要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "missingNgForOfLet": "suppress"
      }
    }
  }
}

有关更多信息，请参阅扩展诊断配置。
