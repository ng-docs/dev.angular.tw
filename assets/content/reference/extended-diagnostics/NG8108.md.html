ngSkipHydration 应该是一个静态属性

edit

ngSkipHydration 是一个特殊属性，指示 Angular 某个组件应选择退出 水合。 此诊断确保此属性 ngSkipHydration 被静态设置，且值设为 "true" 或为空值。

      
import {Component} from '@angular/core';
@Component({
  template: `<user-viewer ngSkipHydration="hasUser" />`,
})
class MyComponent {
  hasUser = true;
}

那有什么问题？

作为 Angular 实现的一个特殊属性， ngSkipHydration 需要是可静态分析的，以便 Angular 在编译时知道某个组件是否需要水合。

我该怎么办？

使用 ngSkipHydration 时，请确保将其设置为静态属性（即不要使用 Angular 模板绑定语法）。

      
import {Component} from '@angular/core';
@Component({
  template: `
    <user-viewer ngSkipHydration />
    <user-viewer ngSkipHydration="true" />
  `,
})
class MyComponent {}

如果需要条件判断，可以将组件包裹在 *ngIf 中。

      
import {Component} from '@angular/core';
@Component({
  template: `
    <div *ngIf="hasUser; else noUser">
      <user-viewer ngSkipHydration />
    </div>
    <ng-template #noUser>
      <user-viewer />
    </ng-template>
  `,
})
class MyComponent {}

配置要求

strictTemplates 必须启用，才能发出任何扩展诊断。 skipHydrationNotStatic 除了 strictTemplates 之外没有其他附加要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "skipHydrationNotStatic": "suppress"
      }
    }
  }
}

有关更多信息，请参阅扩展诊断配置。
