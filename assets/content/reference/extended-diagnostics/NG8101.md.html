無效的香蕉盒

edit

此診斷檢測 雙向繫結 的「banana-in-box」語法反向。

      
<user-viewer ([user])="loggedInUser" />

有什麼問題？

就其本身而言，([var]) 實際上是一個名稱為 [var] 的 事件繫結。 作者可能打算將這作為一個雙向繫結到一個名為 var 的變數，但依據所寫的內容，此繫結沒有任何效果。

我該怎麼做？

修正錯別字。 顧名思義，香蕉 ( 應放在盒子 [] 裡面。 在此情況下：

      
<user-viewer [(user)]="loggedInUser" />

組態需求

必須啟用 strictTemplates 才能發出任何擴充診斷。 invalidBananaInBox 除了 strictTemplates 之外沒有其他額外需求。

如果我無法避免這件事怎麼辦？

這個診斷可以透過編輯專案的 tsconfig.json 檔案停用：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "invalidBananaInBox": "suppress"
      }
    }
  }
}
