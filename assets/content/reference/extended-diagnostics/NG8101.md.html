無效的 Banana-in-Box

edit

此診斷檢測到用於雙向繫結的反向 『banana-in-box』 語法。

      
<user-viewer ([user])="loggedInUser" />

那有什麼問題？

就目前而言，([var]) 實際上是一個名為 [var] 的事件繫結。作者可能打算將其作為名為 var 的變數的雙向繫結，但就目前而言，此繫結不起作用。

我應該怎麼做？

修復筆誤。 顧名思義，香蕉 ( 應該放在方框 [] 裡面。 在這種情況下：

      
<user-viewer [(user)]="loggedInUser" />

設定要求

必須啟用strictTemplates才能發出任何擴充套件診斷。 invalidBananaInBox 除了 strictTemplates 之外沒有其他要求。

如果我無法避免這種情況怎麼辦？

可以透過編輯專案的 tsconfig.json 檔案來停用此診斷：

      
{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "invalidBananaInBox": "suppress"      }    }  }}

有關更多訊息，請參閱擴充套件診斷設定。
