无效的 Banana-in-Box 语法

edit

此诊断检测到向后的 "banana-in-box" 语法用于 双向绑定。

      
<user-viewer ([user])="loggedInUser" />

那有什么问题？

就目前而言， ([var]) 实际上是一个具有名称 [var] 的 事件绑定。 作者可能本意是将其作为一个名为 var 的变量的双向绑定，但按现有写法，此绑定没有任何效果。

我该怎么办？

修复键入失误。顾名思义，香蕉 ( 应该放在盒子 [] 中。在这种情况下：

      
<user-viewer [(user)]="loggedInUser" />

配置要求

strictTemplates 必须启用，才能发出任何扩展诊断。 invalidBananaInBox 除了 strictTemplates 之外没有其他要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "invalidBananaInBox": "suppress"
      }
    }
  }
}

有关更多信息，请参阅扩展诊断配置。
