文字屬性未繫結

edit

此診斷確保帶有 『特殊』 Angular 繫結字首（attr.、style. 和 class.）的 Attribute 被解釋為繫結。

      
<div attr.id="my-id"></div>

那樣有什麼問題？

在此範例中，attr.id 被解釋為常規 Attribute，並將按原樣出現在最終 HTML 中 (<div attr.id="my-id"></div>)。這可能不是開發者的本意。 相反，其本意可能是要設定 id Attribute (<div id="my-id"></div>)。

我應該怎麼做？

當繫結到 attr.、class. 或 style. 時，請確保你使用了 Angular 範本繫結語法 ([])。

      
<div [attr.id]="my-id"></div><div [style.color]="red"></div><div [class.large]="true"></div>

設定要求

必須啟用 strictTemplates 才能發出任何擴充套件診斷。 除了 strictTemplates 之外，textAttributeNotBinding 沒有其他要求。

如果我無法避免這種情況怎麼辦？

可以透過編輯專案的 tsconfig.json 檔案來停用此診斷：

      
{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "textAttributeNotBinding": "suppress"      }    }  }}

有關更多訊息，請參閱擴充套件診斷設定。
