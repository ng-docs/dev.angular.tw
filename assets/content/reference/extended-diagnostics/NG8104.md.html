文字屬性不繫結

edit

這個診斷確保具有「特殊」Angular 繫結字首 (attr., style., 和 class.) 的屬性被解釋為繫結。

      
<div attr.id="my-id"></div>

有什麼問題？

在此範例中，attr.id 被解讀為常規屬性，並會以原樣出現在最終的 HTML 中 (<div attr.id="my-id"></div>)。這可能不是開發人員的本意。反之，本意可能是設定 id 屬性 (<div id="my-id"></div>)。

我該怎麼做？

在繫結到「attr.」、「class.」或「style.」時，請務必使用 Angular 範本繫結語法 ([])。

      
<div [attr.id]="my-id"></div>
<div [style.color]="red"></div>
<div [class.large]="true"></div>

組態需求

必須啟用 strictTemplates 才能發出任何延伸診斷。 textAttributeNotBinding 除了 strictTemplates 之外沒有其他需求。

如果我無法避免這件事怎麼辦？

這個診斷可以透過編輯專案的 tsconfig.json 檔案停用：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "textAttributeNotBinding": "suppress"
      }
    }
  }
}
