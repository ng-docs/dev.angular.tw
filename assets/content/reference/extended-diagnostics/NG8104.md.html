文本属性未绑定

edit

此诊断确保具有「特殊」Angular 绑定前缀（ attr.、 style. 和 class.）的属性会被解释为绑定。

      
<div attr.id="my-id"></div>

那有什么问题？

在此示例中， attr.id 被解释为常规属性，并将按原样出现在最终的 HTML 中（ <div attr.id="my-id"></div>）。这可能不是开发者的意图。相反，意图可能是设置 id 属性（ <div id="my-id"></div>）。

我该怎么办？

在绑定到 attr.、 class. 或 style. 时，请确保使用 Angular 模板绑定语法（ []）。

      
<div [attr.id]="my-id"></div>
<div [style.color]="red"></div>
<div [class.large]="true"></div>

配置要求

strictTemplates 必须启用，任何扩展诊断才会发出。 textAttributeNotBinding 除了 strictTemplates 之外没有其他要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "textAttributeNotBinding": "suppress"
      }
    }
  }
}

有关更多信息，请参阅扩展诊断配置。
