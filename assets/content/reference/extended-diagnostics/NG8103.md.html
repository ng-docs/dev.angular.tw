缺少控制流指令

edit

此诊断确保在模板中使用已知控制流指令（如 *ngIf、 *ngFor 或 *ngSwitch）的独立组件也要单独导入这些指令或通过导入 CommonModule 来导入它们。

      
import {Component} from '@angular/core';
@Component({
  standalone: true,
  // Template uses `*ngIf`, but no corresponding directive imported.
  imports: [],
  template: `<div *ngIf="visible">Hi</div>`,
})
class MyComponent {}

那有什么问题？

在未导入控制流指令的情况下使用它会在运行时失败，因为 Angular 试图绑定到 HTML 元素的 ngIf 属性，但该属性并不存在。

我该怎么办？

确保导入了相应的控制流指令。

指令可以单独导入：

      
import {Component} from '@angular/core';
import {NgIf} from '@angular/common';
@Component({
  standalone: true,
  imports: [NgIf],
  template: `<div *ngIf="visible">Hi</div>`,
})
class MyComponent {}

或者你可以导入整个 CommonModule，其中包含所有控制流指令：

      
import {Component} from '@angular/core';
import {CommonModule} from '@angular/common';
@Component({
  standalone: true,
  imports: [CommonModule],
  template: `<div *ngIf="visible">Hi</div>`,
})
class MyComponent {}

配置要求

strictTemplates 必须启用，任何扩展诊断才会发出。 missingControlFlowDirective 除了 strictTemplates 之外没有其他要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "missingControlFlowDirective": "suppress"
      }
    }
  }
}

有关更多信息，请参阅扩展诊断配置。
