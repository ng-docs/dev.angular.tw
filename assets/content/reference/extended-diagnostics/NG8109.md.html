Signals 必須在範本內嵌中呼叫。

edit

此診斷檢測範本內嵌中未呼叫的 signals。

      
import {Component, signal, Signal} from '@angular/core';@Component({  template: `<div>{{ mySignal }}/div>`,})class MyComponent {    mySignal: Signal<number> = signal(0);}

那樣有什麼問題？

Angular Signals 是零引數函式 (() => T)。當執行時，它們會回傳 signal 的當前值。 這意味著當在範本內嵌中使用它們來渲染它們的值時，它們旨在被呼叫。

我應該怎麼做？

當你在範本內嵌中使用 signal 來渲染它的值時，請確保呼叫該 signal。

      
import {Component, signal, Signal} from '@angular/core';@Component({  template: `<div>{{ mySignal() }}/div>`,})class MyComponent {  mySignal: Signal<number> = signal(0)}

設定要求

必須啟用 strictTemplates 才能發出任何擴充套件診斷。 interpolatedSignalNotInvoked 除了 strictTemplates 之外沒有額外的要求。

如果我無法避免這種情況怎麼辦？

可以透過編輯專案的 tsconfig.json 檔案來停用此診斷：

      
{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "interpolatedSignalNotInvoked": "suppress"      }    }  }}

請參閱 擴充套件診斷設定 以獲取更多訊息。
