信号必须在模板插值中被调用。

edit

此诊断检测模板插值中未被调用的信号。

      
import {Component, signal, Signal} from '@angular/core';
@Component({
  template: `<div>{{ mySignal }}/div>`,
})
class MyComponent {
    mySignal: Signal<number> = signal(0);
}

那有什么问题？

Angular 信号是零参数函数（ () => T）。执行时，它们返回信号的当前值。 这意味着它们在模板插值中使用时应被调用以渲染其值。

我该怎么办？

确保在模板插值中使用信号时调用它以渲染其值。

      
import {Component, signal, Signal} from '@angular/core';
@Component({
  template: `<div>{{ mySignal() }}/div>`,
})
class MyComponent {
  mySignal: Signal<number> = signal(0)
}

配置要求

strictTemplates 必须启用，才能发出任何扩展诊断。 interpolatedSignalNotInvoked 除了 strictTemplates 之外没有其他附加要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "interpolatedSignalNotInvoked": "suppress"
      }
    }
  }
}

有关更多信息，请参阅扩展诊断配置。
