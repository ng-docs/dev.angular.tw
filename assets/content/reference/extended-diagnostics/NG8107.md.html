可選鏈不可為空

edit

此診斷在 Angular 樣板中檢測到可選鏈操作 (.?) 的左側型別不包含 null 或 undefined 時。

      
import {Component} from '@angular/core';@Component({  // Print the user's name safely, even if `user` is `null` or `undefined`.  template: `<div>User name: {{ user?.name }}</div>`,})class MyComponent {  // `user` is declared as an object which *cannot* be `null` or `undefined`.  user: { name: string } = { name: 'Angelino' };}

那有什麼問題？

對不可為空的輸入使用可選鏈運算子沒有效果，並且表明值的允許型別與其在樣板中的呈現方式之間存在差異。 開發人員可能合理地認為可選鏈運算子的輸出可能是 null 或 undefined，但實際上它永遠不會是這些值中的任何一個。 這可能會導致對程式預期輸出的困惑。

我應該怎麼做？

更新樣板和宣告的型別以保持同步。 仔細檢查輸入的型別，並確認它是否真的應該可以為空。

如果輸入應該可以為空，請將 null 或 undefined 新增到其型別中以表明這一點。

      
import {Component} from '@angular/core';@Component({  // If `user` is nullish, `name` won't be evaluated and the expression will  // return the nullish value (`null` or `undefined`).  template: `<div>{{ user?.name }}</div>`,})class MyComponent {  user: { name: string } | null = { name: 'Angelino' };}

如果輸入不應該可以為空，請刪除 ? 運算子。

      
import {Component} from '@angular/core';@Component({  // Template always displays `name` as `user` is guaranteed to never be `null`  // or `undefined`.  template: `<div>{{ foo.bar }}</div>`,})class MyComponent {  user: { name: string } = { name: 'Angelino' };}

設定要求

必須啟用 strictTemplates 才能發出任何擴充套件診斷。 還必須啟用 strictNullChecks 才能發出 optionalChainNotNullable 診斷。

如果我無法避免這種情況怎麼辦？

可以透過編輯專案的 tsconfig.json 檔案來停用此診斷：

      
{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "optionalChainNotNullable": "suppress"      }    }  }}

有關更多訊息，請參閱擴充套件診斷設定。
