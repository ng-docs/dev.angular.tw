可选链不可为空

edit

此诊断会检测可选链操作符（.?）的左侧有没有在其 Angular 模板中包括 null 或 undefined 类型。

      
import {Component} from '@angular/core';
@Component({
  // Print the user's name safely, even if `user` is `null` or `undefined`.
  template: `<div>User name: {{ user?.name }}</div>`,
})
class MyComponent {
  // `user` is declared as an object which *cannot* be `null` or `undefined`.
  user: { name: string } = { name: 'Angelino' };
}

那有什么问题？

使用可选链操作符与不可为空的输入没有效果，这表明值的允许类型与其在模板中的呈现方式之间存在差异。开发者可能会合理地认为，可选链操作符的输出可能是 null 或 undefined，但实际上它永远不会是这两个值中的任何一个。这可能会导致对程序预期输出的混淆。

我该怎么办？

更新模板和声明的类型以同步两者。仔细检查输入的类型，并确认它是否真是可空的。

如果输入应该是可空的，请在其类型中添加 null 或 undefined 以表明这一点。

      
import {Component} from '@angular/core';
@Component({
  // If `user` is nullish, `name` won't be evaluated and the expression will
  // return the nullish value (`null` or `undefined`).
  template: `<div>{{ user?.name }}</div>`,
})
class MyComponent {
  user: { name: string } | null = { name: 'Angelino' };
}

如果输入不应该为空，则删除 ? 操作符。

      
import {Component} from '@angular/core';
@Component({
  // Template always displays `name` as `user` is guaranteed to never be `null`
  // or `undefined`.
  template: `<div>{{ foo.bar }}</div>`,
})
class MyComponent {
  user: { name: string } = { name: 'Angelino' };
}

配置要求

strictTemplates 必须启用，才能发出任何扩展诊断。 strictNullChecks 也必须启用，才能发出 optionalChainNotNullable 诊断。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

      
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "optionalChainNotNullable": "suppress"
      }
    }
  }
}

有关更多信息，请参阅扩展诊断配置。
