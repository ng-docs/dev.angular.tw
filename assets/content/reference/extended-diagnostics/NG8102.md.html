空值合併（??）不可為空

edit

此診斷會檢測 Angular 樣板中無用的空值合併運算符 (??) 字元。具體來說，它會查詢輸入『不可為空』的操作，這意味著其型別不包含 null 或 undefined。對於此類值，永遠不會使用 ?? 的右側。

      
import {Component} from '@angular/core';@Component({  // Template displays `username` if present, falls back to 'root' if it is  // `null` or `undefined`.  template: `<div>{{ username ?? 'root' }}</div>`,})class MyComponent {  // `username` is declared as a `string` which *cannot* be `null` or  // `undefined`.  username: string = 'Angelino';}

這樣做有什麼問題？

對不可為空的輸入使用空值合併運算符沒有任何效果，並且表明值的允許型別與其在樣板中的呈現方式之間存在差異。開發人員可能理所當然地認為在某些情況下會顯示空值合併運算符的右側，但實際上永遠不會顯示。這可能會導致對程式預期輸出的困惑。

我應該怎麼做？

更新樣板和宣告的型別以保持同步。仔細檢查輸入的型別，並確認它是否真的應該是可為空的。

如果輸入應該是可為空的，請將 null 或 undefined 新增到其型別中以表明這一點。

      
import {Component} from '@angular/core';@Component({  template: `<div>{{ username ?? 'root' }}</div>`,})class MyComponent {  // `username` is now nullable. If it is ever set to `null`, 'root' will be  // displayed.  username: string | null = 'Angelino';}

如果輸入不應該是可為空的，請刪除 ?? 運算符及其右操作數，因為 TypeScript 保證該值始終不可為空。

      
import {Component} from '@angular/core';@Component({  // Template always displays `username`, which is guaranteed to never be `null`  // or `undefined`.  template: `<div>{{ username }}</div>`,})class MyComponent {  username: string = 'Angelino';}

設定要求

必須啟用 strictTemplates 才能發出任何擴充套件診斷訊息。還必須啟用 strictNullChecks 才能發出 nullishCoalescingNotNullable 診斷訊息。

如果我無法避免這種情況怎麼辦？

可以透過編輯專案的 tsconfig.json 檔案來停用此診斷：

      
{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "nullishCoalescingNotNullable": "suppress"      }    }  }}

有關更多訊息，請參閱擴充套件診斷設定。
