Angular 路線圖

edit

瞭解 Angular 團隊如何在 Web 上建立勢頭。

作為一個開源專案，Angular 的每日送出、PR 和發展勢頭都可在 GitHub 上追蹤。為了提高這種日常工作與框架未來之間聯絡的透明度，我們的路線圖彙集了團隊當前和未來計劃的願景。

以下專案與特定的 Angular 版本無關。我們將在完成後發布它們，並且根據我們的發布計劃，遵循語義化版本控制，它們將成為特定版本的一部分。例如，如果屬性包含重大更改，我們會在完成後的下一個小版本或下一個主版本中發布。

目前，Angular 框架有兩個目標：

  1. 改進 Angular 開發者體驗 以及
  2. 改進框架的效能。

繼續閱讀以瞭解我們計劃如何透過具體的專案工作來實現這些目標。

交付 Angular Signals

這個專案透過引入 Signals 作為回應式原始型別，重新思考了 Angular 的回應式模型。最初的規劃促成了數百次討論、與開發者的對話、反饋會議、使用者體驗研究以及一系列 RFC，這些 RFC 收到了 1,000 多條評論。

作為 v17 版本發布的一部分，我們將 Angular Signals 函式庫從開發者預覽版升級。在 v19 中，我們將基於 signal 的查詢、輸入和模型輸入移至穩定版。接下來，我們需要在完成此項目之前最終確定副作用。

Zoneless Angular

在 v18 中，我們在 Angular 中發布了實驗性的 zoneless 支援。它使開發者無需在其捆綁包中包含 zone.js 即可使用該框架，從而提高了效能、除錯體驗和互操作性。作為初始版本的一部分，我們還在 Angular CDK 和 Angular Material 中引入了 zoneless 支援。

在 v19 中，我們在服務器端渲染中引入了 zoneless 支援，解決了一些極端情況，並建立了一個原理圖來搭建 zoneless 專案。我們將 Google Fonts 遷移到 zoneless，這提高了效能、開發者體驗，並使我們能夠識別在將此屬性移至開發者預覽版之前需要解決的差距。請繼續關注未來幾個月的更多更新。

Signal 整合

我們正在努力改進諸如表單、HTTP 和路由器等基本 Angular 套件與 Signals 的整合。作為此項目的一部分，我們將尋求機會引入便捷的基於 signal 的 API 或套件裝器，以改善整體開發者體驗。

Angular DevTools 中的 Signal 除錯

隨著 Angular 中 Signals 的發展，我們正在開發更好的工具來除錯它們。優先級清單中較高的是用於檢查和除錯 signal 的 UI。

改進 HMR（熱模組過載）

我們正在透過啟用熱模組替換來努力實現更快的編輯/重新整理週期。
