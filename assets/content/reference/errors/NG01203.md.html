缺少值访问器

edit

对于所有自定义表单控件，你必须注册一个值访问器。

这是一个如何提供它的示例：

      
providers: [
  {
    provide: NG_VALUE_ACCESSOR,
    useExisting: forwardRef(() => MyInputField),
    multi: true,
  }
]

调试此错误

如前所述，你的控件应该有一个值访问器，但它却没有。不过，在实践中发生这种情况的原因有很多。下面是会导致此错误的一些已知问题的列表。

  1. 如果你定义了一个自定义表单控件，你是否忘了提供一个值访问器？
  2. 你是否在没有值的元素或 无效元素（例如 <div [(ngModel)]="foo">）上放置了 ngModel？
  3. 你是否在 NgModule 内声明了自定义表单控件？如果是，请确保你 导入了 NgModule。
  4. 你是否在使用第三方自定义表单控件时使用了 ngModel？检查该控件是否提供了值访问器。如果没有，在控件的元素上使用 ngDefaultControl。
  5. 你在测试自定义表单控件吗？确保配置你的测试台以便让它知道此控件。你可以用 Testbed.configureTestingModule 来实现。
  6. 你是否在使用 Nx 和模块联合（Module Federation）与 Webpack？你的 webpack.config.js 可能需要 额外配置以确保共享表单包。
