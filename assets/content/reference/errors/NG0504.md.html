延遲水合旗標套用至無效節點

edit

當將 ngSkipHydration 屬性新增至不適當的 DOM 節點時，會發生此錯誤。ngSkipHydration 屬性只能直接在範本中或透過主機繫結套用至元件主機節點。無法套用至其他 DOM 節點，若這麼做除了造成此錯誤之外，不會有任何效果。

有關水分補充的更多資訊，請參閱 本指南。

以下範例會觸發錯誤。

範例 1

在此範例中，ngSkipHydration 屬性套用至 <div> 使用指令的宿主繫結。由於 <div> 不作為元件宿主節點，Angular 將拋出錯誤。

      
@Directive({
  standalone: true,
  selector: '[dir]',
  host: {ngSkipHydration: 'true'},
})
class Dir {
}
@Component({
  standalone: true,
  selector: 'app',
  imports: [Dir],
  template: `
    <div dir></div>
  `,
})
class SimpleComponent {
}

範例 2

在此範例中，ngSkipHydration 透過範本套用至 <div> 作為屬性。 由於 <div> 不作為元件主機節點，Angular 將會擲回錯誤。

      
@Component({
  standalone: true,
  selector: 'app',
  template: `
    <div ngSkipHydration></div>
  `,
})
class SimpleComponent {
}

偵錯錯誤

從任何無效的 DOM 節點中移除 ngSkipHydration 屬性。或者，將 ngSkipHydration 屬性移至範本或透過 host binding 中的元件主機節點。
