CLI 參考

edit
指令 別名 說明
add 將外部程式庫支援新增至您的專案。
analytics 設定 Angular CLI 使用量指標的收集。
build b 將 Angular 應用程式或程式庫編譯至給定輸出路徑中的 dist/ 名為輸出的目錄。
cache 設定持久性磁碟快取並擷取快取統計資料。
completion 為您的終端機設定 Angular CLI 自動完成。
config 擷取或設定工作區 angular.json 檔案中的 Angular 設定值。
deploy 為工作區中的預設專案或指定專案呼叫部署建構器。
doc d 在瀏覽器中開啟官方 Angular 文件 (angular.io) 並搜尋給定的關鍵字。
e2e e 建置並提供 Angular 應用程式，然後執行端對端測試。
extract-i18n 從原始程式碼中萃取 i18n 訊息。
generate g 根據架構來產生和/或修改檔案。
lint 在給定的專案資料夾中針對 Angular 應用程式程式碼執行程式碼檢查工具。
new n 建立新的 Angular 工作區。
run 執行架構目標，其中包含專案中定義的選用自訂建構器設定。
serve s 建置並提供您的應用程式，並在檔案變更時重新建置。
test t 在專案中執行單元測試。
update 更新您的工作區及其相依性。請參閱 https://update.angular.io/。
version v 輸出 Angular CLI 版本。
