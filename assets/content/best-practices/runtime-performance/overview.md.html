執行時效能最佳化

edit

快速渲染對於 Angular 至關重要，我們在建立此框架時就考慮了很多最佳化，以幫助你開發高效能的應用。為了更好地理解你的應用的效能，我們提供了 Angular DevTools 和一個關於如何使用 Chrome DevTools 進行效能分析的 影片指南。在本節中，我們將介紹最常見的效能最佳化技術。

變更檢測是 Angular 用來檢查你的應用狀態是否已更改以及是否需要更新任何 DOM 的過程。從高層次來看，Angular 會從上到下遍歷你的元件，以查詢變更。Angular 會定期執行其變更檢測機制，以便將資料模型的更改反映在應用的檢視中。可以手動或透過非同步事件（例如，使用者互動或 XMLHttpRequest 完成）觸發變更檢測。

變更檢測經過了高度最佳化且效能良好，但如果應用執行得過於頻繁，仍然可能導致速度減慢。

在本指南中，你將學習如何透過跳過應用的某些部分並在必要時才執行變更檢測來控制和最佳化變更檢測機制。
