讓你的 Angular 專案保持更新

edit

正如 Web 和整個 Web 生態系統一樣，Angular 也在持續改進。Angular 在持續改進的同時，也非常注重穩定性，併力求讓更新變得簡單直接。讓你的 Angular 應用保持更新，就能讓你享用前沿的新屬性，以及各種最佳化和缺陷修復。

本文件包含一些訊息和資源，以幫助你讓你的 Angular 應用和函式庫保持更新。

關於我們的版本控制策略和實踐（包括支援和棄用實踐，以及發布時間表）的訊息，請參閱 Angular 版本控制與發布。

提示： 如果你目前正在使用 AngularJS，請參閱 從 AngularJS 升級。 AngularJS 是所有 v1.x 版本的 Angular 的名稱。

獲取新版本發布的通知

要在新版本發布時收到通知，請在 X（以前的 Twitter）上關注 @angular，或訂閱 Angular 部落格。

瞭解新屬性

有哪些新東西？有哪些變化？我們在 Angular 部落格的 版本發布公告 中分享你需要瞭解的最重要的訊息。

要檢視按版本組織的完整更改清單，請參閱 Angular 變更日誌。

檢查你的 Angular 版本

要檢查你的應用的 Angular 版本，請在你的專案目錄中使用 ng version 指令。

查詢 Angular 的當前版本

Angular 最新的穩定發布版本顯示在 npm 的 『Version』 下。例如，16.2.4。

你還可以使用 CLI 指令 ng update 找到 Angular 的最新版本。 預設情況下，ng update（不帶附加引數）會列出可用於你的更新。

更新你的環境和應用

為了使更新變得簡單，我們在互動式的 Angular 更新指南 中提供了完整的說明。

Angular 更新指南根據你指定的當前版本和目標版本提供自定義的更新說明。它包括基本和高階更新路徑，以對應你的應用的複雜性。它還包括故障排除訊息和任何推薦的手動更改，以幫助你充分利用新版本。

對於簡單的更新，你只需要 CLI 指令 ng update。不帶附加引數，ng update 會列出可用於你的更新，並提供將你的應用更新到最新版本的推薦步驟。

Angular 版本控制與發布 描述了你可以根據版本的版本號預期的更改級別。它還描述了受支援的更新路徑。
