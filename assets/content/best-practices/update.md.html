持续更新你的 Angular 项目

edit

就像 Web 及其整个生态系统一样，Angular 也在持续改进中。Angular 平衡了持续改进与强调稳定性之间的冲突，努力让升级变得更简单。让 Angular 应用始终保持最新，可以让你获得前沿的新特性所带来的好处，还有各种优化和 Bug 修复。

本文档包含一些信息和资源，来帮助你持续更新 Angular 的应用和库。

关于版本控制策略和实践的信息（包括支持和弃用实践，以及发布计划），请参见 Angular 版本控制和发布。

有帮助的：如果你当前正在使用 AngularJS，请参见 从 AngularJS 升级。 AngularJS 是所有 v1.x 版本的名称。

获得新版本的发布通知

要在有新版本发布时收到通知，请关注 @angular 的 Twitter 或订阅 Angular 博客。

了解最新特性

有什么新内容？有什么变化？我们会在 Angular 博客的 发布公告 中分享你需要知道的最重要的信息。

要审阅按版本组织的完整变更列表，请参阅 Angular 变更日志。

检查你的 Angular 版本

要检查你的应用的 Angular 版本，请在项目目录中使用 ng version 命令。

查看 Angular 的当前版本

Angular 最新的稳定发布版本会显示在 npm 上的 “Version” 下。例如， 16.2.4。

你还可以使用 CLI 命令 ng update发现 Angular 的最新版本。默认情况下，ng update（不带其它参数）会列出你可用的所有更新。

升级你的环境和应用

为了使更新变得简单，我们在交互式的 Angular 更新指南 中提供了完整的说明。

Angular 升级指南提供了自定义的升级指令，基于当前版本和你指定的目标版本。它包括简单的和高级的升级路径，以适应你应用中的各种复杂情况。它还包括诊断信息和建议的手动修改动作，以帮助你获得最新版本。

对于简单的升级工作，你所要做的一切就是执行一下 CLI 命令 ng update。不带额外参数时，ng update 会列出你可用的更新版本，并提供建议的升级步骤，来把你的应用升级到最新版本上来。

Angular 版本控制和发布 描述了你可以根据发布版本号预期的变更级别。它还描述了受支持的更新路径。
