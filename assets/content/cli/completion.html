CLI

completion

指令
ng completion

建立自動補全功能會設定你的終端，這樣在輸入時，按下 <TAB> 鍵就會顯示可用的各種指令和選項。這讓你無需死記硬背就能輕鬆發現和使用 CLI 指令。

自動建立

首次使用 CLI 時（v14+），CLI 應該會提示並詢問是否為你建立自動補全功能。只需回答『是』，CLI 就會處理剩下的事情。

          
$ ng serve? Would you like to enable autocompletion? This will set up your terminal so pressing TAB while typing Angular CLI commands will show possible options and autocomplete arguments. (Enabling autocompletion will modify configuration files in your home directory.) YesAppended `source <(ng completion script)` to `/home/my-username/.bashrc`. Restart your terminal or run:source <(ng completion script)to autocomplete `ng` commands.# Serve output...

如果你之前拒絕了提示，它就不會再次詢問。但是你可以執行 ng completion 來自動執行相同的操作。

這將修改你的終端環境以載入 Angular CLI 自動補全功能，但無法更新你當前的終端會話。可以重啟終端，或者直接執行 source <(ng completion script) 以在你當前的會話中啟用自動補全功能。

透過輸入 ng ser<TAB> 來測試一下，它應該會自動補全為 ng serve。有歧義的引數將顯示所有可能的選項及其文件，例如 ng generate <TAB>。

手動建立

有些使用者可能具有高度自定義的終端設定，其設定檔案可能會以固定的結構檢入到原始碼控制中。ng completion 始終只會將 Angular 的設定附加到當前 Shell 的現有設定檔案中，或者在不存在設定檔案時建立一個。如果你想更精確地控制此設定檔案的位置，你可以透過讓你的 Shell 在啟動時執行以下指令來手動建立它：

          
source <(ng completion script)

這等效於 ng completion 將自動建立的內容，並在需要時為高階使用者提供更高的環境靈活性。

平台支援

Angular CLI 支援 MacOS 和 Linux 作業系統上 Bash 和 Zsh Shell 的自動補全功能。在 Windows 上，支援使用 Bash 或 Zsh 的 Git Bash 和 Windows Subsystem for Linux。

全域安裝

自動補全的工作原理是設定你的終端，以便在啟動時呼叫 Angular CLI 來載入建立指令碼。這意味著終端必須能夠找到並執行 Angular CLI，通常是透過全域安裝將二進位檔案放在使用者的 $PATH 中。如果你收到 command not found: ng，請確保 CLI 已全域安裝，你可以使用 -g 標誌來完成全域安裝：

          
npm install -g @angular/cli
