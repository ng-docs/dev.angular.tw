CLI

completion

Command
ng completion

设置自动补全功能会配置你的终端，这样在输入过程中按下 <TAB> 键时会显示各种可用的命令和选项。这使得用户在不需要大量记忆的情况下，可以非常容易地发现和使用 CLI 命令。

自动化设置

CLI 在你第一次使用它时（v14+），应该会提示并询问是否为你设置自动补全。只需回答『是』，CLI 会处理好剩下的部分。

          
            $ ng serve? Would you like to enable autocompletion? This will set up your terminal so pressing TAB while typing Angular CLI commands will show possible options and autocomplete arguments. (Enabling autocompletion will modify configuration files in your home directory.) YesAppended `source <(ng completion script)` to `/home/my-username/.bashrc`. Restart your terminal or run:source <(ng completion script)to autocomplete `ng` commands.# Serve output...

如果你已经拒绝了提示，它将不会再询问。但你可以运行 ng completion 来自动完成同样的操作。

这会修改你的终端环境以加载 Angular CLI 自动补全，但不能更新当前终端会话。要么重启它，要么直接运行 source <(ng completion script) 以在当前会话中启用自动补全。

通过输入 ng ser<TAB> 来测试一下，它应该会自动补全为 ng serve。对于模糊参数，会显示所有可能的选项及其文档，例如 ng generate <TAB>。

手动设置

一些用户可能有高度自定义的终端设置，可能有配置文件被检入源代码控制并具有特定结构。 ng completion 只会将 Angular 的设置附加到当前 shell 的现有配置文件中，或者如果不存在则创建一个。如果你想更精确地控制此配置的存放位置，可以通过在 shell 启动时运行以下命令手动设置：

          
            source <(ng completion script)

这相当于 ng completion 自动设置的内容，并在需要时为高级用户在其环境中提供更多灵活性。

平台支持

Angular CLI 支持在 MacOS 和 Linux 操作系统上的 Bash 和 Zsh shell 自动补全。在 Windows 上，支持 Git Bash 和 Windows Subsystem for Linux 使用 Bash 或 Zsh。

全局安装

自动补全通过配置你的终端在启动时调用 Angular CLI 来加载设置脚本。这意味着终端必须能够找到并执行 Angular CLI，通常通过全局安装将二进制文件放在用户的 $PATH 中。如果你收到 command not found: ng 错误，请确保 CLI 已全局安装，你可以使用 -g 标志来进行安装：

          
            npm install -g @angular/cli
