CLI

new

指令
ng new
ng n

建立並初始化一個新的 Angular 應用，該應用是新工作區的預設專案。

為可選設定提供互動式提示，例如新增路由支援。 所有提示都可以安全地使用預設值。

  • 新的工作區資料夾被賦予指定的專案名稱，並在頂層包含設定檔案。

  • 預設情況下，新的初始應用（與工作區同名）的檔案放在 src/ 子資料夾中。

  • 新應用的設定會出現在 angular.json 工作區設定檔案的 projects 部分中，位於其專案名稱下。

  • 你在工作區中生成的後續應用都位於 projects/ 子資料夾中。

如果你計劃在工作區中擁有多個應用，你可以使用 --no-create-application 選項建立一個空工作區。 然後，你可以使用 ng generate application 建立一個初始應用。 這允許工作區名稱與初始應用名稱不同，並確保所有應用都位於 /projects 子資料夾中，與設定檔案的結構對應。

引數

name

新工作區和初始專案的名稱。此名稱將用於根目錄和整個專案中的各種識別符號。

值型別string

選項

collection
別名c

用於生成初始應用的原理圖集合。

值型別string
commit

為新倉庫設定初始 Git 送出。

值型別boolean預設值true
create-application

在新工作區中建立一個新的初始應用專案。如果為 false，則建立一個沒有初始應用的空工作區。然後，你可以使用 ng generate application 指令在 projects 目錄中建立應用。

值型別boolean預設值true
defaults

停用具有預設選項的互動式輸入提示。

值型別boolean預設值false
directory

應在其中建立新工作區和專案的目錄。如果未指定，則將在當前目錄中建立工作區。

值型別string
dry-run
別名d

執行並報告活動，但不寫出結果。

值型別boolean預設值false
experimental-zoneless

建立一個不使用 zone.js 的初始應用。

值型別boolean預設值false
force

強制覆蓋現有檔案。

值型別boolean預設值false
help

在控制台中顯示此指令的幫助訊息。

值型別boolean
inline-style
別名s

將初始應用的根元件的樣式直接包含在 app.component.ts 檔案中。預設情況下，會建立一個單獨的樣式表檔案（例如 app.component.css）。

值型別boolean
inline-template
別名t

將初始應用的根元件的 HTML 樣板直接包含在 app.component.ts 檔案中。預設情況下，會建立一個單獨的樣板檔案（例如 app.component.html）。

值型別boolean
interactive

啟用互動式輸入提示。

值型別boolean預設值true
minimal

生成一個不帶任何測試框架的最小 Angular 工作區。這旨在用於學習目的和簡單的實驗，而不是用於生產應用。

值型別boolean預設值false
new-project-root

將在工作區內建立新專案的路徑，相對於工作區根目錄。預設情況下，新專案在 projects 目錄中建立。

值型別string預設值projects
package-manager

用於安裝依賴套件的套件管理器。

值型別string
prefix
別名p

應用於初始專案的生成的選擇器的字首。例如，如果字首是 my-app 並且你生成了一個名為 my-component 的元件，則選擇器將是 my-app-my-component。

值型別string預設值app
routing

在初始應用專案中啟用路由。這將建立必要的檔案和模組，以管理應用中不同檢視之間的導向。

值型別boolean
server-routing

在初始專案中使用伺服器路由和 App Engine API（開發者預覽版）建立一個伺服器應用。

值型別boolean
skip-git
別名g

不要在新工作區中初始化 Git 倉庫。預設情況下，會初始化 Git 倉庫以幫助你跟蹤對專案的更改。

值型別boolean預設值false
skip-install

跳過自動安裝軟體套件。你將需要在以後手動安裝依賴項。

值型別boolean預設值false
skip-tests
別名S

跳過單元測試檔案 spec.ts 的生成。

值型別boolean預設值false
ssr

為伺服器端渲染 (SSR) 和靜態站點生成 (SSG/預渲染) 設定初始應用。

值型別boolean
standalone

基於獨立 API 建立一個應用，無需 NgModule。

值型別boolean預設值true
strict

啟用更嚴格的型別檢查和更嚴格的捆綁包預算設定。此設定有助於提高可維護性並預先捕獲錯誤。有關更多訊息，請參閱 https://angular.dev/tools/cli/template-typecheck#strict-mode

值型別boolean預設值true
style

要在初始專案中為元件建立的樣式表檔案的型別。

值型別string
view-encapsulation

為初始專案中的元件設定檢視封裝模式。這決定了元件樣式的範圍和應用方式。選項包括：Emulated（預設值，樣式限定於元件）、None（樣式是全域的）和 ShadowDom（樣式使用 Shadow DOM 封裝）。

值型別string
