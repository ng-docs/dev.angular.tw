CLI

generate guard

指令
ng generate guard
ng generate g

在你的專案中建立一個新的路由守衛。路由守衛用於在路由啟用之前檢查某些條件，以控制對應用某些部分的訪問。本原理圖使用指定的名稱、型別和選項生成一個新的守衛。

引數

name

新路由守衛的名稱。這將用於建立守衛的類別檔案和 spec 檔案（例如，my-guard.guard.ts 和 my-guard.guard.spec.ts）。

值型別string

選項

flat

在當前專案的頂層建立新的守衛檔案。如果設定為 false，將建立一個以守衛名稱命名的新檔案夾來包含這些檔案。

值型別boolean預設值true
functional

將守衛生成為函式而不是類。對於基本場景，函式式守衛可能更簡單。

值型別boolean預設值true
implements
別名guardType

指定要建立的守衛型別。你可以選擇以下一個或多個：CanActivate（控制對路由的訪問）、CanActivateChild（控制對子路由的訪問）、CanDeactivate（在離開路由之前請求確認）、CanMatch（確定是否可以對應路由）。

值型別array
project

要在其中建立守衛的專案名稱。如果未指定，CLI 將從當前目錄確定專案。

值型別string
skip-tests

跳過為新守衛生成單元測試檔案 spec.ts。

值型別boolean預設值false
