CLI

generate interceptor

指令
ng generate interceptor

在你的專案中建立一個新的攔截器。攔截器用於在 HTTP 請求和回應到達其目的地之前對其進行攔截和修改。這使你可以執行諸如新增身份驗證標頭、處理錯誤或記錄請求之類別的任務。此原理圖會為新的攔截器生成必要的檔案和範本程式碼。

引數

name

新攔截器的名稱。這將用於建立攔截器的類別和 spec 檔案（例如，my-interceptor.interceptor.ts 和 my-interceptor.interceptor.spec.ts）。

值型別string

選項

flat

在當前專案的頂層建立新的攔截器檔案。如果設定為 false，將建立一個以攔截器名稱命名的新檔案夾來包含這些檔案。

值型別boolean預設值true
functional

將攔截器建立為一個 HttpInterceptorFn 函式而不是一個類別。對於基本場景，函式式攔截器可能更簡單。

值型別boolean預設值true
project

要在其中建立攔截器的專案的名稱。如果未指定，CLI 將從當前目錄確定專案。

值型別string
skip-tests

跳過為新攔截器生成單元測試檔案 spec.ts。

值型別boolean預設值false
