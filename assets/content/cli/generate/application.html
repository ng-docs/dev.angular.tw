CLI

生成應用

指令
ng generate application
ng generate app

在你的工作區內生成一個新的 Angular 應用。本原理圖會建立你專案的基本結構，包括根元件、模組和設定檔案。你可以自定義應用的各個方面，例如路由、樣式和測試。

引數

name

新應用的名稱。此名稱將用於專案目錄以及整個應用程式碼中的各種識別符號。

值型別string

選項

experimental-zoneless

生成一個不使用 zone.js 的應用。

值型別boolean預設值false
inline-style
別名s

將根元件的樣式直接包含在 app.component.ts 檔案中。只有 CSS 樣式可以行內包含。預設情況下，會建立一個單獨的樣式表檔案（例如 app.component.css）。

值型別boolean
inline-template
別名t

將根元件的 HTML 樣板直接包含在 app.component.ts 檔案中。預設情況下，會建立一個單獨的樣板檔案（例如 app.component.html）。

值型別boolean
minimal

生成一個不帶任何測試框架的最小化專案。這適用於學習目的和簡單的實驗，不適用於生產應用。

值型別boolean預設值false
prefix
別名p

要新增到此應用中生成的元件的選擇器上的字首。例如，如果字首是 my-app 並且你生成了一個名為 my-component 的元件，則選擇器將是 my-app-my-component。

值型別string預設值app
project-root

相對於工作區根目錄，將在其中建立新應用檔案的目錄。如果未指定，則將在 projects 目錄中的子資料夾中使用應用的名稱建立應用。

值型別string
routing

生成一個已設定路由的應用。這將建立必要的檔案和模組，以管理應用中不同檢視之間的導向。

值型別boolean預設值true
server-routing

使用伺服器路由和 App Engine API（開發者預覽版）建立伺服器應用。

值型別boolean
skip-install

跳過自動安裝軟體套件。你將需要稍後手動安裝依賴項。

值型別boolean預設值false
skip-package-json

不要將依賴項新增到 package.json 檔案中。

值型別boolean預設值false
skip-tests
別名S

跳過生成單元測試檔案 spec.ts。

值型別boolean預設值false
ssr

為伺服器端渲染 (SSR) 和靜態站點生成 (SSG/預渲染) 設定應用。

值型別boolean預設值false
standalone

建立一個利用獨立 API 的應用，從而無需 NgModule。這可以簡化你的應用結構。

值型別boolean預設值true
strict

為應用啟用更嚴格的捆綁包預算設定。這有助於保持你的應用捆綁包大小較小並提高效能。有關更多訊息，請參閱 https://angular.dev/tools/cli/template-typecheck#strict-mode

值型別boolean預設值true
style

要為應用中的元件建立的樣式表檔案的型別。

值型別string預設值css
view-encapsulation

設定應用元件的檢視封裝模式。這將決定元件樣式的範圍和應用方式。

值型別string
