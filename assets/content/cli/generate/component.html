CLI

生成元件

指令
ng generate component
ng generate c

建立一個新的 Angular 元件。元件是 Angular 應用的基本元素。每個元件都由一個 TypeScript 類別、一個 HTML 範本和一個可選的 CSS 樣式表組成。使用此原理圖在你的專案中生成一個新元件。

引數

name

新元件的名稱。這將用於建立元件的類別、範本和樣式表檔案。例如，如果你提供 my-component，則檔案將被命名為 my-component.component.ts、my-component.component.html 和 my-component.component.css。

值型別string

選項

change-detection
別名c

設定元件的變更檢測策略。

值型別string預設值Default
display-block
別名b

將 :host { display: block; } 新增到元件的樣式表，確保元件渲染為區塊級元素。這對於佈局很有用。

值型別boolean預設值false
export

自動從指定的 NgModule 匯出元件，使其可供應用中的其他模組訪問。

值型別boolean預設值false
export-default

在元件的 TypeScript 檔案中使用預設匯出而不是命名匯出。

值型別boolean預設值false
flat

直接在專案的 src/app 目錄中建立元件檔案，而不是為它們建立一個新檔案夾。

值型別boolean預設值false
inline-style
別名s

將元件的樣式直接包含在 component.ts 檔案中。預設情況下，會建立一個單獨的樣式表檔案（例如，my-component.component.css）。

值型別boolean預設值false
inline-template
別名t

將元件的 HTML 範本直接包含在 component.ts 檔案中。預設情況下，會建立一個單獨的範本檔案（例如，my-component.component.html）。

值型別boolean預設值false
module
別名m

指定應該在其中宣告元件的 NgModule。如果未提供，CLI 將嘗試在元件的路徑中找到最近的 NgModule。

值型別string
prefix
別名p

要新增到元件選擇器的字首。例如，如果字首是 app 且元件名稱是 my-component，則選擇器將是 app-my-component。

值型別string
project

應在其中新增元件的專案的名稱。如果未指定，CLI 將從當前目錄確定專案。

值型別string
selector

用於此元件的 HTML 選擇器。如果未提供，將根據元件名稱生成一個選擇器（例如，app-my-component）。

值型別string
skip-import

不要自動將新元件匯入到其最近的 NgModule 中。

值型別boolean預設值false
skip-selector

跳過為元件生成 HTML 選擇器。

值型別boolean預設值false
skip-tests

跳過生成單元測試檔案 spec.ts。

值型別boolean預設值false
standalone

生成一個獨立元件。獨立元件是自包含的，不需要在 NgModule 中宣告。它們可以獨立使用，也可以直接匯入到其他獨立元件中。

值型別boolean預設值true
style

指定要為元件建立的樣式表型別，或者 none 以跳過建立樣式表。

值型別string預設值css
type

將自定義類型附加到元件的檔案名。例如，如果將型別設定為 container，則檔案將命名為 my-component.container.ts。

值型別string預設值Component
view-encapsulation
別名v

設定元件的檢視封裝模式。這決定了元件樣式的範圍和應用方式。

值型別string
