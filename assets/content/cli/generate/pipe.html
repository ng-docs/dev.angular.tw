CLI

生成通道

指令
ng generate pipe
ng generate p

在你的專案中建立一個新的通道。通道用於轉換資料，以便在範本中顯示。它們接受輸入值並應用特定的轉換，例如格式化日期、貨幣或過濾陣列。此原理圖會為新通道生成必要的檔案和範本程式碼。

引數

name

新通道的名稱。這將用於建立通道的類別檔案和 spec 檔案（例如，my-pipe.pipe.ts 和 my-pipe.pipe.spec.ts）。

值型別string

選項

export

自動從指定的 NgModule 匯出此通道，使其可用於應用中的其他模組。

值型別boolean預設值false
flat

在當前專案的頂層建立新的通道檔案。如果設定為 false，將建立一個以通道名稱命名的新檔案夾來包含這些檔案。

值型別boolean預設值true
module
別名m

指定應該在其中宣告通道的 NgModule。如果未提供，CLI 將嘗試在通道的路徑中查詢最近的 NgModule。

值型別string
project

要在其中建立通道的專案的名稱。如果未指定，CLI 將從當前目錄確定專案。

值型別string
skip-import

不要自動將新通道匯入到其最近的 NgModule 中。

值型別boolean預設值false
skip-tests

阻止為新通道生成單元測試檔案 spec.ts。

值型別boolean預設值false
standalone

生成一個獨立的通道。獨立通道是自包含的，不需要在 NgModule 中宣告。它們可以獨立使用，也可以直接匯入到其他獨立的元件、指令或通道中。

值型別boolean預設值true
