CLI

生成指令

指令
ng generate directive
ng generate d

在你的專案中建立一個新的指令。指令用於擴充套件 HTML 元素和元件的行為或外觀。它們允許你操縱 DOM，新增自定義屬性（Attribute），以及回應事件。這個原理圖為新的指令生成必要的檔案和樣板程式碼。

引數

name

新指令的名稱。這將用於建立指令的類別檔案和 spec 檔案（例如，my-directive.directive.ts 和 my-directive.directive.spec.ts）。

值型別string

選項

export

自動從指定的 NgModule 匯出該指令，使其可用於應用中的其他模組。

值型別boolean預設值false
flat

在當前專案的頂層建立新的指令檔案。如果設定為 false，將建立一個以指令名稱命名的新檔案夾來包含這些檔案。

值型別boolean預設值true
module
別名m

指定應該在其中宣告指令的 NgModule。如果未提供，CLI 將嘗試在指令的路徑中找到最近的 NgModule。

值型別string
prefix
別名p

要新增到指令選擇器中的字首。例如，如果字首是 app 且指令名稱是 highlight，則選擇器將是 appHighlight。

值型別string
project

應該在其中新增指令的專案名稱。如果未指定，CLI 將從當前目錄確定專案。

值型別string
selector

用於此指令的 HTML 選擇器。如果未提供，將根據指令的名稱生成一個選擇器（例如，appHighlight）。

值型別string
skip-import

不要自動將新的指令匯入到其最近的 NgModule 中。

值型別boolean預設值false
skip-tests

跳過為新指令生成單元測試檔案 spec.ts。

值型別boolean預設值false
standalone

生成一個獨立的指令。獨立指令是自包含的，不需要在 NgModule 中宣告。它們可以獨立使用，也可以直接匯入到其他獨立元件或指令中。

值型別boolean預設值true
