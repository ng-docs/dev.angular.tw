CLI

生成解析器

指令
ng generate resolver
ng generate r

在你的專案中建立一個新的解析器。解析器用於在路由啟用之前預先獲取資料，以確保在元件顯示之前必要的資料是可用的。這可以透過防止延遲和載入狀態來改善使用者體驗。此原理圖使用指定的名稱和選項生成一個新的解析器。

引數

name

新解析器的名稱。這將用於建立解析器的類別和 spec 檔案（例如，my-resolver.resolver.ts 和 my-resolver.resolver.spec.ts）。

值型別string

選項

flat

在當前專案的頂層建立新的解析器檔案。如果設定為 false，將建立一個以解析器名稱命名的新檔案夾來包含這些檔案。

值型別boolean預設值true
functional

將解析器建立為函式 ResolveFn 而不是類。對於基本場景，函式式解析器可能更簡單。

值型別boolean預設值true
project

要在其中建立解析器的專案的名稱。如果未指定，CLI 將從當前目錄確定專案。

值型別string
skip-tests

跳過為新解析器生成單元測試檔案 spec.ts。

值型別boolean預設值false
