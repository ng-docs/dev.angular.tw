CLI

生成函式庫

指令
ng generate library
ng generate lib

在你的 Angular 工作區中建立一個新的函式庫專案。函式庫是可複用的元件、服務和其他 Angular 工件的集合，可以在多個應用之間共享。這個原理圖簡化了使用必要的檔案和設定生成新函式庫的過程。

引數

name

新函式庫的名稱。此名稱將用於專案目錄和函式庫程式碼中的各種識別符號。

值型別string

選項

entry-file

函式庫的公共 API 檔案的路徑，相對於工作區根目錄。此檔案定義了函式庫的哪些部分可以被匯入它的應用訪問。

值型別string預設值public-api
prefix
別名p

要新增到此函式庫中生成的元件的選擇器上的字首。例如，如果字首是 my-lib 並且你生成了一個名為 my-component 的元件，則選擇器將是 my-lib-my-component。

值型別string預設值lib
project-root

新函式庫的根目錄，相對於工作區根目錄。如果未指定，則將在 projects 目錄中的子資料夾中使用函式庫的名稱建立函式庫。

值型別string
skip-install

跳過自動安裝軟體套件。你將需要稍後手動安裝依賴項。

值型別boolean預設值false
skip-package-json

不要自動將依賴項新增到 package.json 檔案中。

值型別boolean預設值false
skip-ts-config

不要更新工作區 tsconfig.json 檔案以新增新函式庫的路徑對映。路徑對映是在應用中使用函式庫所必需的，但可以在此處停用以簡化開發。

值型別boolean預設值false
standalone

建立一個利用獨立 API 的函式庫，從而無需 NgModule。這可以簡化函式庫的結構及其在應用中的用法。

值型別boolean預設值true
