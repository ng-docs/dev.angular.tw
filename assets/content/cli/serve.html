CLI

serve

指令
ng serve
ng dev
ng s

建立併為你的應用啟動伺服器，並在檔案更改時重新建立。

引數

project

要建立的專案名稱。可以是一個應用或一個函式庫。

值型別string

選項

allowed-hosts

允許訪問開發伺服器的宿主清單。

值型別array
build-target

要啟動伺服器的建構式目標，格式為 project:target[:configuration]。你也可以傳入多個設定名稱，以逗號分隔的清單形式。範例：project:target:production,staging。

值型別string
configuration
別名c

一個或多個已命名的建構式設定，以逗號分隔的清單形式，在 angular.json 的 "configurations" 部分中指定。 建構式使用已命名的設定來執行給定的目標。 欲知詳情，請參閱 https://angular.dev/reference/configs/workspace-config#alternate-build-configurations。

值型別string
disable-host-check

不要驗證已連線的用戶端是否屬於允許的宿主。

值型別boolean預設值false
force-esbuild

強制開發伺服器在建立時使用『browser-esbuild』建構式。

值型別boolean預設值false
headers

要新增到所有回應的自定義 HTTP 標頭。

值型別array
headers

要新增到所有回應的自定義 HTTP 標頭。

值型別string
help

在控制台中顯示此指令的幫助訊息。

值型別boolean
hmr

啟用熱模組替換。

值型別boolean
host

要監聽的地址。

值型別string預設值localhost
inspect

啟用除錯檢查器。此選項僅在啟用『SSR』或『SSG』時有效。

值型別string
live-reload

是否在更改時重新載入頁面，使用動態重新載入。

值型別boolean預設值true
open
別名o

在預設瀏覽器中開啟 URL。

值型別boolean預設值false
poll

啟用並定義檔案監視輪詢時間段，以毫秒為單位。

值型別number
port

要監聽的連接埠。

值型別number預設值4200
prebundle

啟用和控制基於 Vite 的開發伺服器的預捆綁功能。要啟用預捆綁，還必須啟用 Angular CLI 快取。當使用『browser』或其他基於 Webpack 的建構式時，此選項無效。

值型別boolean
proxy-config

代理設定檔案。欲知詳情，請參閱 https://angular.dev/tools/cli/serve#proxying-to-a-backend-server。

值型別string
public-host

瀏覽器用戶端（或動態重新載入用戶端，如果已啟用）應用於連線到開發伺服器的 URL。用於複雜的開發伺服器設定，例如具有反向代理的設定。當使用『application』或其他基於 esbuild 的建構式時，此選項無效。

值型別string
serve-path

應用將被伺服的路徑名。

值型別string
ssl

使用 HTTPS 伺服。

值型別boolean預設值false
ssl-cert

用於使用 HTTPS 伺服的 SSL 證書。

值型別string
ssl-key

用於使用 HTTPS 伺服的 SSL 金鑰。

值型別string
verbose

向輸出日誌新增更多詳細訊息。

值型別boolean
watch

更改時重新建立。

值型別boolean預設值true
