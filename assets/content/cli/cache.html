CLI

cache

Command
ng cache

Angular CLI 默认会将一些可缓存操作保存到磁盘。

当你重新运行相同的构建时，构建系统会恢复上一次构建的状态并复用之前执行的操作，从而减少构建和测试应用及库所需的时间。

要修改默认缓存设置，请将 cli.cache 对象添加到你的 工作区配置中。 该对象位于文件的顶层 cli.cache 下， projects 部分的外部。

          
            {  "$schema": "./node_modules/@angular/cli/lib/config/schema.json",  "version": 1,  "cli": {    "cache": {      // ...    },  },  "projects": {},}

欲了解更多信息，请参阅 缓存选项。

缓存环境

默认情况下，仅为本地环境启用磁盘缓存。环境的值可以是以下之一：

  • all - 允许在所有机器上使用磁盘缓存。
  • local - 仅允许在开发机器上使用磁盘缓存。
  • ci - 仅允许在持续集成（CI）系统上使用磁盘缓存。

要将环境设置更改为 all，运行以下命令：

          
            ng config cli.cache.environment all

欲了解更多信息，请参阅 缓存选项中的 environment。

Angular CLI 检查 CI 环境变量的存在与否及其值，以确定其运行的环境。

缓存路径

默认情况下， .angular/cache 作为存储缓存结果的基础目录。

要将此路径更改为 .cache/ng，请运行以下命令：

          
            ng config cli.cache.path ".cache/ng"

子命令

此命令有以下子命令

  • clean
  • disable
  • enable
  • info
