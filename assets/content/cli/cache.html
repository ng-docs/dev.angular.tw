CLI

cache

指令
ng cache

預設情況下，Angular CLI 會在磁碟上儲存許多可快取的操作。

當你重新執行相同的建立時，建立體系會恢復先前建立的狀態，並複用先前執行的操作，從而減少建立和測試你的應用和函式庫所需的時間。

要修改預設快取設定，請將 cli.cache 物件新增到你的工作區設定中。 該物件位於檔案頂層 cli.cache 下，在 projects 部分之外。

          
{  "$schema": "./node_modules/@angular/cli/lib/config/schema.json",  "version": 1,  "cli": {    "cache": {      // ...    },  },  "projects": {},}

欲知詳情，請參閱快取選項。

快取環境

預設情況下，磁碟快取僅對本地環境啟用。環境的值可以是以下之一：

  • all - 允許在所有機器上使用磁碟快取。
  • local - 僅允許在開發機器上使用磁碟快取。
  • ci - 僅允許在持續整合 (CI) 系統上使用磁碟快取。

要將環境設定更改為 all，請執行以下指令：

          
ng config cli.cache.environment all

欲知詳情，請參閱快取選項中的 environment。

Angular CLI 檢查 CI 環境變數的存在與否及其值，以確定它在哪個環境中執行。

快取路徑

預設情況下，.angular/cache 用作儲存快取結果的基本目錄。

要將此路徑更改為 .cache/ng，請執行以下指令：

          
ng config cli.cache.path ".cache/ng"

子指令

此指令具有以下子指令

  • clean
  • disable
  • enable
  • info

選項

help

在控制台中顯示此指令的幫助訊息。

值型別boolean
