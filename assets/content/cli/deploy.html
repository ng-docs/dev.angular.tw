CLI

deploy

指令
ng deploy

本指令接受一個可選的專案名稱，正如 angular.json 工作區設定檔案的 projects 部分所指定的。 當未提供專案名稱時，則為預設專案執行 deploy 建構式。

要使用 ng deploy 指令，請使用 ng add 新增一個軟體套件，以便為你的首選平台實現部署功能。 新增軟體套件會自動更新你的工作區設定，並新增一個部署CLI 建構式。 例如：

          
"projects": {  "my-project": {    ...    "architect": {      ...      "deploy": {        "builder": "@angular/fire:deploy",        "options": {}      }    }  }}

引數

project

要建立的專案的名稱。可以是一個應用或一個函式庫。

值型別string

選項

configuration
別名c

一個或多個命名建構式設定，以逗號分隔的清單形式，在 angular.json 中的 "configurations" 部分中指定。 建構式使用這些命名設定來執行給定的目標。 更多訊息，請參閱 https://angular.dev/reference/configs/workspace-config#alternate-build-configurations。

值型別string
help

在控制台中顯示此指令的幫助訊息。

值型別boolean
