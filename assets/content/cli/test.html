CLI

test

指令
ng test
ng t

接受專案名稱，該名稱在 angular.json 工作區設定檔案的 projects 部分中指定。 當未提供專案名稱時，它將為所有專案執行。

引數

project

要建立的專案的名稱。可以是應用或函式庫。

值型別string

選項

aot

使用預先編譯執行測試。

值型別boolean預設值false
browsers

覆蓋針對哪些瀏覽器執行測試。設定為 false 以不使用任何瀏覽器。

值型別string
builder-mode

確定如何建立被測程式碼。如果設定為『detect』，則嘗試遵循開發建構式。

值型別string預設值browser
code-coverage

輸出程式碼覆蓋率報告。

值型別boolean預設值false
code-coverage-exclude

從程式碼覆蓋率中排除的 Globs。

值型別array
configuration
別名c

一個或多個命名的建構式設定，以逗號分隔的清單形式在 angular.json 的 『configurations』 部分中指定。 建構式使用命名的設定來執行給定的目標。 更多訊息，請參閱 https://angular.dev/reference/configs/workspace-config#alternate-build-configurations。

值型別string
exclude

要排除的檔案 Globs，相對於專案根目錄。

值型別array
help

在控制台中顯示此指令的幫助訊息。

值型別boolean
include

要包含的檔案 Globs，相對於專案根目錄。 有 2 種特殊情況：

  • 當提供目錄路徑時，將包含所有以 『.spec.@(ts|tsx)』 結尾的 spec 檔案
  • 當提供檔案路徑，並且存在對應的 spec 檔案時，將改為包含該檔案。
值型別array
inline-style-language

用於應用的行內元件樣式的樣式表語言。

值型別string預設值css
karma-config

Karma 設定檔案的名稱。

值型別string
main

主入口點檔案的名稱。

值型別string
poll

啟用並定義檔案監視輪詢時間週期，以毫秒為單位。

值型別number
polyfills

要包含在建立中的膩子指令碼。

值型別string
preserve-symlinks

解析模組時不要使用真實路徑。如果未設定，則當 NodeJS 選項 --preserve-symlinks 設定時，將預設為 true。

值型別boolean
progress

在建立時將進度記錄到控制台。

值型別boolean預設值true
reporters

要使用的 Karma 報告器。直接傳遞給 karma 執行器。

值型別array
source-map

為指令碼和樣式輸出原始碼對映。更多訊息，請參閱 https://angular.dev/reference/configs/workspace-config#source-map-configuration。

值型別boolean預設值true
ts-config

TypeScript 設定檔案的名稱。

值型別string
watch

當檔案更改時運行建立。

值型別boolean
web-worker-ts-config

Web Worker 模組的 TypeScript 設定。

值型別string
