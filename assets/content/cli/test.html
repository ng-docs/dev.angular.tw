CLI

test

Command
ng test
ng t

以项目名称为参数，该名称在 angular.json 工作区配置文件的 projects 部分中指定。 如果不提供项目名称，则将针对所有项目执行。

参数

project

要构建的项目名称。可以是应用或库。

Value Typestring

选项

browsers

覆盖要运行测试的浏览器。设置为 false 表示不使用任何浏览器。

Value Typestring
code-coverage

输出代码覆盖率报告。

Value TypebooleanDefaultfalse
code-coverage-exclude

排除在代码覆盖率之外的文件。

Value Typearray
configuration
Aliasc

一个或多个命名构建器配置，以逗号分隔的列表形式，如 angular.json 中的「configurations」部分所指定。构建器使用命名配置来运行给定的目标。有关更多信息，请参阅 https://angular.dev/reference/configs/workspace-config#alternate-build-configurations。

Value Typestring
exclude

要排除的文件的文件路径模式，相对于项目根目录。

Value Typearray
help

在控制台显示关于本命令的帮助信息。

Value Typeboolean
include

要包含的文件的文件路径模式，相对于项目根目录。 有两种特殊情况：

  • 当提供目录路径时，将包括所有以 ".spec.@(ts|tsx)" 结尾的规范文件。
  • 当提供文件路径时，并且存在匹配的规范文件时，将包含该规范文件。
Value Typearray
inline-style-language

用于应用的内联组件样式的样式表语言。

Value TypestringDefaultcss
karma-config

Karma 配置文件的名称。

Value Typestring
main

主入口文件的名称。

Value Typestring
poll

启用并定义文件监视的轮询时间周期（以毫秒为单位）。

Value Typenumber
polyfills

要包含在构建中的腻子脚本。

Value Typestring
preserve-symlinks

在解析模块时不使用实际路径。如果未设置，则在 NodeJS 选项 --preserve-symlinks 设置时默认为 true。

Value Typeboolean
progress

在构建过程中将进度记录到控制台。

Value TypebooleanDefaulttrue
reporters

要使用的 Karma 报告者。直接传递给 karma 运行器。

Value Typearray
source-map

为脚本和样式输出源码映射。欲了解更多信息，请参见 https://angular.dev/reference/configs/workspace-config#source-map-configuration。

Value TypebooleanDefaulttrue
ts-config

TypeScript 配置文件的名称。

Value Typestring
watch

文件更改时运行构建。

Value Typeboolean
web-worker-ts-config

Web Worker 模块的 TypeScript 配置。

Value Typestring
