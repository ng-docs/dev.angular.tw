CLI

build

Command
ng build
ng b

此命令可用于构建类型为「应用」或「库」的项目。当用于构建库时，会调用不同的构建器，并且仅应用 ts-config、 configuration、 poll 和 watch 选项。所有其他选项仅适用于构建应用。

应用构建器使用 esbuild 构建工具，其默认配置选项在工作区配置文件（ angular.json）中指定，或使用命名的替代配置。当你使用 CLI 创建项目时，会默认创建一个「开发」配置，你可以通过指定 --configuration development 来使用该配置。

配置选项通常对应于命令选项。你可以通过在命令行中指定相应的选项来覆盖单个配置默认值。此命令可以接受以短横线分隔的选项名称。请注意，在配置文件中，你必须使用 camelCase 指定名称。

一些附加选项只能通过配置文件设置，可以直接编辑或使用 ng config 命令设置。这些选项包括 assets、 styles 和 scripts 对象，这些对象提供在项目中包含的全局运行时资源。CSS 中的资源（如图像和字体）会自动写入并在输出文件夹的根目录处进行指纹识别。

有关详细信息，请参阅 工作区配置。
