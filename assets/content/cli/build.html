CLI

build

指令
ng build
ng b

該指令可用於建立型別為『application』或『library』的專案。 當用於建立庫時，將呼叫不同的建構式，並且僅應用 ts-config、configuration、poll 和 watch 選項。 所有其他選項僅適用於建立應用。

應用建構式使用 esbuild 建立工具，預設設定選項在工作區設定檔案 (angular.json) 或命名的備用設定中指定。 當您使用 CLI 建立專案時，預設情況下會建立一個『development』設定，您可以透過指定 --configuration development 來使用該設定。

設定選項通常與指令選項相對應。 您可以透過在指令行上指定相應的選項來重寫各個設定預設值。 該指令可以接受以短劃線命名法給出的選項名稱。 請注意，在設定檔案中，您必須以駝峰命名法指定名稱。

某些其他選項只能透過設定檔案設定，可以透過直接編輯或使用 ng config 指令。 這些選項包括 assets、styles 和 scripts 物件，它們提供執行時全域資源以包含在專案中。 CSS 中的資源（例如影象和字型）會自動寫入輸出資料夾的根目錄並進行指紋標記。

有關更多詳細訊息，請參閱 工作區設定。
