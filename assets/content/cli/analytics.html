指令
ng analytics

你可以允許 Angular 團隊向 Google 傳送指令行工具的使用統計資料，以幫助他們確定屬性和改進的優先級。 除非你明確選擇加入，否則 Angular 團隊不會收集使用情況統計訊息。安裝 Angular CLI 時，系統會提示你是否允許全域收集使用情況統計訊息。 如果你選擇『否』或跳過提示，則不會收集任何資料。

收集哪些內容？

使用情況分析包括每次執行的指令和選定的標誌。 使用情況分析可能包括以下訊息：

  • 你的作業系統（macOS、Linux 發行版、Windows）及其版本。
  • 套件管理器名稱和版本（僅限本地版本）。
  • Node.js 版本（僅限本地版本）。
  • Angular CLI 版本（僅限本地版本）。
  • 執行的指令名稱。
  • 工作區訊息，應用和函式庫專案的數量。
  • 對於原理圖指令（add、generate 和 new），原理圖集合和名稱以及選定的標誌清單。
  • 對於建立指令（build、serve），建構式名稱、捆綁包（初始和延遲載入）的數量和大小、編譯單元、建立和重建所花費的時間以及基本的 Angular 特定 API 使用情況。

僅報告 Angular 擁有和開發的原理圖和建構式。 第三方原理圖和建構式不會向 Angular 團隊傳送資料。
