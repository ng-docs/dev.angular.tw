analytics

Command
ng analytics

你可以通过允许 Angular 团队将命令行命令使用统计数据发送给 Google，来帮助 Angular 团队优先处理特性和改进。 Angular 团队不会收集使用统计，除非你明确选择加入。在安装 Angular CLI 时，你会被提示是否允许全局收集使用统计。 如果你选择不允许或跳过提示，则不会收集任何数据。

收集哪些数据？

使用分析包括每次执行的命令和选定的标志。 使用分析可能包括以下信息：

  • 你的操作系统（macOS、Linux 发行版、Windows）及其版本。
  • 包管理器名称和版本（仅本地版本）。
  • Node.js 版本（仅本地版本）。
  • Angular CLI 版本（仅本地版本）。
  • 运行的命令名称。
  • 工作区信息，应用和库项目的数量。
  • 对于原理图命令（add、generate 和 new），原理图集合和名称以及选定标志的列表。
  • 对于构建命令（build、serve），构建器名称，捆绑包数量和大小（初始和延迟），编译单元，构建和重建所需时间，以及基本的 Angular 特定 API 使用情况。

仅报告 Angular 自有和开发的原理图和构建器。第三方原理图和构建器不会向 Angular 团队发送数据。
