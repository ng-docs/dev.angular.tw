Angular 依赖注入

edit

依赖注入（DI）是 Angular 中的基本概念之一。DI 被装配进 Angular 框架，并允许带有 Angular 装饰器的类（例如组件、指令、管道和可注入对象）配置它们所需的依赖项。

DI 系统中存在两个主要角色：依赖使用者和依赖提供者。

Angular 使用一个称为 Injector 的抽象来简化依赖消费者和依赖提供者之间的交互。当请求一个依赖时，注入器会检查其注册表中是否已有可用实例。如果没有，则创建一个新实例并存储在注册表中。Angular 在应用启动过程（也称为「根」注入器）中创建一个应用范围的注入器。在大多数情况下，你不需要手动创建注入器，但应该知道有一个连接提供者和消费者的层。

本主题涵盖了类作为依赖的基本场景。Angular 还允许你使用函数、对象、字符串或布尔值等原语类型，或任何其他类型作为依赖。有关更多信息，请参见 依赖提供者。

提供依赖

考虑一个名为 HeroService 的类，需要在组件中作为依赖。

第一步是添加 @Injectable 装饰器，以表明该类可以被注入。

      
@Injectable()
class HeroService {}

下一步是在 DI 中提供它，使其可用。 可以在多个位置提供依赖：

  • 首选：在应用根级别使用 providedIn。
  • 在组件级别。
  • 在应用根级别使用 ApplicationConfig。
  • 基于 NgModule 的应用。

首选：在应用的根级别使用 providedIn

在应用根级别使用 providedIn 提供服务可以将该服务注入到所有其他类中。 使用 providedIn 使 Angular 和 JavaScript 代码优化器能够有效移除未使用的服务（称为摇树优化）。

你可以在 @Injectable 装饰器中使用 providedIn: 'root' 来提供一个服务：

      
@Injectable({
  providedIn: 'root'
})
class HeroService {}

当你在根级别提供服务时，Angular 会创建一个 HeroService 的单一共享实例，并将其注入到任何请求它的类中。

在组件级别

你可以通过使用 @Component 装饰器的 providers 字段在 @Component 级别提供服务。在这种情况下， HeroService 可以被此组件的所有实例以及模板中使用的其他组件和指令访问。

比如：

      
@Component({
  standalone: true,
  selector: 'hero-list',
  template: '...',
  providers: [HeroService]
})
class HeroListComponent {}

当你在组件级别注册提供者时，该组件的每个新实例都会获得一个新的服务实例。

注意： 这样声明一个服务会导致 HeroService 始终包含在你的应用中——即使该服务未被使用。

在应用的根级别使用 ApplicationConfig

你可以使用 ApplicationConfig（传递给 bootstrapApplication 函数）的 providers 字段在应用级别提供服务或其他 Injectable。

在下面的示例中， HeroService 可用于所有组件、指令和管道：

      
export const appConfig: ApplicationConfig = {
    providers: [
      { provide: HeroService },
    ]
};

然后，在 main.ts 中：

      
bootstrapApplication(AppComponent, appConfig)

注意： 这样声明一个服务会导致 HeroService 始终包含在你的应用中——即使该服务未被使用。

基于 NgModule 的应用

基于 @NgModule 的应用使用 @NgModule 装饰器的 providers 字段在应用级别提供服务或其他 Injectable。

在模块中提供的服务对该模块的所有声明或共享相同 ModuleInjector 的任何其他模块可用。 要了解所有边界情况，请参阅 分层注入器。

注意： 使用 providers 声明服务会导致该服务包含在你的应用中——即使该服务未被使用。

注入/消费依赖

注入依赖项的最常见方法是在类的构造函数中声明它。当 Angular 创建组件、指令或管道类的新实例时，它会通过查看构造函数的参数类型来确定该类需要哪些服务或其他依赖项。例如，如果 HeroListComponent 要用 HeroService，则构造函数可以如下所示：

      
@Component({ … })
class HeroListComponent {
  constructor(private service: HeroService) {}
}

另一种选择是使用 inject 方法：

      
@Component({ … })
class HeroListComponent {
  private service = inject(HeroService);
}

当 Angular 发现一个组件依赖于一项服务时，它会首先检查注入器中是否已有该服务的任何现有实例。如果所请求的服务实例尚不存在，注入器就会使用注册的提供者创建一个，并在将服务返回给 Angular 之前将其添加到注入器中。

当所有请求的服务都已解析并返回时，Angular 就可以用这些服务实例为参数，调用该组件的构造函数。
