依賴注入的應用

edit

本指南探討了 Angular 中依賴注入的其他屬性。

使用 @Inject 的自定義提供者

使用自定義提供者，你可以為隱式依賴（例如內建瀏覽器 API）提供具體的實現。 以下範例使用 InjectionToken 在 BrowserStorageService 中把 localStorage 瀏覽器 API 作為依賴項來提供：

src/app/storage.service.ts

      
import { Inject, Injectable, InjectionToken } from '@angular/core';export const BROWSER_STORAGE = new InjectionToken<Storage>('Browser Storage', {  providedIn: 'root',  factory: () => localStorage});@Injectable({  providedIn: 'root'})export class BrowserStorageService {  public storage = inject(BROWSER_STORAGE);  get(key: string) {    return this.storage.getItem(key);  }  set(key: string, value: string) {    this.storage.setItem(key, value);  }}

factory 函式回傳附著在瀏覽器 window 物件上的 localStorage 屬性。 inject 函式使用令牌的實例初始化 storage 屬性。

現在，可以在測試期間使用 localStorage 的模擬 API 來重寫此自定義提供者，而無需與真實的瀏覽器 API 互動。

注入元件的 DOM 元素

儘管開發人員力求避免這種情況，但某些視覺效果和第三方工具還是需要直接訪問 DOM。 因此，你可能需要訪問元件的 DOM 元素。

Angular 透過使用 ElementRef 注入令牌進行注入，暴露 @Component 或 @Directive 的底層元素：

      
import { Directive, ElementRef } from '@angular/core';@Directive({  selector: '[appHighlight]'})export class HighlightDirective {  private element = inject(ElementRef)  update() {    this.element.nativeElement.style.color = 'red';  }}

使用 forward reference 求解迴圈依賴

類別的物件宣告順序在 TypeScript 中很重要。 在類被定義之前，你不能直接引用它。

這通常不是問題，特別是當你遵循推薦的每個檔案一個類規則時。 但有時迴圈引用是不可避免的。 例如，當類『A』引用類『B』，而『B』又引用『A』時，其中一個必須先定義。

Angular 的 forwardRef() 函式建立一個間接引用，Angular 可以在稍後對其求解。

當一個類引用自身時，你也會面臨類似的問題。 例如，在其 providers 陣列中。 providers 陣列是 @Component() 裝飾器函式的屬性，它必須出現在類別定義之前。 你可以透過使用 forwardRef 來打破這種迴圈引用。

app.component.ts

      
providers: [  {    provide: PARENT_MENU_ITEM,    useExisting: forwardRef(() => MenuItem),  },],
