元件宿主元素

edit

提示： 本指南假設你已經閱讀過《要點指南》。如果你是 Angular 新手，請先閱讀該指南。

對於每個與元件選擇器對應的 HTML 元素，Angular 都會建立該組件的一個實例。與元件選擇器對應的 DOM 元素就是該組件的宿主元素。元件範本的內容會在其宿主元素內部渲染。

      
// Component source@Component({  selector: 'profile-photo',  template: `    <img src="profile-photo.jpg" alt="Your profile photo" />  `,})export class ProfilePhoto {}

    
      
<!-- Using the component --><h3>Your profile photo</h3><profile-photo /><button>Upload a new profile photo</button>

    
      
<!-- Rendered DOM --><h3>Your profile photo</h3><profile-photo>  <img src="profile-photo.jpg" alt="Your profile photo" /></profile-photo><button>Upload a new profile photo</button>

在上面的範例中，<profile-photo> 是 ProfilePhoto 元件的宿主元素。

繫結到宿主元素

元件可以將其屬性（Property）、Attribute 和事件繫結到其宿主元素。這與元件範本內部元素的繫結行為相同，只不過宿主元素的繫結是在 @Component 裝飾器中使用 host 屬性定義的：

      
@Component({  ...,  host: {    'role': 'slider',    '[attr.aria-valuenow]': 'value',    '[class.active]': 'isActive()',    '[tabIndex]': 'disabled ? -1 : 0',    '(keydown)': 'updateValue($event)',  },})export class CustomSlider {  value: number = 0;  disabled: boolean = false;  isActive = signal(false);  updateValue(event: KeyboardEvent) { /* ... */ }  /* ... */}

@HostBinding 和 @HostListener 裝飾器

你也可以透過將 @HostBinding 和 @HostListener 裝飾器應用於類別成員來繫結到宿主元素。

@HostBinding 允許你將宿主屬性（Property）和 Attribute 繫結到屬性（property）和方法：

      
@Component({  /* ... */})export class CustomSlider {  @HostBinding('attr.aria-valuenow')  value: number = 0;  @HostBinding('tabIndex')  getTabIndex() {    return this.disabled ? -1 : 0;  }  /* ... */}

@HostListener 允許你將事件監聽器附著到宿主元素。該裝飾器接受一個事件名稱和一個可選的引數陣列：

      
export class CustomSlider {  @HostListener('keydown', ['$event'])  updateValue(event: KeyboardEvent) {    /* ... */  }}

始終優先使用 host 屬性而不是 @HostBinding 和 @HostListener。 這些裝飾器的存在僅僅是為了向後相容。

繫結衝突

當你在範本中使用元件時，你可以向該組件實例的元素新增繫結。元件也可以為相同的屬性（Property）或 Attribute 定義宿主繫結。

      
@Component({  ...,  host: {    'role': 'presentation',    '[id]': 'id',  }})export class ProfilePhoto { /* ... */ }

    
      
<profile-photo role="group" [id]="otherId" />

在這種情況下，以下規則決定哪個值生效：

  • 如果兩個值都是靜態的，則實例繫結生效。
  • 如果一個值是靜態的，另一個值是動態的，則動態值生效。
  • 如果兩個值都是動態的，則元件的宿主繫結生效。
