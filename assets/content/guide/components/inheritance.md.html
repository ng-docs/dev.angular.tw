繼承

edit

提示：本指南假設你已經閱讀過《要點指南》。如果你是 Angular 新手，請先閱讀該指南。

Angular 元件是 TypeScript 類，並遵循標準的 JavaScript 繼承語義。

元件可以擴充套件任何基底類別：

      
export class ListboxBase {  value: string;}@Component({ ... })export class CustomListbox extends ListboxBase {  // CustomListbox inherits the `value` property.}

擴充套件其他元件和指令

當元件擴充套件另一個元件或指令時，它會繼承基底類別的裝飾器和基底類別的帶裝飾器成員中定義的某些元資料。 這包括宿主繫結、輸入、輸出、生命週期方法。

      
@Component({  selector: 'base-listbox',  template: `    ...  `,  host: {    '(keydown)': 'handleKey($event)',  },})export class ListboxBase {  @Input() value: string;  handleKey(event: KeyboardEvent) {    /* ... */  }}@Component({  selector: 'custom-listbox',  template: `    ...  `,  host: {    '(click)': 'focusActiveOption()',  },})export class CustomListbox extends ListboxBase {  @Input() disabled = false;  focusActiveOption() {    /* ... */  }}

在上面的例子中，CustomListbox 繼承了與 ListboxBase 關聯的所有訊息，並使用自己的值重寫了選擇器和樣板。CustomListbox 有兩個輸入（value 和 disabled）和兩個事件監聽器（keydown 和 click）。

子類別最終會獲得其所有祖先的輸入、輸出和宿主繫結以及其自身的並集。

轉發注入的依賴

如果基底類別將依賴項作為建構式函式引數注入，則子類別必須顯式地將這些依賴項傳遞給 super。

      
@Component({ ... })export class ListboxBase {  constructor(private element: ElementRef) { }}@Component({ ... })export class CustomListbox extends ListboxBase {  constructor(element: ElementRef) {    super(element);  }}

重寫生命週期方法

如果基底類別定義了一個生命週期方法，例如 ngOnInit，那麼也實現了 ngOnInit 的子類別會重寫基底類別的實現。 如果你想保留基底類別的生命週期方法，請使用 super 顯式呼叫該方法：

      
@Component({ ... })export class ListboxBase {  protected isInitialized = false;  ngOnInit() {    this.isInitialized = true;  }}@Component({ ... })export class CustomListbox extends ListboxBase {  override ngOnInit() {    super.ngOnInit();    /* ... */  }}
