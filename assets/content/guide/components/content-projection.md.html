使用 ng-content 进行内容投影

edit

提示：本指南假定你已经阅读了 基本指南。如果你是 Angular 的新手，请先阅读该指南。

你经常需要创建组件来作为不同类型内容的容器。例如，你可能想要创建一个自定义的卡片组件：

      
@Component({
  selector: 'custom-card',
  template: '<div class="card-shadow"> <!-- card content goes here --> </div>',
})
export class CustomCard {/* ... */}

你可以使用 <ng-content> 元素作为占位符以标记内容应该放置的位置：

      
@Component({
  selector: 'custom-card',
  template: '<div class="card-shadow"> <ng-content></ng-content> </div>',
})
export class CustomCard {/* ... */}

提示： <ng-content> 与 原生 <slot> 元素类似，但具有一些特定于 Angular 的功能。

当你使用带有 <ng-content> 的组件时，该组件宿主元素的任何子元素都会在 <ng-content> 的位置被渲染，或 投影：

      
// Component source
@Component({
  selector: 'custom-card',
  template: `
    <div class="card-shadow">
      <ng-content />
    </div>
  `,
})
export class CustomCard {/* ... */}

    
      
<!-- Using the component -->
<custom-card>
  <p>This is the projected content</p>
</custom-card>

    
      
<!-- The rendered DOM -->
<custom-card>
  <div class="card-shadow">
    <p>This is the projected content</p>
  </div>
</custom-card>

Angular 将通过这种方式传递给组件的任何子元素称为该组件的 内容。这与组件的 视图（即模板中定义的元素）是不同的。

<ng-content> 元素既不是组件也不是 DOM 元素。相反，它是一个特殊的占位符，用于告诉 Angular 在哪里渲染内容。Angular 的编译器在构建时会处理所有 <ng-content> 元素。你不能在运行时插入、删除或修改 <ng-content>。你不能向 <ng-content> 添加 指令、样式或任意属性。

你不应该使用 ngIf、 ngFor 或 ngSwitch 有条件地包含 <ng-content>。有关组件内容的条件渲染，请参阅 模板片段。

多个内容占位符

Angular 支持基于 CSS 选择器将多个不同的元素投影到不同的 <ng-content> 占位符中。扩展上面的卡片示例，你可以使用 select 属性为卡片标题和卡片正文创建两个占位符：

      
<!-- Component template -->
<div class="card-shadow">
  <ng-content select="card-title"></ng-content>
  <div class="card-divider"></div>
  <ng-content select="card-body"></ng-content>
</div>

    
      
<!-- Using the component -->
<custom-card>
  <card-title>Hello</card-title>
  <card-body>Welcome to the example</card-body>
</custom-card>

    
      
<!-- Rendered DOM -->
<custom-card>
  <div class="card-shadow">
    <card-title>Hello</card-title>
    <div class="card-divider"></div>
    <card-body>Welcome to the example</card-body>
  </div>
</custom-card>

<ng-content> 占位符支持与 组件选择器相同的 CSS 选择器。

如果你包含一个或多个带有 select 属性的 <ng-content> 占位符和一个没有 select 属性的 <ng-content> 占位符，后者将捕获所有未匹配 select 属性的元素：

      
<!-- Component template -->
<div class="card-shadow">
  <ng-content select="card-title"></ng-content>
  <div class="card-divider"></div>
  <!-- capture anything except "card-title" -->
  <ng-content></ng-content>
</div>

    
      
<!-- Using the component -->
<custom-card>
  <card-title>Hello</card-title>
  <img src="..." />
  <p>Welcome to the example</p>
</custom-card>

    
      
<!-- Rendered DOM -->
<custom-card>
  <div class="card-shadow">
    <card-title>Hello</card-title>
    <div class="card-divider"></div>
    <img src="..." />
    <p>Welcome to the example></p>
  </div>
</custom-card>

如果一个组件不包含没有 select 属性的 <ng-content> 占位符，则任何不匹配该组件占位符的元素将不会被渲染到 DOM 中。

内容投影的别名

Angular 支持一个特殊属性 ngProjectAs，允许你在任何元素上指定一个 CSS 选择器。每当带有 ngProjectAs 的元素与 <ng-content> 占位符进行匹配时，Angular 将会对比 ngProjectAs 的值而不是元素本身的身份：

      
<!-- Component template -->
<div class="card-shadow">
  <ng-content select="card-title"></ng-content>
  <div class="card-divider"></div>
  <ng-content></ng-content>
</div>

    
      
<!-- Using the component -->
<custom-card>
  <h3 ngProjectAs="card-title">Hello</h3>
  <p>Welcome to the example</p>
</custom-card>

    
      
<!-- Rendered DOM -->
<custom-card>
  <div class="card-shadow">
    <h3>Hello</h3>
    <div class="card-divider"></div>
    <p>Welcome to the example></p>
  </div>
</custom-card>

ngProjectAs 仅支持静态值，不能绑定到动态表达式。
