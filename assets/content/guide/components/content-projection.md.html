使用 ng-content 的內容投影

edit

提示： 本指南假定你已閱讀過要點指南。如果你是 Angular 新手，請先閱讀它。

你經常需要建立一些元件，充當不同型別內容的容器。例如，你可能想要建立一個自定義卡片元件：

      
@Component({  selector: 'custom-card',  template: '<div class="card-shadow"> <!-- card content goes here --> </div>',})export class CustomCard {/* ... */}

你可以使用 <ng-content> 元素作為佔位符來標記內容應該放置的位置：

      
@Component({  selector: 'custom-card',  template: '<div class="card-shadow"> <ng-content></ng-content> </div>',})export class CustomCard {/* ... */}

提示： <ng-content> 的工作方式類似於原生 <slot> 元素，但具有一些 Angular 特有的功能。

當你使用帶有 <ng-content> 的元件時，元件宿主元素的任何子元素都會在 <ng-content> 的位置被渲染或投影到那裡：

      
// Component source@Component({  selector: 'custom-card',  template: `    <div class="card-shadow">      <ng-content />    </div>  `,})export class CustomCard {/* ... */}

    
      
<!-- Using the component --><custom-card>  <p>This is the projected content</p></custom-card>

    
      
<!-- The rendered DOM --><custom-card>  <div class="card-shadow">    <p>This is the projected content</p>  </div></custom-card>

Angular 將以這種方式傳遞的元件的任何子元素都稱為該組件的內容。這與元件的檢視截然不同，後者指的是元件樣板中定義的元素。

<ng-content> 元素既不是元件也不是 DOM 元素。相反，它是一個特殊的佔位符，告訴 Angular 在哪裡渲染內容。Angular 的編譯器在建立時處理所有 <ng-content> 元素。你不能在執行時插入、刪除或修改 <ng-content>。你不能向 <ng-content> 新增指令、樣式或任意屬性。

你不應該有條件地將 <ng-content> 與 @if、@for 或 @switch 一起使用。即使 <ng-content> 佔位符是隱藏的，Angular 始終會為渲染到 <ng-content> 佔位符的內容實例化並建立 DOM 節點。對於元件內容的條件渲染，請參閱樣板片段。

多個內容佔位符

Angular 支援基於 CSS 選擇器將多個不同的元素投影到不同的 <ng-content> 佔位符中。擴充套件上面的卡片範例，你可以透過使用 select 屬性為卡片標題和卡片主體建立兩個佔位符：

      
<!-- Component template --><div class="card-shadow">  <ng-content select="card-title"></ng-content>  <div class="card-divider"></div>  <ng-content select="card-body"></ng-content></div>

    
      
<!-- Using the component --><custom-card>  <card-title>Hello</card-title>  <card-body>Welcome to the example</card-body></custom-card>

    
      
<!-- Rendered DOM --><custom-card>  <div class="card-shadow">    <card-title>Hello</card-title>    <div class="card-divider"></div>    <card-body>Welcome to the example</card-body>  </div></custom-card>

<ng-content> 佔位符支援與元件選擇器相同的 CSS 選擇器。

如果你包含一個或多個帶有 select 屬性的 <ng-content> 佔位符，以及一個沒有 select 屬性的 <ng-content> 佔位符，則後者會捕獲所有不對應 select 屬性的元素：

      
<!-- Component template --><div class="card-shadow">  <ng-content select="card-title"></ng-content>  <div class="card-divider"></div>  <!-- capture anything except "card-title" -->  <ng-content></ng-content></div>

    
      
<!-- Using the component --><custom-card>  <card-title>Hello</card-title>  <img src="..." />  <p>Welcome to the example</p></custom-card>

    
      
<!-- Rendered DOM --><custom-card>  <div class="card-shadow">    <card-title>Hello</card-title>    <div class="card-divider"></div>    <img src="..." />    <p>Welcome to the example</p>  </div></custom-card>

如果元件不包含沒有 select 屬性的 <ng-content> 佔位符，則任何不對應元件佔位符的元素都不會渲染到 DOM 中。

回退內容

如果元件沒有任何對應的子內容，Angular 可以為元件的 <ng-content> 佔位符顯示回退內容。你可以透過向 <ng-content> 元素本身新增子內容來指定回退內容。

      
<!-- Component template --><div class="card-shadow">  <ng-content select="card-title">Default Title</ng-content>  <div class="card-divider"></div>  <ng-content select="card-body">Default Body</ng-content></div>

    
      
<!-- Using the component --><custom-card>  <card-title>Hello</card-title>  <!-- No card-body provided --></custom-card>

    
      
<!-- Rendered DOM --><custom-card>  <div class="card-shadow">    Hello    <div class="card-divider"></div>    Default Body  </div></custom-card>

為投影設定內容別名

Angular 支援一個特殊的屬性 ngProjectAs，它允許你在任何元素上指定 CSS 選擇器。每當檢查帶有 ngProjectAs 的元素是否與 <ng-content> 佔位符對應時，Angular 會將 ngProjectAs 值與元素標識進行比較，而不是元素本身的標識：

      
<!-- Component template --><div class="card-shadow">  <ng-content select="card-title"></ng-content>  <div class="card-divider"></div>  <ng-content></ng-content></div>

    
      
<!-- Using the component --><custom-card>  <h3 ngProjectAs="card-title">Hello</h3>  <p>Welcome to the example</p></custom-card>

    
      
<!-- Rendered DOM --><custom-card>  <div class="card-shadow">    <h3>Hello</h3>    <div class="card-divider"></div>    <p>Welcome to the example</p>  </div></custom-card>

ngProjectAs 僅支援靜態值，不能繫結到動態表示式。
