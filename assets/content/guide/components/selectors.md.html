元件選擇器

edit

提示： 本指南假定你已閱讀過要點指南。 如果你是 Angular 新手，請先閱讀它。

每個元件都定義了一個 CSS 選擇器，它決定了元件的使用方式：

      
@Component({  selector: 'profile-photo',  ...})export class ProfilePhoto { }

你可以透過在其他元件的範本中建立一個對應的 HTML 元素來使用元件：

      
@Component({  template: `    <profile-photo />    <button>Upload a new profile photo</button>`,  ...,})export class UserProfile { }

Angular 在編譯時靜態地對應選擇器。 在執行時更改 DOM（透過 Angular 繫結或使用 DOM API）不會影響渲染的元件。

一個元素只能精確對應一個元件選擇器。 如果多個元件選擇器對應單個元素，Angular 將報告錯誤。

元件選擇器區分大小寫。

選擇器的型別

在元件選擇器中，Angular 支援有限的 基本 CSS 選擇器型別的子集：

選擇器型別 描述 範例
型別選擇器 根據元素的 HTML 標籤名或節點名進行對應。 profile-photo
屬性選擇器 根據 HTML 屬性的存在與否以及（可選）該屬性的精確值來對應元素。 [dropzone] [type="reset"]
類別選擇器 根據 CSS 類別的存在與否來對應元素。 .menu-item

對於屬性值，Angular 支援使用等於 (=) 運算子對應精確的屬性值。 Angular 不支援其他屬性值運算子。

Angular 元件選擇器不支援組合器，包括 後代組合器 或 子組合器。

Angular 元件選擇器不支援指定 命名空間。

:not 偽類別

Angular 支援 :not 偽類別。 你可以將此偽類別附加到任何其他選擇器，以縮小元件的選擇器對應的元素範圍。 例如，你可以定義一個 [dropzone] 屬性選擇器，並阻止對應 textarea 元素：

      
@Component({  selector: '[dropzone]:not(textarea)',  ...})export class DropZone { }

Angular 在元件選擇器中不支援任何其他偽類別或偽元素。

組合選擇器

你可以透過連線多個選擇器來組合它們。 例如，你可以對應指定了 type="reset" 的 <button> 元素：

      
@Component({  selector: 'button[type="reset"]',  ...})export class ResetButton { }

你還可以使用逗號分隔的清單定義多個選擇器：

      
@Component({  selector: 'drop-zone, [dropzone]',  ...})export class DropZone { }

對於清單中與任何選擇器對應的每個元素，Angular 都會建立一個元件。

選擇選擇器

絕大多數元件都應使用自定義元素名稱作為其選擇器。 所有自定義元素名稱都應包含連字元，如 HTML 規範中所述。 預設情況下，如果 Angular 遇到與任何可用元件都不對應的自定義標籤名稱，則會報告錯誤，從而防止因元件名稱拼寫錯誤而導致的錯誤。

有關在 Angular 範本中使用 原生自定義元素的詳細訊息，請參閱高階元件設定。

選擇器字首

Angular 團隊建議為專案中定義的所有自定義元件使用簡短且一致的字首。 例如，如果你要使用 Angular 建立 YouTube，則可以將元件字首設定為 yt-，元件如 yt-menu、yt-player 等。 像這樣對選擇器進行命名空間劃分可以立即清楚地表明特定元件的來源。 預設情況下，Angular CLI 使用 app-。

Angular 將 ng 選擇器字首用於其自己的框架 API。 永遠不要將 ng 用作你自己的自定義元件的選擇器字首。

何時使用屬性選擇器

當你想在標準原生元素上建立元件時，應該考慮使用屬性選擇器。 例如，如果你想建立一個自定義按鈕元件，則可以透過使用屬性選擇器來利用標準 <button> 元素：

      
@Component({  selector: 'button[yt-upload]',   ...})export class YouTubeUploadButton { }

這種方法允許元件的使用者直接使用元素的所有標準 API，而無需額外的工作。 這對於 ARIA 屬性（如 aria-label）尤其有價值。

當 Angular 遇到與可用元件不對應的自定義屬性時，不會報告錯誤。 當將元件與屬性選擇器一起使用時，使用者可能會忘記匯入元件或其 NgModule，從而導致元件無法渲染。 有關更多訊息，請參閱匯入和使用元件。

定義屬性選擇器的元件應使用小寫的、短劃線分隔的屬性。 你可以遵循上面描述的相同字首建議。
