複雜動畫序列

edit

到目前為止，我們學習了單個 HTML 元素的簡單動畫。 Angular 還允許你為協調的序列製作動畫，例如當元素進入和離開頁面時，為整個網格或元素清單製作動畫。 你可以選擇並行執行多個動畫，或者順序執行離散動畫，一個接一個地執行。

控制複雜動畫序列的函式包括：

函式 詳情
query() 查詢一個或多個內部 HTML 元素。
stagger() 將級聯延遲應用於多個元素的動畫。
group() 並行執行多個動畫步驟。
sequence() 一個接一個地執行動畫步驟。

query() 函式

大多數複雜動畫都依賴 query() 函式來查詢子元素並對其應用動畫，此類基本範例包括：

範例 詳情
先是 query()，然後是 animate() 用於查詢簡單的 HTML 元素並直接對其應用動畫。
先是 query()，然後是 animateChild() 用於查詢子元素，這些子元素本身已應用了動畫元資料並觸發此類動畫（否則將被當前/父元素的動畫阻止）。

query() 的第一個引數是一個 css 選擇器字串，它還可以包含以下 Angular 特定的標記：

標記 詳情
:enter
:leave		 用於進入/離開的元素。
:animating 用於當前正在製作動畫的元素。
@*
@triggerName		 用於具有任何觸發器或特定觸發器的元素。
:self 正在製作動畫的元素本身。

進入和離開元素

並非所有子元素實際上都被視為進入/離開；有時，這可能違反直覺並且令人困惑。請參閱 query api 文件以獲取更多訊息。

你還可以在動畫範例（在動畫簡介部分中介紹）的『查詢』頁籤下看到對此的闡釋。

使用 query() 和 stagger() 函式為多個元素製作動畫

透過 query() 查詢子元素後，stagger() 函式允許你定義每個被查詢的條目（將要製作動畫）之間的時序間隔，從而在元素之間以延遲的方式製作動畫。

以下範例示範了如何使用 query() 和 stagger() 函式為清單（英雄清單）製作動畫，方法是從上到下按順序新增每個英雄，並略有延遲。

  • 使用 query() 查詢符合特定條件的進入頁面的元素

  • 對於每個此類元素，使用 style() 為該元素設定相同的初始樣式。 使其透明，並使用 transform 將其移出位置，以便它可以滑動到位。

  • 使用 stagger() 將每個動畫延遲 30 毫秒

  • 使用自定義定義的緩動曲線為螢幕上的每個元素製作 0.5 秒的動畫，同時使其淡入並取消變換

src/app/hero-list-page.component.ts

      
import {Component, HostBinding, OnInit} from '@angular/core';import {trigger, transition, animate, style, query, stagger} from '@angular/animations';import {HEROES} from './mock-heroes';import {Hero} from './hero';import {NgFor} from '@angular/common';@Component({  imports: [NgFor],  selector: 'app-hero-list-page',  templateUrl: 'hero-list-page.component.html',  styleUrls: ['hero-list-page.component.css'],  animations: [    trigger('pageAnimations', [      transition(':enter', [        query('.hero', [          style({opacity: 0, transform: 'translateY(-100px)'}),          stagger(30, [            animate('500ms cubic-bezier(0.35, 0, 0.25, 1)', style({opacity: 1, transform: 'none'})),          ]),        ]),      ]),    ]),    trigger('filterAnimation', [      transition(':enter, * => 0, * => -1', []),      transition(':increment', [        query(          ':enter',          [            style({opacity: 0, width: 0}),            stagger(50, [animate('300ms ease-out', style({opacity: 1, width: '*'}))]),          ],          {optional: true},        ),      ]),      transition(':decrement', [        query(':leave', [stagger(50, [animate('300ms ease-out', style({opacity: 0, width: 0}))])]),      ]),    ]),  ],})export class HeroListPageComponent implements OnInit {  @HostBinding('@pageAnimations')  public animatePage = true;  heroesTotal = -1;  get heroes() {    return this._heroes;  }  private _heroes: Hero[] = [];  ngOnInit() {    this._heroes = HEROES;  }  updateCriteria(criteria: string) {    criteria = criteria ? criteria.trim() : '';    this._heroes = HEROES.filter((hero) =>      hero.name.toLowerCase().includes(criteria.toLowerCase()),    );    const newTotal = this.heroes.length;    if (this.heroesTotal !== newTotal) {      this.heroesTotal = newTotal;    } else if (!criteria) {      this.heroesTotal = -1;    }  }}

使用 group() 函式的並行動畫

你已經瞭解瞭如何在每個連續動畫之間新增延遲。 但是，你可能還希望設定並行發生的動畫。 例如，你可能想要為同一元素的兩個 CSS 屬性製作動畫，但每個屬性使用不同的 easing 函式。 為此，你可以使用動畫 group() 函式。

提示： group() 函式用於對動畫步驟進行分組，而不是對動畫元素進行分組。

以下範例在 :enter 和 :leave 上都使用了 group()，用於兩種不同的時序設定，從而將兩個獨立的動畫並行應用於同一元素。

src/app/hero-list-groups.component.ts（節選）

      
import {Component, Input, Output, EventEmitter} from '@angular/core';import {trigger, state, style, animate, transition, group} from '@angular/animations';import {Hero} from './hero';import {NgFor} from '@angular/common';@Component({  selector: 'app-hero-list-groups',  template: `    <ul class="heroes">      @for (hero of heroes; track hero) {        <li [@flyInOut]="'in'">          <button class="inner" type="button" (click)="removeHero(hero.id)">            <span class="badge">{{ hero.id }}</span>            <span class="name">{{ hero.name }}</span>          </button>        </li>      }    </ul>  `,  styleUrls: ['./hero-list-page.component.css'],  imports: [NgFor],  animations: [    trigger('flyInOut', [      state(        'in',        style({          width: '*',          transform: 'translateX(0)',          opacity: 1,        }),      ),      transition(':enter', [        style({width: 10, transform: 'translateX(50px)', opacity: 0}),        group([          animate(            '0.3s 0.1s ease',            style({              transform: 'translateX(0)',              width: '*',            }),          ),          animate(            '0.3s ease',            style({              opacity: 1,            }),          ),        ]),      ]),      transition(':leave', [        group([          animate(            '0.3s ease',            style({              transform: 'translateX(50px)',              width: 10,            }),          ),          animate(            '0.3s 0.2s ease',            style({              opacity: 0,            }),          ),        ]),      ]),    ]),  ],})export class HeroListGroupsComponent {  @Input() heroes: Hero[] = [];  @Output() remove = new EventEmitter<number>();  removeHero(id: number) {    this.remove.emit(id);  }}

順序動畫與並行動畫

複雜動畫可以同時發生很多事情。 但是，如果你想建立一個動畫，其中涉及多個動畫一個接一個地發生，該怎麼辦？之前你使用了 group() 來同時並行執行多個動畫。

第二個名為 sequence() 的函式使你可以一個接一個地執行這些相同的動畫。 在 sequence() 中，動畫步驟由 style() 或 animate() 函式呼叫組成。

  • 使用 style() 立即應用提供的樣式資料。
  • 使用 animate() 在給定的時間間隔內應用樣式資料。

篩選器動畫範例

看看範例頁面上的另一個動畫。 在『篩選器/交錯』頁籤下，在搜尋英雄文字框中輸入一些文字，例如 Magnet 或 tornado。

篩選器在你鍵入時實時工作。 當你鍵入每個新字母時，元素會離開頁面，並且篩選器會逐漸變得更嚴格。 當你刪除篩選器框中的每個字母時，英雄清單會逐漸重新進入頁面。

HTML 樣板包含一個名為 filterAnimation 的觸發器。

src/app/hero-list-page.component.html

      
<h2>Filter/Stagger</h2><label for="search">Search heroes: </label><input type="text" id="search" #criteria       (input)="updateCriteria(criteria.value)"       placeholder="Search heroes"><ul class="heroes" [@filterAnimation]="heroesTotal">  @for (hero of heroes; track hero) {    <li class="hero">      <div class="inner">        <span class="badge">{{ hero.id }}</span>        <span class="name">{{ hero.name }}</span>      </div>    </li>  }</ul>

元件裝飾器中的 filterAnimation 包含三個轉場。

src/app/hero-list-page.component.ts

      
import {Component, HostBinding, OnInit} from '@angular/core';import {trigger, transition, animate, style, query, stagger} from '@angular/animations';import {HEROES} from './mock-heroes';import {Hero} from './hero';import {NgFor} from '@angular/common';@Component({  imports: [NgFor],  selector: 'app-hero-list-page',  templateUrl: 'hero-list-page.component.html',  styleUrls: ['hero-list-page.component.css'],  animations: [    trigger('pageAnimations', [      transition(':enter', [        query('.hero', [          style({opacity: 0, transform: 'translateY(-100px)'}),          stagger(30, [            animate('500ms cubic-bezier(0.35, 0, 0.25, 1)', style({opacity: 1, transform: 'none'})),          ]),        ]),      ]),    ]),    trigger('filterAnimation', [      transition(':enter, * => 0, * => -1', []),      transition(':increment', [        query(          ':enter',          [            style({opacity: 0, width: 0}),            stagger(50, [animate('300ms ease-out', style({opacity: 1, width: '*'}))]),          ],          {optional: true},        ),      ]),      transition(':decrement', [        query(':leave', [stagger(50, [animate('300ms ease-out', style({opacity: 0, width: 0}))])]),      ]),    ]),  ],})export class HeroListPageComponent implements OnInit {  @HostBinding('@pageAnimations')  public animatePage = true;  heroesTotal = -1;  get heroes() {    return this._heroes;  }  private _heroes: Hero[] = [];  ngOnInit() {    this._heroes = HEROES;  }  updateCriteria(criteria: string) {    criteria = criteria ? criteria.trim() : '';    this._heroes = HEROES.filter((hero) =>      hero.name.toLowerCase().includes(criteria.toLowerCase()),    );    const newTotal = this.heroes.length;    if (this.heroesTotal !== newTotal) {      this.heroesTotal = newTotal;    } else if (!criteria) {      this.heroesTotal = -1;    }  }}

此範例中的程式碼執行以下任務：

  • 當用戶首次開啟或導向到此頁面時跳過動畫（篩選器動畫會窄化已存在的內容，因此它僅適用於 DOM 中已存在的元素）
  • 根據搜尋輸入的值篩選英雄

對於每次更改：

  • 透過將其不透明度和寬度設定為 0 來隱藏離開 DOM 的元素

  • 在 300 毫秒內，為一個進入 DOM 的元素新增動畫效果。 在動畫期間，該元素呈現其預設寬度和不透明度。

  • 如果有多個元素進入或離開 DOM，則會錯開每個動畫，從頁面頂部開始，每個元素之間延遲 50 毫秒

為重新排序的清單的條目新增動畫效果

儘管 Angular 可以開箱即用地為 *ngFor 清單條目正確地新增動畫效果，但如果它們的順序發生變化，它將無法做到這一點。 這是因為它會失去對哪個元素是哪個元素的跟蹤，從而導致動畫中斷。 幫助 Angular 跟蹤此類元素的唯一方法是將 TrackByFunction 賦值給 NgForOf 指令。 這確保了 Angular 始終知道哪個元素是哪個元素，從而使其始終能夠將正確的動畫應用於正確的元素。

重要提示： 如果你需要為 *ngFor 清單的條目新增動畫效果，並且這些條目的順序有可能在執行時發生變化，請始終使用 TrackByFunction。

動畫和元件檢視封裝

Angular 動畫基於元件的 DOM 結構，並且不直接考慮 檢視封裝，這意味著使用 ViewEncapsulation.Emulated 的元件的行為方式與它們使用 ViewEncapsulation.None 完全相同（ViewEncapsulation.ShadowDom 的行為方式有所不同，我們稍後會討論）。

例如，如果 query() 函式（你將在本動畫指南的其餘部分看到更多相關內容）要應用於使用模擬檢視封裝的元件樹的頂部，則此類查詢將能夠識別（並因此為）樹中任何深度的 DOM 元素新增動畫效果。

另一方面，ViewEncapsulation.ShadowDom 透過將 DOM 元素『隱藏』在 ShadowRoot 元素內部來更改元件的 DOM 結構。這種 DOM 操作確實會阻止某些動畫實現正常工作，因為它依賴於簡單的 DOM 結構，並且沒有考慮 ShadowRoot 元素。因此，建議避免將動畫應用於包含使用 ShadowDom 檢視封裝的元件的檢視。

動畫序列摘要

用於為多個元素新增動畫效果的 Angular 函式以 query() 開頭，以查詢內部元素；例如，收集 <div> 中的所有影象。 其餘函式 stagger()、group() 和 sequence() 應用級聯或讓你控制如何應用多個動畫步驟。
