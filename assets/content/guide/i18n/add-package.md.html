添加 localize 包

edit

要利用 Angular 的本地化特性，请使用 Angular CLI 将 @angular/localize 包添加到你的项目中。

要添加 @angular/localize 包，请使用以下命令更新项目中的 package.json 和 TypeScript 配置文件。

      
ng add @angular/localize
ng extract-i18n
ng extract-i18n --output-path src/locale
ng extract-i18n --format=xlf
ng extract-i18n --format=xlf2
ng extract-i18n --format=xmb
ng extract-i18n --format=json
ng extract-i18n --format=arb
ng extract-i18n --out-file source.xlf
ng build --localize
ng serve --configuration=fr
ng build --configuration=production,fr

它在 TypeScript 配置文件中添加 types: ["@angular/localize"]，并在 main.ts 文件顶部添加对 @angular/localize 类型定义的引用。

有帮助的: 有关 package.json 和 tsconfig.json 文件的更多信息，请参阅 工作区 npm 依赖项 和 TypeScript 配置。

如果没有安装 @angular/localize，并且你尝试构建项目的本地化版本（例如，在模板中使用 i18n 属性）， Angular CLI 会生成一个错误，其中会包含启用项目 i18n 的步骤。

选项

OPTION DESCRIPTION 值类型 默认值
--project 项目的名称。 string
--use-at-runtime 如果设置了，那么 $localize 可以在运行时使用。同时 @angular/localize 会被包含在 package.json 的 dependencies 部分中，而不是默认的 devDependencies。 boolean false

有关更多可用选项，请参阅 ng add 在 Angular CLI 中的说明。
