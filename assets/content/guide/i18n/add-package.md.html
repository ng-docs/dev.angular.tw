新增 localize 套件

edit

要利用 Angular 的本地化屬性，請使用 Angular CLI 將 @angular/localize 套件新增到你的專案中。

要新增 @angular/localize 套件，請使用以下指令來更新專案中的 package.json 和 TypeScript 設定檔案。

      
ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

它會在 TypeScript 設定檔案中新增 types: ["@angular/localize"]。 它還會在 main.ts 檔案的頂部新增 /// <reference types="@angular/localize" /> 行，這是對型別定義的引用。

提示： 有關 package.json 和 tsconfig.json 檔案的更多訊息，請參閱 Workspace npm 依賴 和 TypeScript 設定。要了解有關三斜線指令的更多訊息，請訪問 Typescript Handbook。

如果未安裝 @angular/localize 並且你嘗試建立專案的本地化版本（例如，當在樣板中使用 i18n 屬性時），Angular CLI 將生成一個錯誤，其中包含你可以採取的步驟，以便為你的專案啟用 i18n。

選項

OPTION DESCRIPTION 值型別 預設值
--project 專案的名稱。 string
--use-at-runtime 如果設定，則可以在執行時使用 $localize。此外，@angular/localize 會包含在 package.json 的 dependencies 部分中，而不是預設的 devDependencies 中。 boolean false

有關更多可用選項，請參閱 Angular CLI 中的 ng add。
