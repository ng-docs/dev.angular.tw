使用自定義 ID 管理標記文字

edit

對於以下每種情況，Angular 提取器都會生成一個包含翻譯單元條目的檔案。

  • 元件範本中的每個 i18n 屬性
  • 元件程式碼中每個用 $localize 標記的訊息字串

正如含義如何控制文字提取和合並中所述，Angular 會為每個翻譯單元賦予一個唯一的 ID。

以下範例展示了帶有唯一 ID 的翻譯單元。

當你更改可翻譯文字時，提取器會為該翻譯單元生成一個新的 ID。 在大多數情況下，源文字的更改也需要更改翻譯。 因此，使用新的 ID 可以使文字更改與翻譯保持同步。

但是，某些翻譯系統對 ID 的格式或語法有特定要求。 為了滿足此要求，可以使用自定義 ID 來標記文字。 大多數開發人員不需要使用自定義 ID。 如果你想使用獨特的語法來傳達額外的元資料，請使用自定義 ID。 額外的元資料可能包括文字出現位置的應用的函式庫、元件或區域。

要在 i18n 屬性或 $localize 標記的訊息字串中指定自定義 ID，請使用 @@ 字首。 以下範例在標題元素中定義了 introductionHeader 自定義 ID。

app/app.component.html

      
<h1>Hello i18n!</h1><h1 i18n>Hello i18n!</h1><h1 i18n="An introduction header for this sample">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample">Hello i18n!</h1><h1 i18n="An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="@@introductionHeader">Hello i18n!</h1><img [src]="logo" title="Angular logo" alt="Angular logo"><h3 i18n="@@myId">Hello</h3><!-- ... --><p i18n="@@myId">Good bye</p>

以下範例為一個變數定義了 introductionHeader 自定義 ID。

      
variableText1 = $localize`:@@introductionHeader:Hello i18n!`;

當你指定自定義 ID 時，提取器會生成一個帶有該自定義 ID 的翻譯單元。

如果更改文字，提取器不會更改 ID。 因此，你不必採取額外的步驟來更新翻譯。 使用自定義 ID 的缺點是，如果更改文字，你的翻譯可能與新更改的源文字不同步。

將自定義 ID 與描述一起使用

將自定義 ID 與描述和含義結合使用，以進一步幫助翻譯人員。

以下範例包含描述，後跟自定義 ID。

app/app.component.html

      
<h1>Hello i18n!</h1><h1 i18n>Hello i18n!</h1><h1 i18n="An introduction header for this sample">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample">Hello i18n!</h1><h1 i18n="An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="@@introductionHeader">Hello i18n!</h1><img [src]="logo" title="Angular logo" alt="Angular logo"><h3 i18n="@@myId">Hello</h3><!-- ... --><p i18n="@@myId">Good bye</p>

以下範例為一個變數定義了 introductionHeader 自定義 ID 和描述。

      
variableText2 = $localize`:An introduction header for this sample@@introductionHeader:Hello i18n!`;

以下範例新增了一個含義。

variableText3 = $localize`:site header|An introduction header for this sample@@introductionHeader:Hello i18n!`;

定義唯一的自定義 ID

請務必定義唯一的自定義 ID。 如果對兩個不同的文字元素使用相同的 ID，則提取工具只會提取第一個，並且 Angular 會使用該翻譯來代替這兩個原始文字元素。

例如，在以下程式碼片段中，為兩個不同的文字元素定義了相同的 myId 自定義 ID。

app/app.component.html

      
<h1>Hello i18n!</h1><h1 i18n>Hello i18n!</h1><h1 i18n="An introduction header for this sample">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample">Hello i18n!</h1><h1 i18n="An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="@@introductionHeader">Hello i18n!</h1><img [src]="logo" title="Angular logo" alt="Angular logo"><h3 i18n="@@myId">Hello</h3><!-- ... --><p i18n="@@myId">Good bye</p>
