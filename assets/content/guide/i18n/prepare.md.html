準備要翻譯的元件

edit

要準備你的專案以進行翻譯，請完成以下操作。

  • 使用 i18n 屬性標記元件範本中的文字
  • 使用 i18n- 屬性標記元件範本中的屬性文字字串
  • 使用 $localize 標記的訊息字串來標記元件程式碼中的文字字串

標記元件範本中的文字

在元件範本中，i18n 元資料是 i18n 屬性的值。

      
<element i18n="{i18n_metadata}">{string_to_translate}</element>

使用 i18n 屬性在元件範本中標記靜態文字訊息以進行翻譯。 將其放在每個包含要翻譯的固定文字的元素標籤上。

提示： i18n 屬性是 Angular 工具和編譯器可識別的自定義屬性。

i18n 範例

以下 <h1> 標籤顯示了一個簡單的英語問候語『Hello i18n!』。

src/app/app.component.html

      
<h1>Hello i18n!</h1><h1 i18n>Hello i18n!</h1><h1 i18n="An introduction header for this sample">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample">Hello i18n!</h1><h1 i18n="An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="@@introductionHeader">Hello i18n!</h1><img [src]="logo" title="Angular logo" alt="Angular logo"><h3 i18n="@@myId">Hello</h3><!-- ... --><p i18n="@@myId">Good bye</p>

要標記問候語以進行翻譯，請將 i18n 屬性新增到 <h1> 標籤。

src/app/app.component.ts

      
import {Component, computed, signal} from '@angular/core';import {$localize} from '@angular/localize/init';@Component({  selector: 'app-root',  templateUrl: './app.component.html',})export class AppComponent {  minutes = 0;  gender = 'female';  fly = true;  logo = '${this.baseUrl}/angular.svg';  toggle = signal(false);  toggleAriaLabel = computed(() => {    return this.toggle()      ? $localize`:Toggle Button|A button to toggle status:Show`      : $localize`:Toggle Button|A button to toggle status:Hide`;  });  inc(i: number) {    this.minutes = Math.min(5, Math.max(0, this.minutes + i));  }  male() {    this.gender = 'male';  }  female() {    this.gender = 'female';  }  other() {    this.gender = 'other';  }  toggleDisplay() {    this.toggle.update((toggle) => !toggle);  }}

翻譯不帶 HTML 元素的行內文字

使用 <ng-container> 元素來關聯特定文字的翻譯行為，而無需更改文字的顯示方式。

提示： 每個 HTML 元素都會建立一個新的 DOM 元素。 為避免建立新的 DOM 元素，請將文字包裝在 <ng-container> 元素中。 以下範例顯示了 <ng-container> 元素如何轉換為未顯示的 HTML 註解。

      
<h1 i18n="User welcome|An introduction header for this sample@@introductionHeader">  Hello i18n!</h1><ng-container i18n>I don't output any element</ng-container><br /><img [src]="logo" i18n-title title="Angular logo" alt="Angular logo"/><br><button type="button" (click)="inc(1)">+</button> <button type="button" (click)="inc(-1)">-</button><span i18n>Updated {minutes, plural, =0 {just now} =1 {one minute ago} other {{{ minutes }} minutes ago}}</span>({{ minutes }})<br><br><button type="button" (click)="male()">♂</button><button type="button" (click)="female()">♀</button><button type="button" (click)="other()">⚧</button><span i18n>The author is {gender, select, male {male} female {female} other {other}}</span><br><br><span i18n>Updated: {minutes, plural,  =0 {just now}  =1 {one minute ago}  other {{{ minutes }} minutes ago by {gender, select, male {male} female {female} other {other}}}}</span><br><br><button type="button" (click)="toggleDisplay()">Toggle</button><div i18n [attr.aria-label]="toggleAriaLabel()">{{toggle()}}</div>

標記元素屬性以進行翻譯

在元件範本中，i18n 元資料是 i18n-{attribute_name} 屬性的值。

      
<element i18n-{attribute_name}="{i18n_metadata}" {attribute_name}="{attribute_value}" />

HTML 元素的屬性包含應與元件範本中其餘顯示的文字一起翻譯的文字。

將 i18n-{attribute_name} 與任何元素的任何屬性一起使用，並將 {attribute_name} 替換為屬性的名稱。 使用以下語法來賦予含義、描述和自定義 ID。

      
i18n-{attribute_name}="{meaning}|{description}@@{id}"

標記元件程式碼中的文字

在元件程式碼中，翻譯源文字和元資料都用反引號 (`) 字元括起來。

使用 $localize 標記的訊息字串來標記程式碼中的字串以進行翻譯。

      
$localize`string_to_translate`;

i18n 元資料用冒號 (:) 字元括起來，並位於翻譯源文字的前面。

      
$localize`:{i18n_metadata}:string_to_translate`

包含內嵌文字

在 $localize 標記的訊息字串中包含內嵌。

      
$localize`string_to_translate ${variable_name}`;

命名內嵌佔位符 

      
$localize`string_to_translate ${variable_name}:placeholder_name:`;

用於翻譯的條件語法 

      
return this.show ? $localize`Show Tabs` : $localize`Hide tabs`;

用於翻譯的 i18n 元資料 

      
{meaning}|{description}@@{custom_id}

以下引數提供上下文和其他訊息，以減少翻譯人員的困惑。

元資料引數 詳細訊息
自定義 ID 提供自定義識別符號
描述 提供其他訊息或上下文
含義 在特定上下文中提供文字的含義或意圖

有關自定義 ID 的更多訊息，請參閱使用自定義 ID 管理已標記的文字。

新增有用的描述和含義

為了準確地翻譯文字訊息，請為翻譯人員提供其他訊息或上下文。

將文字訊息的描述新增為 i18n 屬性或 $localize 標記的訊息字串的值。

以下範例顯示了 i18n 屬性的值。

翻譯人員可能還需要知道此特定應用上下文中文本訊息的含義或意圖，以便將其翻譯成與其他具有相同含義的文字相同的方式。 使用含義開始 i18n 屬性值，並使用 | 字元將其與描述分隔開：{meaning}|{description}。

meaning 如何控制文字的提取和合並

Angular 提取工具為範本中每個 i18n 屬性生成一個翻譯單元條目。 Angular 提取工具根據 meaning 和 description 為每個翻譯單元賦予一個唯一的 ID。

提示： 有關 Angular 提取工具的更多訊息，請參閱使用翻譯檔案。

具有不同 meanings 的相同文字元素會使用不同的 ID 提取。 例如，如果單詞 "right" 在兩個不同的位置使用以下兩個定義，則該單詞將被不同地翻譯，並作為不同的翻譯條目合併回應用中。

  • correct，如『你是對的』
  • direction，如『向右轉』

如果相同的文字元素滿足以下條件，則這些文字元素僅提取一次並使用相同的 ID。

  • 相同的 meaning 或定義
  • 不同的 description

無論在應用中何處出現相同的文字元素，該翻譯條目都會被合併回應用中。

ICU 表示式

ICU 表示式可幫助你在元件範本中標記備用文字以滿足條件。 ICU 表示式包括元件屬性（Property）、ICU 子句以及用左花括號 ({) 和右花括號 (}) 字元括起來的 case 語句。

      
{ component_property, icu_clause, case_statements }

元件屬性（Property）定義變數 ICU 子句定義條件文字的型別。

ICU 子句 詳細訊息
plural 標記複數數字的使用
select 根據你定義的字串值，標記備用文字的選項

為了簡化翻譯，請將 Unicode 國際元件子句（ICU 子句）與正則表示式結合使用。

提示： ICU 子句遵循 CLDR 複數規則中指定的 ICU 訊息格式。

標記複數

不同的語言有不同的複數規則，這增加了翻譯的難度。 由於其他語言環境表示基數的方式不同，你可能需要設定與英語不一致的複數類別。 使用 plural 子句來標記逐字翻譯可能沒有意義的表示式。

      
{ component_property, plural, pluralization_categories }

在複數類別之後，輸入用左花括號 ({) 和右花括號 (}) 字元括起來的預設文字（英語）。

      
pluralization_category { }

以下複數類別適用於英語，並且可能會根據語言環境而變化。

複數類別 詳細訊息 範例
zero 數量為零 =0 { }
zero { }
one 數量為 1 =1 { }
one { }
two 數量為 2 =2 { }
two { }
few 數量為 2 或更多 few { }
many 數量很大 many { }
other 預設數量 other { }

如果所有複數類別都不對應，Angular 將使用 other 來對應缺失類別的標準回退。

      
other { default_quantity }

提示： 有關複數類別的更多訊息，請參閱 CLDR - Unicode 通用語言環境資料儲存庫中的選擇複數類別名稱。

許多語言環境不支援某些複數類別。 預設語言環境 (en-US) 使用非常簡單的 plural() 函式，該函式不支援 few 複數類別。 另一個具有簡單 plural() 函式的語言環境是 es。 以下程式碼範例展示了 en-US plural() 函式。

      
function plural(n: number): number {  let i = Math.floor(Math.abs(n)),    v = n.toString().replace(/^[^.]*\.?/, '').length;  if (i === 1 && v === 0) return 1;  return 5;}

plural() 函式僅回傳 1 (one) 或 5 (other)。 few 類別永遠不對應。
