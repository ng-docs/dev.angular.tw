匯入語言環境資料的全域變體

edit

如果你使用帶有 --localize 選項執行 ng build 指令，Angular CLI 會自動包含語言環境資料。

      
ng build --localize

提示： Angular 的初始安裝已經包含美國英語 (en-US) 的語言環境資料。 當你將 --localize 選項與 ng build 指令一起使用時，Angular CLI 會自動包含語言環境資料並設定 LOCALE_ID 值。

npm 上的 @angular/common 包包含語言環境資料檔案。 語言環境資料的全域變體在 @angular/common/locales/global 中可用。

法語的 import 範例

例如，你可以在引導啟動應用的 main.ts 中匯入法語 (fr) 的全域變體。

src/main.ts (匯入語言環境)

      
import '@angular/common/locales/global/fr';import {provideProtractorTestingSupport} from '@angular/platform-browser';import {bootstrapApplication} from '@angular/platform-browser';import {AppComponent} from './app/app.component';bootstrapApplication(AppComponent, {  providers: [    provideProtractorTestingSupport(), // essential for e2e testing  ],});

提示： 在 NgModules 應用中，你需要在你的 app.module 中匯入它。
