导入语言环境数据的全局变量

edit

Angular CLI 在你运行 ng build 命令并带有 --localize 选项时，会自动包含语言环境数据。

      
ng build --localize

提示：Angular 的初始安装已经包含了美国英语（ en-US）的语言环境数据。 Angular CLI 会自动包含这些语言环境数据，并在你使用 --localize 选项运行 ng build 命令时设置 LOCALE_ID 值。

npm 上的 @angular/common 包含语言环境数据文件。 语言环境数据的全局变体可以在 @angular/common/locales/global 中找到。

法语的 import 示例

例如，你可以在 main.ts 中导入法语（ fr）的全局变体，在该文件中启动应用。

src/main.ts（导入语言环境）

      
import '@angular/common/locales/global/fr';
import {provideProtractorTestingSupport} from '@angular/platform-browser';
import {bootstrapApplication} from '@angular/platform-browser';
import {AppComponent} from './app/app.component';
bootstrapApplication(AppComponent, {
  providers: [
    provideProtractorTestingSupport(), // essential for e2e testing
  ],
});

提示：在 NgModules 应用中，你可以在 app.module 中导入它。
