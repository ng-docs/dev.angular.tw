將翻譯合併到應用中

edit

要將完成的翻譯合併到你的專案中，請完成以下操作

  1. 使用 Angular CLI 建立專案可分發檔案的副本
  2. 使用 "localize" 選項將所有 i18n 訊息替換為有效的翻譯，並建立本地化的變體應用。 變體應用是已翻譯成單一語言環境的應用的可分發檔案的完整副本。

合併翻譯後，使用伺服器端語言檢測或不同的子目錄為應用的每個可分發副本啟動伺服器。

提示： 有關如何為應用的每個可分發副本啟動伺服器的更多訊息，請參閱 部署多種語言環境。

對於應用的編譯時翻譯，建立過程使用預先編譯 (AoT) 來生成一個小的、快速的、可立即執行的應用。

提示： 有關建立過程的詳細說明，請參閱 建立和為 Angular 應用啟動伺服器。 建立過程適用於 .xlf 格式或 Angular 可理解的其他格式（如 .xtb）的翻譯檔案。 有關 Angular 使用的翻譯檔案格式的更多訊息，請參閱 更改源語言檔案格式

要為每個語言環境建立單獨的應用可分發副本，請在專案的 angular.json 工作區建立設定檔案中的 建立設定中定義語言環境。

此方法透過消除為每個語言環境執行完整應用建立的需求來縮短建立過程。

要為每個語言環境生成應用變體，請使用 angular.json 工作區建立設定檔案中的 "localize" 選項。 此外，要從命令行建立，請將 build Angular CLI 指令與 --localize 選項結合使用。

提示： （可選）為自定義語言環境設定僅為一個語言環境應用特定的建立選項。

在建立設定中定義語言環境

使用專案的 angular.json 工作區建立設定檔案中的 i18n 專案選項來為專案定義語言環境。

以下子選項標識源語言，並告知編譯器在哪裡可以找到專案的受支援翻譯。

子選項 詳細訊息
sourceLocale 你在應用原始碼中使用的語言環境（預設為 en-US）
locales 語言環境識別符號到翻譯檔案的對映表

為每個語言環境生成應用變體

要在建立設定中使用你的語言環境定義，請使用 angular.json 工作區建立設定檔案中的 "localize" 選項來告知 CLI 要為建立設定生成哪些語言環境。

  • 對於建立設定中先前定義的所有語言環境，將 "localize" 設定為 true。
  • 將 "localize" 設定為先前定義的語言環境識別符號子集的陣列，以僅建立這些語言環境版本。
  • 將 "localize" 設定為 false 以停用本地化，並且不生成任何特定於語言環境的版本。

提示： 本地化元件樣板需要預先編譯 (AoT)。

如果你更改了此設定，請將 "aot" 設定為 true 以使用 AoT。

提示： 由於 i18n 的部署複雜性以及需要最大限度地縮短重建時間，開發伺服器一次僅支援本地化單個語言環境。 如果你將 "localize" 選項設定為 true，定義了多個語言環境，並使用 ng serve；則會發生錯誤。 如果你想針對特定語言環境進行開發，請將 "localize" 選項設定為特定語言環境。 例如，對於法語 (fr)，請指定 "localize": ["fr"]。

CLI 載入並註冊語言環境資料，將每個生成的版本放在特定於語言環境的目錄中，以使其與其他語言環境版本分開，並將這些目錄放在專案中設定的 outputPath 中。 對於每個應用變體，html 元素的 lang 屬性都設定為語言環境。 CLI 還會透過將語言環境新增到設定的 baseHref 中來調整每個應用版本的 HTML base HREF。

將 "localize" 屬性設定為共享設定，以便有效地為所有設定繼承。 此外，設定該屬性以覆蓋其他設定。

從命令行建立

此外，將 --localize 選項與 ng build 指令和你現有的 production 設定一起使用。 CLI 建立建立設定中定義的所有語言環境。 如果你在建立設定中設定了語言環境，則這類似於將 "localize" 選項設定為 true。

提示： 有關如何設定語言環境的更多訊息，請參閱 為每個語言環境生成應用變體。

      
ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

僅為一個語言環境應用特定的建立選項

要僅為一個語言環境應用特定的建立選項，請指定單個語言環境以建立自定義的特定於語言環境的設定。

重要提示： 僅將 Angular CLI 開發伺服器 (ng serve) 與單個語言環境一起使用。

報告缺失的翻譯

當翻譯缺失時，建立會成功，但會生成警告，例如 Missing translation for message "{translation_text}"。 要設定 Angular 編譯器生成的警告級別，請指定以下級別之一。

警告級別 詳細訊息 輸出
error 丟擲錯誤並且建立失敗 不適用
ignore 不執行任何操作 不適用
warning 在控制台或 Shell 中顯示預設警告 Missing translation for message "{translation_text}"

在 angular.json 工作區建立設定檔案的 options 部分中，為 build 目標指定警告級別。
