把翻译结果合并回应用

edit

要将完成的翻译结果合并到你的项目中，请完成以下操作

  1. 使用 Angular CLI 构建项目的可分发文件副本
  2. 使用 "localize" 选项替换所有 i18n 消息为有效翻译，并构建一个本地化变体应用。 变体应用是一个完整的可分发文件副本，为单个语言环境进行了翻译。

合并翻译后，可使用服务端语言检测或不同的子目录来提供（serve）应用程序的每个可分发副本。

提示： 有关如何为应用的每个可分发副本启动服务器的更多信息，请参见 部署多个语言环境。

对于应用的编译时翻译，构建过程使用预先编译（AOT）产生一个小型、快速、开箱即用的应用。

提示：要了解构建过程的详细解释，请参见 构建和提供 Angular 应用程序。 构建过程适用于以 .xlf 格式或 Angular 理解的其他格式（例如 .xtb）的翻译文件。 有关 Angular 使用的翻译文件格式的更多信息，请参见 更改源语言文件格式

要为每个语言环境构建一个单独的可分发应用副本，请在项目的 构建配置中定义该语言环境，在 angular.json 工作区构建配置文件中。

此方法不需要为每个语言环境执行完整的应用程序构建，从而缩短了构建过程。

要为每个语言环境 生成应用变体，请在 angular.json 工作区构建配置文件中使用 "localize" 选项。 另外，要 从命令行构建，请使用带有 --localize 选项的 build Angular CLI 命令。

提示：可选择为自定义语言环境配置 应用特定的构建选项。

定义构建配置中的语言环境

在项目的 angular.json 工作区构建配置文件中，使用 i18n 项目选项来定义项目的语言环境。

以下子选项标识源语言并告诉编译器在哪里可以找到项目支持的翻译。

子选项 详情
sourceLocale 你在应用程序源代码中使用的语言环境（默认为 en-US）
locales 语言环境标识符到翻译文件的映射表

为每个语言环境生成应用程序变体

要在构建配置中使用你的语言环境定义，请在 angular.json 工作区构建配置文件中使用 "localize" 选项，告诉 CLI 要为哪些语言环境生成构建配置。

  • 对于先前在构建配置中定义的所有语言环境，将 "localize" 设置为 true。
  • 将 "localize" 设置为先前定义的语言环境标识符子集的数组，以单独构建那些语言环境版本。
  • 将 "localize" 设置为 false 表示禁用本地化，不生成任何特定于本地环境的版本。

提示：需要预先编译（AOT）以本地化组件模板。

如果你更改了此设置，请将 "aot" 设置为 true 以使用 AOT。

提示：由于 i18n 的部署复杂性和需要最小化重建时间，开发服务器仅支持一次本地化一个语言环境。 如果将 "localize"选项设置为 true，定义多个语言环境并使用 ng serve，将出现错误。 如果要针对特定语言环境开发，请将 "localize"选项设置为特定语言环境。 例如，对于法语（ fr），请指定 "localize": ["fr"]。

CLI 加载并注册语言环境数据，将每个生成的版本放置在特定语言环境的目录中以使其与其他语言环境版本分开，并将其目录放在为此项目配置的 outputPath 中。对于每个应用程序变体，将 html 元素的 lang 属性设置为其语言环境。CLI 还通过将语言环境添加到所配置的 baseHref 中来调整每个应用程序版本的 HTML baseHref。

将 "localize" 属性设置为共享配置以有效继承所有配置。此外，会将该属性设置为覆盖其他配置。

从命令行构建

另外，使用 ng build 命令和现有 production 配置的 --localize 选项。 CLI 将构建配置中定义的所有语言环境。 如果你在构建配置中设置了语言环境，这类似于将 "localize" 选项设置为 true。

提示：要了解如何设置语言环境的更多信息，请参见 生成每个语言环境的应用程序变体。

      
ng add @angular/localize
ng extract-i18n
ng extract-i18n --output-path src/locale
ng extract-i18n --format=xlf
ng extract-i18n --format=xlf2
ng extract-i18n --format=xmb
ng extract-i18n --format=json
ng extract-i18n --format=arb
ng extract-i18n --out-file source.xlf
ng build --localize
ng serve --configuration=fr
ng build --configuration=production,fr

仅适用于一个语言环境的特定构建选项

要将特定构建选项仅应用于一个语言环境，请指定单个语言环境以创建自定义的语言环境专有配置。

重要：使用 Angular CLI 开发服务器 ( ng serve) 仅使用单个区域设置。

报告遗漏的翻译

当缺少翻译时，可以构建成功但会生成警告，比如 Missing translation for message "{translation_text}"。要配置 Angular 编译器生成的警告级别，请指定以下级别之一。

警告级别 详情 输出
error 抛出错误并构建失败 不适用
ignore 不执行任何操作 不适用
warning 在控制台或 shell 中显示默认警告 Missing translation for message "{translation_text}"

在你的 angular.json 工作区构建配置文件的 build 目标的 options 部分中指定警告级别。

有用：当你将 Angular 项目编译为 Angular 应用时， i18n 属性的实例将被替换为 $localize 标记的消息字符串的实例。 这意味着你的 Angular 应用会在编译后被翻译。 这也意味着你可以为你的 Angular 应用创建本地化版本，而无需为每个区域设置重新编译整个 Angular 项目。

当你翻译你的 Angular 应用时， 翻译转换 将使用来自翻译集合的字符串替换和重新排序模板字面字符串的部分（静态字符串和表达式）。 有关更多信息，请参见 $localize。

简而言之：编译一次，然后为每个区域设置进行翻译。
