使用翻譯檔案

edit

在為元件準備好翻譯之後，可以使用 extract-i18n Angular CLI 指令，將元件中標記的文字提取到源語言檔案中。

標記的文字包括用 i18n 標記的文字、用 i18n-attribute 標記的屬性（Attribute），以及用 $localize 標記的文字，如為元件準備翻譯中所述。

要為你的專案建立和更新翻譯檔案，請完成以下步驟。

  1. 提取源語言檔案。
    1. （可選）更改位置、格式和名稱。
  2. 將源語言檔案複製到為每種語言建立翻譯檔案。
  3. 翻譯每個翻譯檔案。
  4. 分別翻譯複數和備用表示式。
    1. 翻譯複數。
    2. 翻譯備用表示式。
    3. 翻譯巢狀表示式。

提取源語言檔案

要提取源語言檔案，請完成以下操作。

  1. 開啟終端視窗。
  2. 切換到你的專案根目錄。
  3. 執行以下 CLI 指令。
          
    ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

extract-i18n 指令會在你的專案根目錄中建立一個名為 messages.xlf 的源語言檔案。 有關 XML 本地化交換檔案格式（XLIFF，1.2 版）的更多訊息，請參閱 XLIFF。

使用以下 extract-i18n 指令選項來更改源語言檔案的位置、格式和檔案名。

指令選項 詳情
--format 設定輸出檔案的格式
--out-file 設定輸出檔案的名稱
--output-path 設定輸出目錄的路徑

更改源語言檔案位置

要在 src/locale 目錄中建立檔案，請將輸出路徑指定為一個選項。

extract-i18n --output-path 範例

以下範例將輸出路徑指定為一個選項。

      
ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

更改源語言檔案格式

extract-i18n 指令以以下翻譯格式建立檔案。

翻譯格式 詳情 檔案副檔名
ARB 應用資源捆綁包 .arb
JSON JavaScript 物件表示法 .json
XLIFF 1.2 XML 本地化交換檔案格式，1.2 版 .xlf
XLIFF 2 XML 本地化交換檔案格式，第 2 版 .xlf
XMB XML 訊息捆綁包 .xmb (.xtb)

使用 --format 指令選項顯式指定翻譯格式。

提示： XMB 格式生成 .xmb 源語言檔案，但使用 .xtb 翻譯檔案。

extract-i18n --format 範例

以下範例示範了幾種翻譯格式。

      
ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

更改源語言檔案名

要更改提取工具生成的源語言檔案的名稱，請使用 --out-file 指令選項。

extract-i18n --out-file 範例

以下範例示範了如何命名輸出檔案。

      
ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

為每種語言建立翻譯檔案

要為語言環境或語言建立翻譯檔案，請完成以下操作。

  1. 提取源語言檔案。

  2. 製作源語言檔案的副本，以便為每種語言建立一個翻譯檔案。

  3. 重新命名翻譯檔案以新增語言環境。

          messages.xlf --> messages.{locale}.xlf

  4. 在專案根目錄下建立一個名為 locale 的新目錄。

          src/locale

  5. 將翻譯檔案移動到新目錄。

  6. 將翻譯檔案傳送給你的翻譯人員。

  7. 對於要新增到應用中的每種語言，重複上述步驟。

extract-i18n 法語範例

例如，要建立法語翻譯檔案，請完成以下操作。

  1. 執行 extract-i18n 指令。
  2. 製作 messages.xlf 源語言檔案的副本。
  3. 將副本重新命名為 messages.fr.xlf 以進行法語 (fr) 翻譯。
  4. 將 fr 翻譯檔案移動到 src/locale 目錄。
  5. 將 fr 翻譯檔案傳送給翻譯人員。

翻譯每個翻譯檔案

除非你精通該語言並有時間編輯翻譯，否則你很可能需要完成以下步驟。

  1. 將每個翻譯檔案傳送給翻譯人員。
  2. 翻譯人員使用 XLIFF 檔案編輯器來完成以下操作。
    1. 建立翻譯。
    2. 編輯翻譯。

翻譯複數

根據每種語言的需要新增或刪除複數情況。

有幫助的： 有關語言複數規則，請參閱 CLDR 複數規則。
