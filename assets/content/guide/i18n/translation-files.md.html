处理翻译文件

edit

准备组件进行翻译后，使用 extract-i18n Angular CLI命令将组件中标记的文本提取到一个 源语言文件中。

标记的文本包括用 i18n 标记的文本、用 i18n- attribute 标记的属性和用 $localize 标记的文本，如 准备组件进行翻译 中所述。

完成以下步骤为你的项目创建和更新翻译文件。

  1. Extract the source language file.
    1. （可选）更改位置、格式和名称。
  2. 将源语言文件复制到 为每种语言创建翻译文件。
  3. 翻译每个翻译文件。
  4. Translate plurals and alternate expressions separately.
    1. 翻译复数形式。
    2. 翻译替代表达式.
    3. 翻译嵌套表达式.

提取源语言文件

要提取源语言文件，请完成以下操作。

  1. 打开终端窗口。
  2. 切换到你的项目根目录。
  3. 运行以下 CLI 命令。
          
    ng add @angular/localize
    ng extract-i18n
    ng extract-i18n --output-path src/locale
    ng extract-i18n --format=xlf
    ng extract-i18n --format=xlf2
    ng extract-i18n --format=xmb
    ng extract-i18n --format=json
    ng extract-i18n --format=arb
    ng extract-i18n --out-file source.xlf
    ng build --localize
    ng serve --configuration=fr
    ng build --configuration=production,fr

extract-i18n 命令在项目的根目录中创建一个名为 messages.xlf 的源语言文件。 有关 XML 本地化交换文件格式（XLIFF 1.2 版）的更多信息，请参见 XLIFF。

使用以下 extract-i18n 命令选项来更改源语言文件的位置、格式和文件名。

命令选项 详情
--format 设置输出文件的格式
--out-file 设置输出文件的名称
--output-path 设置输出目录的路径

更改源语言文件的位置

要在 src/locale 目录中创建文件，请将输出路径指定为选项。

extract-i18n --output-path 示例

以下示例将输出路径指定给选项。

      
ng add @angular/localize
ng extract-i18n
ng extract-i18n --output-path src/locale
ng extract-i18n --format=xlf
ng extract-i18n --format=xlf2
ng extract-i18n --format=xmb
ng extract-i18n --format=json
ng extract-i18n --format=arb
ng extract-i18n --out-file source.xlf
ng build --localize
ng serve --configuration=fr
ng build --configuration=production,fr

更改源语言文件格式

extract-i18n 命令会创建如下翻译格式的文件。

翻译格式 详情 文件扩展名
ARB 应用资源捆绑 .arb
JSON JavaScript 对象表示法 .json
XLIFF 1.2 XML 本地化互换文件格式，版本 1.2 .xlf
XLIFF 2 XML 本地化互换文件格式，版本 2 .xlf
XMB XML 消息捆绑 .xmb (.xtb)

使用 --format 命令选项明确指定转换格式。

有用: XMB 格式生成 .xmb 源语言文件，但使用 .xtb 翻译文件。

extract-i18n --format 示例

以下示例演示了几种翻译格式。

      
ng add @angular/localize
ng extract-i18n
ng extract-i18n --output-path src/locale
ng extract-i18n --format=xlf
ng extract-i18n --format=xlf2
ng extract-i18n --format=xmb
ng extract-i18n --format=json
ng extract-i18n --format=arb
ng extract-i18n --out-file source.xlf
ng build --localize
ng serve --configuration=fr
ng build --configuration=production,fr

为每种语言创建一个翻译文件

要为语言环境或语言创建翻译文件，请完成以下操作。

  1. 提取源语言文件。

  2. 复制源语言文件，为每种语言创建一个翻译文件。

  3. 重命名翻译文件以添加语言环境标识。

          
    messages.xlf --> messages.{locale}.xlf

  4. 在你的项目根目录创建一个名为 locale 的新目录。

          
    src/locale

  5. 将翻译文件移到新目录。

  6. 将翻译文件发送给你的翻译人员。

  7. 对要添加到应用程序中的每种语言重复上述步骤。

法语的 extract-i18n 示例

比如，要创建法语翻译文件，请完成以下操作。

  1. 运行 extract-i18n 命令。
  2. 复制 messages.xlf 源语言文件。
  3. 将副本重命名为 messages.fr.xlf 以进行法语（fr）翻译。
  4. 将 fr 翻译文件移动到 src/locale 目录。
  5. 将 fr 翻译文件发送给翻译人员。

翻译每个翻译文件

除非你精通该语言并有时间编辑翻译，否则你可能会完成以下步骤。

  1. 将每个翻译文件发送给翻译人员。
  2. The translator uses an XLIFF file editor to complete the following actions.
    1. 创建翻译。
    2. 编辑翻译。

法语翻译流程示例

为了演示这个过程，请审阅 messages.fr.xlf 文件在 示例 Angular 国际化应用 中。 该 示例 Angular 国际化应用 包含一个法语翻译，你可以在没有特殊 XLIFF 编辑器或法语知识的情况下进行编辑。

以下操作描述了法语的翻译过程。

  1. 打开 messages.fr.xlf，找到第一个 <trans-unit> 元素。这是一个 翻译单元，也称为 文本节点，表示之前标记为 i18n 属性的 <h1> 问候标记的翻译。

    id="introductionHeader" 是一个 自定义 ID，但没有源 HTML 中所需的 @@ 前缀。

  2. 复制文本节点中的 <source>...</source> 元素，将其重命名为 target，然后将内容替换为法语文本。

    在更复杂的翻译中， 描述和含义元素中的信息和上下文可帮助你选择正确的翻译词语。

  3. 翻译其他文本节点。以下示例展示了翻译的方式。

重要: 不要更改翻译单元的 ID。每个 id 属性由 Angular 生成，并取决于组件文本内容和分配的含义。

如果更改文本或含义， id 属性会发生变化。有关管理文本更新和 ID 的更多信息，请参阅 自定义 ID。

翻译替代表达式

Angular 还会提取备用的 select ICU 表达式作为单独的翻译单元。

gender select 示例

以下示例在组件模板中显示了一个 select ICU 表达式。

src/app/app.component.html

      
<h1 i18n="User welcome|An introduction header for this sample@@introductionHeader">
  Hello i18n!
</h1>
<ng-container i18n>I don't output any element</ng-container>
<br />
<img [src]="logo" i18n-title title="Angular logo" alt="Angular logo"/>
<br>
<button type="button" (click)="inc(1)">+</button> <button type="button" (click)="inc(-1)">-</button>
<span i18n>Updated {minutes, plural, =0 {just now} =1 {one minute ago} other {{{ minutes }} minutes ago}}</span>
({{ minutes }})
<br><br>
<button type="button" (click)="male()">♂</button>
<button type="button" (click)="female()">♀</button>
<button type="button" (click)="other()">⚧</button>
<span i18n>The author is {gender, select, male {male} female {female} other {other}}</span>
<br><br>
<span i18n>Updated: {minutes, plural,
  =0 {just now}
  =1 {one minute ago}
  other {{{ minutes }} minutes ago by {gender, select, male {male} female {female} other {other}}}}
</span>

在这个例子中，Angular 将表达式提取到两个翻译单元中。第一个包含 select 子句之外的文本，并为 select 使用占位符（<x id="ICU">）：

重要提示：在翻译文本时，如有必要，请移动占位符，但不要删除它。 如果删除占位符，则会从你翻译的应用程序中移除 ICU 表达式。
