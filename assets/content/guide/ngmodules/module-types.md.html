模組（NgModule）建立指南

edit

本主題提供了有關你可以建立的不同 NgModules 類別的概念概述，以便以模組化結構組織程式碼。這些類別並非鐵板一塊，它們只是建議。你可能想為其他目的建立 NgModules，或者將某些類別的特徵組合在一起。

NgModules 是組織應用程式並保持與特定功能或屬性相關的程式碼與其他程式碼分開的絕佳方式。使用 NgModules 將元件、指令和通道合併為功能性區塊。將每個區塊專注於一個屬性或業務域、一個工作流程或導向流、一個常見的實用工具集，或者一個或多個服務的提供者。

模組類別彙總

所有應用程式都是透過 啟動根 NgModule 開始的。你可以按照自己的方式組織其他的 NgModules。

本主題為下列常見模組類別提供了一些指導：

領域模組

使用領域 NgModule 提供專門針對特定功能或應用領域的使用者體驗，例如編輯客戶或下訂單。

領域模組用來組織與特定功能有關的程式碼，裡面包含構成此功能的所有元件、路由和範本。領域模組中的頂級元件是該屬性或領域的根，是你要匯出的唯一元件。各種私有的支撐子元件都是它的後代。

領域模組要匯入到另一個模組中一次並且只能一次，比如一個領域模組，或者一個只包含少量模組的應用的根模組（AppModule）中。

領域模組主要由可宣告物件組成，很少會在此提供服務。如果一定要提供，那麼這些服務的生命週期應和該模組的生命週期一致。

路由定義模組

使用路由 NgModule 為領域 NgModule 提供路由設定，從而將路由關注點與其伴隨的領域 NgModule 分離開來。

有用提示: 欲瞭解有關路由的概述和詳細訊息，請參閱 應用內導向：路由到檢視。

使用路由定義模組來完成如下任務：

  • 定義路由
  • 透過 imports 向 NgModule 新增路由設定
  • 往模組的提供者清單中新增路由守衛和解析器（resolver）提供者

路由 NgModule 的名稱應與其伴隨的 NgModule 名稱相對應，使用字尾 Routing。 例如，考慮在 contact.module.ts 中有一個名為 ContactModule 的模組，則在 contact-routing.module.ts 中有一個名為 ContactRoutingModule 的路由 NgModule。

僅將路由 NgModule 匯入其伴隨的 NgModule 中。 如果伴隨的 NgModule 是根 AppModule，則 AppRoutingModule 使用 RouterModule.forRoot(routes) 向其匯入新增路由設定。 所有其他路由 NgModule 都是子級，使用 RouterModule.forChild(routes) 進行匯入。

在你的路由 NgModule 中重新匯出 RouterModule，以便伴隨的 NgModule 的元件可以訪問路由指令，如 RouterLink 和 RouterOutlet。

不要在路由定義模組中使用可宣告物件。元件、指令和通道都是伴生領域模組的責任，而不是路由定義模組的。

服務模組

使用服務 NgModule 提供諸如資料訪問或訊息傳遞等實用服務。 理想的服務 NgModule 完全由提供者組成，沒有宣告。 Angular 的 HttpClientModule 就是一個好的服務 NgModule 範例。

只能使用根模組 AppModule 來匯入各種服務模組。

小部件模組

使用小部件模組可以把元件、指令或通道提供給外部模組使用。把小部件模組匯入到任何需要在範本使用這些小部件的模組中。很多第三方 UI 元件庫都是作為小部件模組提供的。

小部件模組應該完全由可宣告物件組成，其中大部分都是匯出的。服務提供者非常罕見。

共享模組

將常用指令、通道和元件放入一個 NgModule 中，通常命名為 SharedModule，然後在應用程式的其他部分需要時僅匯入該 NgModule。 可以在領域 NgModule 中匯入共享 NgModule，包括 延遲載入的 NgModules。

注意: 共享 NgModule 不應包含提供者，也不應包含其匯入或重新匯出的任何 NgModule 中的提供者。

要了解如何使用共享模組組織和簡化程式碼，請參閱 在應用中共享 NgModules。
