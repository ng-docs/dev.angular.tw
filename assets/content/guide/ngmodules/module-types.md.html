模块（NgModule）创建指南

edit

本主题提供了有关你可以创建的不同 NgModules 类别的概念概述，以便以模块化结构组织代码。这些类别并非铁板一块，它们只是建议。你可能想为其他目的创建 NgModules，或者将某些类别的特征组合在一起。

NgModules 是组织应用程序并保持与特定功能或特性相关的代码与其他代码分开的绝佳方式。使用 NgModules 将组件、指令和管道合并为功能性块。将每个块专注于一个特性或业务域、一个工作流程或导航流、一个常见的实用工具集，或者一个或多个服务的提供者。

模块类别汇总

所有应用程序都是通过 启动根 NgModule 开始的。你可以按照自己的方式组织其他的 NgModules。

本主题为下列常见模块类别提供了一些指导：

分类 详情
领域 围绕功能、业务领域或用户体验组织。
路由 为另一个 NgModule 提供路由配置。
服务 提供诸如数据访问和消息传递等实用服务。
小部件 使组件、指令或管道可供其他 NgModules 使用。
共享 使一组组件、指令和管道可供其他 NgModules 使用。

下表总结了每个类别的主要特性。

NgModule 声明 提供者 导出 被导入
领域 是 罕见 顶级组件 另一个领域， AppModule
带路由的 是 罕见 不 无
路由 不 是（守卫） RouterModule 另一个领域（用于路由）
服务 不 是 不 AppModule
小部件 是 罕见 是 另一个领域
共享 是 不 是 另一个领域

领域模块

使用领域 NgModule 提供专门针对特定功能或应用领域的用户体验，例如编辑客户或下订单。

领域模块用来组织与特定功能有关的代码，里面包含构成此功能的所有组件、路由和模板。领域模块中的顶级组件是该特性或领域的根，是你要导出的唯一组件。各种私有的支撑子组件都是它的后代。

领域模块要导入到另一个模块中一次并且只能一次，比如一个领域模块，或者一个只包含少量模块的应用的根模块（AppModule）中。

领域模块主要由可声明对象组成，很少会在此提供服务。如果一定要提供，那么这些服务的生命周期应和该模块的生命周期一致。

路由定义模块

使用路由 NgModule 为领域 NgModule 提供路由配置，从而将路由关注点与其伴随的领域 NgModule 分离开来。

有用提示: 欲了解有关路由的概述和详细信息，请参阅 应用内导航：路由到视图。

使用路由定义模块来完成如下任务：

  • 定义路由
  • 通过 imports 向 NgModule 添加路由配置
  • 往模块的提供者列表中添加路由守卫和解析器（resolver）提供者

路由 NgModule 的名称应与其伴随的 NgModule 名称相对应，使用后缀 Routing。 例如，考虑在 contact.module.ts 中有一个名为 ContactModule 的模块，则在 contact-routing.module.ts 中有一个名为 ContactRoutingModule 的路由 NgModule。

仅将路由 NgModule 导入其伴随的 NgModule 中。 如果伴随的 NgModule 是根 AppModule，则 AppRoutingModule 使用 RouterModule.forRoot(routes) 向其导入添加路由配置。 所有其他路由 NgModule 都是子级，使用 RouterModule.forChild(routes) 进行导入。

在你的路由 NgModule 中重新导出 RouterModule，以便伴随的 NgModule 的组件可以访问路由指令，如 RouterLink 和 RouterOutlet。

不要在路由定义模块中使用可声明对象。组件、指令和管道都是伴生领域模块的责任，而不是路由定义模块的。

服务模块

使用服务 NgModule 提供诸如数据访问或消息传递等实用服务。 理想的服务 NgModule 完全由提供者组成，没有声明。 Angular 的 HttpClientModule 就是一个好的服务 NgModule 示例。

只能使用根模块 AppModule 来导入各种服务模块。

小部件模块

使用小部件模块可以把组件、指令或管道提供给外部模块使用。把小部件模块导入到任何需要在模板使用这些小部件的模块中。很多第三方 UI 组件库都是作为小部件模块提供的。

小部件模块应该完全由可声明对象组成，其中大部分都是导出的。服务提供者非常罕见。

共享模块

将常用指令、管道和组件放入一个 NgModule 中，通常命名为 SharedModule，然后在应用程序的其他部分需要时仅导入该 NgModule。 可以在领域 NgModule 中导入共享 NgModule，包括 延迟加载的 NgModules。

注意: 共享 NgModule 不应包含提供者，也不应包含其导入或重新导出的任何 NgModule 中的提供者。

要了解如何使用共享模块组织和简化代码，请参阅 在应用中共享 NgModules。
