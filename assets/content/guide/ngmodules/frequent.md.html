常用模块

edit

Angular 应用至少需要一个充当根模块使用的模块。 如果你要把某些特性添加到应用中，可以通过添加模块来实现。 下列是一些常用的 Angular 模块，其中带有一些其内容物的例子：

NgModule 从这里导入 使用它的原因
BrowserModule @angular/platform-browser 在浏览器中运行你的应用。
CommonModule @angular/common 使用 NgIf 和 NgFor。
FormsModule @angular/forms 构建模板驱动表单（包括 NgModel）。
ReactiveFormsModule @angular/forms 构建响应式表单。
RouterModule @angular/router 使用 RouterLink、 .forRoot() 和 .forChild()。
HttpClientModule @angular/common/http 使用 HTTP 协议与服务器通信。

导入模块

当你使用这些 Angular 模块时，在 AppModule 或相应的特性模块中导入它们，并将它们列在 @NgModule 的 imports 数组中。

例如，在由 Angular CLI 生成的新应用中， BrowserModule 被导入到 AppModule 中。

      
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { AppComponent } from './app.component';
@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    /* add modules here so Angular knows to use them */
    BrowserModule,
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

数组顶部的导入是 JavaScript 导入语句，而 @NgModule 中的 imports 数组是 Angular 专用的。有关区别的更多信息，请参见 JavaScript 模块与 NgModules。

BrowserModule 和 CommonModule

BrowserModule 重新导出了 CommonModule，该模块暴露了许多常见的指令，如 ngIf 和 ngFor。 由于重新导出，这些指令对于任何导入此浏览器模块的模块都是可用的。

对于在浏览器中运行的应用，在根 AppModule 中导入 BrowserModule，因为它提供了在浏览器中启动和渲染应用所需的服务。

注意： BrowserModule 的提供者是为整个应用服务的，所以它应该只在根模块中使用，而不应该在特性模块中使用。特性模块只需要 CommonModule 中的常见指令；它们不需要重新安装应用范围的提供者。
