常用模組

edit

Angular 應用至少需要一個充當根模組使用的模組。 如果你要把某些屬性新增到應用中，可以透過新增模組來實現。 下列是一些常用的 Angular 模組，其中帶有一些其內容物的例子：

NgModule 從這裡匯入 使用它的原因
BrowserModule @angular/platform-browser 在瀏覽器中執行你的應用。
CommonModule @angular/common 使用 NgIf 和 NgFor。
FormsModule @angular/forms 建立範本驅動表單（包括 NgModel）。
ReactiveFormsModule @angular/forms 建立回應式表單。
RouterModule @angular/router 使用 RouterLink、 .forRoot() 和 .forChild()。
HttpClientModule @angular/common/http 使用 HTTP 協定與伺服器通訊。

匯入模組

當你使用這些 Angular 模組時，在 AppModule 或相應的屬性模組中匯入它們，並將它們列在 @NgModule 的 imports 陣列中。

例如，在由 Angular CLI 生成的新應用中， BrowserModule 被匯入到 AppModule 中。

      
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { AppComponent } from './app.component';
@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    /* add modules here so Angular knows to use them */
    BrowserModule,
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

陣列頂部的匯入是 JavaScript 匯入語句，而 @NgModule 中的 imports 陣列是 Angular 專用的。有關區別的更多訊息，請參見 JavaScript 模組與 NgModules。

BrowserModule 和 CommonModule

BrowserModule 重新匯出了 CommonModule，該模組暴露了許多常見的指令，如 ngIf 和 ngFor。 由於重新匯出，這些指令對於任何匯入此瀏覽器模組的模組都是可用的。

對於在瀏覽器中執行的應用，在根 AppModule 中匯入 BrowserModule，因為它提供了在瀏覽器中啟動和渲染應用所需的服務。

注意： BrowserModule 的提供者是為整個應用服務的，所以它應該只在根模組中使用，而不應該在屬性模組中使用。屬性模組只需要 CommonModule 中的常見指令；它們不需要重新安裝應用範圍的提供者。
