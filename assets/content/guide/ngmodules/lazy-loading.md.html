延遲載入屬性模組

edit

預設情況下，NgModule 都是急性載入的。意思是它會在應用載入時儘快載入，所有模組都是如此，無論是否立即要用。對於帶有很多路由的大型應用，考慮使用延遲載入 —— 一種按需載入 NgModule 的模式。延遲載入可以減小初始套件的尺寸，從而減少載入時間。

延遲載入入門

本節介紹設定延遲載入路由的基本步驟。 有關分步範例，請參閱本頁的 分步設定 部分。

要延遲載入 Angular 模組，請在 AppRoutingModule routes 中使用 loadChildren 代替 component 進行設定，程式碼如下。

AppRoutingModule（摘錄）

      
const routes: Routes = [
  {
    path: 'items',
    loadChildren: () => import('./items/items.module').then(m => m.ItemsModule)
  }
];

在延遲載入模組的路由模組中，新增一個指向該組件的路由。

延遲載入模組的路由模組（摘錄）

      
const routes: Routes = [
  {
    path: '',
    component: ItemsComponent
  }
];

還要確保從 AppModule 中移除了 ItemsModule。想要一個關於延遲載入模組的分步操作指南，請繼續檢視本頁的後續章節。

分步設定

建立延遲載入的屬性模組需要兩個主要步驟：

  1. 使用 --route 標誌，用 Angular CLI 建立屬性模組。
  2. 設定相關路由。

建立應用

如果你還沒有應用，請按照以下步驟使用 Angular CLI 建立一個。 如果你已經有了應用，則跳到 設定路由。

輸入下列指令，其中的 customer-app 表示你的應用名稱。

      
ng new customer-app --no-standalone --routing

這將建立一個名為 customer-app 的應用， --no-standalone 標誌使應用基於模組，並且 --routing 標誌生成一個名為 app-routing.module.ts 的檔案。 這是你為屬性模組設定延遲載入所需的檔案之一。 透過執行指令 cd customer-app 導向到專案中。

建立帶有路由的屬性模組

接下來，你將需要一個包含路由的目標元件的屬性模組。要建立它，在指令行工具中輸入如下指令，其中 customers 是屬性模組的名稱。載入 customers 屬性模組的路徑也是 customers，因為它是透過 --route 選項指定的：

      
ng generate module customers --route customers --module app.module

這將建立一個 customers目錄，其中包含在 customers.module.ts檔案中定義的新延遲載入屬性模組 CustomersModule，以及在 customers-routing.module.ts檔案中定義的路由模組 CustomersRoutingModule。該指令會自動在新屬性模組中宣告 CustomersComponent並匯入 CustomersRoutingModule。

因為這個新模組想要延遲載入，所以該指令不會在應用的根模組 app.module.ts 中新增對新屬性模組的引用。相反，它將宣告的路由 customers 新增到以 --module 選項指定的模組中宣告的 routes 陣列中。

src/app/app-routing.module.ts

      
const routes: Routes = [
  {
    path: 'customers',
    loadChildren: () => import('./customers/customers.module').then(m => m.CustomersModule)
  }
];

注意，延遲載入語法使用 loadChildren，其後是一個使用瀏覽器內建的 import('...') 語法進行動態匯入的函式。其匯入路徑是到當前模組的相對路徑。

基於字串的延遲載入

在 Angular 版本 8 中， loadChildren 路由規範的字串語法已被棄用，改為使用 import() 語法。 你可以透過在 tsconfig 檔案中包含延遲載入的路由，選擇使用基於字串的延遲載入（ loadChildren: './path/to/module#Module'），這會將延遲載入的檔案包含在編譯中。

預設情況下，會用 Angular CLI 生成專案，這些專案將更嚴格地包含旨在與 import() 語法一起使用的檔案。

新增另一個屬性模組

使用同樣的指令建立第二個帶路由的延遲載入屬性模組及其樁元件。

      
ng generate module orders --route orders --module app.module

這將建立一個名為 orders 的新目錄，其中包含 OrdersModule 和 OrdersRoutingModule 以及新的 OrdersComponent 源檔案。使用 --route 選項指定的 orders 路由，用延遲載入語法新增到了 app-routing.module.ts 檔案內的 routes 陣列中。

src/app/app-routing.module.ts

      
const routes: Routes = [
  {
    path: 'customers',
    loadChildren: () => import('./customers/customers.module').then(m => m.CustomersModule)
  },
  {
    path: 'orders',
    loadChildren: () => import('./orders/orders.module').then(m => m.OrdersModule)
  }
];

建立 UI

雖然你也可以在地址欄中輸入 URL，不過導向 UI 會更好用，也更常見。把 app.component.html 中的佔位指令碼替換成一個自定義的導向，以便你在瀏覽器中能在模組之間導向。

src/app/app.component.html

      
<h1>
  {{title}}
</h1>
<button type="button" routerLink="/customers">Customers</button>
<button type="button" routerLink="/orders">Orders</button>
<button type="button" routerLink="">Home</button>
<router-outlet></router-outlet>

要想在瀏覽器中看到你的應用，就在指令行工具視窗中輸入下列指令：

      
ng serve

然後，跳轉到 localhost:4200，這時你應該看到 "customer-app" 和三個按鈕。

這些按鈕之所以能工作，是因為 Angular CLI 自動將屬性模組的路由新增到了 app-routing.module.ts 中的 routes 陣列。

匯入與路由設定

Angular CLI 會將每個屬性模組自動新增到應用級的路由對映表中。透過新增預設路由來最終完成這些步驟。在 app-routing.module.ts 檔案中，使用如下指令更新 routes 陣列：

src/app/app-routing.module.ts

      
const routes: Routes = [
  {
    path: 'customers',
    loadChildren: () => import('./customers/customers.module').then(m => m.CustomersModule)
  },
  {
    path: 'orders',
    loadChildren: () => import('./orders/orders.module').then(m => m.OrdersModule)
  },
  {
    path: '',
    redirectTo: '',
    pathMatch: 'full'
  }
];

前兩個路徑是到 CustomersModule 和 OrdersModule 的路由。最後一個條目則定義了預設路由。空路徑對應所有不對應先前路徑的內容。

屬性模組內部

接下來，仔細看看 customers.module.ts 檔案。如果你使用的是 Angular CLI，並按照此頁面中的步驟進行操作，則無需在此處執行任何操作。

src/app/customers/customers.module.ts

      
import { NgModule } from '@angular/core';
import { CommonModule } from '@angular/common';
import { CustomersRoutingModule } from './customers-routing.module';
import { CustomersComponent } from './customers.component';
@NgModule({
  imports: [
    CommonModule,
    CustomersRoutingModule
  ],
  declarations: [CustomersComponent]
})
export class CustomersModule { }

customers.module.ts 檔案匯入了 customers-routing.module.ts 和 customers.component.ts 檔案。 CustomersRoutingModule 被列在 @NgModule 的 imports 陣列中，使 CustomersModule 能訪問其自己的路由模組。 CustomersComponent 在 declarations 陣列中，這意味著 CustomersComponent 屬於 CustomersModule。

然後，app-routing.module.ts 會使用 JavaScript 的動態匯入功能來匯入屬性模組 customers.module.ts。

專屬於屬性模組的路由定義檔案 customers-routing.module.ts 將匯入在 customers.component.ts 檔案中定義的自有屬性元件，以及其它 JavaScript 匯入語句。然後將空路徑對映到 CustomersComponent。

src/app/customers/customers-routing.module.ts

      
import { NgModule } from '@angular/core';
import { Routes, RouterModule } from '@angular/router';
import { CustomersComponent } from './customers.component';
const routes: Routes = [
  {
    path: '',
    component: CustomersComponent
  }
];
@NgModule({
  imports: [RouterModule.forChild(routes)],
  exports: [RouterModule]
})
export class CustomersRoutingModule { }

這裡的 path 設定為空字串，因為 AppRoutingModule 中的路徑已經設定為 customers，因此，CustomersRoutingModule 中的此路由已經位於 customers 這個上下文中。此路由模組中的每個路由都是其子路由。

另一個屬性模組中路由模組的設定也類似。

src/app/orders/orders-routing.module.ts（摘錄）

      
import { OrdersComponent } from './orders.component';
const routes: Routes = [
  {
    path: '',
    component: OrdersComponent
  }
];

驗證延遲載入

你可以使用 Chrome 開發者工具驗證某個模組是否確實在延遲載入。在 Chrome 中，按 ⌘ Cmd+ Option+ i（Mac）或 Ctrl+ Shift+ j（PC）開啟開發者工具，然後轉到 Network 標籤頁。

點選 Orders 或 Customers 按鈕。如果你看到某個 chunk 檔案出現了，就表示一切就緒，屬性模組被延遲載入成功了。Orders 和 Customers 都應該出現一次 chunk，並且它們各自只應該出現一次。

要再次檢視或在更改後進行測試，請點選網路頁籤左上角的帶線圈的圓圈：

然後根據你的平台，按 ⌘ Cmd+ R 或 Ctrl+ R 重新載入。

forRoot() 與 forChild()

你可能已經注意到 Angular CLI 將 RouterModule.forRoot(routes) 新增到 AppRoutingModule 的 imports 陣列中。這讓 Angular 知道 AppRoutingModule 是一個路由模組， forRoot() 指定這是根路由模組。它設定你傳遞給它的所有路由，賦予你對路由器指令的訪問許可權，並註冊 Router 服務。在應用中只使用一次 forRoot()，放在 AppRoutingModule 內。

Angular CLI 還會把 RouterModule.forChild(routes) 新增到各個屬性模組中。這種方式下 Angular 就會知道這個路由清單只負責提供額外的路由並且其設計意圖是作為屬性模組使用。你可以在多個模組中使用 forChild()。

forRoot()方法負責 Router 的 全域注入器設定。 forChild()方法沒有注入器設定。 它使用 RouterOutlet和 RouterLink等指令。更多訊息，請參見單例服務指南的 forRoot()模式部分。

預載入

預載入透過在後台載入部分應用來改進使用者體驗。你可以預載入模組、獨立元件或元件資料。

預載入模組和預載入獨立元件

預載入模組和預載入獨立元件透過在後台載入部分應用來改善使用者體驗。這樣一來，使用者在啟用路由時就無需等待下載這些元素。

要啟用所有延遲載入模組和獨立元件的預載入，從 Angular router 匯入 PreloadAllModules 標記。

基於模組的應用程式

AppRoutingModule（摘錄）

      
import { PreloadAllModules } from '@angular/router';

然後，在 AppRoutingModule 的 RouterModule.forRoot() 呼叫中指定你的預載入策略。

AppRoutingModule（摘錄）

      
RouterModule.forRoot(
  appRoutes,
  {
    preloadingStrategy: PreloadAllModules
  }
)

獨立應用

對於獨立應用，透過在 app.config.ts 中將 withPreloading 新增到 provideRouter 的 RouterFeatures 來設定預載入策略。

app.config.ts

      
import { ApplicationConfig } from '@angular/core';
import {
  PreloadAllModules,
  provideRouter
  withPreloading,
} from '@angular/router';
import { routes } from './app.routes';
export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(
      routes,
      withPreloading(PreloadAllModules)
    ),
  ],
};

預載入元件資料

要預載入元件資料，可以用 resolver 守衛。解析器透過阻止頁面載入來改進使用者體驗，直到顯示頁面時的全部必要資料都可用。

解析器

建立一個解析器服務。透過 Angular CLI，建立服務的指令如下：

      
ng generate service <service-name>

在新創建的服務中，實現 @angular/router 套件提供的 Resolve 介面：

解析器服務（節選）

      
import { Resolve } from '@angular/router';
…
/*An interface that represents your data model*/
export interface Crisis {
  id: number;
  name: string;
}
export class CrisisDetailResolverService implements Resolve<Crisis> {
  resolve(route: ActivatedRouteSnapshot, state: RouterStateSnapshot): Observable<Crisis> {
    // your logic goes here
  }
}

把這個解析器匯入此模組的路由模組。

屬性模組的路由模組（節選）

      
import { CrisisDetailResolverService } from './crisis-detail-resolver.service';

在元件的 route 設定中新增一個 resolve 物件。

屬性模組的路由模組（節選）

      
{
  path: '/your-path',
  component: YourComponent,
  resolve: {
    crisis: CrisisDetailResolverService
  }
}

在元件的建構式函式中，注入代表當前路由的 ActivatedRoute 類別的實例。

元件的建構式函式（節選）

      
import { ActivatedRoute } from '@angular/router';
@Component({ … })
class YourComponent {
  constructor(private route: ActivatedRoute) {}
}

使用注入的 ActivatedRoute 類別的實例來訪問與給定路由相關聯的 data。

元件的 ngOnInit 生命週期鉤子（節選）

      
import { ActivatedRoute } from '@angular/router';
@Component({ … })
class YourComponent {
  constructor(private route: ActivatedRoute) {}
  ngOnInit() {
    this.route.data
      .subscribe(data => {
        const crisis: Crisis = data.crisis;
        // …
      });
  }
}

對延遲載入模組進行故障排查

延遲載入模組時常見的錯誤之一，就是在應用程式中的多個位置匯入通用模組。可以先用 Angular CLI 生成模組幷包括 --route route-name 引數，來測試這種情況，其中 route-name 是模組的名稱。接下來，建立不帶 --route 引數的模組。如果你呼叫 ng generate module 時帶上了 --route 引數，就會回傳一個錯誤，否則它便可以正確執行，這樣一來你就可能會在多個位置匯入相同的模組。

請記住，許多常見的 Angular 模組都應該匯入應用的基礎模組中。

有關 Angular 模組的更多訊息，請參見 NgModules。
