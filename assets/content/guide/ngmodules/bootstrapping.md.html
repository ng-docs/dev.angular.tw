使用根模块启动应用

edit

一个 NgModule 描述了应用各部分如何组合在一起。 每个应用至少有一个 Angular 模块， 根 模块，它必须存在以便在启动时引导应用。 按照惯例和默认情况下，这个 NgModule 名为 AppModule。

当使用 Angular CLI的 ng new命令生成应用时，默认的 AppModule如下所示：

      
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { AppComponent } from './app.component';
@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

@NgModule装饰器将 AppModule标识为一个 NgModule类。 @NgModule接受一个元数据对象，告诉 Angular 如何编译和启动应用程序。

元数据字段 详情
declarations 包括 根应用程序组件。
imports 导入 BrowserModule 以启用特定于浏览器的服务（例如 DOM 渲染、消毒）
providers 服务提供者。
bootstrap Angular 创建并插入到 index.html 主机网页中的 根 组件。

declarations 数组

该模块的 declarations 数组告诉 Angular 哪些组件属于该模块。当你创建更多组件时，也要把它们添加到 declarations 中。

declarations 数组仅接受可声明项。 可声明项包括 组件、 指令 和 管道。 一个模块的所有可声明项必须在 declarations 数组中。 可声明项必须属于正好一个模块。 如果在多个模块中声明相同的类，编译器会返回错误。

这些已声明的类在模块内可用，但对于不同模块的组件是私有的，除非它们从此模块中导出并且其他模块导入此模块。

下面是一个 declarations 数组中包含内容的示例：

      
declarations: [
  YourComponent,
  YourPipe,
  YourDirective
],

在 @NgModule 中使用指令

使用 declarations 数组声明指令。在模块中使用指令、组件或管道的步骤如下：

  1. 从编写它的 TypeScript 文件中导出它
  2. 将其导入到包含 @NgModule 类的适当文件中。
  3. 在 @NgModule 的 declarations 数组中声明它。

这三个步骤看起来像以下。在创建指令的文件中，导出它。 以下示例展示了一个名为 ItemDirective 的空指令。

src/app/item.directive.ts

      
import { Directive } from '@angular/core';
@Directive({
  selector: '[appItem]'
})
export class ItemDirective {
  // your code here
}

关键点在于你必须导出它，这样你才能在其他地方导入它。 接下来，将其导入到包含你的 NgModule 的文件中。在本例中，这是 app.module.ts 文件。

src/app/app.module.ts

      
import { ItemDirective } from './item.directive';

并且在同一文件中，将其添加到 @NgModule 的 declarations 数组中：

src/app/app.module.ts

      
declarations: [
    AppComponent,
    ItemDirective
  ],

现在你可以在组件中使用 ItemDirective。 此示例使用了 AppModule，但对于功能模块，你将按照相同步骤进行。 关于指令的更多信息，请参见 属性型指令 和 结构型指令。 对于 管道 和 组件，你也将使用相同的技术。

记住：组件、指令和管道都只能属于一个模块。你在应用中也只需要声明它们一次，因为你还可以通过导入必要的模块来使用它们。这能节省你的时间，并且帮助你的应用保持精简。

imports 数组

模块在 @NgModule 元数据对象中接受一个 imports 数组。 它告诉 Angular 关于其他 NgModules，这个特定模块需要以正常运行。

src/app/app.module.ts

      
imports: [
    BrowserModule,
    FormsModule,
    HttpClientModule
  ],

这些模块列表是那些导出组件、指令或管道，这些组件模板在该模块中引用。 在这种情况下，组件是 AppComponent，它引用了 BrowserModule、 FormsModule 或 HttpClientModule 中的组件、指令或管道。 当组件模板引用另一个组件、指令或管道时，若被引用的类在该模块中声明，或者该类从另一个模块导入，则可以引用。

providers 数组

提供者数组是你列出应用程序需要的服务的地方。 当你在这里列出服务时，它们在整个应用程序中可用。 在使用功能模块和延迟加载时，你可以对其进行作用域限制。 更多信息，请参见 模块中的提供者。

bootstrap 数组

应用程序通过引导根 AppModule来启动。 引导过程会创建在 bootstrap数组中列出的组件，并将每个组件插入到浏览器DOM中，如果找到与组件的 selector匹配的元素。

每个引导组件都是其自己组件树的基础。 插入一个引导组件通常会触发一系列组件创建的级联过程，来构建该树。 虽然你可以在一个宿主网页上放置多个组件树，但大多数应用程序只有一个组件树，并引导单个根组件。

根组件通常称为 AppComponent，位于根模块的 bootstrap数组中。

在想要基于API响应引导组件的情况下， 或者想要挂载 AppComponent到不匹配组件选择器的不同DOM节点时，请参考 ApplicationRef.bootstrap()文档。
