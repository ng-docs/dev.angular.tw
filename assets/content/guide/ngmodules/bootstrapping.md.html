使用根模組啟動應用

edit

一個 NgModule 描述了應用各部分如何組合在一起。 每個應用至少有一個 Angular 模組， 根 模組，它必須存在以便在啟動時引導應用。 按照慣例和預設情況下，這個 NgModule 名為 AppModule。

當使用 Angular CLI的 ng new指令生成應用時，預設的 AppModule如下所示：

      
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { AppComponent } from './app.component';
@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

@NgModule裝飾器將 AppModule標識為一個 NgModule類。 @NgModule接受一個元資料物件，告訴 Angular 如何編譯和啟動應用程式。

元資料欄位 詳情
declarations 包括 根應用程式元件。
imports 匯入 BrowserModule 以啟用特定於瀏覽器的服務（例如 DOM 渲染、消毒）
providers 服務提供者。
bootstrap Angular 建立並插入到 index.html 主機網頁中的 根 元件。

declarations 陣列

該模組的 declarations 陣列告訴 Angular 哪些元件屬於該模組。當你建立更多元件時，也要把它們新增到 declarations 中。

declarations 陣列僅接受可宣告項。 可宣告項包括 元件、 指令 和 通道。 一個模組的所有可宣告項必須在 declarations 陣列中。 可宣告項必須屬於正好一個模組。 如果在多個模組中宣告相同的類，編譯器會回傳錯誤。

這些已宣告的類在模組內可用，但對於不同模組的元件是私有的，除非它們從此模組中匯出並且其他模組匯入此模組。

下面是一個 declarations 陣列中包含內容的範例：

      
declarations: [
  YourComponent,
  YourPipe,
  YourDirective
],

在 @NgModule 中使用指令

使用 declarations 陣列宣告指令。在模組中使用指令、元件或通道的步驟如下：

  1. 從編寫它的 TypeScript 檔案中匯出它
  2. 將其匯入到包含 @NgModule 類別的適當檔案中。
  3. 在 @NgModule 的 declarations 陣列中宣告它。

這三個步驟看起來像以下。在建立指令的檔案中，匯出它。 以下範例展示了一個名為 ItemDirective 的空指令。

src/app/item.directive.ts

      
import { Directive } from '@angular/core';
@Directive({
  selector: '[appItem]'
})
export class ItemDirective {
  // your code here
}

關鍵點在於你必須匯出它，這樣你才能在其他地方匯入它。 接下來，將其匯入到包含你的 NgModule 的檔案中。在本例中，這是 app.module.ts 檔案。

src/app/app.module.ts

      
import { ItemDirective } from './item.directive';

並且在同一檔案中，將其新增到 @NgModule 的 declarations 陣列中：

src/app/app.module.ts

      
declarations: [
    AppComponent,
    ItemDirective
  ],

現在你可以在元件中使用 ItemDirective。 此範例使用了 AppModule，但對於功能模組，你將按照相同步驟進行。 關於指令的更多訊息，請參見 屬性型指令 和 結構型指令。 對於 通道 和 元件，你也將使用相同的技術。

記住：元件、指令和通道都只能屬於一個模組。你在應用中也只需要宣告它們一次，因為你還可以透過匯入必要的模組來使用它們。這能節省你的時間，並且幫助你的應用保持精簡。

imports 陣列

模組在 @NgModule 元資料物件中接受一個 imports 陣列。 它告訴 Angular 關於其他 NgModules，這個特定模組需要以正常執行。

src/app/app.module.ts

      
imports: [
    BrowserModule,
    FormsModule,
    HttpClientModule
  ],

這些模組清單是那些匯出元件、指令或通道，這些元件樣板在該模組中引用。 在這種情況下，元件是 AppComponent，它引用了 BrowserModule、 FormsModule 或 HttpClientModule 中的元件、指令或通道。 當元件樣板引用另一個元件、指令或通道時，若被引用的類在該模組中宣告，或者該類別從另一個模組匯入，則可以引用。

providers 陣列

提供者陣列是你列出應用程式需要的服務的地方。 當你在這裡列出服務時，它們在整個應用程式中可用。 在使用功能模組和延遲載入時，你可以對其進行作用域限制。 更多訊息，請參見 模組中的提供者。

bootstrap 陣列

應用程式透過引導根 AppModule來啟動。 引導過程會建立在 bootstrap陣列中列出的元件，並將每個元件插入到瀏覽器DOM中，如果找到與元件的 selector對應的元素。

每個引導元件都是其自己元件樹的基礎。 插入一個引導元件通常會觸發一系列元件建立的級聯過程，來建立該樹。 雖然你可以在一個宿主網頁上放置多個元件樹，但大多數應用程式只有一個元件樹，並引導單個根元件。

根元件通常稱為 AppComponent，位於根模組的 bootstrap陣列中。

在想要基於API回應引導元件的情況下， 或者想要掛載 AppComponent到不對應元件選擇器的不同DOM節點時，請參考 ApplicationRef.bootstrap()文件。
