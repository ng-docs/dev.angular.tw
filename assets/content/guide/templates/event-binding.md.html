事件繫結

edit

透過事件繫結，你可以偵聽並回應使用者操作，比如按鍵、滑鼠移動、點選和觸控。

繫結到事件

提示: 關於屬性繫結的訊息，請參閱 屬性繫結。

要繫結到事件，你可以使用 Angular 事件繫結語法。此語法由等號左側括號中的目標事件名稱和右側帶引號的範本語句組成。

建立以下範例；目標事件名是 click，範本語句是 onSave()。

事件繫結語法

      
<button (click)="onSave()">Save</button>

事件繫結偵聽按鈕的單擊事件，並在發生單擊時調用元件的 onSave()。

確定事件目標

為了確定事件目標，Angular 會檢查目標事件的名稱是否與已知指令的 event 屬性對應。

建立以下範例：（Angular 會檢查 myClick 是否是自定義 ClickDirective 上的事件）

src/app/app.component.html

      
<h1 id="event-binding">Event Binding</h1>
<div class="group">
  <h3>Target event</h3>
  <button type="button" (click)="onSave($event)">Save</button>
  <button type="button" on-enter="onSave()">on-click Save</button>
  <h4>myClick is an event on the custom ClickDirective:</h4>
  <button type="button" (myClick)="clickMessage=$event" clickable>click with myClick</button>
  {{ clickMessage }}
</div>
<div class="group">
  <h3>$event and event handling statements</h3>
  <h4>Result: {{ currentItem.name }}</h4>
  <input [value]="currentItem.name"
         (input)="currentItem.name=getValue($event)">
  without NgModel
</div>
<div class="group">
  <h3>Binding to a nested component</h3>
  <h4>Custom events with EventEmitter</h4>
  <app-item-detail (deleteRequest)="deleteItem($event)" [item]="currentItem"></app-item-detail>
  <h4>Click to see event target class:</h4>
<div class="parent-div" (click)="onClickMe($event)" clickable>Click me (parent)
  <div class="child-div">Click me too! (child) </div>
</div>
<h3>Saves only once:</h3>
<div (click)="onSave()" clickable>
  <button type="button" (click)="onSave($event)">Save, no propagation</button>
</div>
<h3>Saves twice:</h3>
<div (click)="onSave()" clickable>
  <button type="button" (click)="onSave()">Save with propagation</button>
</div>

如果目標事件名稱 myClick 與 ClickDirective 的輸出屬性不對應，Angular 將改為繫結到基礎 DOM 元素上的 myClick 事件。

繫結到鍵盤事件

你可以使用 Angular 的綁定語法來繫結鍵盤事件。你可以指定要繫結到鍵盤事件的鍵或程式碼。 key和 code欄位是瀏覽器鍵盤事件物件的本機部分。預設情況下，事件繫結假定你想要使用鍵盤事件的 key欄位。你也可以使用 code欄位。

鍵的組合可以用點（.）分隔。例如，keydown.enter 將允許你將事件繫結到 enter 鍵。你還可以用修飾鍵，例如 shift 、 alt 、 control 和 Mac 中的 command 鍵。以下範例展示了如何將鍵盤事件繫結到 keydown.shift.t。

<input (keydown.shift.t)="onKeydown($event)" />

根據作業系統的不同，某些組合鍵可能會建立特殊字元，而不是你期望的組合鍵。例如，當你同時使用 option 和 shift 鍵時，MacOS 會建立特殊字元。如果你繫結到 keydown.shift.alt.t，在 macOS 上，該組合會生成 ˇ 而不是 t，它與繫結不對應，也不會觸發你的事件處理程式。要繫結到 macOS 上的 keydown.shift.alt.t，請使用 code 鍵盤事件欄位來獲取正確的行為，例如此範例中顯示的 keydown.code.shiftleft.altleft.keyt。

<input (keydown.code.shiftleft.altleft.keyt)="onKeydown($event)" />

code 欄位比 key 欄位更具體。key 欄位總是會報告 shift，而 code 欄位將指明 leftshift 或 rightshift。使用 code 欄位時，你可能需要新增單獨的繫結以捕獲你想要的所有行為。使用 code 欄位時無需處理作業系統特有的行為，例如 macOS 上的 shift + option 行為。

欲瞭解更多訊息，請訪問 key和 code的完整參考，以幫助建立你的事件字串。

繫結到被動事件

Angular 還支援 被動事件監聽器。

這是一種高階技術，對大多數應用程式來說並非必需。如果需要最佳化處理頻繁發生事件而導致效能問題的情況，你可能會發現這很有用。

例如，使用以下步驟將捲動事件設為被動事件。

  1. 在 src 目錄下建立一個檔案 zone-flags.ts。

  2. 將以下行新增到此檔案中。

          
    (window as any)['__zone_symbol__PASSIVE_EVENTS'] = ['scroll'];

  3. 在 src/polyfills.ts 檔案中，在匯入 zone.js 之前，先導入新創建的 zone-flags。

          
    import './zone-flags';
    import 'zone.js';  // Included with Angular CLI.

在這些步驟之後，如果你為 scroll 事件新增事件偵聽器，偵聽器就會是 passive 的。
