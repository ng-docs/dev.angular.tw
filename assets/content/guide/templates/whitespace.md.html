範本中的空白符

edit

預設情況下，Angular 範本不會保留框架認為不必要的空白符。這通常發生在兩種情況下：元素之間的空白符和文字內部可摺疊的空白符。

元素之間的空白符

大多數開發人員喜歡使用換行符和縮排符來格式化他們的範本，以提高範本的可讀性：

      
<section>  <h3>User profile</p>  <label>    User name    <input>  </label></section>

此範本在所有元素之間都包含空白符。以下程式碼段落顯示了相同的 HTML，其中每個空白字元都替換為井號 (#) 字元，以突出顯示存在多少空白符：

      
<!-- Total Whitespace: 20 --><section>###<h3>User profile</p>###<label>#####User name#####<input>###</label>#</section>

保留範本中編寫的空白符會導致許多不必要的 文字節點，並增加頁面渲染開銷。透過忽略元素之間的這些空白符，Angular 在頁面上渲染範本時執行的工作更少，從而提高了整體性能。

文字內部可摺疊的空白符

當你的 Web 瀏覽器在頁面上渲染 HTML 時，它會將多個連續的空白字元摺疊為一個字元：

      
<!-- What it looks like in the template --><p>Hello         world</p>

在此範例中，瀏覽器僅在 『Hello』 和 『world』 之間顯示一個空格。

      
<!-- What shows up in the browser --><p>Hello world</p>

有關其工作原理的更多上下文，請參閱 HTML、CSS 和 DOM 如何處理空白符。

Angular 透過在編譯範本時將這些不必要的空白字元摺疊為一個字元，從而從一開始就避免將它們傳送到瀏覽器。

保留空白符

你可以透過在範本的 @Component 裝飾器中指定 preserveWhitespaces: true 來告訴 Angular 保留範本中的空白符。

      
@Component({  /* ... */,  preserveWhitespaces: true,  template: `    <p>Hello         world</p>  `})

除非絕對必要，否則請避免設定此選項。保留空白符可能會導致 Angular 在渲染時生成明顯更多的節點，從而減慢你的應用。

你還可以使用 Angular 特有的特殊 HTML 實體 &ngsp;。此實體生成一個空格字元，該字元會保留在編譯後的輸出中。
