表示式語法

edit

Angular 表示式基於 JavaScript，但在一些關鍵方面有所不同。本指南將介紹 Angular 表示式和標準 JavaScript 之間的異同。

字面量值

Angular 支援 JavaScript 的 字面量值 的子集。

支援的字面量值

字面量型別 範例值
字串 'Hello', "World"
布林值 true, false
數字 123, 3.14
物件 {name: 'Alice'}
陣列 ['Onion', 'Cheese', 'Garlic']
null null
樣板字串 `Hello ${name}`

不支援的字面量

字面量型別 範例值
RegExp /\d+/
帶標籤的樣板字串 tag`Hello ${name}`

全域變數

Angular 表示式支援以下全域變數：

  • undefined
  • $any

不支援其他 JavaScript 全域變數。常見的 JavaScript 全域變數包括 Number、Boolean、NaN、Infinity、parseInt 等。

區域性變數

在特定上下文中，Angular 會自動提供特殊的區域性變數以在表示式中使用。這些特殊變數始終以美元符號 ($) 開頭。

例如，@for 控制區塊會建立多個與迴圈訊息相對應的區域性變數，例如 $index。

支援哪些運算子？

支援的運算子

Angular 支援來自標準 JavaScript 的以下運算子。

運算子 範例
加法 / 連線 1 + 2
減法 52 - 3
乘法 41 * 6
除法 20 / 4
求餘（取模） 17 % 5
圓括號 9 * (8 + 4)
條件（三元） a > b ? true : false
與（邏輯與） &&
或（邏輯或） ||
非（邏輯非） !
空值合併 const foo = null ?? 'default'
比較運算符 <、<=、>、>=、==、===、!==
一元取反 const y = -x
一元加 const x = +y
屬性訪問器 person['name'] = 'Mirabel'

此外，Angular 表示式還支援以下非標準運算子：

運算子 範例
通道 {{ total | currency }}
可選鏈式呼叫* someObj.someProp?.nestedProp
非空斷言（TypeScript） someObj!.someProp

*注意：可選鏈式呼叫的行為與標準 JavaScript 版本不同，如果 Angular 可選鏈式呼叫運算子的左側是 null 或 undefined，它將回傳 null 而不是 undefined。

不支援的運算子

運算子 範例
所有位運算符 &、&=、~、|=、^= 等。
賦值運算符 =
物件解構 const { name } = person
陣列解構 const [firstItem] = items
逗號運算符 x = (x++, x)
typeof typeof 42
void void 1
in 'model' in car
instanceof car instanceof Automobile
new new Car()

表示式的詞法上下文

Angular 表示式是在元件類別的上下文中求值的，以及任何相關的 樣板變數、區域性變數和全域變數。

當引用類別成員時，總是預設使用 this。

物件宣告

一般來說，Angular 表示式不支援物件宣告。這包括但不限於：

物件宣告 範例
變數 let label = 'abc'，const item = 'apple'
函式 function myCustomFunction() { }
箭頭函式 () => { }
類 class Rectangle { }

事件監聽器語句

edit

事件處理程式是語句而不是表示式。雖然它們支援與 Angular 表示式相同的所有語法，但有兩個主要區別：

  1. 語句確實支援賦值運算符（但不支援解構賦值）
  2. 語句不支援通道
