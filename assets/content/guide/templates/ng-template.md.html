使用 ng-template 建立樣板片段

edit

受原生 <template> 元素的啟發，<ng-template> 元素允許你宣告一個樣板片段 —— 一段你可以動態或以程式設計方式渲染的內容。

建立一個樣板片段

你可以在任何元件樣板內使用 <ng-template> 元素建立一個樣板片段：

      
<p>This is a normal element</p><ng-template>  <p>This is a template fragment</p></ng-template>

當上述程式碼渲染時，<ng-template> 元素的內容不會在頁面上渲染。相反，你可以獲得對樣板片段的引用，並編寫程式碼來動態渲染它。

片段的繫結上下文

樣板片段可能包含帶有動態表示式的繫結：

      
@Component({  /* ... */,  template: `<ng-template>You've selected {{count}} items.</ng-template>`,})export class ItemCounter {  count: number = 0;}

樣板片段中的表示式或語句是針對宣告該片段的元件求值的，無論該片段在何處渲染。

獲取對樣板片段的引用

你可以透過以下三種方式之一獲取對樣板片段的引用：

  • 透過在 <ng-template> 元素上聲明一個樣板引用變數
  • 透過使用元件或指令查詢查詢片段
  • 透過在直接應用於 <ng-template> 元素的指令中注入片段。

在這三種情況下，片段都由 TemplateRef 物件表示。

使用樣板引用變數引用樣板片段

你可以向 <ng-template> 元素新增一個樣板引用變數，以便在同一樣板檔案的其他部分引用該樣板片段：

      
<p>This is a normal element</p><ng-template #myFragment>  <p>This is a template fragment</p></ng-template>

然後，你可以透過 myFragment 變數在樣板中的任何其他位置引用此片段。

使用查詢引用樣板片段

你可以使用任何元件或指令查詢 API來獲取對樣板片段的引用。

例如，如果你的樣板只有一個樣板片段，你可以使用 @ViewChild 查詢直接查詢 TemplateRef 物件：

      
@Component({  /* ... */,  template: `    <p>This is a normal element</p>    <ng-template>      <p>This is a template fragment</p>    </ng-template>  `,})export class ComponentWithFragment {  @ViewChild(TemplateRef) myFragment: TemplateRef<unknown> | undefined;}

然後，你可以像引用任何其他類別成員一樣，在元件程式碼或元件的樣板中引用此片段。

如果一個樣板包含多個片段，你可以透過向每個 <ng-template> 元素新增一個樣板引用變數並根據該名稱查詢片段，從而為每個片段分配一個名稱：

      
@Component({  /* ... */,  template: `    <p>This is a normal element</p>    <ng-template #fragmentOne>      <p>This is one template fragment</p>    </ng-template>    <ng-template #fragmentTwo>      <p>This is another template fragment</p>    </ng-template>  `,})export class ComponentWithFragment {  // When querying by name, you can use the `read` option to specify that you want to get the  // TemplateRef object associated with the element.  @ViewChild('fragmentOne', {read: TemplateRef}) fragmentOne: TemplateRef<unknown> | undefined;  @ViewChild('fragmentTwo', {read: TemplateRef}) fragmentTwo: TemplateRef<unknown> | undefined;}

同樣，然後你可以像引用任何其他類別成員一樣，在元件程式碼或元件的樣板中引用這些片段。

注入樣板片段

如果指令直接應用於 <ng-template> 元素，則該指令可以注入 TemplateRef：

      
@Directive({  selector: '[myDirective]'})export class MyDirective {  private fragment = inject(TemplateRef);}

    
      
<ng-template myDirective>  <p>This is one template fragment</p></ng-template>

然後，你可以像引用任何其他類別成員一樣，在指令程式碼中引用此片段。

渲染樣板片段

一旦你獲得了對樣板片段的 TemplateRef 物件的引用，你就可以透過兩種方式之一渲染片段：在你的樣板中使用 NgTemplateOutlet 指令，或者在你的 TypeScript 程式碼中使用 ViewContainerRef。

使用 NgTemplateOutlet

來自 @angular/common 的 NgTemplateOutlet 指令接受一個 TemplateRef，並將該片段渲染為帶有出口元素的同級元素。你通常應該在 <ng-container> 元素上使用 NgTemplateOutlet。

首先，匯入 NgTemplateOutlet：

      
import { NgTemplateOutlet } from '@angular/common';

以下範例宣告了一個樣板片段，並使用 NgTemplateOutlet 將該片段渲染到 <ng-container> 元素：

      
<p>This is a normal element</p><ng-template #myFragment>  <p>This is a fragment</p></ng-template><ng-container *ngTemplateOutlet="myFragment"></ng-container>

此範例生成以下渲染後的 DOM：

      
<p>This is a normal element</p><p>This is a fragment</p>

使用 ViewContainerRef

檢視容器是 Angular 元件樹中的一個節點，可以包含內容。任何元件或指令都可以注入 ViewContainerRef 以獲得對與該組件或指令在 DOM 中的位置相對應的檢視容器的引用。

你可以使用 ViewContainerRef 上的 createEmbeddedView 方法來動態渲染樣板片段。當你使用 ViewContainerRef 渲染片段時，Angular 會將其作為注入 ViewContainerRef 的元件或指令的下一個同級元素追加到 DOM 中。

以下範例展示了一個元件，該組件接受對樣板片段的引用作為輸入，並在按鈕單擊時將該片段渲染到 DOM 中。

      
@Component({  /* ... */,  selector: 'component-with-fragment',  template: `    <h2>Component with a fragment</h2>    <ng-template #myFragment>      <p>This is the fragment</p>    </ng-template>    <my-outlet [fragment]="myFragment" />  `,})export class ComponentWithFragment { }@Component({  /* ... */,  selector: 'my-outlet',  template: `<button (click)="showFragment()">Show</button>`,})export class MyOutlet {  private viewContainer = inject(ViewContainerRef);  @Input() fragment: TemplateRef<unknown> | undefined;  showFragment() {    if (this.fragment) {      this.viewContainer.createEmbeddedView(this.fragment);    }  }}

在上面的範例中，單擊『Show』按鈕會產生以下輸出：

      
<component-with-fragment>  <h2>Component with a fragment>  <my-outlet>    <button>Show</button>  </my-outlet>  <p>This is the fragment</p></component-with-fragment>

在渲染樣板片段時傳遞引數

當使用 <ng-template> 宣告樣板片段時，你可以額外宣告片段接受的引數。當你渲染片段時，你可以選擇性地傳遞與這些引數對應的 context 物件。除了引用宣告片段的元件中的資料外，你還可以在繫結表示式和語句中使用來自此 context 物件的資料。

每個引數都寫成一個以 let- 為字首的屬性，其值與 context 物件中的屬性名稱對應：

      
<ng-template let-pizzaTopping="topping">  <p>You selected: {{pizzaTopping}}</p></ng-template>

使用 NgTemplateOutlet

你可以將 context 物件繫結到 ngTemplateOutletContext 輸入：

      
<ng-template #myFragment let-pizzaTopping="topping">  <p>You selected: {{pizzaTopping}}</p></ng-template><ng-container  [ngTemplateOutlet]="myFragment"  [ngTemplateOutletContext]="{topping: 'onion'}"/>

使用 ViewContainerRef

你可以將 context 物件作為第二個引數傳遞給 createEmbeddedView：

      
this.viewContainer.createEmbeddedView(this.myFragment, {topping: 'onion'});

結構型指令

結構型指令是任何滿足以下條件的指令：

  • 注入 TemplateRef
  • 注入 ViewContainerRef 並以程式設計方式渲染注入的 TemplateRef

Angular 為結構型指令提供了一種特殊的便捷語法。如果將指令應用於元素，並在指令的選擇器前加上星號 (*) 字元，Angular 會將整個元素及其所有內容解釋為樣板片段：

      
<section *myDirective>  <p>This is a fragment</p></section>

這等效於：

      
<ng-template myDirective>  <section>    <p>This is a fragment</p>  </section></ng-template>

開發人員通常使用結構型指令來有條件地渲染片段或多次渲染片段。

有關更多詳細訊息，請參閱結構型指令。
