範本語法

edit

在 Angular 中，範本是一段 HTML。 在範本中使用特殊的語法可以充分利用 Angular 的許多屬性。

提示：在深入學習本綜合指南之前，請檢視 Angular 的要點。

每個 Angular 元件都有一個範本，用於定義元件渲染到頁面上的 DOM。透過使用範本，Angular 能夠在資料更改時自動保持頁面最新。

範本通常位於 *.component.ts 檔案的 template 屬性或 *.component.html 檔案中。要了解更多訊息，請檢視深入元件指南。

範本是如何工作的？

範本基於 HTML 語法，並具有其他屬性，例如內建範本函式、資料繫結、事件監聽、變數等等。

Angular 將範本編譯成 JavaScript，以便建立對你的應用的內部理解。這樣做的好處之一是 Angular 會自動將內建的渲染最佳化應用於你的應用。

與標準 HTML 的差異

範本和標準 HTML 語法之間的一些差異包括：

  • 範本原始碼中的註解不包含在渲染後的輸出中
  • 元件和指令元素可以是自閉合的（例如，<UserProfile />）
  • 具有某些字元（即 []、() 等）的 Attribute 對 Angular 具有特殊含義。有關更多訊息，請參閱繫結文件和新增事件監聽器文件。
  • @ 字元對於 Angular 具有特殊含義，用於向範本新增動態行為，例如控制區塊。你可以透過將其轉義為 HTML 實體程式碼（&commat; 或 &#64;）來包含字面量 @ 字元。
  • Angular 會忽略和摺疊不必要的空白字元。有關更多詳細訊息，請參閱範本中的空白。
  • Angular 可能會向頁面新增註解節點，作為動態內容的佔位符，但開發人員可以忽略這些註解節點。

此外，雖然大多數 HTML 語法都是有效的範本語法，但 Angular 不支援範本中的 <script> 元素。有關更多訊息，請參閱安全性頁面。
