通道

edit

概覽

通道是 Angular 樣板表示式中的一種特殊運算子，允許你在樣板中宣告式地轉換資料。通道允許你宣告一次轉換函式，然後在多個樣板中複用該轉換。Angular 通道使用豎線字元 (|)，其靈感來源於 Unix 通道。

注意： Angular 的通道語法與標準 JavaScript 不同，標準 JavaScript 使用豎線字元作為 按位或運算符。Angular 樣板表示式不支援按位運算符。

這裡有一個範例，展示了 Angular 提供的一些內建通道：

      
import { Component } from '@angular/core';import { CurrencyPipe, DatePipe, TitleCasePipe } from '@angular/common';@Component({  selector: 'app-root',  imports: [CurrencyPipe, DatePipe, TitleCasePipe],  template: `    <main>       <!-- Transform the company name to title-case and       transform the purchasedOn date to a locale-formatted string --><h1>Purchases from {{ company | titlecase }} on {{ purchasedOn | date }}</h1>	    <!-- Transform the amount to a currency-formatted string -->      <p>Total: {{ amount | currency }}</p>    </main>  `,})export class ShoppingCartComponent {  amount = 123.45;  company = 'acme corporation';  purchasedOn = '2024-07-08';}

當 Angular 渲染元件時，它將確保適當的日期格式和貨幣基於使用者的語言環境。如果使用者在美國，它將渲染為：

      
<main>  <h1>Purchases from Acme Corporation on Jul 8, 2024</h1>  <p>Total: $123.45</p></main>

請參閱 關於 i18n 的深入指南，以瞭解更多關於 Angular 如何本地化值的訊息。

內建通道

Angular 在 @angular/common 套件中包含了一組內建通道：

名稱 描述
AsyncPipe 從 Promise 或 RxJS Observable 中讀取值。
CurrencyPipe 根據語言環境規則，將數字轉換為貨幣字串。
DatePipe 根據語言環境規則，格式化 Date 值。
DecimalPipe 根據語言環境規則，將數字轉換為帶小數點的字串。
I18nPluralPipe 根據語言環境規則，將值對映到對該值進行複數化的字串。
I18nSelectPipe 將鍵對映到回傳所需值的自定義選擇器。
JsonPipe 透過 JSON.stringify 將物件轉換為字串表示形式，用於除錯。
KeyValuePipe 將 Object 或 Map 轉換為鍵值對陣列。
LowerCasePipe 將文字轉換為全小寫。
PercentPipe 根據語言環境規則，將數字轉換為百分比字串。
SlicePipe 建立一個新的 Array 或 String，其中包含元素的子集（切片）。
TitleCasePipe 將文字轉換為首字母大寫。
UpperCasePipe 將文字轉換為全大寫。

使用通道

Angular 的通道運算子在樣板表示式中使用豎線字元 (|)。通道運算子是一個二元運算符 —— 左側操作數是傳遞給轉換函式的值，右側操作數是通道的名稱和任何附加引數（如下所述）。

      
<p>Total: {{ amount | currency }}</p>

在此範例中，amount 的值被傳遞到 CurrencyPipe 中，其中通道名稱為 currency。然後，它為使用者的語言環境渲染預設貨幣。

在同一表示式中組合多個通道

你可以使用多個通道運算子對一個值應用多個轉換。Angular 從左到右執行通道。

以下範例示範了通道的組合，以全大寫形式顯示本地化的日期：

      
<p>The event will occur on {{ scheduledOn | date | uppercase }}.</p>

向通道傳遞引數

一些通道接受引數來設定轉換。要指定引數，請在通道名稱後附加一個冒號 (:)，後跟引數值。

例如，DatePipe 能夠接受引數以特定方式格式化日期。

      
<p>The event will occur at {{ scheduledOn | date:'hh:mm' }}.</p>

一些通道可能接受多個引數。你可以指定其他引數值，並用冒號字元 (:) 分隔。

例如，我們還可以傳遞第二個可選引數來控制時區。

      
<p>The event will occur at {{ scheduledOn | date:'hh:mm':'UTC' }}.</p>

通道的工作原理

從概念上講，通道是接受輸入值並回傳轉換後值的函式。

      
import { Component } from '@angular/core';import { CurrencyPipe} from '@angular/common';@Component({  selector: 'app-root',  imports: [CurrencyPipe],  template: `    <main>      <p>Total: {{ amount | currency }}</p>    </main>  `,})export class AppComponent {  amount = 123.45;}

在此範例中：

  1. CurrencyPipe 從 @angular/common 匯入
  2. CurrencyPipe 被新增到 imports 陣列中
  3. amount 資料被傳遞到 currency 通道

通道運算子優先級

通道運算子的優先級低於其他二元運算符，包括 +、-、*、/、%、&&、|| 和 ??。

      
<!-- firstName and lastName are concatenated before the result is passed to the uppercase pipe -->{{ firstName + lastName | uppercase }}

通道運算子的優先級高於條件（三元）運算符。

      
{{ (isAdmin ? 'Access granted' : 'Access denied') | uppercase }}

如果在沒有括號的情況下編寫相同的表示式：

      
{{ isAdmin ? 'Access granted' : 'Access denied' | uppercase }}

它將被解析為：

      
{{ isAdmin ? 'Access granted' : ('Access denied' | uppercase) }}

當運算子優先級可能不明確時，請始終在表示式中使用括號。

使用通道進行變更檢測

預設情況下，所有通道都被認為是純通道，這意味著它僅在原始輸入值（例如 String、Number、Boolean 或 Symbol）或更改的物件引用（例如 Array、Object、Function 或 Date）發生變化時執行。純通道提供了效能優勢，因為如果傳遞的值沒有更改，Angular 可以避免呼叫轉換函式。

因此，這意味著除非整個物件或陣列引用被替換為不同的實例，否則不會檢測到對物件屬性或陣列條目的修改。如果你想要這種級別的變更檢測，請參閱 檢測陣列或物件內部的更改。

建立自定義通道

你可以透過使用 @Pipe 裝飾器實現一個 TypeScript 類來定義自定義通道。一個通道必須具備以下兩點：

  • 一個名稱，在通道裝飾器中指定
  • 一個名為 transform 的方法，用於執行值轉換。

TypeScript 類還應該實現 PipeTransform 介面，以確保它滿足通道的型別簽名。

這是一個自定義通道的範例，它將字串轉換為 kebab case：

      
// kebab-case.pipe.tsimport { Pipe, PipeTransform } from '@angular/core';@Pipe({  name: 'kebabCase',})export class KebabCasePipe implements PipeTransform {  transform(value: string): string {    return value.toLowerCase().replace(/ /g, '-');  }}

使用 @Pipe 裝飾器

當建立自定義通道時，從 @angular/core 套件中匯入 Pipe，並將其用作 TypeScript 類別的裝飾器。

      
import { Pipe } from '@angular/core';@Pipe({  name: 'myCustomTransformation',})export class MyCustomTransformationPipe {}

@Pipe 裝飾器需要一個 name，用於控制通道在樣板中的使用方式。

自定義通道的命名約定

自定義通道的命名約定由兩個約定組成：

  • name - 推薦使用 camelCase。不要使用連字元。
  • class name - name 的 PascalCase 版本，並在末尾附加 Pipe

實現 PipeTransform 介面

除了 @Pipe 裝飾器之外，自定義通道應該始終實現來自 @angular/core 的 PipeTransform 介面。

      
import { Pipe, PipeTransform } from '@angular/core';@Pipe({  name: 'myCustomTransformation',})export class MyCustomTransformationPipe implements PipeTransform {}

實現此介面可確保你的通道類具有正確的結構。

轉換通道的值

每次轉換都由 transform 方法呼叫，第一個引數是要傳入的值，回傳值是轉換後的值。

      
import { Pipe, PipeTransform } from '@angular/core';@Pipe({  name: 'myCustomTransformation',})export class MyCustomTransformationPipe implements PipeTransform {  transform(value: string): string {    return `My custom transformation of ${value}.`  }}

向自定義通道新增引數

你可以透過向 transform 方法新增額外的引數，來向你的轉換新增引數：

      
import { Pipe, PipeTransform } from '@angular/core';@Pipe({  name: 'myCustomTransformation',})export class MyCustomTransformationPipe implements PipeTransform {  transform(value: string, format: string): string {    let msg = `My custom transformation of ${value}.`    if (format === 'uppercase') {      return msg.toUpperCase()    } else {      return msg    }  }}

檢測陣列或物件內部的變化

當你希望通道檢測陣列或物件內部的變化時，必須透過傳遞 pure 標誌並將其值設為 false，將其標記為非純函式。

除非絕對必要，否則應避免建立非純通道，因為如果不小心使用，它們可能會導致顯著的效能損失。

      
import { Pipe, PipeTransform } from '@angular/core';@Pipe({  name: 'joinNamesImpure',  pure: false,})export class JoinNamesImpurePipe implements PipeTransform {  transform(names: string[]): string {    return names.join();  }}

Angular 開發人員通常採用在通道 name 和類別名稱中包含 Impure 的約定，以向其他開發人員表明潛在的效能陷阱。
