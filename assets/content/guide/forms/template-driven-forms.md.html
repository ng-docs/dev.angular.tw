建立範本驅動型表單

edit

本教學課程向你展示如何建立一個範本驅動型表單。表單中的控制元件元素繫結到具有輸入驗證的資料屬性（Property）。輸入驗證有助於維護資料完整性和樣式，從而改善使用者體驗。

範本驅動型表單使用雙向資料繫結，以便在範本中進行更改時更新元件中的資料模型，反之亦然。

範本驅動型表單 vs 回應式表單

Angular 支援互動式表單的兩種設計方法。範本驅動型表單允許你在 Angular 範本中使用特定於表單的指令。回應式表單提供了一種模型驅動的方法來建立表單。

範本驅動型表單是小型或簡單表單的絕佳選擇，而回應式表單更具延展性，更適合複雜表單。有關這兩種方法的比較，請參閱選擇一種方法

你可以使用 Angular 範本建立幾乎任何型別的表單 —— 登入表單、聯絡表單以及幾乎任何業務表單。 你可以創造性地佈置控制元件並將它們繫結到物件模型中的資料。 你可以指定驗證規則並顯示驗證錯誤，有條件地允許來自特定控制元件的輸入，觸發內建的可視反饋等等。

目標

本教學課程教你如何執行以下操作：

  • 使用元件和範本建立 Angular 表單
  • 使用 ngModel 建立雙向資料繫結，以讀取和寫入輸入控制元件值
  • 使用特殊的 CSS 類別提供視覺反饋，以跟蹤控制元件的狀態
  • 向用戶顯示驗證錯誤，並根據表單狀態有條件地允許來自表單控制元件的輸入
  • 使用範本引用變數在 HTML 元素之間共享訊息

建立範本驅動型表單

範本驅動型表單依賴於 FormsModule 中定義的指令。

指令 詳情
NgModel 協調附著的表單元素中的值更改與資料模型中的更改，從而允許你使用輸入驗證和錯誤處理來回應使用者輸入。
NgForm 建立一個頂級的 FormGroup 實例，並將其繫結到 <form> 元素，以跟蹤聚合的表單值和驗證狀態。一旦你匯入了 FormsModule，此指令就會預設在所有 <form> 標籤上啟用。你無需新增特殊的選擇器。
NgModelGroup 建立 FormGroup 實例並將其繫結到 DOM 元素。

步驟概覽

在本教學課程中，你將透過以下步驟將範例表單繫結到資料並處理使用者輸入。

  1. 建立基本表單。
    • 定義範例資料模型
    • 包含必要的 инфраструктура，例如 FormsModule
  2. 使用 ngModel 指令和雙向資料繫結語法將表單控制元件繫結到資料屬性（Property）。
    • 瞭解 ngModel 如何使用 CSS 類別報告控制元件狀態
    • 命名控制元件，使它們可以被 ngModel 訪問
  3. 使用 ngModel 跟蹤輸入的有效性和控制元件狀態。
    • 新增自定義 CSS 以提供關於狀態的視覺反饋
    • 顯示和隱藏驗證錯誤訊息
  4. 透過新增到模型資料來回應原生 HTML 按鈕點選事件。
  5. 使用表單的 ngSubmit 輸出屬性處理表單送出。
    • 停用 Submit 按鈕，直到表單有效
    • 送出後，將完成的表單換成頁面上的不同內容

將輸入控制元件繫結到資料屬性（Property）

下一步是將輸入控制元件與雙向資料繫結繫結到相應的 Actor 屬性（Property），以便它們透過更新資料模型來回應使用者輸入，並透過更新顯示來回應資料中的程式化更改。

在 FormsModule 中宣告的 ngModel 指令允許你將範本驅動表單中的控制元件繫結到資料模型中的屬性（Property）。 當你使用雙向資料繫結的語法 [(ngModel)] 包含該指令時，Angular 可以跟蹤控制元件的值和使用者互動，並保持檢視與模型同步。

  1. 編輯範本檔案 actor-form.component.html。
  2. 找到 Name 標籤旁邊的 <input> 標籤。
  3. 新增 ngModel 指令，使用雙向資料繫結語法 [(ngModel)]="..."。

跟蹤表單狀態

在表單送出後，Angular 會將 ng-submitted 類別應用於 form 元素。此 CSS 類別可用於更改表單送出後的樣式。

跟蹤控制元件狀態

將 NgModel 指令新增到控制元件會為控制元件新增描述其狀態的 CSS 類別名稱。 這些 CSS 類別可用於根據控制元件的狀態更改其樣式。

下表描述了 Angular 根據控制元件狀態應用的 CSS 類別名稱。

狀態 為真時的類別 為假時的類別
控制元件已被訪問過。 ng-touched ng-untouched
控制元件的值已更改。 ng-dirty ng-pristine
控制元件的值有效。 ng-valid ng-invalid

送出時，Angular 也會將 ng-submitted 類別應用於 form 元素，但不會應用於 form 元素內的控制元件。

你可以使用這些 CSS 類別來根據控制元件的狀態定義其樣式。

要檢視框架如何新增和刪除 CSS 類別，請開啟瀏覽器的開發者工具，並檢查代表演員姓名的 <input> 元素。

  1. 使用瀏覽器的開發者工具，找到與 Name 輸入框對應的 <input> 元素。 你可以看到，除了 "form-control" 之外，該元素還有多個 CSS 類別。

  2. 首次開啟時，這些 CSS 類別表明它具有有效值，該值自初始化或重置後未更改，並且自初始化或重置後控制元件未被訪問過。

          
    <input class="form-control ng-untouched ng-pristine ng-valid">;

  3. 對 Name <input> 框執行以下操作，並觀察出現哪些 CSS 類別。

    • 看，但不要觸碰。 這些 CSS 類別表明它是未觸碰的 (untouched)、原始的 (pristine) 且有效的 (valid)。

    • 在姓名框內單擊，然後在框外單擊。 控制元件現在已被訪問過，並且該元素具有 ng-touched 類別，而不是 ng-untouched 類別。

    • 在姓名末尾新增斜槓。 現在它是已觸碰的 (touched) 且髒的 (dirty)。

    • 擦除姓名。 這會使值變為無效，因此 ng-invalid 類別將替換 ng-valid 類別。

總結

本頁討論的 Angular 表單利用了以下框架屬性來提供對資料修改、驗證等的支援。

  • 一個 Angular HTML 表單範本
  • 一個帶有 @Component 裝飾器的表單元件類別
  • 透過繫結到 NgForm.ngSubmit 事件屬性來處理表單送出
  • 範本引用變數，例如 #actorForm 和 #name
  • 用於雙向資料繫結的 [(ngModel)] 語法
  • 使用 name 屬性進行驗證和表單元素變更跟蹤
  • 輸入控制元件上引用變數的 valid 屬性表明控制元件是否有效，或者是否應該顯示錯誤訊息
  • 透過繫結到 NgForm 有效性來控制 Submit 按鈕的啟用狀態
  • 自定義 CSS 類別，用於向用戶提供關於無效控制元件的視覺化反饋
