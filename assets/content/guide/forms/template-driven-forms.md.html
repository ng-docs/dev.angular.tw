构建模板驱动表单

edit

本教程会向你展示如何创建模板驱动的表单。表单中的控件元素绑定到具有输入验证功能的数据属性。输入验证有助于保持数据的完整性，而样式可以改善用户体验。

模板驱动表单使用 双向数据绑定，在模板中进行更改时更新组件中的数据模型，反之亦然。

模板驱动表单 vs 响应式表单

Angular支持两种交互表单的设计方法。模板驱动表单允许你在Angular模板中使用特定于表单的指令。响应式表单提供了一种基于模型的构建表单的方法。

模板驱动表单适用于小型或简单表单，而响应式表单更具伸缩性，适用于复杂表单。有关这两种方法的比较，请参见 选择方法

你可以用 Angular 模板来构建各种表单，比如登录表单、联系人表单和几乎所有的业务表单。你可以创造性地对控件进行布局并把它们绑定到对象模型的数据上。你可以指定验证规则并显示验证错误，根据条件启用或禁用特定控件，触发内置的视觉反馈等等。

目标

本教程将教你如何执行以下操作：

  • 使用组件和模板构建一个 Angular 表单
  • 使用 ngModel创建读取和写入输入控件值的双向数据绑定
  • 使用跟踪控件状态的特殊 CSS 类来提供视觉反馈
  • 向用户显示验证错误，并根据表单状态有条件地允许来自表单控件的输入
  • 使用 模板引用变量在HTML元素之间共享信息

构建模板驱动表单

基于模板的表单依赖于 FormsModule中定义的指令。

指令 详情
NgModel 将附加表单元素中的值变化与数据模型中的变化进行协调，允许你通过输入验证和错误处理来响应用户输入。
NgForm 创建一个顶层 FormGroup实例并将其绑定到一个 <form>元素上，以跟踪聚合的表单值和验证状态。只要导入 FormsModule，此指令默认在所有 <form>标签上启用。你不需要添加特殊的选择器。
NgModelGroup 创建并绑定一个 FormGroup实例到一个DOM元素。

步骤概述

在本教程中，你将使用以下步骤将一个范例表单绑定到数据并处理用户输入。

  1. Build the basic form.
    • 定义一个范例数据模型
    • 包括必需的基础设施，如 FormsModule
  2. Bind form controls to data properties using the ngModel directive and two-way data-binding syntax.
    • 查看如何使用CSS类别 ngModel报告控件状态
    • 为控件命名，以使它们对 ngModel可访问
  3. Track input validity and control status using ngModel.
    • 添加自定义 CSS 来根据状态提供可视化反馈
    • 显示和隐藏验证错误信息
  4. 通过添加到模型数据来响应原生 HTML 按钮的单击事件。
  5. Handle form submission using the ngSubmit output property of the form.
    • 在表单生效之前，先禁用 Submit 按钮
    • 在提交完成后，把已完成的表单替换成页面上不同的内容

  2. 表单的布局和细节在 ActorFormComponent类中定义。

    src/app/actor-form/actor-form.component.ts（v1）

          
    import {Component} from '@angular/core';
    import {Actor} from '../actor';
    @Component({
      selector: 'app-actor-form',
      templateUrl: './actor-form.component.html',
    })
    export class ActorFormComponent {
      skills = ['Method Acting', 'Singing', 'Dancing', 'Swordfighting'];
      model = new Actor(18, 'Tom Cruise', this.skills[3], 'CW Productions');
      submitted = false;
      onSubmit() {
        this.submitted = true;
      }
      newActor() {
        this.model = new Actor(42, '', '');
      }
      heroine(): Actor {
        const myActress = new Actor(42, 'Marilyn Monroe', 'Singing');
        console.log('My actress is called ' + myActress.name); // "My actress is called Marilyn"
        return myActress;
      }
      //////// NOT SHOWN IN DOCS ////////
      // Reveal in html:
      //   Name via form.controls = {{showFormControls(actorForm)}}
      showFormControls(form: any) {
        return form && form.controls.name && form.controls.name.value; // Tom Cruise
      }
      /////////////////////////////
    }

    组件的 selector值为"app-actor-form"，这意味着你可以通过 <app-actor-form>标签在父模板中放置此表单。

把输入控件绑定到数据属性

跟踪表单状态

提交表单后，Angular 会将 ng-submitted 类应用于 form 元素。此类可用于在提交后更改表单的样式。

跟踪控件状态

向控件添加 NgModel指令会向控件添加描述其状态的类名。这些类名可用于根据其状态更改控件的样式。

下表描述了 Angular 根据控件状态应用的类名。

状态 如果为真的类 如果为假的类
该控件已被访问。 ng-touched ng-untouched
该控件的值已更改。 ng-dirty ng-pristine
该控件的值有效。 ng-valid ng-invalid

Angular 还会在提交时将 ng-submitted 类应用于 form 元素，但不会应用于此 form 元素中的控件。

你可以用这些 CSS 类来根据控件的状态定义其样式。

观察控件状态

要查看框架如何添加和移除这些类，请打开浏览器的开发者工具并检查代表演员姓名的 <input>元素。

  1. 使用浏览器的开发者工具，找到对应于 名称输入框的 <input>元素。你会发现该元素除了"form-control"之外还有多个 CSS 类。

  2. 当你第一次启动它的时候，这些类表明它是一个有效的值，该值在初始化或重置之后还没有改变过，并且在该控件自初始化或重置后也没有被访问过。

          
    <input class="form-control ng-untouched ng-pristine ng-valid">;

  3. 在 Name <input> 框中执行以下操作，看看会出现哪些类。

    • 看看但不要触碰。这些类表示它是未触摸、原始的和有效的。

    • 点击姓名框内，然后再点击框外。现在该控件已被访问，该元素具有 ng-touched类，而不再有 ng-untouched类。

    • 擦除姓名。这使得值无效，因此 ng-invalid类替代了 ng-valid类。

你可能希望消息仅在用户进行无效更改时显示。在控件处于 原始状态时隐藏消息可以实现这一目标。当你在下一步中向表单添加新演员时，你会看到这种选择的重要性。

总结

本页讨论的 Angular 表单利用了下列框架特性来支持数据修改，验证等工作。

  • 一个 Angular HTML 表单模板
  • 一个带有 @Component装饰器的表单组件类
  • 通过绑定 NgForm.ngSubmit事件属性来处理表单提交
  • 模板引用变量，例如 #actorForm和 #name
  • [(ngModel)]语法用于双向数据绑定
  • name 属性的用途是验证和表单元素的变更跟踪
  • 输入控件上的引用变量的 valid 属性会表明控件是有效的还是应该显示错误消息
  • 通过绑定 NgForm有效性来控制 提交按钮的启用状态
  • 自定义 CSS 类，向用户提供有关无效控件的视觉反馈
