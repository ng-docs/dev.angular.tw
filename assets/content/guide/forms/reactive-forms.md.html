回應式表單

edit

回應式表單提供了一種模型驅動的方式來處理表單輸入，這些輸入的值會隨時間而變化。 本指南將向你展示如何建立和更新基本的表單控制元件，逐步學習如何在組中使用多個控制元件、驗證表單值，以及建立動態表單，在動態表單中，你可以在執行時新增或刪除控制元件。

回應式表單概覽

回應式表單使用顯式且不可變的方式來管理表單在給定時間點的狀態。 表單狀態的每次更改都會回傳一個新狀態，從而在更改之間保持模型的完整性。 回應式表單圍繞可觀察者流建立，其中表單輸入和值作為輸入值流提供，可以同步訪問。

回應式表單還為測試提供了一條直接的途徑，因為你可以確信你的資料在被請求時是一致且可預測的。 流的任何使用者都可以安全地操縱這些資料。

回應式表單與樣板驅動表單的區別在於其獨特的方式。 回應式表單透過可觀察者運算子提供對資料模型的同步訪問和不變性，並透過可觀察者流進行變更跟蹤。

樣板驅動表單允許直接訪問來修改樣板中的資料，但不如回應式表單那樣顯式，因為它們依賴於嵌入在樣板中的指令以及可變資料來非同步跟蹤更改。 有關這兩種範例之間詳細的比較，請參閱表單概覽。

新增基本的表單控制元件

使用表單控制元件有三個步驟。

  1. 生成一個新的元件並註冊回應式表單模組。此模組宣告了使用回應式表單所需的回應式表單指令。
  2. 實例化一個新的 FormControl。
  3. 在樣板中註冊 FormControl。

然後，你可以透過將元件新增到樣板來顯示錶單。

以下範例展示了如何新增單個表單控制元件。 在該範例中，使用者在輸入欄位中輸入他們的姓名，捕獲該輸入值，並顯示錶單控制元件元素的當前值。

  1. 生成一個新的元件並匯入 ReactiveFormsModule

    使用 CLI 指令 ng generate component 在你的專案中生成一個元件，並從 @angular/forms 套件中匯入 ReactiveFormsModule，並將其新增到你的元件的 imports 陣列中。

    src/app/name-editor/name-editor.component.ts (節選)

          
    import {Component} from '@angular/core';import {FormControl, ReactiveFormsModule} from '@angular/forms';@Component({  selector: 'app-name-editor',  templateUrl: './name-editor.component.html',  styleUrls: ['./name-editor.component.css'],  imports: [ReactiveFormsModule],})export class NameEditorComponent {  name = new FormControl('');  updateName() {    this.name.setValue('Nancy');  }}

  2. 宣告一個 FormControl 實例

    使用 FormControl 的建構式函式來設定其初始值，在本例中是一個空字串。透過在元件類別中建立這些控制元件，你可以立即訪問以監聽、更新和驗證表單輸入的狀態。

    src/app/name-editor/name-editor.component.ts

          
    import {Component} from '@angular/core';import {FormControl, ReactiveFormsModule} from '@angular/forms';@Component({  selector: 'app-name-editor',  templateUrl: './name-editor.component.html',  styleUrls: ['./name-editor.component.css'],  imports: [ReactiveFormsModule],})export class NameEditorComponent {  name = new FormControl('');  updateName() {    this.name.setValue('Nancy');  }}

  3. 在樣板中註冊控制元件

    在元件類別中建立控制元件後，你必須將其與樣板中的表單控制元件元素關聯起來。使用 FormControlDirective 提供的 formControl 繫結來更新樣板中的表單控制元件，FormControlDirective 也包含在 ReactiveFormsModule 中。

    src/app/name-editor/name-editor.component.html

          
    <label for="name">Name: </label><input id="name" type="text" [formControl]="name"><p>Value: {{ name.value }}</p><button type="button" (click)="updateName()">Update Name</button>

    使用樣板繫結語法，表單控制元件現在已註冊到樣板中的 name 輸入元素。表單控制元件和 DOM 元素相互通訊：檢視反映模型中的更改，模型反映檢視中的更改。

  4. 顯示元件

    當 <app-name-editor> 元件新增到樣板時，將顯示賦值給 name 屬性的 FormControl。

    src/app/app.component.html (名稱編輯器)

          
    <h1>Reactive Forms</h1><app-name-editor /><app-profile-editor />

顯示錶單控制元件值

你可以透過以下方式顯示該值。

  • 透過 valueChanges 可觀察者，你可以在樣板中使用 AsyncPipe 或在元件類別中使用 subscribe() 方法來監聽表單值的變化
  • 使用 value 屬性，它可以為你提供當前值的快照

以下範例向你展示如何在樣板中使用內嵌來顯示當前值。

src/app/name-editor/name-editor.component.html (控制元件值)

      
<label for="name">Name: </label><input id="name" type="text" [formControl]="name"><p>Value: {{ name.value }}</p><button type="button" (click)="updateName()">Update Name</button>

當你更新表單控制元件元素時，顯示的值也會隨之更改。

回應式表單允許透過每個實例提供的屬性和方法來訪問有關給定控制元件的訊息。 底層 AbstractControl 類別的這些屬性和方法用於控制表單狀態，並確定在處理輸入驗證時何時顯示訊息。

在API 參考中閱讀有關其他 FormControl 屬性和方法的訊息。

替換表單控制元件值

回應式表單具有以程式設計方式更改控制元件值的方法，這使你可以靈活地在沒有用戶互動的情況下更新值。 表單控制元件實例提供了一個 setValue() 方法，該方法更新表單控制元件的值，並根據控制元件的結構驗證所提供值的結構。 例如，當從後端 API 或服務檢索表單資料時，使用 setValue() 方法將控制元件更新為其新值，從而完全替換舊值。

以下範例向元件類別新增了一個方法，以使用 setValue() 方法將控制元件的值更新為 Nancy。

src/app/name-editor/name-editor.component.ts (更新值)

      
import {Component} from '@angular/core';import {FormControl, ReactiveFormsModule} from '@angular/forms';@Component({  selector: 'app-name-editor',  templateUrl: './name-editor.component.html',  styleUrls: ['./name-editor.component.css'],  imports: [ReactiveFormsModule],})export class NameEditorComponent {  name = new FormControl('');  updateName() {    this.name.setValue('Nancy');  }}

使用按鈕更新樣板以模擬名稱更新。 當你單擊 Update Name 按鈕時，在表單控制元件元素中輸入的值將反映為其當前值。

src/app/name-editor/name-editor.component.html (更新值)

      
<label for="name">Name: </label><input id="name" type="text" [formControl]="name"><p>Value: {{ name.value }}</p><button type="button" (click)="updateName()">Update Name</button>

表單模型是控制元件的事實之源，因此當你單擊按鈕時，輸入的的值將在元件類別中更改，從而覆蓋其當前值。

提示： 在此範例中，你正在使用單個控制元件。 當將 setValue() 方法與表單組或表單陣列實例一起使用時，該值需要與組或陣列的結構對應。

對錶單控制元件進行分組

表單通常包含多個相關的控制元件。 回應式表單提供了兩種將多個相關控制元件分組到單個輸入表單中的方法。

表單組 詳情
表單組 定義了一個具有固定控制元件集的表單，你可以一起管理這些控制元件。本節討論了表單組的基礎知識。你還可以巢狀表單組以建立更復雜的表單。
表單陣列 定義了一個動態表單，你可以在執行時新增和刪除控制元件。你還可以巢狀表單陣列以建立更復雜的表單。有關此選項的更多訊息，請參閱建立動態表單。

正如表單控制元件實例使你可以控制單個輸入欄位一樣，表單組實例跟蹤一組表單控制元件實例（例如，表單）的表單狀態。 在建立表單組時，表單組實例中的每個控制元件都按名稱進行跟蹤。 以下範例展示了如何在單個組中管理多個表單控制元件實例。

生成一個 ProfileEditor 元件，並從 @angular/forms 套件中匯入 FormGroup 和 FormControl 類。

      
ng generate component ProfileEditor

src/app/profile-editor/profile-editor.component.ts (匯入)

      
import {Component} from '@angular/core';import {FormGroup, FormControl, ReactiveFormsModule} from '@angular/forms';@Component({  selector: 'app-profile-editor',  templateUrl: './profile-editor.component.html',  styleUrls: ['./profile-editor.component.css'],  imports: [ReactiveFormsModule],})export class ProfileEditorComponent {  profileForm = new FormGroup({    firstName: new FormControl(''),    lastName: new FormControl(''),    address: new FormGroup({      street: new FormControl(''),      city: new FormControl(''),      state: new FormControl(''),      zip: new FormControl(''),    }),  });  updateProfile() {    this.profileForm.patchValue({      firstName: 'Nancy',      address: {        street: '123 Drew Street',      },    });  }}

要向此元件新增表單組，請執行以下步驟。

  1. 建立一個 FormGroup 實例。
  2. 關聯 FormGroup 模型和檢視。
  3. 儲存表單資料。

  1. 建立一個 FormGroup 實例

    在元件類別中建立一個名為 profileForm 的屬性，並將該屬性設定為一個新的表單組實例。要初始化此表單組，請使用一個物件為建構式函式賦值，該物件的具名鍵會對映到它們各自的控制元件。

    對於個人訊息表單，新增兩個表單控制元件實例，名稱分別為 firstName 和 lastName

    src/app/profile-editor/profile-editor.component.ts (表單組)

          
    import {Component} from '@angular/core';import {FormGroup, FormControl, ReactiveFormsModule} from '@angular/forms';@Component({  selector: 'app-profile-editor',  templateUrl: './profile-editor.component.html',  styleUrls: ['./profile-editor.component.css'],  imports: [ReactiveFormsModule],})export class ProfileEditorComponent {  profileForm = new FormGroup({    firstName: new FormControl(''),    lastName: new FormControl(''),    address: new FormGroup({      street: new FormControl(''),      city: new FormControl(''),      state: new FormControl(''),      zip: new FormControl(''),    }),  });  updateProfile() {    this.profileForm.patchValue({      firstName: 'Nancy',      address: {        street: '123 Drew Street',      },    });  }}

    各個表單控制元件現在被收集在一個組中。FormGroup 實例會將其模型值提供為一個物件，該物件由組中每個控制元件的值歸約而來。表單組實例與表單控制元件實例具有相同的屬性（例如 value 和 untouched）和方法（例如 setValue()）。

  2. 關聯 FormGroup 模型和檢視

    表單組會跟蹤其每個控制元件的狀態和變更，因此，如果其中一個控制元件發生更改，則父控制元件也會發出新的狀態或值變更。組的模型由其成員維護。定義模型後，你必須更新樣板以在檢視中反映該模型。

    src/app/profile-editor/profile-editor.component.html (樣板表單組)

          
    <form [formGroup]="profileForm">  <label for="first-name">First Name: </label>  <input id="first-name" type="text" formControlName="firstName" />  <label for="last-name">Last Name: </label>  <input id="last-name" type="text" formControlName="lastName" />  <div formGroupName="address">    <h2>Address</h2>    <label for="street">Street: </label>    <input id="street" type="text" formControlName="street" />    <label for="city">City: </label>    <input id="city" type="text" formControlName="city" />    <label for="state">State: </label>    <input id="state" type="text" formControlName="state" />    <label for="zip">Zip Code: </label>    <input id="zip" type="text" formControlName="zip" />  </div>  <div formArrayName="aliases">    <h2>Aliases</h2>    <button type="button" (click)="addAlias()">+ Add another alias</button>    @for(alias of aliases.controls; track $index; let i = $index) {    <div>      <!-- The repeated alias template -->      <label for="alias-{{ i }}">Alias: </label>      <input id="alias-{{ i }}" type="text" [formControlName]="i" />    </div>    }  </div></form><p>Form Value: {{ profileForm.value | json }}</p><button type="button" (click)="updateProfile()">Update Profile</button>

    正如表單組包含一組控制元件一樣，profileForm FormGroup 也使用 FormGroup 指令繫結到 form 元素，從而在模型和包含輸入的表單之間建立了一個通訊層。FormControlName 指令提供的 formControlName 輸入將每個單獨的輸入繫結到 FormGroup 中定義的表單控制元件。表單控制元件與其各自的元素進行通訊。它們還會將更改傳遞給表單組實例，後者為模型值提供事實之源。

  3. 儲存表單資料

    ProfileEditor 元件接受來自使用者的輸入，但在實際場景中，你希望捕獲表單值，並使其可在元件外部進行進一步處理。FormGroup 指令偵聽 form 元素發出的 submit 事件，併發出你可以繫結到回呼函式的 ngSubmit 事件。將 ngSubmit 事件偵聽器新增到帶有 onSubmit() 回呼方法的 form 標籤。

    src/app/profile-editor/profile-editor.component.html (送出事件)

    ProfileEditor 元件中的 onSubmit() 方法會捕獲 profileForm 的當前值。使用 EventEmitter 以保持表單的封裝性，並在元件外部提供表單值。以下範例使用 console.warn 向瀏覽器控制台記錄訊息。

    src/app/profile-editor/profile-editor.component.ts (送出方法)

    ProfileEditor 使你可以管理表單組實例中 firstName 和 lastName 控制元件的表單控制元件實例。

    建立巢狀表單組

    表單組可以接受單個表單控制元件實例和其他表單組實例作為子項。 這使得組合複雜的表單模型更易於維護和邏輯分組。

    在建立複雜表單時，在較小的部分中管理訊息的不同區域更容易。 使用巢狀的表單組實例可讓你將大型表單組分解為更小、更易於管理的表單組。

    要建立更復雜的表單，請使用以下步驟。

    1. 建立一個巢狀組。
    2. 在樣板中對巢狀表單進行分組。

    某些型別的訊息自然會歸入同一組。 姓名和地址是此類巢狀組的典型範例，並在以下範例中使用。

    要在 `profileForm` 中建立巢狀組，請將巢狀的 `address` 元素新增到表單組實例。

    src/app/profile-editor/profile-editor.component.ts (巢狀表單組)

          
    import {Component} from '@angular/core';import {FormGroup, FormControl, ReactiveFormsModule} from '@angular/forms';@Component({  selector: 'app-profile-editor',  templateUrl: './profile-editor.component.html',  styleUrls: ['./profile-editor.component.css'],  imports: [ReactiveFormsModule],})export class ProfileEditorComponent {  profileForm = new FormGroup({    firstName: new FormControl(''),    lastName: new FormControl(''),    address: new FormGroup({      street: new FormControl(''),      city: new FormControl(''),      state: new FormControl(''),      zip: new FormControl(''),    }),  });  updateProfile() {    this.profileForm.patchValue({      firstName: 'Nancy',      address: {        street: '123 Drew Street',      },    });  }}

    在此範例中，address group 將當前的 firstName 和 lastName 控制元件與新的 street、city、state 和 zip 控制元件群組合在一起。即使表單組中的 address 元素是表單組中整體 profileForm 元素的子元素，但相同的規則也適用於值和狀態更改。來自巢狀表單組的狀態和值更改會傳播到父表單組，從而保持與整體模型的一致性。

  5. 在樣板中對巢狀表單進行分組

有兩種更新模型值的方法：

方法 詳情
setValue() 為單個控制元件設定新值。setValue() 方法嚴格遵守表單組的結構，並替換控制元件的整個值。
patchValue() 替換物件中定義的在表單模型中已更改的任何屬性。

setValue() 方法的嚴格檢查有助於捕獲複雜表單中的巢狀錯誤，而 patchValue() 在這些錯誤上會靜默失敗。

使用 FormBuilder 服務生成控制元件

在處理多個表單時，手動建立表單控制元件實例可能會變得重複。 FormBuilder 服務提供了用於生成控制元件的便捷方法。

使用以下步驟來利用此服務。

  1. 匯入 FormBuilder 類。
  2. 注入 FormBuilder 服務。
  3. 生成表單內容。

建立動態表單

FormArray 是 FormGroup 的替代品，用於管理任意數量的未命名控制元件。 與表單組實例一樣，你可以從表單陣列實例中動態插入和移除控制元件，並且表單陣列實例的值和驗證狀態是根據其子控制元件計算得出的。 但是，你不需要按名稱為每個控制元件定義鍵，因此如果你事先不知道子值的數量，這是一個很好的選擇。

  4. 在樣板中顯示錶單數組。

  4. 在樣板中顯示錶單數組

    要附著來自你的表單模型的別名，你必須將其新增到樣板中。與 FormGroupNameDirective 提供的 formGroupName 輸入類似，formArrayName 使用 FormArrayNameDirective 將來自表單陣列實例的通訊繫結到樣板。

    在 <div> 關閉 formGroupName 元素後，新增以下樣板 HTML。

    src/app/profile-editor/profile-editor.component.html (別名 form array 樣板)

          
    <form [formGroup]="profileForm" (ngSubmit)="onSubmit()">  <label for="first-name">First Name: </label>  <input id="first-name" type="text" formControlName="firstName" required />  <label for="last-name">Last Name: </label>  <input id="last-name" type="text" formControlName="lastName" />  <div formGroupName="address">    <h2>Address</h2>    <label for="street">Street: </label>    <input id="street" type="text" formControlName="street" />    <label for="city">City: </label>    <input id="city" type="text" formControlName="city" />    <label for="state">State: </label>    <input id="state" type="text" formControlName="state" />    <label for="zip">Zip Code: </label>    <input id="zip" type="text" formControlName="zip" />  </div>  <div formArrayName="aliases">    <h2>Aliases</h2>    <button type="button" (click)="addAlias()">+ Add another alias</button>    @for (alias of aliases.controls; track $index; let i = $index) {      <div>        <!-- The repeated alias template -->        <label for="alias-{{ i }}">Alias:</label>        <input id="alias-{{ i }}" type="text" [formControlName]="i" />      </div>    }  </div>  <p>Complete the form to enable button.</p>  <button type="submit" [disabled]="!profileForm.valid">Submit</button></form><hr><p>Form Value: {{ profileForm.value | json }}</p><p>Form Status: {{ profileForm.status }}</p><button type="button" (click)="updateProfile()">Update Profile</button>

    *ngFor 指令迭代 aliases 表單陣列實例提供的每個表單控制元件實例。由於表單陣列元素是未命名的，因此你將索引賦值給 i 變數，並將其傳遞給每個控制元件以將其繫結到 formControlName 輸入。

    每次新增新的別名實例時，新的表單陣列實例都會根據索引提供其控制元件。這使你可以在計算根控制元件的狀態和值時跟蹤每個單獨的控制元件。

  5. 新增別名

    最初，表單包含一個 Alias 欄位。要新增另一個欄位，請單擊 Add Alias 按鈕。你還可以驗證樣板底部的 Form Value 顯示的表單模型報告的別名陣列。你可以使用其他欄位組合另一個表單組實例，而不是為每個別名使用表單控制元件實例。為每個條目定義控制元件的過程是相同的。

回應式表單 API 概覽

下表列出了用於建立和管理回應式表單控制元件的基礎類和服務。 有關完整的語法詳細訊息，請參閱 Forms 套件的 API 參考文件。

類

類 詳情
AbstractControl 具體表單控制元件類 FormControl、FormGroup 和 FormArray 的抽象基底類別。它提供了它們的通用行為和屬性（Property）。
FormControl 管理單個表單控制元件的值和有效性狀態。它對應於 HTML 表單控制元件，例如 <input> 或 <select>。
FormGroup 管理一組 AbstractControl 實例的值和有效性狀態。該組的屬性（Property）包括其子控制元件。元件中的頂層表單是 FormGroup。
FormArray 管理 AbstractControl 實例的數字索引陣列的值和有效性狀態。
FormBuilder 一個可注入的服務，提供用於建立控制元件實例的工廠方法。
FormRecord 跟蹤型別相同的一組 FormControl 實例的值和有效性狀態。

指令

指令 詳情
FormControlDirective 將獨立的 FormControl 實例同步到表單控制元件元素。
FormControlName 透過名稱將現有 FormGroup 實例中的 FormControl 同步到表單控制元件元素。
FormGroupDirective 將現有的 FormGroup 實例同步到 DOM 元素。
FormGroupName 將巢狀的 FormGroup 實例同步到 DOM 元素。
FormArrayName 將巢狀的 FormArray 實例同步到 DOM 元素。
