在單頁應用中使用 Angular 路由

edit

本教學課程介紹如何建立一個使用多個 Angular 路由的單頁應用（SPA）。

在單頁應用（SPA）中，你的應用的所有功能都存在於單個 HTML 頁面中。 當用戶訪問應用的功能時，瀏覽器只需要渲染對使用者重要的部分，而不是載入新頁面。 這種模式可以顯著改善應用的使用者體驗。

要定義使用者如何在你的應用中導向，你可以使用路由。 新增路由以定義使用者如何從應用的某一部分導向到另一部分。 你也可以設定路由以防止意外或未經授權的行為。

目標

  • 將範例應用的功能組織成模組。
  • 定義如何導向到元件。
  • 使用引數將訊息傳遞給元件。
  • 透過巢狀多個路由來建立路由結構。
  • 檢查使用者是否可以訪問路由。
  • 控制應用是否可以放棄未儲存的更改。
  • 透過預取路由資料和延遲載入屬性模組來提高效能。
  • 要求滿足特定條件才能載入元件。

建立一個範例應用

使用 Angular CLI，建立一個新的應用，angular-router-sample。 此應用將有兩個元件：crisis-list 和 heroes-list。

  1. 建立一個新的 Angular 專案，angular-router-sample。

          
    ng new angular-router-sample

    當提示 Would you like to add Angular routing? 時，選擇 N。

    當提示 Which stylesheet format would you like to use? 時，選擇 CSS。

    稍等片刻，新的專案 angular-router-sample 就準備好了。

  2. 從終端中，導向到 angular-router-sample 目錄。

  3. 建立一個元件，crisis-list。

          
    ng generate component crisis-list

  4. 在你的程式碼編輯器中，找到檔案 crisis-list.component.html，並將佔位符內容替換為以下 HTML。

    src/app/crisis-list/crisis-list.component.html

          
    <h3>CRISIS CENTER</h3><p>Get your crisis here</p>

  5. 建立第二個元件，heroes-list。

          
    ng generate component heroes-list

  6. 在你的程式碼編輯器中，找到檔案 heroes-list.component.html，並將佔位符內容替換為以下 HTML。

    src/app/heroes-list/heroes-list.component.html

          
    <h3>HEROES</h3><p>Get your heroes here</p>

  7. 在你的程式碼編輯器中，開啟檔案 app.component.html，並將其內容替換為以下 HTML。

    src/app/app.component.html

          
    <h1>Angular Router Sample</h1><nav>  <a class="button"     routerLink="/crisis-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Crisis Center  </a> |  <a class="button"     routerLink="/heroes-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Heroes  </a></nav><router-outlet></router-outlet><div style="display: none;"><!-- This HTML represents the initial state for the tutorial. It is not intended to appear in the app. --><app-crisis-list></app-crisis-list><app-heroes-list></app-heroes-list><!-- This HTML snippet is for when the user first adds the routerlink navigation. --><nav>  <a class="button" routerLink="/crisis-list">Crisis Center</a> |  <a class="button" routerLink="/heroes-list">Heroes</a></nav></div>

  8. 透過執行 ng serve 指令，驗證你的新應用是否按預期執行。

          
    ng serve

  9. 在瀏覽器中開啟 http://localhost:4200。

    你應該看到一個單頁，其中包含標題和你兩個元件的 HTML。

定義你的路由

在本節中，你將定義兩個路由：

  • 路由 /crisis-center 開啟 crisis-center 元件。
  • 路由 /heroes-list 開啟 heroes-list 元件。

路由定義是一個 JavaScript 物件。 每個路由通常有兩個屬性。 第一個屬性 path 是一個字串，用於指定路由的 URL 路徑。 第二個屬性 component 是一個字串，用於指定你的應用應該為該路徑顯示的元件。

  1. 在你的程式碼編輯器中，建立並開啟 app.routes.ts 檔案。

  2. 為你的應用建立並匯出一個路由清單：

    import {Routes} from '@angular/router';

export const routes = [];

  3. 為你的前兩個元件新增兩個路由：

    {path: 'crisis-list', component: CrisisListComponent},
{path: 'heroes-list', component: HeroesListComponent},

此路由清單是一個 JavaScript 物件陣列，每個物件都定義了一個路由的屬性。

從 @angular/router 匯入 provideRouter

路由允許你根據 URL 路徑顯示應用的特定檢視。 要將此功能新增到你的範例應用，你需要更新 app.config.ts 檔案以使用路由提供者函式 provideRouter。 你從 @angular/router 匯入此提供者函式。

  1. 在你的程式碼編輯器中，開啟 app.config.ts 檔案。

  2. 新增以下匯入語句：

    import { provideRouter } from '@angular/router';
import { routes } from './app.routes';

  3. 更新 appConfig 中的 providers：

    providers: [provideRouter(routes)]

對於基於 NgModule 的應用，將 provideRouter 放在 AppModule 的 providers 清單中，或者傳遞給應用中 bootstrapModule 的任何模組。

使用 router-outlet 更新你的元件

此時，你已經為你的應用定義了兩個路由。 但是，你的應用仍然在 app.component.html 範本中硬編碼了 crisis-list 和 heroes-list 元件。 為了使你的路由工作，你需要更新你的範本以根據 URL 路徑動態載入元件。

要實現此功能，你需要將 router-outlet 指令新增到你的範本檔案中。

  1. 在你的程式碼編輯器中，開啟 app.component.html 檔案。

  2. 刪除以下行。

    src/app/app.component.html

It is not intended to appear in the app. --><app-crisis-list></app-crisis-list><app-heroes-list></app-heroes-list><!-- This HTML snippet is for when the user first adds the routerlink navigation. --><nav>  <a class="button" routerLink="/crisis-list">Crisis Center</a> |  <a class="button" routerLink="/heroes-list">Heroes</a></nav></div>

  3. 新增 router-outlet 指令。

    src/app/app.component.html

          
    <h1>Angular Router Sample</h1><nav>  <a class="button"     routerLink="/crisis-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Crisis Center  </a> |  <a class="button"     routerLink="/heroes-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Heroes  </a></nav><router-outlet></router-outlet><div style="display: none;"><!-- This HTML represents the initial state for the tutorial. It is not intended to appear in the app. --><app-crisis-list></app-crisis-list><app-heroes-list></app-heroes-list><!-- This HTML snippet is for when the user first adds the routerlink navigation. --><nav>  <a class="button" routerLink="/crisis-list">Crisis Center</a> |  <a class="button" routerLink="/heroes-list">Heroes</a></nav></div>

  4. 在 app.component.ts 中，把 RouterOutlet 新增到 AppComponent 的 imports 中

    imports: [RouterOutlet],

在瀏覽器中檢視你更新後的應用。 你應該只看到應用標題。 要檢視 crisis-list 元件，請將 crisis-list 新增到瀏覽器地址欄中路徑的末尾。 例如：

      
http://localhost:4200/crisis-list

請注意，crisis-list 元件會顯示出來。 Angular 正在使用你定義的路由來動態載入該組件。 你可以用同樣的方式載入 heroes-list 元件：

      
http://localhost:4200/heroes-list

使用 UI 元素控制導向

目前，你的應用支援兩個路由。 然而，使用這些路由的唯一方法是讓使用者在瀏覽器的地址欄中手動輸入路徑。 在本節中，你將新增兩個連結，使用者可以單擊這些連結在 heroes-list 和 crisis-list 元件之間導向。 你還將新增一些 CSS 樣式。 雖然這些樣式不是必需的，但它們可以更容易地識別當前顯示元件的連結。 你將在下一節中新增該功能。

  1. 開啟 app.component.html 檔案，並在標題下方新增以下 HTML。

    src/app/app.component.html

          
    <h1>Angular Router Sample</h1><nav>  <a class="button"     routerLink="/crisis-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Crisis Center  </a> |  <a class="button"     routerLink="/heroes-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Heroes  </a></nav><router-outlet></router-outlet><div style="display: none;"><!-- This HTML represents the initial state for the tutorial. It is not intended to appear in the app. --><app-crisis-list></app-crisis-list><app-heroes-list></app-heroes-list><!-- This HTML snippet is for when the user first adds the routerlink navigation. --><nav>  <a class="button" routerLink="/crisis-list">Crisis Center</a> |  <a class="button" routerLink="/heroes-list">Heroes</a></nav></div>

    此 HTML 使用了一個 Angular 指令 routerLink。 此指令將你定義的路由連線到你的範本檔案。

  2. 將 RouterLink 指令新增到 app.component.ts 中 AppComponent 的 imports 清單中。

  3. 開啟 app.component.css 檔案，並新增以下樣式。

    src/app/app.component.css

          
    .button {    box-shadow: inset 0 1px 0 0 #ffffff;    background: #ffffff linear-gradient(to bottom, #ffffff 5%, #f6f6f6 100%);    border-radius: 6px;    border: 1px solid #dcdcdc;    display: inline-block;    cursor: pointer;    color: #666666;    font-family: Arial, sans-serif;    font-size: 15px;    font-weight: bold;    padding: 6px 24px;    text-decoration: none;    text-shadow: 0 1px 0 #ffffff;    outline: 0;}.activebutton {    box-shadow: inset 0 1px 0 0 #dcecfb;    background: #bddbfa linear-gradient(to bottom, #bddbfa 5%, #80b5ea 100%);    border: 1px solid #84bbf3;    color: #ffffff;    text-shadow: 0 1px 0 #528ecc;}

如果在瀏覽器中檢視你的應用，你應該會看到這兩個連結。 當你單擊連結時，相應的元件就會出現。

識別活動路由

雖然使用者可以使用你在上一節中新增的連結來導向你的應用，但他們沒有直接的方式來識別哪個是活動路由。 使用 Angular 的 routerLinkActive 指令新增此功能。

  1. 在你的程式碼編輯器中，開啟 app.component.html 檔案。
  2. 更新這些錨點標籤，以包含 routerLinkActive 指令。

    src/app/app.component.html

          
    <h1>Angular Router Sample</h1><nav>  <a class="button"     routerLink="/crisis-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Crisis Center  </a> |  <a class="button"     routerLink="/heroes-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Heroes  </a></nav><router-outlet></router-outlet><div style="display: none;"><!-- This HTML represents the initial state for the tutorial. It is not intended to appear in the app. --><app-crisis-list></app-crisis-list><app-heroes-list></app-heroes-list><!-- This HTML snippet is for when the user first adds the routerlink navigation. --><nav>  <a class="button" routerLink="/crisis-list">Crisis Center</a> |  <a class="button" routerLink="/heroes-list">Heroes</a></nav></div>
  3. 將 RouterLinkActive 指令新增到 app.component.ts 中 AppComponent 的 imports 清單中。

再次檢視你的應用。 當你單擊其中一個按鈕時，該按鈕的樣式會自動更新，從而向用戶標識出活動的元件。 透過新增 routerLinkActive 指令，你可以通知你的應用將特定的 CSS 類別應用於活動路由。 在本教學課程中，該 CSS 類別是 activebutton，但你可以使用任何你想要的類別。

請注意，我們還為 routerLinkActive 的 ariaCurrentWhenActive 指定了一個值。這確保了視覺障礙使用者（他們可能無法感知到正在應用的不同樣式）也可以識別出活動按鈕。有關更多訊息，請參閱無障礙功能最佳實踐的活動連結識別部分。

新增重導向

在本教學課程的這一步中，你將新增一個路由，該路由會將使用者重導向以顯示 /heroes-list 元件。

  1. 在你的程式碼編輯器中，開啟 app.routes.ts 檔案。

  2. 按如下所示更新 routes 部分。

    {path: '', redirectTo: '/heroes-list', pathMatch: 'full'},

    請注意，這個新路由使用一個空字串作為其路徑。 此外，它使用兩個新屬性替換了 component 屬性：

    屬性 詳情
    redirectTo 此屬性指示 Angular 從空路徑重導向到 heroes-list 路徑。
    pathMatch 此屬性指示 Angular 如何對應 URL 的多少。對於本教學課程，你應該將此屬性設定為 full。當你的路徑為空字串時，建議使用此策略。有關此屬性的更多訊息，請參閱 Route API 文件。

現在，當你開啟你的應用時，它預設會顯示 heroes-list 元件。

新增 404 頁面

使用者有可能嘗試訪問你尚未定義的路由。 為了應對這種行為，最佳實踐是顯示 404 頁面。 在本節中，你將建立一個 404 頁面並更新你的路由設定，以便為任何未指定的路由顯示該頁面。

  1. 從終端中，建立一個新元件 PageNotFound。

          
    ng generate component page-not-found

  2. 在你的程式碼編輯器中，開啟 page-not-found.component.html 檔案，並將其內容替換為以下 HTML。

    src/app/page-not-found/page-not-found.component.html

          
    <h2>Page Not Found</h2><p>We couldn't find that page! Not even with x-ray vision.</p>

  3. 開啟 app.routes.ts 檔案，並將以下路由新增到路由清單中：

    {path: '**', component: PageNotFoundComponent}

    新路由使用路徑 **。 此路徑是 Angular 識別萬用字元路由的方式。 任何與你的設定中現有路由不對應的路由都將使用此路由。

重要提示： 請注意，萬用字元路由位於陣列的末尾。 你的路由順序非常重要，因為 Angular 會按順序應用路由並使用它找到的第一個對應項。

嘗試導向到你的應用上不存在的路由，例如 http://localhost:4200/powers。 此路由與你在 app.routes.ts 檔案中定義的任何內容都不對應。 但是，由於你定義了一個萬用字元路由，因此應用會自動顯示你的 PageNotFound 元件。

後續步驟

至此，你已經有了一個基本應用，它使用 Angular 的路由屬性來根據 URL 地址更改使用者可以看到的元件。 你已經擴充套件了這些屬性，以包含重導向以及用於顯示自定義 404 頁面的萬用字元路由。
