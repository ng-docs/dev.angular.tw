常見的路由任務

edit

本主題介紹如何實現將 Angular 路由器新增到應用中的許多常見任務。

生成啟用路由的應用

以下指令使用 Angular CLI 生成一個帶有應用路由的基本 Angular 應用。在以下範例中，應用名稱為 routing-app。

      
ng new routing-app

為路由新增元件

要使用 Angular 路由器，應用至少需要有兩個元件，以便可以從一個元件導向到另一個元件。要使用 CLI 建立元件，請在指令行中輸入以下指令，其中 first 是你的元件的名稱：

      
ng generate component first

對第二個元件重複此步驟，但給它一個不同的名稱。這裡，新名稱是 second。

      
ng generate component second

CLI 會自動追加 Component，因此如果你編寫 first-component，你的元件將是 FirstComponentComponent。

base href

本指南適用於 CLI 生成的 Angular 應用。如果你是手動操作，請確保你的 index.html 檔案的 <head> 中有 <base href="/">。 這裡假設 app 資料夾是應用根目錄，並使用 "/"。

匯入你的新元件

要使用你的新元件，請將它們匯入到 app.routes.ts 檔案的頂部，如下所示：

      
import {FirstComponent} from './first/first.component';import {SecondComponent} from './second/second.component';

定義基本路由

建立路由有三個基本建立塊。

將路由匯入到 app.config.ts 中，並將其新增到 provideRouter 函式中。以下是使用 CLI 的預設 ApplicationConfig。

      
export const appConfig: ApplicationConfig = {  providers: [provideRouter(routes)]};

Angular CLI 會為你執行此步驟。但是，如果你是手動建立應用或使用現有的非 CLI 應用，請驗證匯入和設定是否正確。

  1. 為你的路由建立一個 Routes 陣列

    Angular CLI 會自動執行此步驟。

          
    import { Routes } from '@angular/router';export const routes: Routes = [];

  2. 在你的 Routes 陣列中定義你的路由

    此陣列中的每個路由都是一個 JavaScript 物件，其中包含兩個屬性。第一個屬性 path 定義了路由的 URL 路徑。第二個屬性 component 定義了 Angular 應該用於相應路徑的元件。

          
    const routes: Routes = [  { path: 'first-component', component: FirstComponent },  { path: 'second-component', component: SecondComponent },];

  3. 將你的路由新增到你的應用

    現在你已經定義了你的路由，將它們新增到你的應用中。首先，新增指向這兩個元件的連結。將要新增路由的錨標記賦值給 routerLink 屬性。將該屬性的值設定為當用戶單擊每個連結時要顯示的元件。接下來，更新你的元件樣板以包含 <router-outlet>。此元素通知 Angular 使用所選路由的元件來更新應用檢視。

          
    <h1>Angular Router App</h1><nav>  <ul>    <li><a routerLink="/first-component" routerLinkActive="active" ariaCurrentWhenActive="page">First Component</a></li>    <li><a routerLink="/second-component" routerLinkActive="active" ariaCurrentWhenActive="page">Second Component</a></li>  </ul></nav><!-- The routed views render in the <router-outlet>--><router-outlet />

    你還需要將 RouterLink、RouterLinkActive 和 RouterOutlet 新增到 AppComponent 的 imports 陣列中。

          
    @Component({  selector: 'app-root',  imports: [RouterOutlet, RouterLink, RouterLinkActive],  templateUrl: './app.component.html',  styleUrls: ['./app.component.css']})export class AppComponent {  title = 'routing-app';}

路由順序

路由的順序很重要，因為 Router 在對應路由時使用先到先得策略，因此更具體的路由應放在不太具體的路由之上。 首先列出具有靜態路徑的路由，然後是空路徑路由，它對應預設路由。 萬用字元路由放在最後，因為它對應每個 URL，並且僅當沒有其他路由首先對應時，Router 才會選擇它。

獲取路由訊息

通常，當用戶導向你的應用時，你希望將訊息從一個元件傳遞到另一個元件。 例如，考慮一個顯示食品雜貨購物清單的應用。 清單中的每個條目都有一個唯一的 id。 要編輯條目，使用者單擊『編輯』按鈕，這將開啟 EditGroceryItem 元件。 你希望該組件檢索食品雜貨條目的 id，以便它可以向用戶顯示正確的訊息。

使用路由將此類訊息傳遞到你的應用元件。 為此，你可以將 withComponentInputBinding 屬性與 provideRouter 或 RouterModule.forRoot 的 bindToComponentInputs 選項一起使用。

要從路由獲取訊息：

  1. 新增 withComponentInputBinding

    將 withComponentInputBinding 屬性新增到 provideRouter 方法。

          
    providers: [  provideRouter(appRoutes, withComponentInputBinding()),]

  2. 向元件新增一個 Input

    更新元件以使其具有與引數名稱對應的 Input。

          
    @Input()set id(heroId: string) {  this.hero$ = this.service.getHero(heroId);}

    注意：你可以使用鍵值對將所有路由資料繫結到元件輸入：靜態或已求解的路由資料、路徑引數、矩陣引數和查詢引數。 如果你想使用父元件的路由訊息，你需要設定路由器的 paramsInheritanceStrategy 選項：withRouterConfig({paramsInheritanceStrategy: 'always'})

建立萬用字元路由

一個執行良好的應用應該優雅地處理使用者嘗試導向到應用中不存在的部分的情況。 要將此功能新增到你的應用，你可以建立一個萬用字元路由。 每當請求的 URL 與任何路由器路徑都不對應時，Angular 路由器都會選擇此路由。

要建立萬用字元路由，請將以下程式碼新增到你的 routes 定義中。

      
{ path: '**', component: <component-name> }

兩個星號 ** 向 Angular 指示此 routes 定義是萬用字元路由。 對於 component 屬性，你可以定義應用中的任何元件。 常見的選擇包括特定於應用的 PageNotFoundComponent，你可以將其定義為向用戶顯示 404 頁面；或重導向到應用的主元件。 萬用字元路由是最後一個路由，因為它對應任何 URL。 有關為什麼路由順序很重要的更多詳細訊息，請參閱路由順序。

顯示 404 頁面

要顯示 404 頁面，請建立一個萬用字元路由，並將 component 屬性設定為你要用於 404 頁面的元件，如下所示：

      
const routes: Routes = [  { path: 'first-component', component: FirstComponent },  { path: 'second-component', component: SecondComponent },  { path: '**', component: PageNotFoundComponent },  // Wildcard route for a 404 page];

path 為 ** 的最後一個路由是萬用字元路由。 如果請求的 URL 與清單中較早的任何路徑都不對應，則路由器會選擇此路由，並將使用者傳送到 PageNotFoundComponent。

建立重導向

要建立重導向，請設定一個路由，其中包含你要從中重導向的 path、你要重導向到的 component 以及告知路由器如何對應 URL 的 pathMatch 值。

      
const routes: Routes = [  { path: 'first-component', component: FirstComponent },  { path: 'second-component', component: SecondComponent },  { path: '',   redirectTo: '/first-component', pathMatch: 'full' }, // redirect to `first-component`  { path: '**', component: PageNotFoundComponent },  // Wildcard route for a 404 page];

在此範例中，第三個路由是重導向，以便路由器預設為 first-component 路由。 請注意，此重導向在萬用字元路由之前。 在這裡，path: '' 表示使用初始相對 URL ('')。

有時，重導向不是簡單的靜態重導向。redirectTo 屬性也可以是一個函式，其中包含更復雜的邏輯，該邏輯回傳字串或 UrlTree。

      
const routes: Routes = [  { path: "first-component", component: FirstComponent },  {    path: "old-user-page",    redirectTo: ({ queryParams }) => {      const errorHandler = inject(ErrorHandler);      const userIdParam = queryParams['userId'];      if (userIdParam !== undefined) {        return `/user/${userIdParam}`;      } else {        errorHandler.handleError(new Error('Attempted navigation to user page without user ID.'));        return `/not-found`;      }    },  },  { path: "user/:userId", component: OtherComponent },];

巢狀路由

隨著你的應用變得越來越複雜，你可能想要建立相對於根元件以外的元件的路由。 這些型別的巢狀路由稱為子路由。 這意味著你要向你的應用新增第二個 <router-outlet>，因為它是在 AppComponent 中的 <router-outlet> 之外新增的。

在此範例中，有兩個額外的子元件，child-a 和 child-b。 在這裡，除了 AppComponent 中的 <router-outlet> 之外，FirstComponent 還有自己的 <nav> 和第二個 <router-outlet>。

      
<h2>First Component</h2><nav>  <ul>    <li><a routerLink="child-a">Child A</a></li>    <li><a routerLink="child-b">Child B</a></li>  </ul></nav><router-outlet />

子路由就像任何其他路由一樣，因為它既需要 path 也需要 component。 唯一的區別是你將子路由放在父路由中的 children 陣列中。

      
const routes: Routes = [  {    path: 'first-component',    component: FirstComponent, // this is the component with the <router-outlet> in the template    children: [      {        path: 'child-a', // child route path        component: ChildAComponent, // child route component that the router renders      },      {        path: 'child-b',        component: ChildBComponent, // another child route component that the router renders      },    ],  },];

設定頁面標題

應用中的每個頁面都應該有一個唯一的標題，以便可以在瀏覽器歷史記錄中識別它們。 Router 使用 Route 設定中的 title 屬性設定文件的標題。

      
const routes: Routes = [  {    path: 'first-component',    title: 'First component',    component: FirstComponent,  // this is the component with the <router-outlet> in the template    children: [      {        path: 'child-a',  // child route path        title: resolvedChildATitle,        component: ChildAComponent,  // child route component that the router renders      },      {        path: 'child-b',        title: 'child b',        component: ChildBComponent,  // another child route component that the router renders      },    ],  },];const resolvedChildATitle: ResolveFn<string> = () => Promise.resolve('child a');

提示：title 屬性所遵循的規則與靜態路由 data 以及實現了 ResolveFn 的動態值相同。

你還可以透過擴充套件 TitleStrategy 來提供自定義的標題策略。

      
@Injectable({ providedIn: 'root' })export class TemplatePageTitleStrategy extends TitleStrategy {  private readonly title = inject(Title);  override updateTitle(routerState: RouterStateSnapshot) {    const title = this.buildTitle(routerState);    if (title !== undefined) {      this.title.setTitle(`My Application | ${title}`);    }  }}export const appConfig: ApplicationConfig = {  providers: [    provideRouter(routes),    { provide: TitleStrategy, useClass: TemplatePageTitleStrategy },  ]};

使用相對路徑

相對路徑允許你定義相對於當前 URL 段的路徑。 以下範例展示了到另一個元件 second-component 的相對路由。 FirstComponent 和 SecondComponent 在樹中處於同一級別，但是，到 SecondComponent 的連結位於 FirstComponent 中，這意味著路由器必須向上移動一級，然後再進入第二個目錄才能找到 SecondComponent。 與其寫出到達 SecondComponent 的完整路徑，不如使用 ../ 表示法來向上移動一級。

      
<h2>First Component</h2><nav>  <ul>    <li><a routerLink="../second-component">Relative Route to second component</a></li>  </ul></nav><router-outlet />

除了 ../ 之外，還可以使用 ./ 或不加前導斜槓來指定當前級別。

指定相對路由

要指定相對路由，請使用 NavigationExtras 的 relativeTo 屬性。 在元件類別中，從 @angular/router 匯入 NavigationExtras。

然後在你的導向方法中使用 relativeTo。 在連結引數陣列（此處包含 items）之後，新增一個物件，並將 relativeTo 屬性設定為 ActivatedRoute，即 this.route。

      
goToItems() {  this.router.navigate(['items'], { relativeTo: this.route });}

navigate() 引數設定路由器以當前路由為基礎，在其後追加 items。

goToItems() 方法將目標 URI 解釋為相對於已啟用的路由，並導向到 items 路由。

訪問查詢引數和片段

有時，你的應用的某個屬性需要訪問路由的一部分，例如查詢引數或片段。 在本範例中，路由包含一個 id 引數，我們可以使用它來定位特定的英雄頁面。

      
import { ApplicationConfig } from "@angular/core";import { Routes } from '@angular/router';import { HeroListComponent } from './hero-list.component';export const routes: Routes = [  { path: 'hero/:id', component: HeroDetailComponent }];export const appConfig: ApplicationConfig = {  providers: [provideRouter(routes)],};

首先，在你想要從中導向的元件中匯入以下成員。

      
import { inject } from '@angular/core';import { ActivatedRoute } from '@angular/router';import { Observable, switchMap } from 'rxjs';

接下來，注入已啟用的路由服務：

      
private readonly route = inject(ActivatedRoute);

設定該類別，以便你擁有一個可觀察者 heroes$、一個用於儲存英雄 id 號碼的 selectedId 以及 ngOnInit() 中的英雄，新增以下程式碼以獲取所選英雄的 id。 此程式碼段落假定你有一個英雄清單、一個英雄服務、一個獲取英雄的函式以及用於渲染清單和詳情的 HTML，就像在英雄之旅範例中一樣。

      
heroes$: Observable<Hero[]>;selectedId: number;heroes = HEROES;ngOnInit() {  this.heroes$ = this.route.paramMap.pipe(    switchMap(params => {      this.selectedId = Number(params.get('id'));      return this.service.getHeroes();    })  );}

接下來，在你想要導向到的元件中，匯入以下成員。

      
import { Router, ActivatedRoute, ParamMap } from '@angular/router';import { Observable } from 'rxjs';

在元件類別的建構式函式中注入 ActivatedRoute 和 Router，以便它們可用於此元件：

      
private readonly route = inject(ActivatedRoute);private readonly router = inject(Router);hero$: Observable<Hero>;ngOnInit() {  const heroId = this.route.snapshot.paramMap.get('id');  this.hero$ = this.service.getHero(heroId);}gotoItems(hero: Hero) {  const heroId = hero ? hero.id : null;  // Pass along the hero id if available  // so that the HeroList component can select that item.  this.router.navigate(['/heroes', { id: heroId }]);}

延遲載入

你可以設定你的路由以延遲載入模組，這意味著 Angular 只在需要時載入模組，而不是在應用啟動時載入所有模組。 此外，在後台預載入應用的某些部分可以改善使用者體驗。

任何路由都可以透過使用 loadComponent: 延遲載入其路由的獨立元件

延遲載入獨立元件

      
const routes: Routes = [  {    path: 'lazy',    loadComponent: () => import('./lazy.component').then(c => c.LazyComponent)  }];

只要載入的元件是獨立的，這就有效。

阻止未經授權的訪問

使用路由守衛來阻止使用者在未經授權的情況下導向到應用的某些部分。 以下路由守衛在 Angular 中可用：

canActivate canActivateChild canDeactivate canMatch resolve canLoad

要使用路由守衛，請考慮使用無元件路由，因為這有助於保護子路由。

為你的守衛建立一個檔案：

      
ng generate guard your-guard

在你的守衛檔案中，新增你要使用的守衛函式。 以下範例使用 canActivateFn 來守衛路由。

      
export const yourGuardFunction: CanActivateFn = (  next: ActivatedRouteSnapshot,  state: RouterStateSnapshot) => {  // your  logic goes here}

在你的路由模組中，在 routes 設定中使用適當的屬性。 在這裡，canActivate 告訴路由器協調到此特定路由的導向。

      
{  path: '/your-path',  component: YourComponent,  canActivate: [yourGuardFunction],}

連結引數陣列

連結引數陣列包含用於路由器導向的以下要素：

  • 到目標元件的路由路徑
  • 進入路由 URL 的必要和可選路由引數

像這樣將 RouterLink 指令繫結到這樣的陣列：

      
<a [routerLink]="['/heroes']">Heroes</a>

以下是在指定路由引數時的雙元素陣列：

      
<a [routerLink]="['/hero', hero.id]">  <span class="badge">{{ hero.id }}</span>{{ hero.name }}</a>

在物件中提供可選的路由引數，如 { foo: 'foo' }：

      
<a [routerLink]="['/crisis-center', { foo: 'foo' }]">Crisis Center</a>

這三個範例涵蓋了具有一級路由的應用的需求。 但是，對於子路由器，例如危機中心中的子路由器，你將建立新的連結陣列可能性。

以下最小的 RouterLink 範例建立在為危機中心指定的預設子路由之上。

      
<a [routerLink]="['/crisis-center']">Crisis Center</a>

審閱以下內容：

  • 陣列中的第一個條目標識父路由 (/crisis-center)
  • 此父路由沒有引數
  • 子路由沒有預設值，因此你需要選擇一個
  • 你正在導向到 CrisisListComponent，其路由路徑為 /，但你不需要顯式新增斜槓

考慮以下從應用根目錄向下導向到『龍危機』的路由器連結：

      
<a [routerLink]="['/crisis-center', 1]">Dragon Crisis</a>
  • 陣列中的第一個條目標識父路由 (/crisis-center)
  • 此父路由沒有引數
  • 第二個條目標識關於特定危機的子路由詳情 (/:id)
  • 詳情子路由需要一個 id 路由引數
  • 你已將『龍危機』的 id 新增為數組中的第二個條目 (1)
  • 生成的路徑是 /crisis-center/1

你還可以使用專門的危機中心路由重新定義 AppComponent 樣板：

      
@Component({  template: `    <h1 class="title">Angular Router</h1>    <nav>      <a [routerLink]="['/crisis-center']">Crisis Center</a>      <a [routerLink]="['/crisis-center/1', { foo: 'foo' }]">Dragon Crisis</a>      <a [routerLink]="['/crisis-center/2']">Shark Crisis</a>    </nav>    <router-outlet />  `})export class AppComponent {}

總之，你可以編寫具有一級、二級或更多級路由的應用。 連結引數陣列提供了靈活性，可以表示任何路由深度和任何合法的路由路徑序列、(必要的) 路由器引數和（可選的）路由引數物件。

LocationStrategy 和瀏覽器 URL 樣式

當路由器導向到一個新的元件檢視時，它會用該檢視的 URL 更新瀏覽器的位置和歷史記錄。

現代 HTML5 瀏覽器支援 history.pushState，這項技術可以在不觸發伺服器頁面請求的情況下更改瀏覽器的位置和歷史記錄。 路由器可以生成一個『自然的』URL，它與原本需要頁面載入的 URL 難以區分。

這是以『HTML5 pushState』風格表示的危機中心 URL：

      
localhost:3002/crisis-center

除非更改發生在『#』（稱為『雜湊』）之後，否則當位置 URL 更改時，舊版瀏覽器會向伺服器傳送頁面請求。 路由器可以利用此例外，透過使用雜湊值來組合應用內路由 URL。 這是一個路由到危機中心的『雜湊 URL』。

      
localhost:3002/src/#/crisis-center

路由器透過兩個 LocationStrategy 提供者來支援這兩種風格：

提供者 詳情
PathLocationStrategy 預設的『HTML5 pushState』風格。
HashLocationStrategy 『雜湊 URL』風格。

RouterModule.forRoot() 函式將 LocationStrategy 設定為 PathLocationStrategy，這使其成為預設策略。 你還可以選擇在引導過程中使用重寫切換到 HashLocationStrategy。

提示： 有關提供者和引導過程的更多訊息，請參閱依賴注入。

選擇路由策略

你必須在專案開發的早期選擇路由策略，因為一旦應用投入生產環境，你網站的訪問者就會使用並依賴於應用 URL 引用。

幾乎所有 Angular 專案都應該使用預設的 HTML5 風格。 它生成的 URL 更易於使用者理解，並且保留了進行伺服器端渲染的選項。

在服務器上渲染關鍵頁面是一種可以大大提高應用首次載入時的感知回應速度的技術。 原本需要 10 秒或更長時間才能啟動的應用可以在服務器上渲染，並在不到一秒的時間內交付到使用者的裝置。

只有當應用 URL 看起來像沒有雜湊 (#) 字元在中間的普通 Web URL 時，此選項才可用。

<base href>

路由器使用瀏覽器的 history.pushState 進行導向。 pushState 允許你自定義應用內 URL 路徑；例如，localhost:4200/crisis-center。 應用內 URL 可以與伺服器 URL 難以區分。

現代 HTML5 瀏覽器是第一個支援 pushState 的瀏覽器，這就是為什麼許多人將這些 URL 稱為『HTML5 風格』URL。

提示： HTML5 風格導向是路由器的預設設定。 在LocationStrategy 和瀏覽器 URL 風格部分，瞭解為什麼 HTML5 風格更可取、如何調整其行為以及如何在必要時切換到舊的雜湊 (#) 風格。

你必須將 <base href> 元素新增到應用的 index.html 中，以便 pushState 路由正常工作。 當引用 CSS 檔案、指令碼和影象時，瀏覽器使用 <base href> 值來為相對 URL 新增字首。

將 <base> 元素新增到 <head> 標籤之後。 如果 app 資料夾是應用根目錄（就像此應用一樣），請按此處所示在 index.html 中設定 href 值。

src/index.html (base-href)

      
<!DOCTYPE html><html lang="en">  <head>    <!-- Set the base href -->    <base href="/">    <title>Angular Router</title>    <meta charset="UTF-8">    <meta name="viewport" content="width=device-width, initial-scale=1">  </head>  <body>    <app-root></app-root>  </body></html>

HTML5 URL 和 <base href>

以下指南將引用 URL 的不同部分。 此圖表概述了這些部分所指的內容：

      
foo://example.com:8042/over/there?name=ferret#nose\_/   \______________/\_________/ \_________/ \__/ |           |            |            |        |scheme    authority      path        query   fragment

雖然路由器預設使用 HTML5 pushState 風格，但你必須使用 <base href> 設定該策略。

設定此策略的首選方法是在 index.html 的 <head> 中新增 <base href> 元素標籤。

      
<base href="/">

如果沒有該標籤，當『深度連結』到應用中時，瀏覽器可能無法載入資源（影象、CSS、指令碼）。

有些開發人員可能無法新增 <base> 元素，可能是因為他們無權訪問 <head> 或 index.html。

這些開發人員仍然可以透過採取以下兩個步驟來使用 HTML5 URL：

  1. 為路由器提供適當的 APP_BASE_HREF 值。

  2. 對所有 Web 資源（CSS、影象、指令碼和樣板 HTML 檔案）使用根 URL（帶有 authority 的 URL）。

    • <base href> 的 path 應以『/』結尾，因為瀏覽器會忽略 path 中最右側的『/』之後的字元。

    • 如果 <base href> 包含 query 部分，則僅當頁面中連結的 path 為空且沒有 query 時才使用 query。 這意味著 <base href> 中的 query 僅在使用 HashLocationStrategy 時才包含在內。

    • 如果頁面中的連結是根 URL（具有 authority），則不使用 <base href>。 這樣，帶有 authority 的 APP_BASE_HREF 將導致 Angular 建立的所有連結都忽略 <base href> 值。

    • <base href> 中的片段永遠不會持久存在

有關如何使用 <base href> 建立目標 URI 的更完整訊息，請參閱關於轉換引用的 RFC 部分。

HashLocationStrategy

透過在 AppModule 中，將包含 useHash: true 的物件作為 RouterModule.forRoot() 的第二個引數提供，來使用 HashLocationStrategy。

      
providers: [  provideRouter(appRoutes, withHashLocation())]

當使用 RouterModule.forRoot 時，可以使用第二個引數 useHash: true 進行設定：RouterModule.forRoot(routes, {useHash: true})。
