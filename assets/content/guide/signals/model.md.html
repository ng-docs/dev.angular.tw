模型輸入

edit

模型輸入 是一種特殊型別的輸入，使元件能夠將新值傳播回另一個元件。

有幫助的提示： 目前模型輸入處於 開發者預覽階段。

建立元件時，可以像建立標準輸入一樣定義模型輸入。

      
import {Component, model, input} from '@angular/core';
@Component({...})
export class CustomCheckbox {
  // This is a model input.
  checked = model(false);
  // This is a standard input.
  disabled = input(false);
}

兩種型別的輸入都允許繫結一個值到屬性中。然而， 模型輸入允許元件作者將值寫入屬性。

在其他方面，可以像使用標準輸入一樣使用模型輸入。你可以透過呼叫訊號函式讀取值，包括在 computed 和 effect 這樣的回應式上下文中。

      
import {Component, model, input} from '@angular/core';
@Component({
  selector: 'custom-checkbox',
  template: '<div (click)="toggle()"> ... </div>',
})
export class CustomCheckbox {
  checked = model(false);
  disabled = input(false);
  toggle() {
    // While standard inputs are read-only, you can write directly to model inputs.
    this.checked.set(!this.checked());
  }
}

當元件將新值寫入模型輸入時，Angular 可以將新值傳播回繫結該輸入值的元件。這被稱為 雙向繫結，因為值可以雙向流動。

使用訊號的雙向繫結

你可以將可寫訊號繫結到模型輸入。

      
@Component({
  ...,
  // `checked` is a model input.
  // The parenthesis-inside-square-brackets syntax (aka "banana-in-a-box") creates a two-way binding
  template: '<custom-checkbox [(checked)]="isAdmin" />',
})
export class UserProfile {
  protected isAdmin = signal(false);
}

在上述範例中， CustomCheckbox 可以將值寫入其 checked 模型輸入，然後將這些值傳播回 UserProfile 中的 isAdmin 訊號。此繫結保持 checked 和 isAdmin 的值同步。注意，繫結傳遞的是 isAdmin 訊號本身，而不是訊號的 值。

使用普通屬性的雙向繫結

你可以將普通 JavaScript 屬性繫結到模型輸入。

      
@Component({
  ...,
  // `checked` is a model input.
  // The parenthesis-inside-square-brackets syntax (aka "banana-in-a-box") creates a two-way binding
  template: '<custom-checkbox [(checked)]="isAdmin" />',
})
export class UserProfile {
  protected isAdmin = false;
}

在上述範例中， CustomCheckbox 可以將值寫入其 checked 模型輸入，然後將這些值傳播回 UserProfile 中的 isAdmin 屬性。此繫結保持 checked 和 isAdmin 的值同步。

隱式 change 事件

在元件或指令中宣告模型輸入時，Angular 會自動為該模型建立一個對應的 輸出。輸出的名稱是在模型輸入的名稱後加上 "Change" 字尾。

      
@Directive({...})
export class CustomCheckbox {
  // This automatically creates an output named "checkedChange".
  // Can be subscribed to using `(checkedChange)="handler()"` in the template.
  checked = model(false);
}

每當你透過呼叫 set 或 update 方法寫入新值到模型輸入時，Angular 就會觸發此更改事件。

自定義模型輸入

你可以像為 標準輸入 一樣，將模型輸入標記為必要或提供別名。

模型輸入不支援輸入轉換。

model() 和 input() 的區別

input() 和 model() 函式都是在 Angular 中定義基於訊號的輸入的方法，但它們在一些方面有所不同：

  1. model() 定義了 輸入和輸出。輸出的名稱總是輸入名稱字尾 Change，以支援雙向繫結。由你的指令的使用者決定是隻使用輸入，只使用輸出，還是同時使用兩者。
  2. ModelSignal 是 WritableSignal，這意味著可以透過 set 和 update 方法從任何地方更改其值。當賦予新值時， ModelSignal 將向其輸出發出訊號。這與 InputSignal 不同，後者是隻讀的，只能透過範本更改。
  3. 模型輸入不支援輸入轉換，而訊號輸入支援。

何時使用模型輸入

在基於使用者互動修改值的元件中使用模型輸入。自定義表單控制元件（如日期選擇器或組合框）應使用模型輸入作為其主要值。

避免為了方便而使用模型輸入，以免引入一個額外的類別屬性來包含本地狀態。
