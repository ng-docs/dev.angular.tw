測試

edit

測試你的 Angular 應用可以幫助你檢查你的應用是否按照預期工作。

建立測試

Angular CLI 下載並安裝使用 Jasmine 測試框架 測試 Angular 應用所需的一切。

你使用 CLI 建立的專案可以立即進行測試。 只需執行 ng test CLI 指令：

      
ng test

ng test 指令以監聽模式建立應用， 並啟動 Karma 測試執行器。

控制台輸出如下所示：

日誌的最後一行顯示 Karma 執行了三個測試，全部透過。

測試輸出使用 Karma Jasmine HTML Reporter 顯示在瀏覽器中。

單擊測試行可重新執行該測試，或單擊描述可重新執行所選測試組（『測試套件』）中的測試。

同時，ng test 指令正在監視更改。

要檢視實際效果，請對 app.component.ts 稍作更改並儲存。 測試會再次執行，瀏覽器會重新整理，並顯示新的測試結果。

設定

Angular CLI 負責處理 Jasmine 和 Karma 的設定。它會根據 angular.json 檔案中指定的選項在記憶體中建立完整的設定。

如果要自定義 Karma，可以透過執行以下指令來建立 karma.conf.js：

      
ng generate config karma

提示： 在 Karma 設定指南 中閱讀有關 Karma 設定的更多訊息。

其他測試框架

你還可以使用其他測試庫和測試執行器對 Angular 應用進行單元測試。 每個函式庫和執行器都有自己獨特的安裝步驟、設定和語法。

測試檔案名和位置

在 src/app 資料夾中，Angular CLI 為 AppComponent 生成了一個名為 app.component.spec.ts 的測試檔案。

重要提示： 測試檔案副檔名必須是 .spec.ts，以便工具可以將其識別為包含測試的檔案（也稱為spec 檔案）。

app.component.ts 和 app.component.spec.ts 檔案是同一資料夾中的同級檔案。 兩個檔案的根檔案名 (app.component) 相同。

在你的專案中，每種測試檔案都應採用這兩個約定。

將你的 spec 檔案放在它所測試的檔案旁邊

最好將單元測試 spec 檔案放在與它們測試的應用原始碼檔案相同的資料夾中：

  • 這樣的測試很容易找到
  • 你可以一目瞭然地看到應用的哪個部分缺少測試
  • 附近的測試可以揭示某個部分如何在上下文中工作
  • 當你移動原始碼（不可避免）時，你會記得移動測試
  • 當你重新命名源檔案（不可避免）時，你會記得重新命名測試檔案

將你的 spec 檔案放在 test 資料夾中

應用整合 spec 可以測試分佈在資料夾和模組中的多個部分的互動。 它們實際上不屬於任何特定的部分，因此它們在任何一個檔案旁邊都沒有自然的歸宿。

通常最好在 tests 目錄中為它們建立一個合適的資料夾。

當然，測試測試輔助函式的 spec 屬於 test 資料夾， 與其對應的輔助函式檔案放在一起。

在持續整合中進行測試

保持專案沒有缺陷的最佳方法之一是透過測試套件，但你可能會忘記一直執行測試。

持續整合 (CI) 伺服器允許你建立專案倉庫，以便你的測試在每次送出和拉取請求時運行。

要在持續整合 (CI) 中測試你的 Angular CLI 應用，請執行以下指令：

      
ng test --no-watch --no-progress --browsers=ChromeHeadless
