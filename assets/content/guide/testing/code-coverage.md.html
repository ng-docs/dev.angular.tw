瞭解你正在測試多少程式碼

edit

Angular CLI 可以執行單元測試並建立程式碼覆蓋率報告。 程式碼覆蓋率報告會顯示你的程式碼庫中可能未被單元測試充分測試到的任何部分。

要在你的專案根目錄下生成覆蓋率報告，請執行以下指令。

      
ng test --no-watch --code-coverage

當測試完成時，該指令會在專案中建立一個新的 /coverage 目錄。 開啟 index.html 檔案以檢視包含你的原始碼和程式碼覆蓋率值的報告。

如果你想在每次測試時都建立程式碼覆蓋率報告，請在 Angular CLI 設定檔案 angular.json 中設定以下選項：

      
"test": {  "options": {    "codeCoverage": true  }}

程式碼覆蓋率強制執行

程式碼覆蓋率百分比讓你估計你的程式碼有多少被測試過。 如果你的團隊決定了要進行單元測試的最小程式碼量，請使用 Angular CLI 強制執行此最小值。

例如，假設你希望程式碼庫的程式碼覆蓋率至少達到 80%。 要啟用此功能，請開啟 Karma 測試平台設定檔案 karma.conf.js，並在 coverageReporter: 鍵中新增 check 屬性。

      
coverageReporter: {  dir: require('path').join(__dirname, './coverage/<project-name>'),  subdir: '.',  reporters: [    { type: 'html' },    { type: 'text-summary' }  ],  check: {    global: {      statements: 80,      branches: 80,      functions: 80,      lines: 80    }  }}

提示： 請在測試指南中閱讀更多關於建立和微調 Karma 設定的訊息。

當在專案中執行單元測試時，check 屬性會導致該工具強制執行至少 80% 的程式碼覆蓋率。

請在 karma coverage 文件中閱讀更多關於覆蓋率設定選項的訊息。
