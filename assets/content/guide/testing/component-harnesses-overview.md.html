元件測試工具概覽

edit

元件測試工具是一個類，它允許測試以最終使用者的方式，透過受支援的 API 與元件進行互動。你可以為任何元件建立測試工具，範圍從小型的可複用小部件到完整的頁面。

測試工具提供了以下幾個優點：

  • 它們透過將自身與元件的實現細節（例如其 DOM 結構）隔離開來，使測試不易出錯
  • 它們使測試更具可讀性，並且更易於維護
  • 它們可以跨多個測試環境使用
      
// Example of test with a harness for a component called MyButtonComponentit('should load button with exact text', async () => {  const button = await loader.getHarness(MyButtonComponentHarness);  expect(await button.getText()).toBe('Confirm');});

元件測試工具對於共享 UI 小部件尤其有用。開發人員經常編寫依賴於小部件的私有實現細節（例如 DOM 結構和 CSS 類）的測試。這些依賴關係使測試變得脆弱且難以維護。測試工具提供了一種替代方案——一個受支援的 API，它以最終使用者的方式與小部件進行互動。現在，小部件實現的更改不太可能破壞使用者測試。例如，Angular Material 為庫中的每個元件都提供了一個測試工具。

元件測試工具支援多個測試環境。你可以在單元測試和端到端測試中使用相同的測試工具實現。測試編寫者只需要學習一個 API，而元件作者不必維護單獨的單元和端到端測試實現。

許多開發人員可以根據以下開發人員型別類別之一進行分類：測試編寫者、元件測試工具作者和測試工具環境作者。使用下表根據這些類別查詢本指南中最相關的部分：

測試編寫者 使用其他人編寫的元件測試工具來測試其應用的開發人員。例如，這可能是一個應用開發人員，他使用第三方選單元件，並且需要在單元測試中與該選單進行互動。 在測試中使用元件測試工具
元件測試工具作者 維護一些可複用的 Angular 元件，並希望為其使用者建立一個測試工具以在他們的測試中使用的開發人員。例如，第三方 Angular 元件庫的作者或為大型 Angular 應用維護一組通用元件的開發人員。 為你的元件建立元件測試工具
