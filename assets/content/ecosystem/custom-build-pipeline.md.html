自定義建立通道

edit

在建立 Angular 應用時，我們強烈建議你使用 Angular CLI，以利用其依賴於結構的更新功能和建立系統抽象。 這樣，你的專案就可以從最新的安全性、效能和 API 改進以及透明的建立改進中獲益。

本頁探討了當你需要不使用 Angular CLI 的自定義建立通道時的罕見用例。 下面列出的所有工具都是由 Angular 社群成員維護的開源建立外掛。 要了解有關其支援模式和維護狀態的更多訊息，請檢視其文件和 GitHub 倉庫 URL。

何時應該使用自定義建立通道？

在某些特定的用例中，你可能希望維護自定義建立通道。 例如：

  • 你有一個使用不同工具鏈的現有應用，並且你想向其中新增 Angular
  • 你與 module federation 緊密耦合，並且無法採用與捆綁器無關的 native federation
  • 你想使用你最喜歡的建立工具建立一個短期的實驗

有哪些選項？

目前，有兩個得到良好支援的社群工具，使你能夠使用 Vite 外掛 和 Rspack 外掛 建立自定義建立通道。 它們都使用為 Angular CLI 提供支援的底層抽象。 它們允許你建立靈活的建立通道，並且需要手動維護，沒有自動更新體驗。

Rspack

Rspack 是一個基於 Rust 的捆綁器，旨在提供與 webpack 外掛生態系統的相容性。

如果你的專案與 webpack 生態系統緊密耦合，並且嚴重依賴自定義 webpack 設定，則可以利用 Rspack 來縮短建立時間。

你可以在該專案的文件網站上找到有關 Angular Rspack 的更多訊息。

Vite

Vite 是一種前端建立工具，旨在為現代 Web 專案提供更快、更精簡的開發體驗。 Vite 還可透過其外掛系統進行擴充套件，該系統允許生態系統建立與 Vite 的整合，例如用於單元和瀏覽器測試的 Vitest、用於隔離創作元件的 Storybook 等。 Angular CLI 也使用 Vite 作為其開發伺服器。

AnalogJS Vite plugin for Angular 使 Angular 能夠與使用 Vite 或建立於 Vite 之上的專案或框架一起使用。 這可以包括直接使用 Vite 開發和建立 Angular 專案，或將 Angular 新增到現有專案或通道。 一個範例是將 Angular UI 元件整合到使用 Astro and Starlight 的文件網站中。

你可以透過其文件頁面瞭解有關 AnalogJS 以及如何使用該外掛的更多訊息。
